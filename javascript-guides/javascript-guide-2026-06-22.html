R1 — ResizeObserver & PerformanceObserver Browser APIs · 8 days overdue · rep 0
Quick reminder: ResizeObserver fires whenever an observed element’s dimensions change. PerformanceObserver watches performance entries — e.g., "largest-contentful-paint", "layout-shift", "longtask". Both are callback-based and need explicit cleanup to avoid leaks.
Key Pattern — ResizeObserver
JavaScript
const ro = new ResizeObserver(entries => {
  for (const entry of entries) {
    console.log(entry.contentRect.width, entry.contentRect.height);
  }
});
ro.observe(document.querySelector('.container'));

// Cleanup when done
// ro.unobserve(element);  -- stop watching one element
// ro.disconnect();        -- stop watching all elements
800 600 800 450 (after resize)
Key Pattern — PerformanceObserver
JavaScript
const po = new PerformanceObserver(list => {
  for (const entry of list.getEntries()) {
    console.log(entry.name, entry.startTime, entry.duration);
  }
});
po.observe({ type: 'largest-contentful-paint', buffered: true });

// buffered: true  -- also get entries that fired before observation started
// po.disconnect() -- stop observing
largest-contentful-paint 1234.5 0
Challenge

Create a ResizeObserver that logs when a <textarea> changes size, then automatically calls unobserve() after 5 resize events. Track the count in a closure variable.

Pitfall: Forgetting to call disconnect() or unobserve() leads to memory leaks — the observer keeps a strong reference to every observed element. Always clean up in component teardown (e.g., disconnectedCallback, React’s useEffect cleanup, etc.).
R2 — Fetch Advanced: Interceptors, Streaming & Cloning Async & Network · 8 days overdue · rep 0
Quick reminder: response.clone() before consuming the body — the body is a one-shot ReadableStream. You can pipe response.body for streaming reads. Build interceptors with a wrapper function that decorates fetch.
Key Pattern — Clone before consuming
JavaScript
const response = await fetch(url);
const clone = response.clone();

const json = await response.json();   // consumes original body
const text = await clone.text();       // can still read from clone

console.log(json);
console.log(text.slice(0, 50));
{ "id": 1, "name": "Alice" } {"id":1,"name":"Alice"}
Key Pattern — Streaming response body
JavaScript
const response = await fetch('/large-file.json');
const reader = response.body.getReader();
const decoder = new TextDecoder();
let result = '';

while (true) {
  const { value, done } = await reader.read();
  if (done) break;
  result += decoder.decode(value, { stream: true });
  console.log(`Received ${result.length} bytes so far...`);
}
console.log('Download complete');
Received 16384 bytes so far... Received 32768 bytes so far... Download complete
Challenge

Write a fetchWithRetry(url, maxRetries) function that retries on 5xx status codes up to maxRetries times with exponential backoff (delay = 2^attempt * 100 ms). Return the successful response or throw after all retries are exhausted.

Pitfall: Reading the body twice without cloning throws — TypeError: body stream already read. The body is a one-shot stream. Always clone() before the first read if you need the body more than once.
R3 — Web Storage & IndexedDB Browser APIs · 8 days overdue · rep 0
Quick reminder: localStorage.setItem/getItem/removeItem — synchronous, ~5 MB limit, strings only. IndexedDB is asynchronous, stores structured objects, uses transactions with onsuccess/onerror callbacks. sessionStorage is per-tab and clears on tab close.
Key Pattern — localStorage with JSON
JavaScript
// Always JSON.stringify on write, JSON.parse on read
localStorage.setItem('user', JSON.stringify({ name: 'Alice', age: 30 }));

const user = JSON.parse(localStorage.getItem('user'));
console.log(user.name);  // "Alice"

// Remove a single key
localStorage.removeItem('user');

// Clear all keys for this origin
localStorage.clear();
Alice
Key Pattern — IndexedDB basics
JavaScript
const request = indexedDB.open('myDB', 1);

request.onupgradeneeded = (e) => {
  const db = e.target.result;
  db.createObjectStore('users', { keyPath: 'id' });
};

request.onsuccess = (e) => {
  const db = e.target.result;
  const tx = db.transaction('users', 'readwrite');
  const store = tx.objectStore('users');
  store.put({ id: 1, name: 'Alice', email: 'alice@example.com' });

  tx.oncomplete = () => console.log('Saved!');
};
Saved!
Challenge

Write a simple key-value store class that wraps IndexedDB with async get(key), set(key, value), and delete(key) methods using Promises. The constructor should accept a database name and store name.

Pitfall: localStorage stores strings only — forgetting JSON.stringify / JSON.parse leads to storing "[object Object]" instead of actual data. Always serialize/deserialize.
R4 — Streams API Browser APIs · 6 days overdue · rep 1
Quick reminder: ReadableStream, WritableStream, TransformStream. Pull-based: reader calls read(), gets { value, done }. Pipe chains with readable.pipeThrough(transform).pipeTo(writable).
Key Pattern — Creating a ReadableStream
JavaScript
const stream = new ReadableStream({
  start(controller) {
    controller.enqueue('Hello');
    controller.enqueue(' World');
    controller.close();
  }
});

const reader = stream.getReader();
let result = await reader.read(); // { value: 'Hello', done: false }
console.log(result.value);

result = await reader.read();     // { value: ' World', done: false }
console.log(result.value);

result = await reader.read();     // { value: undefined, done: true }
console.log(result.done);
Hello World true
Key Pattern — TransformStream pipe chain
JavaScript
const upper = new TransformStream({
  transform(chunk, controller) {
    controller.enqueue(chunk.toUpperCase());
  }
});

const readable = new ReadableStream({
  start(controller) {
    controller.enqueue('hello');
    controller.enqueue('streams');
    controller.close();
  }
});

const transformed = readable.pipeThrough(upper);
const reader = transformed.getReader();

console.log((await reader.read()).value);
console.log((await reader.read()).value);
HELLO STREAMS
Challenge

Create a TransformStream that converts chunks to uppercase, and pipe a ReadableStream of strings through it. Read and log each transformed chunk from the resulting readable side.

Pitfall: Forgetting to call controller.close() — the reader hangs forever waiting for more data. Always close the controller when you have no more chunks to enqueue.
R5 — Web Workers Browser APIs · 6 days overdue · rep 1
Quick reminder: new Worker('worker.js') spawns a background thread. Communicate via postMessage / onmessage. Workers have no DOM access. Transfer ownership of large buffers with Transferable objects for zero-copy performance.
Key Pattern — Basic Worker communication
JavaScript — main.js
// main.js
const worker = new Worker('worker.js');
worker.postMessage({ task: 'sort', data: [3, 1, 2] });
worker.onmessage = (e) => console.log(e.data);  // [1, 2, 3]
worker.onerror = (e) => console.error(e.message);
JavaScript — worker.js
// worker.js
self.onmessage = (e) => {
  const { task, data } = e.data;
  if (task === 'sort') {
    self.postMessage(data.slice().sort((a, b) => a - b));
  }
};
[1, 2, 3]
Key Pattern — Inline Worker via Blob URL
JavaScript
const code = `
  self.onmessage = (e) => {
    const n = e.data;
    let a = 0, b = 1;
    for (let i = 2; i <= n; i++) [a, b] = [b, a + b];
    self.postMessage(n <= 0 ? 0 : b);
  };
`;
const blob = new Blob([code], { type: 'application/javascript' });
const worker = new Worker(URL.createObjectURL(blob));

worker.postMessage(10);
worker.onmessage = (e) => {
  console.log('Fibonacci(10) =', e.data);
  worker.terminate();
};
Fibonacci(10) = 55
Challenge

Create an inline worker using a Blob URL that computes Fibonacci(n) and posts the result back. After receiving the result, call worker.terminate() to free the thread.

Pitfall: Workers copy data by default via structured clone. Transferring large ArrayBuffers is faster but makes them unusable in the sender — worker.postMessage(buffer, [buffer]) transfers ownership. After transfer, buffer.byteLength === 0 in the sender.
R6 — Service Workers Browser APIs · 6 days overdue · rep 1
Quick reminder: Lifecycle: install → waiting → activate → fetch. Cache-first strategy: check cache, fallback to network. Register with navigator.serviceWorker.register('/sw.js'). Must be served over HTTPS (or localhost).
Key Pattern — Cache-first fetch handler
JavaScript — sw.js
// sw.js
self.addEventListener('fetch', (event) => {
  event.respondWith(
    caches.match(event.request)
      .then(cached => cached || fetch(event.request))
  );
});
Key Pattern — Pre-caching in the install event
JavaScript — sw.js
const CACHE_NAME = 'app-v1';
const URLS_TO_CACHE = ['/index.html', '/styles.css', '/app.js'];

self.addEventListener('install', (event) => {
  event.waitUntil(
    caches.open(CACHE_NAME)
      .then(cache => cache.addAll(URLS_TO_CACHE))
  );
});

// Activate: clean up old caches
self.addEventListener('activate', (event) => {
  event.waitUntil(
    caches.keys().then(keys =>
      Promise.all(
        keys.filter(k => k !== CACHE_NAME).map(k => caches.delete(k))
      )
    )
  );
});
Challenge

Write a service worker install handler that pre-caches 3 specific URLs into a named cache called 'static-v1'. Use event.waitUntil() to ensure the install doesn’t complete until caching is done.

Pitfall: Service workers only work on HTTPS (and localhost) — file:// won’t register. Also, a new service worker won’t activate until all tabs using the old version are closed (unless you call self.skipWaiting()).

#1 — Canvas & requestAnimationFrame Browser APIs
Canvas is the go-to for drawing graphics, charts, games, and image processing. requestAnimationFrame gives smooth 60fps animations synced to the display refresh rate — far superior to setInterval for visual work.

The <canvas> element provides a bitmap drawing surface. You get a rendering context (usually '2d') and draw shapes, text, images, and paths programmatically. The canvas is immediate-mode: once you draw something, it’s just pixels — there’s no scene graph to query or manipulate later.

requestAnimationFrame(callback) schedules the callback to run before the next repaint, typically at 60fps. The callback receives a high-resolution timestamp for smooth, time-based animations.

JavaScript — Basic canvas setup
// HTML: <canvas id="myCanvas" width="400" height="300"></canvas>
const canvas = document.getElementById('myCanvas');
const ctx = canvas.getContext('2d');

// Draw a filled rectangle
ctx.fillStyle = '#3fb950';
ctx.fillRect(50, 50, 150, 100);

// Draw a stroked rectangle
ctx.strokeStyle = '#58a6ff';
ctx.lineWidth = 3;
ctx.strokeRect(250, 50, 100, 100);

// Draw a circle (arc)
ctx.beginPath();
ctx.arc(150, 220, 40, 0, Math.PI * 2);
ctx.fillStyle = '#bc8cff';
ctx.fill();
ctx.closePath();
JavaScript — Drawing text and paths
const canvas = document.getElementById('myCanvas');

ctx.lineWidth = 2;

JavaScript — Animation loop with requestAnimationFrame
const canvas = document.getElementById('myCanvas');
const ctx = canvas.getContext('2d');

let x = 0;
const speed = 2;    // pixels per frame
const boxSize = 50;

function animate(timestamp) {
  // 1. Clear the canvas
  ctx.clearRect(0, 0, canvas.width, canvas.height);

  // 2. Draw at current position
  ctx.fillStyle = '#58a6ff';
  ctx.fillRect(x, 125, boxSize, boxSize);

  // 3. Update position (bounce)
  x += speed;
  if (x + boxSize > canvas.width) x = 0;

  // 4. Schedule next frame
  requestAnimationFrame(animate);
}

// Start the loop
requestAnimationFrame(animate);
JavaScript — Time-based animation (frame-rate independent)
const canvas = document.getElementById('myCanvas');
const ctx = canvas.getContext('2d');

let lastTime = 0;
let x = 0;
const SPEED = 200; // pixels per second

function animate(timestamp) {
  const dt = (timestamp - lastTime) / 1000; // delta in seconds
  lastTime = timestamp;

  ctx.clearRect(0, 0, canvas.width, canvas.height);
  ctx.fillStyle = '#3fb950';
  ctx.fillRect(x, 100, 50, 50);

  x += SPEED * dt;  // consistent speed regardless of frame rate
  if (x > canvas.width) x = -50;

  requestAnimationFrame(animate);
}

requestAnimationFrame((ts) => {
  lastTime = ts;
  animate(ts);
});
JavaScript — Pixel manipulation with getImageData / putImageData
const canvas = document.getElementById('myCanvas');
const ctx = canvas.getContext('2d');

// Draw something first
ctx.fillStyle = '#58a6ff';
ctx.fillRect(0, 0, canvas.width, canvas.height);

// Get pixel data (RGBA array)
const imageData = ctx.getImageData(0, 0, canvas.width, canvas.height);
const data = imageData.data; // Uint8ClampedArray [R,G,B,A, R,G,B,A, ...]

// Invert colors
for (let i = 0; i < data.length; i += 4) {
  data[i]     = 255 - data[i];     // R
  data[i + 1] = 255 - data[i + 1]; // G
  data[i + 2] = 255 - data[i + 2]; // B
  // data[i + 3] is alpha, leave it
}

// Write modified pixels back
ctx.putImageData(imageData, 0, 0);
console.log('Pixels inverted:', data.length / 4, 'total pixels');
Pixels inverted: 120000 total pixels
Common Mistakes
Wrong: Using setInterval(draw, 16) for animation — not synced to display refresh, causes jank, and continues running in background tabs (wasting CPU/battery).
Right: Use requestAnimationFrame(draw) — syncs to the display refresh rate, pauses when the tab is hidden, and provides a high-resolution timestamp.
Wrong: Not clearing the canvas between frames — previous frame’s drawing persists, causing smearing/ghosting.
Right: Call ctx.clearRect(0, 0, canvas.width, canvas.height) at the start of each frame before drawing.
Key Rules
  • 2026-07-09 (current — Day 28)
  • getContext('2d') returns the 2D drawing API; getContext('webgl') for 3D
  • Canvas coordinates: (0,0) is top-left, x increases right, y increases down
  • requestAnimationFrame passes a DOMHighResTimeStamp to the callback
  • Use time-based animation (dt * speed) for frame-rate independence
  • getImageData returns an RGBA Uint8ClampedArray — 4 bytes per pixel
  • Setting canvas.width or canvas.height clears the canvas entirely
Practice Exercises
1Draw a simple bar chart on a canvas: given an array [40, 70, 55, 90, 30], draw 5 colored bars with proportional heights and labels underneath.
2Create a bouncing ball animation: a circle that moves diagonally and bounces off all four canvas walls. Use requestAnimationFrame and time-based movement.
3Implement a grayscale filter using getImageData — load an image onto the canvas with drawImage(), read the pixels, average the RGB channels for each pixel, and write back.
Connections: Canvas pixel data produces ArrayBuffers that can be sent to Web Workers (R5) via Transferable for off-main-thread image processing. The Streams API (R4) can pipe processed frames. Pairs naturally with Web Components (#2 below) to encapsulate canvas widgets.
#2 — Web Components Browser APIs
Create reusable, encapsulated custom elements with their own styles and behavior — a framework-free component model built into the browser. No build step needed.

Web Components combine three specs: Custom Elements (define new HTML tags), Shadow DOM (encapsulated styles/markup), and HTML Templates (<template> and <slot> for composable content). Together they let you build self-contained components that work anywhere, regardless of framework.

The lifecycle: constructor() → connectedCallback() (element added to DOM) → disconnectedCallback() (removed) → attributeChangedCallback() (observed attribute changed).

JavaScript — Basic custom element
class GreetingCard extends HTMLElement {
  connectedCallback() {
    const name = this.getAttribute('name') || 'World';
    this.innerHTML = `
      <div style="padding: 16px; border: 2px solid #58a6ff; border-radius: 8px;">
        <h3>Hello, ${name}!</h3>
        <p>Welcome to Web Components.</p>
      </div>
    `;
  }
}

customElements.define('greeting-card', GreetingCard);

// Usage in HTML: <greeting-card name="Alice"></greeting-card>
JavaScript — Shadow DOM for style encapsulation
class StyledButton extends HTMLElement {
  constructor() {
    super();
    this.attachShadow({ mode: 'open' });
  }

  connectedCallback() {
    this.shadowRoot.innerHTML = `
      <style>
        button {
          background: #238636;
          color: white;
          border: none;
          padding: 8px 20px;
          border-radius: 6px;
          cursor: pointer;
          font-size: 14px;
        }
        button:hover { background: #2ea043; }
      </style>
      <button><slot>Click me</slot></button>
    `;
  }
}

customElements.define('styled-button', StyledButton);

// Usage: <styled-button>Save</styled-button>
// The <slot> receives "Save" as its content
// Styles inside Shadow DOM do NOT leak out or get overridden by parent styles

JavaScript — Observed attributes & attributeChangedCallback
class CounterElement extends HTMLElement {
  static get observedAttributes() {
    return ['count'];
  }

  constructor() {
    super();
    this.attachShadow({ mode: 'open' });
  }

  connectedCallback() {
    this.render();
    this.shadowRoot.querySelector('button').addEventListener('click', () => {
      const current = parseInt(this.getAttribute('count') || '0');
      this.setAttribute('count', current + 1);
    });
  }

  attributeChangedCallback(name, oldValue, newValue) {
    if (name === 'count') {
      this.render();
    }
  }

  render() {
    const count = this.getAttribute('count') || '0';
    this.shadowRoot.innerHTML = `
      <style>
        span { font-size: 24px; font-weight: 700; margin-right: 12px; }
        button { padding: 6px 16px; cursor: pointer; }
      </style>
      <span>${count}</span>
      <button>Increment</button>
    `;
    // Re-attach listener after innerHTML (simple approach)
    this.shadowRoot.querySelector('button')?.addEventListener('click', () => {
      this.setAttribute('count', parseInt(count) + 1);
    });
  }
}

customElements.define('my-counter', CounterElement);

// Usage: <my-counter count="0"></my-counter>
JavaScript — Full lifecycle demo
class LifecycleDemo extends HTMLElement {
  static get observedAttributes() {
    return ['label'];
  }

  constructor() {
    super();
    console.log('1. constructor() - element created, no DOM access yet');
    // Do NOT access attributes or children here - they are not available
    this.attachShadow({ mode: 'open' });
  }

  connectedCallback() {
    console.log('2. connectedCallback() - element added to DOM');
    // Safe to access attributes, children, and render
    this.shadowRoot.innerHTML = `<p>${this.getAttribute('label')}</p>`;
  }

  disconnectedCallback() {
    console.log('3. disconnectedCallback() - element removed from DOM');
    // Clean up: remove event listeners, cancel timers, etc.
  }

  attributeChangedCallback(name, oldVal, newVal) {
    console.log(`4. attributeChangedCallback("${name}", "${oldVal}", "${newVal}")`);
    if (this.shadowRoot) {
      this.shadowRoot.innerHTML = `<p>${newVal}</p>`;
    }
  }
}

customElements.define('lifecycle-demo', LifecycleDemo);
1. constructor() - element created, no DOM access yet 2. connectedCallback() - element added to DOM
Common Mistakes
Wrong: Using the constructor() to access attributes or children — they are not yet available at construction time.
Right: Do all DOM work, attribute reads, and rendering in connectedCallback().
Wrong: Custom element name without a hyphen: customElements.define('mybutton', ...) — this will throw a DOMException.
Right: Names MUST contain a hyphen: 'my-button', 'app-nav', 'x-widget'. This prevents conflicts with future HTML elements.
Key Rules
  • Custom element names must contain a hyphen (my-el, not myel)
  • Shadow DOM encapsulates styles — external CSS cannot reach in, internal CSS cannot leak out
  • mode: 'open' — element.shadowRoot is accessible; 'closed' — returns null
  • <slot> projects light DOM children into the shadow DOM
  • Only attributes listed in observedAttributes trigger attributeChangedCallback
  • customElements.whenDefined('my-el') returns a Promise that resolves when the element is registered

3Wrap the Canvas animation from Concept #1 inside a <bouncing-box> custom element. The canvas and animation loop should live entirely inside the Shadow DOM. Expose speed and color as observed attributes.
Connections: Shadow DOM provides the encapsulation that Mixins & Composition (#4) bring to logic. Web Components pair with Canvas (#1) for encapsulated graphics widgets. The disconnectedCallback is where you clean up ResizeObservers (R1) and Workers (R5).
2 Module Architecture
#3 — Module Patterns: Barrel Files, Circular Deps & Tree-Shaking Modules & Patterns
Understanding how bundlers resolve imports affects bundle size, load time, and architecture. Circular dependencies cause subtle bugs. Barrel files can help or hurt depending on how you use them.

A barrel file is an index.js that re-exports from multiple modules, giving consumers a single import path. Tree-shaking is the bundler’s ability to remove unused exports from the final bundle. Circular dependencies occur when module A imports from B and B imports from A — JavaScript handles them via partially-initialized modules, which leads to undefined at the point of use.

JavaScript — Barrel file pattern
// --- src/utils/math.js ---
export function add(a, b) { return a + b; }
export function multiply(a, b) { return a * b; }

// --- src/utils/string.js ---
export function capitalize(str) { return str[0].toUpperCase() + str.slice(1); }
export function slugify(str) { return str.toLowerCase().replace(/\s+/g, '-'); }

// --- src/utils/index.js  (the barrel) ---
export { add, multiply } from './math.js';
export { capitalize, slugify } from './string.js';

// --- Consumer ---
import { add, capitalize } from './utils/index.js';
// Bundler can tree-shake away multiply and slugify
console.log(add(2, 3));
console.log(capitalize('hello'));
5 Hello
JavaScript — Circular dependency problem
// --- a.js ---
import { b } from './b.js';
export const a = 'A is ' + b;
console.log('a.js loaded, a =', a);

// --- b.js ---
import { a } from './a.js';
export const b = 'B is ' + a;
console.log('b.js loaded, b =', b);

// --- main.js ---
import { a } from './a.js';
console.log(a);

// What happens:
// 1. main.js imports a.js
// 2. a.js imports b.js
// 3. b.js imports a.js -- circular! a.js is partially initialized
//    'a' is undefined at this point
// 4. b = "B is undefined"
// 5. a = "A is B is undefined"
b.js loaded, b = B is undefined a.js loaded, a = A is B is undefined A is B is undefined
JavaScript — Breaking circular deps with restructuring
// Solution 1: Extract shared code into a third module
// --- shared.js ---
export const config = { prefix: 'App' };

// --- a.js ---
import { config } from './shared.js';
export const a = config.prefix + '-A';

// --- b.js ---
import { config } from './shared.js';
export const b = config.prefix + '-B';

// Solution 2: Use lazy imports (functions, not top-level)
// --- a.js ---
export function getA() {
  const { getB } = require('./b.js'); // lazy, only runs when called
  return 'A + ' + getB();
}
JavaScript — Tree-shaking: named vs namespace imports
// GOOD: Named imports -- bundler knows exactly what you use
import { add } from './math.js';
// Result: multiply is removed from the bundle

// BAD: Namespace import with dynamic access defeats tree-shaking
import * as MathUtils from './math.js';
const fn = condition ? 'add' : 'multiply';
MathUtils[fn](2, 3); // Bundler cannot statically determine which export is used
// Result: ALL exports are kept in the bundle
JSON — Side effects in package.json
// package.json
{
  "name": "my-lib",
  "sideEffects": false
  // Tells bundler: "Every module is safe to tree-shake"
  // If some files have side effects (e.g., polyfills, CSS imports):
  // "sideEffects": ["./src/polyfill.js", "*.css"]
}

// A "side effect" is code that runs at import time and affects
// something outside the module (global state, DOM, etc.)
// Example of a side-effectful module:
// --- polyfill.js ---
// if (!Array.prototype.flat) {
//   Array.prototype.flat = function() { /* ... */ };
// }
// Importing this file changes global state even if you don't use any export
Common Mistakes
Wrong: Barrel files that re-export everything from huge modules — consumers who import one function pull in the entire barrel, defeating tree-shaking in many bundler configurations.
Right: For performance-critical code, import directly from the source module: import { add } from './utils/math.js' instead of from './utils'.
Wrong: Circular imports with top-level side effects — the partially-initialized module produces undefined values silently.
Right: Restructure to break cycles. Extract shared dependencies into a common module. Use madge or bundler warnings to detect circular deps.
Key Rules
  • Barrel files centralize exports but can bloat bundles if tree-shaking is imperfect
  • import { specific } enables better tree-shaking than import *
  • Circular deps are resolved by partial initialization — exports may be undefined
  • "sideEffects": false in package.json tells bundlers it’s safe to drop unused modules
  • Dynamic import() creates a new chunk — great for code splitting
Practice Exercises
1Create a small project with 3 utility modules and a barrel file. Import only one function in main.js. Use a bundler (esbuild, Rollup, or webpack) to verify that unused exports are removed from the output.
2Intentionally create a circular dependency between two modules. Observe the undefined behavior, then refactor to break the cycle by extracting shared logic into a third module.
3Add "sideEffects": false to a package.json and measure the bundle size difference. Then add a CSS import to see how "sideEffects": ["*.css"] prevents the bundler from dropping it.
Connections: Module resolution directly affects Service Worker (R6) cache strategies — fewer chunks = simpler pre-cache lists. Tree-shaking eliminates dead code from Mixin libraries (#4). Understanding barrel files is essential when publishing libraries that others will import.
#4 — Mixins & Composition Modules & Patterns
JavaScript does not have multiple inheritance, but mixins and composition let you combine behaviors from multiple sources. The principle: composition over inheritance — small, focused pieces that combine, rather than deep class hierarchies.

A mixin adds methods/properties to an existing class or object without using inheritance. There are two main patterns: object mixins (using Object.assign) and class mixin factories (functions that return a subclass of a given base). Composition goes further — instead of “is-a” relationships, objects “have-a” capability by holding references to strategy/delegate objects.

JavaScript — Object mixin with Object.assign
const Serializable = {
  serialize() {
    return JSON.stringify(this);
  },
  deserialize(json) {
    return Object.assign(this, JSON.parse(json));
  }
};

const Loggable = {
  log(message) {
    console.log(`[${this.constructor.name}] ${message}`);
  }
};

class User {
  constructor(name, age) {
    this.name = name;
    this.age = age;
  }
}

// Mix in behaviors
Object.assign(User.prototype, Serializable, Loggable);

const alice = new User('Alice', 30);
alice.log('created');
console.log(alice.serialize());
[User] created {"name":"Alice","age":30}
JavaScript — Class mixin factory pattern
// A mixin factory: takes a Base class, returns a new class extending it
const Timestamped = (Base) => class extends Base {
  constructor(...args) {
    super(...args);
    this.createdAt = new Date();
  }
  getAge() {
    return Date.now() - this.createdAt.getTime();
  }
};

const Activatable = (Base) => class extends Base {
  #active = false;
  activate() { this.#active = true; }
  deactivate() { this.#active = false; }
  get isActive() { return this.#active; }
};

class BaseModel {
  constructor(id) { this.id = id; }
}

// Compose multiple mixins
class UserModel extends Activatable(Timestamped(BaseModel)) {
  constructor(id, name) {
    super(id);
    this.name = name;
  }
}

const u = new UserModel(1, 'Alice');
u.activate();
console.log(u.name, u.isActive, u.createdAt instanceof Date);
Alice true true
JavaScript — Composing multiple mixins with a helper
// Utility to apply multiple mixin factories
function applyMixins(Base, ...mixins) {
  return mixins.reduce((acc, mixin) => mixin(acc), Base);
}

const Draggable = (Base) => class extends Base {
  drag(x, y) {
    this.x = x;
    this.y = y;
    console.log(`Dragged to (${x}, ${y})`);
  }
};

const Resizable = (Base) => class extends Base {
  resize(w, h) {
    this.width = w;
    this.height = h;
    console.log(`Resized to ${w}x${h}`);
  }
};

const Selectable = (Base) => class extends Base {
  #selected = false;
  select() { this.#selected = true; }
  deselect() { this.#selected = false; }
  get isSelected() { return this.#selected; }
};

class Shape {
  constructor(name) { this.name = name; }
}

// Apply all three mixins
class Widget extends applyMixins(Shape, Draggable, Resizable, Selectable) {}

const box = new Widget('box');
box.drag(100, 200);
box.resize(50, 50);
box.select();
console.log(box.name, box.isSelected);
Dragged to (100, 200) Resized to 50x50 box true
JavaScript — Composition with plain objects (no classes)
// Strategy pattern: compose behavior via delegation, not inheritance
function createUser(name) {
  // "Has-a" instead of "Is-a"
  const validator = createValidator();
  const logger = createLogger('User');
  const serializer = createSerializer();

  return {
    name,
    validate() { return validator.validate(this); },
    log(msg) { logger.log(msg); },
    toJSON() { return serializer.serialize(this); },
  };
}

function createValidator() {
  return {
    validate(obj) {
      if (!obj.name || obj.name.length < 2) return { valid: false, error: 'Name too short' };
      return { valid: true };
    }
  };
}

function createLogger(prefix) {
  return {
    log(msg) { console.log(`[${prefix}] ${msg}`); }
  };
}

function createSerializer() {
  return {
    serialize(obj) { return JSON.stringify(obj, (k, v) => typeof v === 'function' ? undefined : v); }
  };
}

const user = createUser('Alice');
user.log('created');
console.log(user.validate());
console.log(user.toJSON());
[User] created { valid: true } {"name":"Alice"}
Common Mistakes
Wrong: Deep inheritance chains (fragile base class problem) — class A extends B extends C extends D makes changes to B break A, C, and D.
Right: Prefer composition. Keep inheritance shallow (1-2 levels max). Use mixins for cross-cutting concerns.
Wrong: Name collisions in mixins without checking — two mixins defining the same method silently overwrite each other.
Right: Use Symbol keys for truly private mixin methods: const _log = Symbol('log'); prevents collisions.
Key Rules
  • Object.assign(target.prototype, mixin) copies enumerable own properties — simple but no super
  • Class mixin factories: (Base) => class extends Base { ... } — supports super, private fields
  • Composition over inheritance: objects hold references to delegates, not parent classes
  • Use Symbols for private mixin methods to avoid name collisions
  • The mixin chain creates a real prototype chain: instanceof works through all layers
Practice Exercises
1Create a Validatable mixin factory that adds a validate() method checking static validationRules on the class. Rules should be an array of { field, test, message } objects.
2Build an EventEmitter mixin using Object.assign that adds on(event, fn), off(event, fn), and emit(event, ...args) to any object. Use a Map of Sets internally.
3Refactor a class hierarchy of 4+ levels into a flat composition model: a factory function that composes the needed behaviors. Compare the before/after in terms of testability (can you test each behavior in isolation?).
Connections: The class mixin pattern is how Web Components (#2) add shared behaviors to different custom elements. Module Patterns (#3) determine how mixin code is imported and tree-shaken. The composition approach mirrors how Streams API (R4) pipes transform stages together.

#5 — Temporal API Modern JavaScript
Date is fundamentally broken — mutable, timezone-confused, no duration type, and months are 0-indexed. Temporal (stage 3 TC39 proposal) fixes all of it with immutable, timezone-aware types. This is the future of date/time in JavaScript.

The Temporal API introduces several distinct types for different use cases:

  • Temporal.PlainDate — a date without time or timezone (birthdays, deadlines)
  • Temporal.PlainTime — a time without date or timezone (alarm time, opening hours)
  • Temporal.PlainDateTime — date + time without timezone (calendar event in “local time”)
  • Temporal.ZonedDateTime — full date + time + timezone (exact moment in a specific zone)
  • Temporal.Instant — an exact point on the UTC timeline (timestamps, logs)
  • Temporal.Duration — a length of time (2 hours 30 minutes)

All Temporal objects are immutable — methods like add() and with() return new instances instead of mutating.

JavaScript — Getting today's date
// Requires polyfill: npm install @js-temporal/polyfill
// import { Temporal } from '@js-temporal/polyfill';

// Get today's date (no time, no timezone confusion)
const today = Temporal.Now.plainDateISO();
console.log(today.toString());
console.log(today.year, today.month, today.day);
console.log(today.dayOfWeek);  // 1=Monday ... 7=Sunday (ISO)

// Note: month is 1-indexed (not 0-indexed like Date!)
// January = 1, December = 12
2026-06-22 2026 6 22 1
JavaScript — Creating dates from strings and components
// From ISO string
const date1 = Temporal.PlainDate.from('2026-06-22');
console.log(date1.toString());

// From components
const date2 = Temporal.PlainDate.from({ year: 2026, month: 12, day: 25 });
console.log(date2.toString());

// PlainDateTime: date + time, no timezone
const dt = Temporal.PlainDateTime.from('2026-06-22T14:30:00');
console.log(dt.toString());
console.log(dt.hour, dt.minute);

// PlainTime: just a time
const time = Temporal.PlainTime.from('09:30:00');
console.log(time.toString());
2026-06-22 2026-12-25 2026-06-22T14:30:00 14 30 09:30:00
JavaScript — ZonedDateTime: timezone-aware
// ZonedDateTime: the only type that knows its timezone
const meeting = Temporal.ZonedDateTime.from({
  year: 2026,
  month: 6,
  day: 22,
  hour: 14,
  minute: 0,
  timeZone: 'America/New_York',
});
console.log(meeting.toString());
console.log('Offset:', meeting.offset);       // "-04:00" (EDT)
console.log('Epoch ms:', meeting.epochMilliseconds);

// Convert to another timezone
const inTokyo = meeting.withTimeZone('Asia/Tokyo');
console.log('Tokyo:', inTokyo.toString());
// Same instant, different wall-clock time
2026-06-22T14:00:00-04:00[America/New_York] Offset: -04:00 Epoch ms: 1782338400000 Tokyo: 2026-06-23T03:00:00+09:00[Asia/Tokyo]
JavaScript — Duration and arithmetic
// Create durations
const duration = Temporal.Duration.from({ hours: 2, minutes: 30 });
console.log(duration.toString());   // "PT2H30M" (ISO 8601)
console.log(duration.total('minutes'));  // 150

// Date arithmetic: add/subtract (returns NEW object, originals unchanged)
const today = Temporal.PlainDate.from('2026-06-22');
const nextWeek = today.add({ days: 7 });
console.log(nextWeek.toString());

const threeMonthsAgo = today.subtract({ months: 3 });
console.log(threeMonthsAgo.toString());

// Difference between dates
const start = Temporal.PlainDate.from('2026-01-01');
const end = Temporal.PlainDate.from('2026-06-22');
const diff = start.until(end);
console.log(diff.toString());       // days between
console.log(diff.total('days'));    // as a number
PT2H30M 150 2026-06-29 2026-03-22 P172D 172
JavaScript — Comparing Temporal objects
const a = Temporal.PlainDate.from('2026-06-22');
const b = Temporal.PlainDate.from('2026-12-25');

// Static compare: returns -1, 0, or 1
console.log(Temporal.PlainDate.compare(a, b));  // -1  (a before b)
console.log(Temporal.PlainDate.compare(b, a));  //  1  (b after a)
console.log(Temporal.PlainDate.compare(a, a));  //  0  (equal)

// Great for sorting
const dates = [
  Temporal.PlainDate.from('2026-12-25'),
  Temporal.PlainDate.from('2026-01-01'),
  Temporal.PlainDate.from('2026-06-22'),
];
dates.sort(Temporal.PlainDate.compare);
console.log(dates.map(d => d.toString()));

// Instance method: equals()
console.log(a.equals(Temporal.PlainDate.from('2026-06-22')));  // true
-1 1 0 ["2026-01-01", "2026-06-22", "2026-12-25"] true
JavaScript — Immutability: with() for partial updates
const date = Temporal.PlainDate.from('2026-06-22');

// with() creates a new date with some fields changed
const sameMonthDifferentDay = date.with({ day: 1 });
console.log(sameMonthDifferentDay.toString());  // first of month

const nextYear = date.with({ year: 2027 });
console.log(nextYear.toString());

// Original is UNCHANGED (immutable)
console.log(date.toString());

// DateTime with()
const dt = Temporal.PlainDateTime.from('2026-06-22T14:30:00');
const morning = dt.with({ hour: 8, minute: 0 });
console.log(morning.toString());
2026-06-01 2027-06-22 2026-06-22 2026-06-22T08:00:00
Common Mistakes
Wrong: Assuming Temporal is available natively everywhere — it is still a stage 3 proposal and not yet shipped in most browsers or Node.js.
Right: Check availability or use the polyfill: npm install @js-temporal/polyfill and import { Temporal } from '@js-temporal/polyfill'.
Wrong: Confusing PlainDate (no timezone) with ZonedDateTime (has timezone) — using PlainDate for a meeting across timezones will show different times than intended.
Right: Pick the right type for your use case. Birthdays → PlainDate. Meetings across timezones → ZonedDateTime. Timestamps/logs → Instant.
Key Rules
  • All Temporal objects are immutable — add(), subtract(), with() return new instances
  • Months are 1-indexed: January = 1 (unlike Date’s 0-indexed months)
  • PlainDate / PlainTime / PlainDateTime have no timezone concept
  • ZonedDateTime is the only type that handles timezone transitions (DST, etc.)
  • Temporal.Duration represents a length of time with add, subtract, total
  • Temporal.PlainDate.compare(a, b) returns -1, 0, or 1 — use for sort()
  • Use @js-temporal/polyfill until native support ships
