R1 — Intl API Functions & Iteration · 6 days since review
Quick reminder: Intl.NumberFormat for locale-aware numbers/currency, Intl.DateTimeFormat for dates, Intl.Collator for locale-sensitive string comparison, and Intl.RelativeTimeFormat for “3 days ago”.
Key Pattern — currency formatting
JavaScript
// Locale + style options drive the formatting
const usd = new Intl.NumberFormat('en-US', {
  style: 'currency',
  currency: 'USD',
});
console.log(usd.format(1234.56));   // "$1,234.56"

// Collator gives locale-correct sorting (handles accents, case)
const names = ['Zoe', 'álvaro', 'ana'];
console.log(names.sort(new Intl.Collator('es').compare));
$1,234.56 [ 'álvaro', 'ana', 'Zoe' ]
Challenge

Format a number as compact notation in the Japanese locale — e.g., 1500000 should render as "150万". Use Intl.NumberFormat with the right notation option and a Japanese locale tag.

#1 Pitfall: Creating a new Intl.NumberFormat on every call. Constructing a formatter is expensive — cache the instance once and reuse it across calls.
R2 — Promise Internals Async Mastery · 6 days since review
Quick reminder: a Promise has 3 states (pending → fulfilled/rejected). .then() returns a new Promise. Promise.all rejects on first failure; Promise.allSettled waits for all; Promise.any resolves on first success.
Key Pattern — allSettled, keep only fulfilled
JavaScript
const tasks = [
  Promise.resolve('ok-A'),
  Promise.reject(new Error('boom')),
  Promise.resolve('ok-C'),
];

// allSettled NEVER rejects — every entry has a status
const results = await Promise.allSettled(tasks);
const fulfilled = results
  .filter(r => r.status === 'fulfilled')
  .map(r => r.value);

console.log(fulfilled);
[ 'ok-A', 'ok-C' ]
Challenge

Implement Promise.any from scratch using Promise.all with inverted logic: a value should be treated as a “rejection” and a rejection as a “success”, then flip the result back. It must resolve on the first fulfilled input and reject with an AggregateError only if all inputs reject.

#1 Pitfall: Leaving an unhandled rejection. Always attach a .catch() or wrap the await in try/catch — an unhandled rejection can crash a Node process.
R3 — async/await Patterns Async Mastery · 5 days since review
Quick reminder: await in series is sequential; Promise.all([f1(), f2()]) runs in parallel. Use for await...of for async iteration. Handle errors with try/catch, not .catch().
Key Pattern — parallel vs sequential
JavaScript
const fetchA = () => Promise.resolve('A');
const fetchB = () => Promise.resolve('B');

// PARALLEL — both start before we await either
const [a, b] = await Promise.all([fetchA(), fetchB()]);
console.log(a, b);

// SEQUENTIAL — b doesn't start until a resolves (slower!)
const a2 = await fetchA();
const b2 = await fetchB();
console.log(a2, b2);
A B A B
Challenge

Write retry(fn, n) that calls an async fn and retries up to n times on rejection, using exponential backoff (wait 2attempt × 100ms between tries). If all attempts fail, re-throw the last error.

#1 Pitfall: Using await inside a for loop when the iterations are independent — that runs them sequentially. Use await Promise.all(items.map(async item => ...)) for parallelism.
R4 — Map & Set Data & Collections · 5 days since review
Quick reminder: Map preserves insertion order, allows any key type, and exposes .size. Set stores unique values. WeakMap/WeakSet let keys be garbage-collected. Prefer Map over a plain object when keys aren't strings.
Key Pattern — iterate a Map, dedupe with a Set
JavaScript
const m = new Map([['a', 1]]);
m.set('b', 2);
for (const [k, v] of m) console.log(k, '=>', v);

// Set dedupes; spread back to an array
const unique = [...new Set([1, 1, 2, 3, 3])];
console.log(unique, 'size', m.size);
a => 1 b => 2 [ 1, 2, 3 ] size 2
Challenge

Implement an LRU cache (get/put, fixed capacity) using a single Map. Hint: Map preserves insertion order, so map.delete(key) followed by map.set(key, val) moves an entry to the “most recently used” end; the least-recently-used key is the first one from map.keys().next().value.

#1 Pitfall: Expecting value equality for object keys. Map compares keys by reference, so m.get({x:1}) won't find an entry set with a different {x:1} literal.
R5 — ES Modules Deep Dive Modules & Patterns · 5 days since review
Quick reminder: import/export are static (hoisted, analyzed at parse time). import() is dynamic and returns a Promise. import.meta.url gives the module's URL. Top-level await works in modules.
Key Pattern — dynamic import
JavaScript
// Load a module by name at runtime (code-splitting / plugins)
const name = 'greeter';
const mod = await import(`./plugins/${name}.js`);
mod.default();   // call the module's default export

// import.meta exposes metadata about THIS module
console.log(new URL('./data.json', import.meta.url).pathname);
/app/plugins/data.json
Challenge

Write a plugin loader: given an array of plugin names, dynamically import() each module from a ./plugins/ directory and call its exported init() function. Load them in parallel and collect any that fail to import into an errors array, without aborting the others.

#1 Pitfall: Circular imports. If module A imports B which imports A, the not-yet-initialized binding reads as undefined at the moment of the circular reference — restructure to break the cycle or defer the use.
R6 — IntersectionObserver Browser APIs & DOM · 5 days since review
Quick reminder: fires a callback when observed elements enter/exit the viewport (or a container). threshold controls trigger points (0 = any pixel visible, 1 = fully visible). Use for lazy-loading images, infinite scroll, and visibility analytics.
Key Pattern — lazy-load images
JavaScript
const obs = new IntersectionObserver((entries) => {
  entries.forEach((e) => {
    if (e.isIntersecting) {
      const img = e.target;
      img.src = img.dataset.src;   // swap placeholder for real src
      obs.unobserve(img);          // stop watching once loaded
    }
  });
}, { threshold: 0.1 });

document.querySelectorAll('img[data-src]').forEach((img) => obs.observe(img));
Challenge

Build an infinite scroll: observe a sentinel element at the bottom of a list. When it becomes visible, load the next page of items and append them. Return a cleanup function that calls observer.disconnect() so it can be torn down on unmount.

#1 Pitfall: Forgetting to unobserve(el) (or disconnect()) after an element has done its job — you keep paying to observe elements that no longer need watching.
1 Modern Web APIs
1. Web Animations API (WAAPI) new
WAAPI gives JavaScript direct control over CSS-like animations — start, pause, reverse, cancel, chain, and read playback state programmatically. It's the right tool when an animation is data-driven or needs orchestration that CSS transitions can't express.

The entry point is element.animate(keyframes, options). It returns an Animation object you can control:

  1. Keyframes come in two shapes: an array of objects ([{ opacity: 0 }, { opacity: 1 }]) where each object is a step, or an object of arrays ({ opacity: [0, 1] }) where each property lists its values across the timeline.
  2. Options control timing: duration (ms), easing ('ease-out', 'cubic-bezier(...)'), iterations (or Infinity), delay, and fill.
  3. The returned Animation object exposes .play(), .pause(), .reverse(), .cancel(), and a .finished Promise that resolves when the animation completes.
  4. Because .finished is a Promise, you can chain animations with await or .then() — slide in, then pulse, then fade out.

The crucial gotcha: by default an animation does not persist its end state. Set fill: 'forwards' so the element keeps the final keyframe instead of snapping back.

Example 1a: basic fade-in

browser console
const box = document.querySelector('#box');

// Two-keyframe fade: from invisible to fully visible
const anim = box.animate(
  [{ opacity: 0 }, { opacity: 1 }],
  { duration: 500, easing: 'ease-out', fill: 'forwards' }
);

// .finished is a Promise — resolves when the animation ends
anim.finished.then(() => console.log('fade-in done, opacity stays at 1'));
fade-in done, opacity stays at 1

Example 1b: multi-step keyframes with offsets

browser console
const box = document.querySelector('#box');

// 'offset' places a keyframe at a fraction of the timeline (0..1)
box.animate(
  [
    { transform: 'translateX(0)',     offset: 0    },
    { transform: 'translateX(200px)', offset: 0.7  },  // fast then slow
    { transform: 'translateX(180px)', offset: 1    },  // slight overshoot back
  ],
  { duration: 1000, easing: 'ease-in-out', fill: 'forwards' }
);

console.log('box slides right with an overshoot');
box slides right with an overshoot

Example 1c: chain animations with await

browser console
async function toast(el) {
  // 1) slide in from the right
  await el.animate(
    { transform: ['translateX(100%)', 'translateX(0)'] },
    { duration: 300, easing: 'ease-out', fill: 'forwards' }
  ).finished;

  // 2) pause 3 seconds (a Promise-based delay)
  await new Promise(r => setTimeout(r, 3000));

  // 3) slide back out
  await el.animate(
    { transform: ['translateX(0)', 'translateX(100%)'] },
    { duration: 300, easing: 'ease-in', fill: 'forwards' }
  ).finished;

  console.log('toast lifecycle complete');
}
toast(document.querySelector('#toast'));
toast lifecycle complete

Example 1d: data-driven progress bar

browser console
function animateProgress(bar, percent) {
  const from = bar.style.width || '0%';
  return bar.animate(
    { width: [from, `${percent}%`] },
    { duration: 400, easing: 'ease-out', fill: 'forwards' }
  ).finished.then(() => {
    bar.style.width = `${percent}%`;   // persist real value beyond the fill
    console.log(`progress now at ${percent}%`);
  });
}
animateProgress(document.querySelector('#bar'), 65);
progress now at 65%
Common Mistakes
❌ Animating to a new state without fill: 'forwards' — the element snaps back to its pre-animation style the instant the animation ends.
✅ Add fill: 'forwards' to persist the final keyframe (and, for layout-affecting props, also write the real value to .style in .finished).
❌ Calling el.animate(...) inside a scroll handler on every scroll tick — you create and discard hundreds of animations per second.
✅ Trigger animations from IntersectionObserver or batch with requestAnimationFrame so they run once when needed.
Key Rules
  • Use WAAPI when you need to control an animation (pause/reverse/cancel) or drive it from data; reach for plain CSS transitions for simple hover/state changes.
  • .finished is a Promise — await it to sequence animations instead of chaining setTimeout.
  • Set fill: 'forwards' whenever the element should keep its end state.
Practice Exercises
1Animate a div from opacity 0 to opacity 1 while sliding up 100px (start at translateY(100px)), over 500ms with ease-out. The final state must persist.
2Create a “typewriter” reveal: animate a clip-path from inset(0 100% 0 0) to inset(0 0 0 0) so text appears left-to-right over 1.2s.
3Build a toast that slides in, pauses 3 seconds, then slides out — using .finished to chain the three phases. Expected: console logs "shown" after phase 1 and "dismissed" after phase 3.
Connections: Builds on .finished being a Promise (R2 Promise Internals) and the await-to-sequence idea from R3 (async/await Patterns). Pairs naturally with R6 IntersectionObserver to trigger animations only when elements scroll into view, and with requestAnimationFrame from earlier performance topics.
2. Broadcast Channel API new
Lets same-origin tabs, windows, iframes, and workers talk to each other without localStorage hacks. It's the clean way to do “log out of all tabs”, sync theme across windows, or keep a shopping cart consistent everywhere the app is open.

The whole API is four members:

  1. new BroadcastChannel(name) — create or join a named channel. Every context that opens the same name on the same origin shares it.
  2. .postMessage(data) — send structured-cloneable data to all other contexts. No JSON.stringify needed; objects, Maps, Dates, etc. survive.
  3. .onmessage = (e) => e.data — receive. Critically, the sender does not receive its own message.
  4. .close() — detach this context from the channel.

Compared with the old window.addEventListener('storage', ...) trick, BroadcastChannel sends structured data directly, doesn't pollute storage, and has a much cleaner mental model.

Example 2a: cross-tab theme sync

browser console (run in 2 tabs)
// Both tabs join the same channel
const themeBus = new BroadcastChannel('theme');

// Receiver: react to a theme change from ANOTHER tab
themeBus.onmessage = (e) => {
  document.documentElement.dataset.theme = e.data.theme;
  console.log('theme changed remotely to', e.data.theme);
};

// Sender: call this from your toggle button
function setTheme(theme) {
  document.documentElement.dataset.theme = theme;  // apply locally
  themeBus.postMessage({ theme });                 // tell other tabs
}
setTheme('dark');
// In the OTHER tab: theme changed remotely to dark

Example 2b: log out across all tabs

browser console
const authBus = new BroadcastChannel('auth');

authBus.onmessage = (e) => {
  if (e.data.type === 'logout') {
    sessionStorage.clear();
    console.log('logged out because another tab logged out');
    // location.href = '/login';
  }
};

function logout() {
  sessionStorage.clear();
  authBus.postMessage({ type: 'logout' });  // notify the others
  console.log('logged out (this tab) + broadcast');
}
logout();
logged out (this tab) + broadcast // other tabs: logged out because another tab logged out

Example 2c: cart sync + cleanup

browser console
const cartBus = new BroadcastChannel('cart');
const cart = new Map();

cartBus.onmessage = (e) => {
  const { id, qty } = e.data;
  cart.set(id, qty);
  console.log('cart synced:', [...cart]);
};

function addToCart(id, qty) {
  cart.set(id, qty);
  cartBus.postMessage({ id, qty });
}

// IMPORTANT: close the channel when the page goes away
window.addEventListener('beforeunload', () => cartBus.close());

addToCart('sku-42', 2);
// other tabs: cart synced: [ [ 'sku-42', 2 ] ]
Common Mistakes
❌ Expecting the sending tab to receive its own postMessage — BroadcastChannel delivers only to other contexts, so local UI never updates.
✅ Apply the change locally and broadcast it: call your handler directly, then channel.postMessage(...) for everyone else.
❌ Never calling .close() — leaked channels keep the context referenced and handlers firing.
✅ Close in beforeunload or on component unmount.
Key Rules
  • Same name + same origin = one shared channel; different origins are isolated.
  • Messages are structured-cloned — no functions, DOM nodes, or class identity, but Maps/Sets/Dates/ArrayBuffers all work.
  • The sender is excluded from delivery — always handle local state separately.
Practice Exercises
1Build a theme toggle that syncs dark/light mode across all open tabs. Toggling in one tab must visibly switch the others.
2Create a “tab counter” that shows how many tabs of your app are currently open. Hint: broadcast a "hello" on open and a "bye" on beforeunload, and have each tab reply to "hello".
3Implement collaborative state: typing into a <textarea> in one tab updates a mirror <textarea> in every other open tab in real time.
Connections: The message payload uses structured clone (Day 7/8 Data & Collections), so the same “no functions, Maps OK” rules apply. It's the multi-tab cousin of postMessage from Web Workers, and complements Web Storage for persistence (broadcast the change, persist it too).
3. File System Access API new
Lets a web app read and write real files on the user's disk (with explicit permission) instead of being limited to downloads. It's what makes browser-based text editors, image editors, and data tools feel native.

The flow is built around handles — permission-bearing references to files or directories:

  1. window.showOpenFilePicker() — opens the OS file dialog; resolves to an array of FileSystemFileHandle.
  2. handle.getFile() — returns a File (a Blob) you can read with .text(), .arrayBuffer(), or .stream().
  3. handle.createWritable() — returns a FileSystemWritableFileStream; .write(data) then .close() commits to disk.
  4. window.showSaveFilePicker() — a save dialog that returns a handle for a new file.
  5. window.showDirectoryPicker() — grants access to a whole folder you can iterate.
  6. Handles are structured-cloneable, so you can persist them in IndexedDB to offer a “recent files” feature across sessions.

All of these require a user gesture (a click) and the page must be served over HTTPS.

Example 3a: open a text file and read it

browser console (in a click handler)
async function openText() {
  // MUST be called from a user gesture (e.g. a button click)
  const [handle] = await window.showOpenFilePicker({
    types: [{ description: 'Text', accept: { 'text/plain': ['.txt', '.md'] } }],
  });

  const file = await handle.getFile();   // a File (Blob)
  const text = await file.text();        // read its contents
  console.log(`read ${file.name}: ${text.length} chars`);
  return { handle, text };
}
document.querySelector('#open').onclick = openText;
read notes.md: 1280 chars

Example 3b: write back to the same file

browser console
async function save(handle, newText) {
  const writable = await handle.createWritable();
  try {
    await writable.write(newText);   // overwrites file contents
  } finally {
    await writable.close();          // ALWAYS close to flush to disk
  }
  console.log('saved', newText.length, 'chars');
}
// reuse the handle from Example 3a — no second dialog needed
// await save(handle, text.toUpperCase());
saved 1280 chars

Example 3c: batch-process a directory

browser console
async function countJson() {
  const dir = await window.showDirectoryPicker();
  let total = 0;

  // async iteration over the directory's entries
  for await (const [name, handle] of dir.entries()) {
    if (handle.kind === 'file' && name.endsWith('.json')) {
      const file = await handle.getFile();
      const data = JSON.parse(await file.text());
      total += Array.isArray(data) ? data.length : 1;
    }
  }
  console.log('total records across .json files:', total);
}
total records across .json files: 347
Common Mistakes
❌ Calling showOpenFilePicker() outside a user gesture (e.g. on page load or in a setTimeout) — the browser silently blocks it.
✅ Always invoke it from a click (or other user-initiated event) handler.
❌ Forgetting to await writable.close() — data may never reach disk and the file can be left locked.
✅ Wrap the write in try/finally and always close() in the finally.
Key Rules
  • Every picker call needs a user gesture and a secure (HTTPS) context.
  • getFile() for reading, createWritable() + close() for writing.
  • Persist handles in IndexedDB to re-open recent files (re-prompting permission as needed) — you can't persist a raw OS path.
Practice Exercises
1Build a minimal text editor: an “Open” button loads a file into a <textarea>, and a “Save” button writes the edited text back to the same file handle.
2Write a converter: open a .csv file, parse it into an array of objects, then use showSaveFilePicker() to write it out as .json.
3Add a “recent files” list by storing each opened FileSystemFileHandle in IndexedDB and re-opening it on click in a later session.
Connections: Reading via file.text() returns a Promise (R2/R3 async), and the directory walk in 3c uses for await...of from async generators. Handles are structured-cloneable (Day 7/8) which is exactly why they can live in IndexedDB.
2 Security & Communication
4. Web Crypto API new
crypto.subtle provides real, audited cryptographic primitives in the browser — hashing, encryption, signing, and key generation — with no third-party library. It's the foundation for any client-side security (tokens, integrity checks, end-to-end encryption).

There are two halves to the API:

  1. crypto.getRandomValues(typedArray) — fills a typed array with cryptographically secure random bytes. Synchronous. Use it for tokens, nonces, and IVs.
  2. crypto.subtle.* — the async, Promise-returning operations that work on ArrayBuffers:
    • 2026-07-09 (Day 28)
    • digest('SHA-256', data) — hashing.
    • generateKey(algo, extractable, usages) — key generation.
    • encrypt/decrypt — symmetric encryption (AES-GCM).
    • sign/verify — HMAC and RSA/ECDSA signatures.

Strings must be turned into bytes first with new TextEncoder().encode(str), and results come back as ArrayBuffers you typically convert to hex or base64.

Example 4a: SHA-256 hash as a hex string

browser console
async function sha256Hex(message) {
  const bytes = new TextEncoder().encode(message);      // string -> Uint8Array
  const digest = await crypto.subtle.digest('SHA-256', bytes);
  // ArrayBuffer -> array of bytes -> 2-char hex each
  return [...new Uint8Array(digest)]
    .map(b => b.toString(16).padStart(2, '0'))
    .join('');
}
console.log(await sha256Hex('hello'));
2cf24dba5fb0a30e26e83b2ac5b9e29e1b161e5c1fa7425e73043362938b9824

Example 4b: secure random token

browser console
function randomToken(bytes = 16) {
  const buf = new Uint8Array(bytes);
  crypto.getRandomValues(buf);    // cryptographically secure — NOT Math.random
  return [...buf].map(b => b.toString(16).padStart(2, '0')).join('');
}
console.log(randomToken());   // 32 hex chars
9f2c41ab7e0d8835c1a6f4b29e7d0a13

Example 4c: AES-GCM encrypt + decrypt

browser console
async function demo() {
  const enc = new TextEncoder();
  const dec = new TextDecoder();

  // 1) generate a symmetric key
  const key = await crypto.subtle.generateKey(
    { name: 'AES-GCM', length: 256 }, true, ['encrypt', 'decrypt']
  );

  // 2) a FRESH random IV per encryption (never reuse one!)
  const iv = crypto.getRandomValues(new Uint8Array(12));

  // 3) encrypt
  const ciphertext = await crypto.subtle.encrypt(
    { name: 'AES-GCM', iv }, key, enc.encode('top secret')
  );

  // 4) decrypt with the SAME key + iv
  const plain = await crypto.subtle.decrypt(
    { name: 'AES-GCM', iv }, key, ciphertext
  );
  console.log(dec.decode(plain));
}
demo();
top secret

Example 4d: HMAC sign & verify

browser console
async function hmacDemo() {
  const enc = new TextEncoder();
  const key = await crypto.subtle.generateKey(
    { name: 'HMAC', hash: 'SHA-256' }, false, ['sign', 'verify']
  );

  const data = enc.encode('msg-123');
  const sig = await crypto.subtle.sign('HMAC', key, data);

  // verify returns a boolean Promise
  const ok = await crypto.subtle.verify('HMAC', key, sig, data);
  const tampered = await crypto.subtle.verify('HMAC', key, sig, enc.encode('msg-999'));
  console.log('valid:', ok, '| tampered:', tampered);
}
hmacDemo();
valid: true | tampered: false
Common Mistakes
❌ Using Math.random() for tokens, session IDs, or IVs — it is not cryptographically secure and is predictable.
✅ Use crypto.getRandomValues() for anything security-sensitive.
❌ Reusing the same IV for multiple AES-GCM encryptions with one key — this is catastrophic and can leak the plaintext.
✅ Generate a fresh random IV for every single encryption and store it alongside the ciphertext (the IV is not secret).
Key Rules
  • Everything under crypto.subtle is async and works on ArrayBuffer — encode strings first with TextEncoder.
  • getRandomValues is the only acceptable randomness source for security values.
  • AES-GCM: one fresh IV per message; the IV travels with the ciphertext, the key does not.
Practice Exercises
1Write hashPassword(pw) that returns the SHA-256 hash of a string as a lowercase hex string. Expected: hashPassword('abc') → a 64-char hex string.
2Build an encrypt/decrypt round-trip: derive an AES-GCM key from a password (research crypto.subtle.importKey + deriveKey with PBKDF2), encrypt a message, then decrypt it back to the original.
3Write a file integrity checker: read a file's bytes, compute its SHA-256, and compare against a known expected hash, returning true/false.
Connections: Every operation returns a Promise (R2/R3), and the data is held in typed arrays / ArrayBuffers (Day 7/8 Data & Collections). Pairs with the File System Access API above for the integrity-checker exercise, and underpins secure payloads sent over Broadcast Channel or Workers.
5. Navigation API new
The Navigation API is the modern replacement for the clunky History API in single-page apps. It gives you a single navigate event you can intercept, typed entries, and built-in scroll restoration — the foundation a real client-side router is built on.

Everything hangs off the global navigation object:

  1. navigation.navigate(url, { state }) — programmatic navigation.
  2. navigation.addEventListener('navigate', handler) — fires for every navigation: link clicks, back/forward, and programmatic calls.
  3. e.intercept({ handler: async () => {...} }) — take over the navigation and load content asynchronously while the URL updates correctly.
  4. navigation.currentEntry — the current URL plus any state; navigation.entries() is the full session history as an array.
  5. navigatesuccess / navigateerror events let you react to completion or failure.

Not every navigation can be intercepted (cross-origin ones can't), so always feature-check e.canIntercept first.

Example 5a: minimal SPA router

browser console
navigation.addEventListener('navigate', (e) => {
  // Skip cross-origin and in-page hash changes
  if (!e.canIntercept || e.hashChange) return;

  const url = new URL(e.destination.url);

  e.intercept({
    async handler() {
      // load the route's content (e.g. fetch a partial)
      const html = await fetch(`/partials${url.pathname}.html`).then(r => r.text());
      document.querySelector('#app').innerHTML = html;
      console.log('navigated to', url.pathname);
    },
  });
});

navigation.navigate('/about');   // triggers the handler above
navigated to /about

Example 5b: success & error handling

browser console
navigation.addEventListener('navigatesuccess', () => {
  console.log('rendered:', location.pathname);
});

// If the intercept handler throws, navigateerror fires
navigation.addEventListener('navigateerror', (e) => {
  console.error('navigation failed:', e.message);
  document.querySelector('#app').innerHTML = '<p>Something went wrong.</p>';
});
rendered: /about

Example 5c: per-entry state & reading history

browser console
// Attach state to the current entry (survives back/forward)
navigation.navigate('/feed', { state: { scrollY: 0 } });

// Read it back later (e.g. on a back navigation)
const { scrollY } = navigation.currentEntry.getState() ?? { scrollY: 0 };
console.log('restoring scroll to', scrollY);

// Inspect the full session history
console.log('history length:', navigation.entries().length);
restoring scroll to 0 history length: 2

Example 5d: feature-detect, fall back to History API

function go(url, state = {}) {
  if ('navigation' in window) {
    navigation.navigate(url, { state });   // modern path
  } else {
    history.pushState(state, '', url);     // fallback
    window.dispatchEvent(new PopStateEvent('popstate', { state }));
  }
}

Common Mistakes
❌ Calling e.intercept() without checking e.canIntercept — it throws for cross-origin navigations and breaks the whole router.
✅ Guard with if (e.canIntercept && !e.hashChange) before intercepting.
❌ Ignoring the navigateerror event — a failed intercept handler leaves the user staring at a half-rendered page.
✅ Listen for navigateerror and render a fallback/error UI.
Key Rules
  • One navigate listener catches link clicks, back/forward, and programmatic navigation — far simpler than juggling popstate + click handlers.
  • Always feature-check e.canIntercept (and skip e.hashChange) before intercepting.
  • intercept({ handler }) accepts an async handler — the URL stays correct while content loads.
Practice Exercises
1Build a minimal SPA router that intercepts link clicks and loads content via fetch based on the URL path, updating a #app container.
2Add scroll restoration: store the current window.scrollY in the entry's state before leaving, and restore it on a back navigation.
3Implement a route guard that prompts “You have unsaved changes — leave anyway?” and cancels the navigation (don't call intercept, or reject inside the handler) when a form is dirty.
Connections: The intercept handler is async (R3 async/await) and typically fetches a partial (Fetch Advanced). It supersedes the older History API patterns from earlier weeks, and entry state is structured-cloned (Day 7/8). Pairs with the Web Animations API above for animated route transitions.
