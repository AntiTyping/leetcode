List, Dict & Set Comprehensions

Comprehensions • 8 days since last review • rep 2

Quick Reminder

[expr for x in iter if cond] for lists, {k: v for ...} for dicts, {expr for ...} for sets. The walrus operator := can capture intermediate values inside comprehensions to avoid computing them twice.

Key Pattern — Nested Comprehension with Condition

# Flatten a matrix, keeping only even numbers
matrix = [[1, 2, 3], [4, 5, 6], [7, 8, 9]]
evens = [val for row in matrix for val in row if val % 2 == 0]
print(evens)  # [2, 4, 6, 8]

# Dict comprehension: word lengths (only long words)
words = ["hello", "hi", "magnificent", "ok", "extraordinary"]
long_words = {w: len(w) for w in words if len(w) > 5}
print(long_words)  # {'magnificent': 11, 'extraordinary': 13}

# Walrus operator to avoid recomputing
import math
values = [2, -1, 16, 0, 81]
roots = {v: r for v in values if (r := math.sqrt(v)) == int(r) and v > 0}
print(roots)  # {16: 4.0, 81: 9.0}
Challenge

Given a list of dicts [{"name": "Alice", "scores": [90, 85]}, {"name": "Bob", "scores": [70, 80, 90]}], create a dict mapping names to their average score using a dict comprehension.

#1 Pitfall: Nested comprehensions read left-to-right (outer loop first). [val for row in matrix for val in row] is equivalent to the nested loop for row in matrix: for val in row:. Reversing the order gives a NameError.

Type Hints — Basic Annotations

Type System • 4 days since last review • rep 1

Quick Reminder

def foo(x: int) -> str:, list[int], dict[str, int], tuple[int, ...] (Python 3.9+ lowercase generics). Type hints are not enforced at runtime — they are for static analysis tools like mypy.

Key Pattern — Annotating Complex Return Types

def parse_scores(raw: list[str]) -> dict[str, list[int]]:
    """Parse 'name:score,score' lines into a structured dict."""
    result: dict[str, list[int]] = {}
    for line in raw:
        name, scores_str = line.split(":")
        result[name] = [int(s) for s in scores_str.split(",")]
    return result

data = ["Alice:90,85,92", "Bob:78,88"]
print(parse_scores(data))
# {'Alice': [90, 85, 92], 'Bob': [78, 88]}
Challenge

Annotate a function that takes a list of tuples (name, score) and returns a dict mapping names to their highest score. Use list[tuple[str, int]] for the input and dict[str, int] for the output.

#1 Pitfall: Using List from typing instead of list in Python 3.9+. The old typing.List, typing.Dict, typing.Tuple imports are unnecessary. Use lowercase builtins: list[int], dict[str, int], tuple[int, ...].

Enum

Data Modeling • 4 days since last review • rep 1

Quick Reminder

class Color(Enum): RED = 1 — use IntEnum for int comparison, Flag for bitwise operations, auto() for auto-assigned values. Access members by name (Color["RED"]) or value (Color(1)).

Key Pattern — Pattern Matching with Enum

from enum import Enum, Flag, auto

class Direction(Enum):
    NORTH = auto()
    SOUTH = auto()
    EAST = auto()
    WEST = auto()

def describe(d: Direction) -> str:
    match d:
        case Direction.NORTH: return "Going up"
        case Direction.SOUTH: return "Going down"
        case Direction.EAST:  return "Going right"
        case Direction.WEST:  return "Going left"

print(describe(Direction.NORTH))  # Going up
print(describe(Direction.EAST))   # Going right

# Iterating all members
for d in Direction:
    print(f"{d.name} = {d.value}", end="  ")
# NORTH = 1  SOUTH = 2  EAST = 3  WEST = 4
Challenge

Create a Permission Flag enum (READ=1, WRITE=2, EXECUTE=4) and write a function checking if a combined permission includes WRITE.

#1 Pitfall: Enum members are not ints by default. Color.RED == 1 is False with plain Enum. Use IntEnum if you need integer comparison. With Flag, use Permission.WRITE in combined to test membership.

Dataclasses

Data Modeling • 4 days since last review • rep 1

Quick Reminder

@dataclass auto-generates __init__, __repr__, __eq__. Use field(default_factory=list) for mutable defaults. frozen=True makes instances immutable. order=True adds comparison methods.

Key Pattern — Computed Field with __post_init__

from dataclasses import dataclass, field

@dataclass(order=True)
class BankAccount:
    sort_index: float = field(init=False, repr=False)
    name: str
    balance: float = 0.0
    transactions: list[str] = field(default_factory=list, repr=False)

    def __post_init__(self):
        self.sort_index = self.balance

    @property
    def is_overdrawn(self) -> bool:
        return self.balance < 0

a1 = BankAccount("Alice", 500.0)
a2 = BankAccount("Bob", -50.0)
print(a1)              # BankAccount(name='Alice', balance=500.0)
print(a2.is_overdrawn) # True
print(a1 > a2)        # True (compares by sort_index = balance)
Challenge

Create a BankAccount dataclass with name, balance, and a computed is_overdrawn property. Make it support comparison by balance using order=True and a sort_index field.

#1 Pitfall: Mutable default arguments. tags: list[str] = [] in a dataclass shares one list across all instances. Always use field(default_factory=list) instead of field(default=[]).

Classes — __init__, __repr__, __str__

OOP • 4 days since last review • rep 1

Quick Reminder

__init__ initializes the instance. __repr__ returns an unambiguous string for debugging (ideally valid Python to reconstruct the object). __str__ returns a human-readable string. repr() falls back to __repr__ if no __str__ is defined.

Key Pattern — __repr__ That Produces Valid Python

class Money:
    def __init__(self, amount: float, currency: str = "USD"):
        self.amount = amount
        self.currency = currency

    def __repr__(self) -> str:
        return f"Money({self.amount!r}, {self.currency!r})"

    def __str__(self) -> str:
        symbols = {"USD": "$", "EUR": "€", "GBP": "£"}
        sym = symbols.get(self.currency, self.currency + " ")
        return f"{sym}{self.amount:.2f} {self.currency}"

m = Money(42.5, "USD")
print(repr(m))  # Money(42.5, 'USD')
print(str(m))   # $42.50 USD
print(m)        # $42.50 USD (print calls __str__)

# repr is reconstructable
m2 = eval(repr(m))
print(m2)  # $42.50 USD
Challenge

Create a Money class with amount and currency that has proper __repr__ (reconstructable) and __str__ (human-readable like "$42.50 USD"). Verify that eval(repr(m)) creates an equivalent object.

#1 Pitfall: Not implementing __repr__. Without it, you get useless <Foo object at 0x...>. Every class you write should have a __repr__. Use !r in f-strings for attributes to get proper quoting: f"Foo({self.name!r})".

Context Managers

Error Handling & I/O • 4 days since last review • rep 0

Quick Reminder

The with statement calls __enter__/__exit__. Use @contextmanager from contextlib for generator-based context managers. The code before yield is the setup, the code after is teardown. Cleanup happens even on exceptions.

Key Pattern — @contextmanager with try/finally

from contextlib import contextmanager
import time

@contextmanager
def timer(label: str = "Block"):
    start = time.perf_counter()
    try:
        yield
    finally:
        elapsed = time.perf_counter() - start
        print(f"{label} took {elapsed:.4f}s")

# Usage
with timer("Sorting"):
    data = sorted(range(100_000, 0, -1))
# Sorting took 0.0045s  (time varies)

# Class-based equivalent
class Timer:
    def __init__(self, label: str = "Block"):
        self.label = label

    def __enter__(self):
        self.start = time.perf_counter()
        return self

    def __exit__(self, exc_type, exc_val, exc_tb):
        elapsed = time.perf_counter() - self.start
        print(f"{self.label} took {elapsed:.4f}s")
        return False  # don't suppress exceptions

with Timer("Summing") as t:
    total = sum(range(1_000_000))
# Summing took 0.0123s  (time varies)
Challenge

Write a context manager timer() that measures and prints the elapsed time of the with block. Implement it both as a class with __enter__/__exit__ and as a generator with @contextmanager.

#1 Pitfall: Forgetting to yield in @contextmanager, or yielding more than once. Exactly one yield is required. Without yield, the body of the with block never executes. Put yield inside try and cleanup in finally.

Virtual Environments & pyproject.toml

Testing & Packaging

Why This Matters

Every Python project needs isolated dependencies to avoid version conflicts. pyproject.toml is the modern standard (PEP 621) replacing setup.py/setup.cfg for project metadata, dependencies, and build configuration.

Creating and Using Virtual Environments

# Create a virtual environment

# Activate it (macOS/Linux)

# Activate it (Windows)
# .venv\Scripts\activate

# Verify isolation
import sys

# Deactivate when done

Design Patterns in Python

Testing & Packaging

Why This Matters

Classic design patterns (Singleton, Factory, Strategy, Observer) have much simpler Pythonic implementations thanks to first-class functions, decorators, and dunder methods. Knowing when not to use a pattern is as important as knowing the pattern itself.

Strategy Pattern — Just Pass a Function

# In Java, you'd need a Strategy interface + concrete classes.
# In Python, just pass a function.

def format_upper(text: str) -> str:
    return text.upper()

def format_title(text: str) -> str:
    return text.title()

def format_snake(text: str) -> str:
    return text.lower().replace(" ", "_")

# Strategy dispatch via dict
strategies = {
    "upper": format_upper,
    "title": format_title,
    "snake": format_snake,
}

def apply_format(text: str, style: str) -> str:
    formatter = strategies.get(style)
    if formatter is None:
        raise ValueError(f"Unknown style: {style}")
    return formatter(text)

print(apply_format("hello world", "upper"))  # HELLO WORLD
print(apply_format("hello world", "title"))  # Hello World
print(apply_format("hello world", "snake"))  # hello_world

Factory Pattern — __init_subclass__ Registry

class Serializer:
    _registry: dict[str, type] = {}

    def __init_subclass__(cls, fmt: str = "", **kwargs):
        super().__init_subclass__(**kwargs)
        if fmt:
            Serializer._registry[fmt] = cls

    @classmethod
    def create(cls, fmt: str) -> "Serializer":
        klass = cls._registry.get(fmt)
        if klass is None:
            raise ValueError(f"Unknown format: {fmt}")
        return klass()

    def serialize(self, data: dict) -> str:
        raise NotImplementedError

class JSONSerializer(Serializer, fmt="json"):
    def serialize(self, data: dict) -> str:
        import json
        return json.dumps(data)

class CSVSerializer(Serializer, fmt="csv"):
    def serialize(self, data: dict) -> str:
        return ",".join(f"{k}={v}" for k, v in data.items())

s = Serializer.create("json")
print(s.serialize({"name": "Alice", "age": 30}))
# {"name": "Alice", "age": 30}

s2 = Serializer.create("csv")
print(s2.serialize({"x": 1, "y": 2}))
# x=1,y=2

print(Serializer._registry)
# {'json': <class 'JSONSerializer'>, 'csv': <class 'CSVSerializer'>}

Observer Pattern — Simple Callbacks

class EventEmitter:
    def __init__(self):
        self._listeners: dict[str, list] = {}

    def on(self, event: str, callback):
        self._listeners.setdefault(event, []).append(callback)

    def emit(self, event: str, *args, **kwargs):
        for cb in self._listeners.get(event, []):
            cb(*args, **kwargs)

emitter = EventEmitter()
emitter.on("login", lambda user: print(f"{user} logged in"))
emitter.on("login", lambda user: print(f"Audit: {user}"))
emitter.emit("login", "Alice")
# Alice logged in
# Audit: Alice

Singleton — A Module IS a Singleton

# config.py (this module IS the singleton)
_settings = {"debug": False, "db_url": "sqlite:///app.db"}

def get(key: str):
    return _settings.get(key)

def set(key: str, value):
    _settings[key] = value

# Any module that imports config gets the same _settings dict.
# No Singleton class needed.
Pythonic
# Strategy: pass a function
sorted(data, key=len)

# Singleton: module variable
# config.py with module-level dict
Over-Engineered
# Java-style Strategy interface
class SortStrategy(ABC):
    @abstractmethod
    def compare(self, a, b): ...

# Singleton with __new__
class Config:
    _instance = None
    def __new__(cls): ...
  • Python's first-class functions replace Strategy, Command, and Template Method patterns.
  • A module IS a singleton. Don't create a Singleton class unless you have a compelling reason.
  • Use __init_subclass__ for registration patterns (Factory, Plugin systems).
  • Use dict dispatch instead of if/elif chains for selecting behavior.

Metaprogramming: __init_subclass__ & __class_getitem__

Advanced Topics

Why This Matters

These hooks let you customize class creation without full metaclasses — 90% of the power, 10% of the complexity. __init_subclass__ runs when a class is subclassed. __class_getitem__ lets you subscript classes like MyClass[int]. __set_name__ gives descriptors automatic attribute names.

__init_subclass__ — Auto-Registration

class Animal:
    _registry: dict[str, type] = {}

    def __init_subclass__(cls, sound: str = "", **kwargs):
        super().__init_subclass__(**kwargs)
        if sound:
            cls.sound = sound
            Animal._registry[cls.__name__.lower()] = cls

    @classmethod
    def make(cls, name: str) -> "Animal":
        return cls._registry[name]()

    def speak(self) -> str:
        return f"{type(self).__name__} says {self.sound}"

class Dog(Animal, sound="woof"): pass
class Cat(Animal, sound="meow"): pass
class Duck(Animal, sound="quack"): pass

print(Animal._registry)
# {'dog': <class 'Dog'>, 'cat': <class 'Cat'>, 'duck': <class 'Duck'>}

animal = Animal.make("cat")
print(animal.speak())  # Cat says meow

__init_subclass__ — Validation

class Serializable:
    def __init_subclass__(cls, **kwargs):
        super().__init_subclass__(**kwargs)
        # Skip abstract base itself
        if not hasattr(cls, 'serialize'):
            raise TypeError(
                f"{cls.__name__} must define a serialize() method"
            )

class JSONDoc(Serializable):
    def serialize(self) -> str:
        return "{}"

# This would raise TypeError at class definition time:
# class BadDoc(Serializable):
#     pass
# TypeError: BadDoc must define a serialize() method

print(JSONDoc().serialize())  # {}

__class_getitem__ — Generic-Looking Classes

class TypedBox:
    def __init__(self, value):
        self.value = value

    def __class_getitem__(cls, item):
        # Called when you write TypedBox[int]
        # Returns a parameterized alias for type-checking
        return type(f"{cls.__name__}[{item.__name__}]", (cls,), {
            "_type": item
        })

    def __repr__(self):
        return f"TypedBox({self.value!r})"

IntBox = TypedBox[int]
box = IntBox(42)
print(box)            # TypedBox(42)
print(IntBox._type)   # <class 'int'>
print(type(box).__name__)  # TypedBox[int]

__set_name__ — Descriptor Gets Its Attribute Name

class TypeChecked:
    def __init__(self, expected_type):
        self.expected_type = expected_type

    def __set_name__(self, owner, name):
        self.attr_name = f"_{name}"
        self.public_name = name

    def __get__(self, obj, objtype=None):
        if obj is None:
            return self
        return getattr(obj, self.attr_name, None)

    def __set__(self, obj, value):
        if not isinstance(value, self.expected_type):
            raise TypeError(
                f"{self.public_name} must be {self.expected_type.__name__}, "
                f"got {type(value).__name__}"
            )
        setattr(obj, self.attr_name, value)

class User:
    name = TypeChecked(str)
    age = TypeChecked(int)

    def __init__(self, name: str, age: int):
        self.name = name
        self.age = age

u = User("Alice", 30)
print(u.name, u.age)  # Alice 30

try:
    u.age = "not a number"
except TypeError as e:
    print(e)  # age must be int, got str
Do
# Use __init_subclass__ for registration
class Plugin:
    def __init_subclass__(cls, **kw):
        registry[cls.__name__] = cls
Don't
# Use a full metaclass for this
class PluginMeta(type):
    def __new__(mcs, name, bases, ns):
        cls = super().__new__(...)
        registry[name] = cls
        return cls
  • Prefer __init_subclass__ over metaclasses — simpler and covers 90% of use cases.
  • __class_getitem__ is primarily for type hints, not runtime behavior.
  • __set_name__ completes the descriptor protocol — no need to repeat the attribute name.
  • Always call super().__init_subclass__(**kwargs) to support cooperative inheritance.

Import System: __import__, importlib, sys.path

Advanced Topics

Why This Matters

Understanding Python's import system helps debug "module not found" errors, implement plugin systems, and control module loading. Imports follow a well-defined chain: finder → loader → module cache (sys.modules).

How sys.path Works

import sys

# sys.path is a list of directories Python searches for modules
print(sys.path[:4])
# ['', '/usr/lib/python311.zip', '/usr/lib/python3.11', ...]
# '' means the current directory

# The search order:
# 1. sys.modules cache (already imported?)
# 2. Built-in modules (sys, os, etc.)
# 3. Directories in sys.path (in order)

Dynamic Imports with importlib

import importlib

# Import a module by name (string)
json_mod = importlib.import_module("json")
data = json_mod.dumps({"key": "value"})
print(data)  # {"key": "value"}

# Import a submodule
path_mod = importlib.import_module("os.path")
print(path_mod.join("/usr", "local", "bin"))
# /usr/local/bin

# Useful for plugin systems where module name is determined at runtime
module_name = "collections"
mod = importlib.import_module(module_name)
print(mod.Counter("hello"))
# Counter({'l': 2, 'h': 1, 'e': 1, 'o': 1})

Module Cache — sys.modules

import sys

# Every imported module is cached in sys.modules
import json
print("json" in sys.modules)  # True

# Importing again returns the cached version (fast!)
import json as j2
print(json is j2)  # True (same object)

# You can inspect the cache
print(len(sys.modules))  # hundreds of modules loaded at startup

Controlling Exports with __all__

# mymodule.py
__all__ = ["public_func", "PublicClass"]

def public_func():
    return "I'm public"

def _private_func():
    return "I'm private by convention"

class PublicClass:
    pass

class _InternalClass:
    pass

# When someone does: from mymodule import *
# Only public_func and PublicClass are imported.
# __all__ explicitly declares the public API.

Circular Imports — The Fix

# Problem: a.py imports b.py, b.py imports a.py
# This causes ImportError or partially initialized modules.

# FIX 1: Move the import inside the function (lazy import)
# b.py
def process():
    from a import helper  # imported only when called
    return helper()

# FIX 2: Restructure into a third module
# common.py -- shared code that both a.py and b.py need

# FIX 3: Import the module, not the name
# Instead of: from a import helper
# Use: import a
# Then: a.helper()  (defers name lookup to runtime)
Do
# Absolute imports
from mypackage.utils import helper
# importlib for dynamic loading
importlib.import_module(name)
Don't
# sys.path hacks at runtime
sys.path.insert(0, "/random/path")
# Shadowing stdlib modules
# json.py in your project!
  • Use absolute imports. Relative imports are acceptable within a package but can confuse readers.
  • Fix circular imports by restructuring, not with sys.path hacks.
  • Use importlib.import_module() for dynamic loading (plugin systems).
  • Define __all__ in every public module to control the from module import * API.

Walrus Operator & Structural Pattern Matching

Advanced Topics

Why This Matters

The walrus operator (:=, Python 3.8+) assigns and returns a value in one expression, eliminating repeated computation. Pattern matching (match/case, Python 3.10+) replaces complex if/elif chains with declarative, readable patterns.

Walrus Operator in Action

# In a while loop -- avoid calling twice
import re

text = "Error: file not found at line 42"
if (match := re.search(r"line (\d+)", text)) is not None:
    print(f"Found line number: {match.group(1)}")
else:

# Sequence patterns (lists/tuples)
def process_command(command: list[str]) -> str:
    match command:
        case ["quit"]:
            return "Goodbye!"
        case ["greet", name]:
            return f"Hello, {name}!"
        case ["move", x, y]:
            return f"Moving to ({x}, {y})"
        case ["add", *items]:
            return f"Adding {len(items)} items: {items}"
        case _:
            return f"Unknown command: {command}"

print(process_command(["quit"]))          # Goodbye!
print(process_command(["greet", "Alice"]))  # Hello, Alice!
print(process_command(["move", "3", "5"]))  # Moving to (3, 5)
print(process_command(["add", "a", "b", "c"]))  # Adding 3 items: ['a', 'b', 'c']

Pattern Matching — Mappings and Guards

# Mapping patterns (dicts)
def handle_event(event: dict) -> str:
    match event:
        case {"type": "click", "x": x, "y": y}:
            return f"Clicked at ({x}, {y})"
        case {"type": "keypress", "key": key} if len(key) == 1:
            return f"Pressed key: {key}"
        case {"type": "keypress", "key": key}:
            return f"Special key: {key}"
        case {"type": t}:
            return f"Unhandled event type: {t}"
        case _:
            return "Invalid event"

print(handle_event({"type": "click", "x": 10, "y": 20}))
# Clicked at (10, 20)
print(handle_event({"type": "keypress", "key": "a"}))
# Pressed key: a
print(handle_event({"type": "keypress", "key": "Enter"}))
# Special key: Enter

from dataclasses import dataclass

@dataclass
class Point:
    x: float
    y: float

def classify_point(p: Point) -> str:
    match p:
        case Point(x=0, y=0):
            return "Origin"
        case Point(x=0, y=y):
            return f"On Y-axis at y={y}"
        case Point(x=x, y=0):
            return f"On X-axis at x={x}"
        case Point(x=x, y=y) if x > 0 and y > 0:
            return f"Quadrant I: ({x}, {y})"
        case _:
            return f"Other: ({p.x}, {p.y})"

print(classify_point(Point(0, 0)))    # Origin
print(classify_point(Point(0, 5)))    # On Y-axis at y=5
print(classify_point(Point(3, 0)))    # On X-axis at x=3
print(classify_point(Point(2, 7)))    # Quadrant I: (2, 7)
print(classify_point(Point(-1, 3)))   # Other: (-1, 3)
