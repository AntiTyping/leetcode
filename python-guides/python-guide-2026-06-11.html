super() Deep Dive

OOP • 6 days overdue • rep 1

Quick Reminder

super() follows the Method Resolution Order (MRO), not just the immediate parent class. In diamond inheritance, super().__init__() calls the next class in the MRO chain, ensuring each class's __init__ runs exactly once. Always forward **kwargs in cooperative multiple inheritance so each class can consume its own arguments.

Key Pattern — Cooperative Multiple Inheritance

class Base:
    def __init__(self, **kwargs):
        print("Base.__init__")

class Left(Base):
    def __init__(self, left_val=0, **kwargs):
        self.left_val = left_val
        print(f"Left.__init__(left_val={left_val})")
        super().__init__(**kwargs)

class Right(Base):
    def __init__(self, right_val=0, **kwargs):
        self.right_val = right_val
        print(f"Right.__init__(right_val={right_val})")
        super().__init__(**kwargs)

class Diamond(Left, Right):
    def __init__(self, **kwargs):
        print("Diamond.__init__")
        super().__init__(**kwargs)

d = Diamond(left_val=1, right_val=2)
print(Diamond.__mro__)
Diamond.__init__ Left.__init__(left_val=1) Right.__init__(right_val=2) Base.__init__ (<class 'Diamond'>, <class 'Left'>, <class 'Right'>, <class 'Base'>, <class 'object'>)
Challenge

Create a diamond with Loggable and Serializable mixins that both inherit from Base. The final class LoggableSerializableModel should call all four __init__ methods exactly once. Print the MRO to verify.

#1 Pitfall: Mixing super().__init__() with explicit ParentClass.__init__(self) in a diamond breaks cooperative inheritance — the shared base may get called twice or not at all. Pick one pattern and be consistent.

Abstract Base Classes & Protocol

OOP • 6 days overdue • rep 1

Quick Reminder

ABC enforces nominal typing — subclasses must inherit from the ABC and implement all @abstractmethods. Protocol (from typing) enforces structural typing: any class with the right methods satisfies the protocol, no inheritance required. Use @runtime_checkable to allow isinstance() checks with Protocols.

Key Pattern — Protocol for Structural Typing

from typing import Protocol, runtime_checkable

@runtime_checkable
class Drawable(Protocol):
    def draw(self) -> str: ...

class Circle:
    def draw(self) -> str:
        return "Drawing circle"

class Square:
    def draw(self) -> str:
        return "Drawing square"

def render(shape: Drawable) -> None:
    print(shape.draw())

render(Circle())    # Works: Circle has draw()
render(Square())    # Works: Square has draw()
print(isinstance(Circle(), Drawable))  # True at runtime
Drawing circle Drawing square True
Challenge

Define a Cacheable protocol with cache_key(self) -> str and ttl(self) -> int methods. Create two unrelated classes (UserProfile and SearchResult) that satisfy the protocol without inheriting from it. Write a function that accepts Cacheable and prints the cache key.

#1 Pitfall: @runtime_checkable Protocol only checks method existence, not signatures. A class with def draw(self, x, y) still passes isinstance(obj, Drawable) even though calling draw() with no args would fail.

Dunder: __getitem__, __len__, __contains__

OOP • 6 days overdue • rep 1

Quick Reminder

Implementing __getitem__ makes objects indexable and iterable (Python falls back to indexing with 0, 1, 2... if __iter__ is missing). __len__ enables len(obj) and truthiness testing. __contains__ enables the in operator — without it, Python falls back to iterating through __iter__ or __getitem__.

Key Pattern — Custom Container with Slicing

class NamedList:
    def __init__(self, name, items):
        self.name = name
        self._items = list(items)

    def __getitem__(self, index):
        if isinstance(index, slice):
            return NamedList(self.name, self._items[index])
        return self._items[index]

    def __len__(self):
        return len(self._items)

    def __contains__(self, item):
        return item in self._items

    def __repr__(self):
        return f"NamedList({self.name!r}, {self._items})"

nl = NamedList("fruits", ["apple", "banana", "cherry", "date"])
print(nl[1])          # banana
print(nl[1:3])        # NamedList('fruits', ['banana', 'cherry'])
print(len(nl))        # 4
print("cherry" in nl) # True
banana NamedList('fruits', ['banana', 'cherry']) 4 True
Challenge

Create a Matrix class that supports m[row, col] indexing (using tuple unpacking inside __getitem__), len(m) returning total element count, and value in m for element search.

#1 Pitfall: Forgetting to handle slice objects in __getitem__. When a user writes obj[1:3], Python passes a slice(1, 3, None) object — you must check isinstance(index, slice) and delegate to the underlying data's slicing.

Advanced typing

Testing • 6 days overdue • rep 0

Quick Reminder

TypeVar creates generic type variables. Generic[T] makes classes parameterized. ParamSpec captures the parameter signature of callables for decorator typing. @overload provides multiple type signatures for a single function.

Key Pattern — Generic Class with Bound TypeVar

from typing import TypeVar, Generic

class Comparable:
    def __lt__(self, other): ...

T = TypeVar("T", bound=Comparable)

class SortedList(Generic[T]):
    def __init__(self) -> None:
        self._items: list[T] = []

    def add(self, item: T) -> None:
        for i, existing in enumerate(self._items):
            if item < existing:
                self._items.insert(i, item)
                return
        self._items.append(item)

    def __repr__(self) -> str:
        return f"SortedList({self._items})"

sl: SortedList[int] = SortedList()
sl.add(3)
sl.add(1)
sl.add(2)
print(sl)  # SortedList([1, 2, 3])
SortedList([1, 2, 3])

Memory Optimization

Performance • due today • rep 1

Quick Reminder

__slots__ eliminates per-instance __dict__, saving 40-60% memory on data-heavy classes. sys.getsizeof() measures shallow size of an object. Generators process large datasets without loading everything into memory. The array module stores homogeneous numeric data more compactly than lists.

Key Pattern — __slots__ on a Data-Heavy Class

import sys

class PointDict:
    def __init__(self, x, y, z):
        self.x, self.y, self.z = x, y, z

class PointSlots:
    __slots__ = ('x', 'y', 'z')
    def __init__(self, x, y, z):
        self.x, self.y, self.z = x, y, z

pd = PointDict(1, 2, 3)
ps = PointSlots(1, 2, 3)
print(f"Dict-based: {sys.getsizeof(pd)} bytes")
print(f"Slots-based: {sys.getsizeof(ps)} bytes")

# Generator vs list for large data
import tracemalloc
tracemalloc.start()
total = sum(x * x for x in range(1_000_000))  # generator - low memory
print(f"Sum: {total}")
print(f"Peak memory: {tracemalloc.get_traced_memory()[1] / 1024:.0f} KB")
tracemalloc.stop()
Dict-based: 152 bytes Slots-based: 64 bytes Sum: 333332833333500000 Peak memory: ~25 KB
Challenge

Create an array.array('d') of 100,000 floats and a regular list of the same floats. Compare their memory usage with sys.getsizeof(). Then time summing both using timeit.

#1 Pitfall: sys.getsizeof() only measures the object's shallow size — for a list, it shows the array of pointers, not the pointed-to objects. Use tracemalloc or pympler.asizeof() for deep/recursive measurement.

copy Module: Shallow vs Deep Copy

Advanced Standard Library • concept_id: copy_deep_shallow

Python's assignment operator (=) never copies objects — it creates a new reference to the same object. The copy module provides copy() for shallow copies and deepcopy() for recursive deep copies. Understanding when each is needed prevents subtle mutation bugs.

Example 1 — Assignment vs Shallow vs Deep

import copy

original = [[1, 2, 3], [4, 5, 6]]

assigned = original            # same object
shallow  = copy.copy(original) # new outer list, same inner lists
deep     = copy.deepcopy(original)  # everything is new

original[0][0] = 999

print(f"assigned: {assigned}")  # [[999, 2, 3], [4, 5, 6]] - affected!
print(f"shallow:  {shallow}")   # [[999, 2, 3], [4, 5, 6]] - affected!
print(f"deep:     {deep}")      # [[1, 2, 3], [4, 5, 6]]   - safe!
assigned: [[999, 2, 3], [4, 5, 6]] shallow: [[999, 2, 3], [4, 5, 6]] deep: [[1, 2, 3], [4, 5, 6]]

Example 2 — When Shallow Copy Is Sufficient

import copy

# Shallow copy works fine for flat structures with immutable elements
nums = [1, 2, 3, 4, 5]
nums_copy = copy.copy(nums)
nums_copy[0] = 999
print(f"original: {nums}")      # [1, 2, 3, 4, 5] - safe (ints are immutable)
print(f"copy:     {nums_copy}") # [999, 2, 3, 4, 5]

# Also fine for dicts with immutable values
config = {"host": "localhost", "port": 8080}
config2 = copy.copy(config)
config2["port"] = 9090
print(f"original: {config}")    # port still 8080
print(f"copy:     {config2}")   # port is 9090
original: [1, 2, 3, 4, 5] copy: [999, 2, 3, 4, 5] original: {'host': 'localhost', 'port': 8080} copy: {'host': 'localhost', 'port': 9090}

Example 3 — Customizing Copy Behavior

import copy

class GameState:
    def __init__(self, board, score, history):
        self.board = board
        self.score = score
        self.history = history

    def __copy__(self):
        """Shallow copy: share history (append-only), copy board."""
        cls = self.__class__
        result = cls.__new__(cls)
        result.board = [row[:] for row in self.board]  # copy 2D board
        result.score = self.score
        result.history = self.history  # shared reference (intentional)
        return result

    def __deepcopy__(self, memo):
        """Deep copy: everything is independent."""
        cls = self.__class__
        result = cls.__new__(cls)
        memo[id(self)] = result
        result.board = copy.deepcopy(self.board, memo)
        result.score = self.score
        result.history = copy.deepcopy(self.history, memo)
        return result

state = GameState([[0, 0], [0, 0]], 100, ["move1"])
shallow = copy.copy(state)
deep = copy.deepcopy(state)

state.board[0][0] = 1
state.history.append("move2")

print(f"shallow board: {shallow.board}")    # [[0, 0], [0, 0]] - board copied
print(f"shallow history: {shallow.history}")# ["move1", "move2"] - shared!
print(f"deep board: {deep.board}")          # [[0, 0], [0, 0]] - independent
print(f"deep history: {deep.history}")      # ["move1"] - independent
shallow board: [[0, 0], [0, 0]] shallow history: ['move1', 'move2'] deep board: [[0, 0], [0, 0]] deep history: ['move1']

Example 4 — Deepcopy and Circular References

import copy

class Node:
    def __init__(self, value):
        self.value = value
        self.next = None

    def __repr__(self):
        return f"Node({self.value})"

# Create circular reference
a = Node("A")
b = Node("B")
a.next = b
b.next = a  # circular!

# deepcopy handles cycles via the memo dict
a_copy = copy.deepcopy(a)
print(a_copy)                        # Node(A)
print(a_copy.next)                   # Node(B)
print(a_copy.next.next)              # Node(A) - same copy, not original
print(a_copy.next.next is a_copy)    # True - cycle preserved correctly
print(a_copy is a)                   # False - it's a copy
Node(A) Node(B) Node(A) True False

threading: Locks, Events & Queues

Advanced Standard Library • concept_id: threading_patterns

Python's threading module provides OS-level threads. Due to the GIL, CPU-bound work doesn't parallelize (use multiprocessing instead), but I/O-bound work (network, disk, sleep) benefits greatly. The key synchronization primitives are Lock, RLock, Event, Semaphore, Barrier, and queue.Queue.

Example 1 — Lock for Mutual Exclusion

import threading

counter = 0
lock = threading.Lock()

def increment(n):
    global counter
    for _ in range(n):
        with lock:  # acquire and release automatically
            counter += 1

threads = [threading.Thread(target=increment, args=(100_000,)) for _ in range(4)]
for t in threads:
    t.start()
for t in threads:
    t.join()

print(f"Counter: {counter}")  # Always 400000 with lock
Counter: 400000

Example 2 — Event for Thread Signaling

import threading
import time

ready = threading.Event()

def worker(name):
    print(f"{name}: waiting for signal...")
    ready.wait()  # blocks until event is set
    print(f"{name}: received signal, starting work!")

threads = [threading.Thread(target=worker, args=(f"Worker-{i}",)) for i in range(3)]
for t in threads:
    t.start()

time.sleep(1)
print("Main: sending signal to all workers")
ready.set()  # unblocks all waiting threads

for t in threads:
    t.join()
Worker-0: waiting for signal... Worker-1: waiting for signal... Worker-2: waiting for signal... Main: sending signal to all workers Worker-0: received signal, starting work! Worker-1: received signal, starting work! Worker-2: received signal, starting work!

Example 3 — Queue for Producer-Consumer

import threading
import queue
import time

task_queue = queue.Queue(maxsize=5)

def producer(items):
    for item in items:
        task_queue.put(item)  # blocks if queue is full
        print(f"Produced: {item}")
    task_queue.put(None)  # sentinel to signal done

def consumer(name):
    while True:
        item = task_queue.get()  # blocks if queue is empty
        if item is None:
            task_queue.put(None)  # re-add sentinel for other consumers
            break
        print(f"{name} processed: {item}")
        task_queue.task_done()

p = threading.Thread(target=producer, args=(range(8),))
c1 = threading.Thread(target=consumer, args=("Consumer-1",))
c2 = threading.Thread(target=consumer, args=("Consumer-2",))

p.start(); c1.start(); c2.start()
p.join(); c1.join(); c2.join()
print("All done!")
Produced: 0 Consumer-1 processed: 0 Produced: 1 Consumer-2 processed: 1 ... (interleaved output) All done!

functools: cached_property & total_ordering

Advanced Standard Library • concept_id: functools_cached_total

@cached_property computes a value once on first access and caches it as an instance attribute. @total_ordering lets you define just __eq__ and one comparison method (__lt__, __le__, __gt__, or __ge__) and auto-generates the remaining five comparison methods.

Example 1 — cached_property Basics

from functools import cached_property
import time

class DataAnalyzer:
    def __init__(self, data):
        self._data = data

    @cached_property
    def statistics(self):
        """Expensive computation, only runs once."""
        print("Computing statistics...")
        time.sleep(0.5)  # simulate heavy work
        return {
            "mean": sum(self._data) / len(self._data),
            "min": min(self._data),
            "max": max(self._data),
            "count": len(self._data),
        }

analyzer = DataAnalyzer(range(1_000_000))
print(analyzer.statistics)  # "Computing statistics..." then result
print(analyzer.statistics)  # instant - cached!
Computing statistics... {'mean': 499999.5, 'min': 0, 'max': 999999, 'count': 1000000} {'mean': 499999.5, 'min': 0, 'max': 999999, 'count': 1000000}

Example 2 — cached_property vs @property + @lru_cache

from functools import cached_property, lru_cache

class Config:
    def __init__(self, path):
        self.path = path

    # Option 1: cached_property (preferred for instance attributes)
    @cached_property
    def settings(self):
        print(f"Reading {self.path}...")
        return {"debug": True}  # simulate file read

    # Option 2: property + lru_cache (DOES NOT WORK as expected!)
    # @property
    # @lru_cache()  # lru_cache on a method caches per (self,) - memory leak!
    # def settings(self):
    #     ...

c = Config("/etc/app.conf")
print(c.settings)         # reads file
print(c.settings)         # cached

# Invalidate by deleting the cached attribute
del c.settings
print(c.settings)         # reads file again

Example 3 — total_ordering Basics

from functools import total_ordering

@total_ordering
class Version:
    def __init__(self, major, minor, patch):
        self.major = major
        self.minor = minor
        self.patch = patch

    def __eq__(self, other):
        if not isinstance(other, Version):
            return NotImplemented
        return (self.major, self.minor, self.patch) == (other.major, other.minor, other.patch)

    def __lt__(self, other):
        if not isinstance(other, Version):
            return NotImplemented
        return (self.major, self.minor, self.patch) < (other.major, other.minor, other.patch)

    def __repr__(self):
        return f"v{self.major}.{self.minor}.{self.patch}"

v1 = Version(1, 2, 3)
v2 = Version(1, 3, 0)
v3 = Version(1, 2, 3)

print(f"{v1} < {v2}: {v1 < v2}")    # True
print(f"{v1} > {v2}: {v1 > v2}")    # False (auto-generated!)
print(f"{v1} <= {v3}: {v1 <= v3}")  # True (auto-generated!)
print(f"{v1} >= {v2}: {v1 >= v2}")  # False (auto-generated!)
print(f"{v1} == {v3}: {v1 == v3}")  # True

# Now sorting works automatically
versions = [Version(2, 0, 0), Version(1, 0, 1), Version(1, 2, 0)]
print(sorted(versions))
v1.2.3 < v1.3.0: True v1.2.3 > v1.3.0: False v1.2.3 <= v1.2.3: True v1.2.3 >= v1.3.0: False v1.2.3 == v1.2.3: True [v1.0.1, v1.2.0, v2.0.0]

Example 4 — When to Use Manual Comparisons Instead

from functools import total_ordering

# total_ordering has a small performance overhead
# For hot code paths, define all comparisons manually:

class FastPoint:
    __slots__ = ('x', 'y')

    def __init__(self, x, y):
        self.x, self.y = x, y

    def __eq__(self, other):
        return self.x == other.x and self.y == other.y

    def __lt__(self, other):
        return (self.x, self.y) < (other.x, other.y)

    def __le__(self, other):
        return (self.x, self.y) <= (other.x, other.y)

    def __gt__(self, other):
        return (self.x, self.y) > (other.x, other.y)

    def __ge__(self, other):
        return (self.x, self.y) >= (other.x, other.y)

# Use total_ordering for convenience, manual for performance-critical code
import timeit

@total_ordering
class SlowPoint:
    def __init__(self, x, y):
        self.x, self.y = x, y
    def __eq__(self, other):
        return self.x == other.x and self.y == other.y
    def __lt__(self, other):
        return (self.x, self.y) < (other.x, other.y)

a_fast = FastPoint(1, 2); b_fast = FastPoint(3, 4)
a_slow = SlowPoint(1, 2); b_slow = SlowPoint(3, 4)

t_fast = timeit.timeit(lambda: a_fast <= b_fast, number=1_000_000)
t_slow = timeit.timeit(lambda: a_slow <= b_slow, number=1_000_000)
print(f"Manual __le__: {t_fast:.3f}s")
print(f"total_ordering __le__: {t_slow:.3f}s")

dis Module & Bytecode Inspection

Python Internals • concept_id: dis_bytecode

CPython compiles Python source to bytecode, which the virtual machine executes. The dis module disassembles bytecode into human-readable instructions. Understanding bytecode helps you reason about performance, debug mysterious behavior, and appreciate what Python does under the hood.

Example 1 — Basic Disassembly

import dis

def add(a, b):
    return a + b

dis.dis(add)
2 0 LOAD_FAST 0 (a) 2 LOAD_FAST 1 (b) 4 BINARY_ADD 6 RETURN_VALUE

LOAD_FAST loads a local variable onto the stack. BINARY_ADD pops two values, adds them, pushes the result. RETURN_VALUE pops the top of the stack and returns it.

Example 2 — Comparing Loop Approaches

import dis

def loop_range():
    total = 0
    for i in range(100):
        total += i
    return total

def loop_while():
    total = 0
    i = 0
    while i < 100:
        total += i
        i += 1
    return total

print("=== for loop ===")
dis.dis(loop_range)
print("\n=== while loop ===")
dis.dis(loop_while)
=== for loop === ... FOR_ITER (to end) STORE_FAST (i) ... === while loop === ... LOAD_FAST (i) LOAD_CONST (100) COMPARE_OP (<) POP_JUMP_IF_FALSE (to end) ...

The for loop uses the optimized FOR_ITER instruction. The while loop requires explicit COMPARE_OP and POP_JUMP_IF_FALSE on every iteration — more bytecode instructions means slower execution.

Example 3 — Why in set Beats in list

import dis

def check_in_list(x):
    return x in [1, 2, 3, 4, 5]

def check_in_set(x):
    return x in {1, 2, 3, 4, 5}

print("=== list membership ===")
dis.dis(check_in_list)
print("\n=== set membership (frozenset constant) ===")
dis.dis(check_in_set)
=== list membership === ... BUILD_LIST 5 CONTAINS_OP 0 ... === set membership (frozenset constant) === ... LOAD_CONST (frozenset({1, 2, 3, 4, 5})) CONTAINS_OP 0 ...

The set literal is optimized to a frozenset constant at compile time — LOAD_CONST is O(1). The list version must BUILD_LIST on every call. Additionally, CONTAINS_OP on a frozenset is O(1) hash lookup vs O(n) linear scan for lists.

Example 4 — Examining Code Objects

def greet(name, greeting="Hello"):
    message = f"{greeting}, {name}!"
    return message

code = greet.__code__

print(f"co_varnames: {code.co_varnames}")   # local variable names
print(f"co_consts:   {code.co_consts}")     # constant values
print(f"co_argcount: {code.co_argcount}")   # number of arguments
print(f"co_nlocals:  {code.co_nlocals}")    # number of local variables
print(f"co_stacksize: {code.co_stacksize}") # max stack depth needed
co_varnames: ('name', 'greeting', 'message') co_consts: (None, ', ', '!') co_argcount: 2 co_nlocals: 3 co_stacksize: 4

... LOAD_CONST (42) STORE_FAST (x) LOAD_CONST ('hellohellohello') STORE_FAST (y) ... Constants: (None, 42, 'hellohellohello')

socket Programming Basics

Advanced Standard Library • concept_id: socket_basics

The socket module provides low-level networking. AF_INET + SOCK_STREAM = TCP (reliable, ordered). AF_INET + SOCK_DGRAM = UDP (fast, no guarantees). Server workflow: bind() → listen() → accept() → recv()/send(). Client: connect() → send()/recv().

Example 1 — Simple TCP Echo Server

import socket

def run_echo_server(host="127.0.0.1", port=65432):
    with socket.socket(socket.AF_INET, socket.SOCK_STREAM) as server:
        server.setsockopt(socket.SOL_SOCKET, socket.SO_REUSEADDR, 1)
        server.bind((host, port))
        server.listen(5)  # backlog of 5 pending connections
        print(f"Server listening on {host}:{port}")

        conn, addr = server.accept()
        with conn:
            print(f"Connected by {addr}")
            while True:
                data = conn.recv(1024)  # receive up to 1024 bytes
                if not data:
                    break
                print(f"Received: {data.decode()}")
                conn.sendall(data)  # echo it back

# run_echo_server()  # uncomment to run

Example 2 — TCP Client

import socket

def echo_client(message, host="127.0.0.1", port=65432):
    with socket.socket(socket.AF_INET, socket.SOCK_STREAM) as client:
        client.connect((host, port))
        client.sendall(message.encode())
        data = client.recv(1024)
        print(f"Sent:     {message}")
        print(f"Received: {data.decode()}")

Example 3 — Multi-Client Server with Threading

import socket
import threading

def handle_client(conn, addr):
    """Handle a single client in its own thread."""
    with conn:
        print(f"[{addr}] connected")
        while True:
            data = conn.recv(1024)
            if not data:
                break
            response = data.decode().upper().encode()
            conn.sendall(response)
        print(f"[{addr}] disconnected")

def run_threaded_server(host="127.0.0.1", port=65432):
    with socket.socket(socket.AF_INET, socket.SOCK_STREAM) as server:
        server.setsockopt(socket.SOL_SOCKET, socket.SO_REUSEADDR, 1)
        server.bind((host, port))
        server.listen()
        print(f"Threaded server on {host}:{port}")

        while True:
            conn, addr = server.accept()
            thread = threading.Thread(target=handle_client, args=(conn, addr))
            thread.daemon = True
            thread.start()

# run_threaded_server()  # uncomment to run

Example 4 — UDP Client and Server

import socket

def udp_server(host="127.0.0.1", port=65433):
    with socket.socket(socket.AF_INET, socket.SOCK_DGRAM) as server:
        server.bind((host, port))
        print(f"UDP server on {host}:{port}")
        while True:
            data, addr = server.recvfrom(1024)
            print(f"From {addr}: {data.decode()}")
            server.sendto(data.upper(), addr)

def udp_client(message, host="127.0.0.1", port=65433):
    with socket.socket(socket.AF_INET, socket.SOCK_DGRAM) as client:
        client.sendto(message.encode(), (host, port))
        data, _ = client.recvfrom(1024)
        print(f"Response: {data.decode()}")

# No connect() needed for UDP - just sendto/recvfrom

Example 5 — Health Check Server (Practical)

import socket
import json
import time

def health_check_server(host="127.0.0.1", port=8080):
    """Simple HTTP health check endpoint."""
    with socket.socket(socket.AF_INET, socket.SOCK_STREAM) as server:
        server.setsockopt(socket.SOL_SOCKET, socket.SO_REUSEADDR, 1)
        server.bind((host, port))
        server.listen(1)
        print(f"Health check on http://{host}:{port}/health")

        while True:
            conn, addr = server.accept()
            with conn:
                request = conn.recv(1024).decode()
                if "GET /health" in request:
                    body = json.dumps({
                        "status": "healthy",
                        "uptime": time.monotonic(),
                    })
                    response = (
                        "HTTP/1.1 200 OK\r\n"
                        "Content-Type: application/json\r\n"
                        f"Content-Length: {len(body)}\r\n"
                        "\r\n"
                        f"{body}"
                    )
                else:
                    response = "HTTP/1.1 404 Not Found\r\n\r\n"

                conn.sendall(response.encode())

# health_check_server()  # try: curl http://127.0.0.1:8080/health
