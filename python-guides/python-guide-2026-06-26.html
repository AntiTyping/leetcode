1. Typing: ParamSpec & TypeVarTuple 15 days overdue rep 0 → 1

A ParamSpec (P) captures a callable's entire parameter signature so a decorator can forward *args/**kwargs with full type fidelity — declared as Callable[P, R] with *args: P.args, **kwargs: P.kwargs. A TypeVarTuple (Ts) plus Unpack types a variadic number of generic positions, e.g. tuple[Unpack[Ts]].

Key Pattern
from typing import ParamSpec, TypeVar, Callable
from functools import wraps

P = ParamSpec("P")
R = TypeVar("R")

def logged(fn: Callable[P, R]) -> Callable[P, R]:
    @wraps(fn)
    def inner(*args: P.args, **kwargs: P.kwargs) -> R:
        print("call", fn.__name__)
        return fn(*args, **kwargs)
    return inner

@logged
def add(a: int, b: int) -> int:
    return a + b

print(add(2, 3))   # call add  ->  5  (signature preserved for checkers)

from typing import TypeVarTuple, Unpack
Ts = TypeVarTuple("Ts")
def pack(*vals: Unpack[Ts]) -> tuple[Unpack[Ts]]:
    return vals
print(pack(1, "a", 3.0))   # (1, 'a', 3.0)
Challenge

Type a retry(times) decorator using ParamSpec so the wrapped function's exact signature survives — a caller passing a wrong argument type should still be flagged by a type checker. Then type a head function that takes *Ts and returns the first element as the right type.

Pitfall

P.args and P.kwargs may only appear together in the wrapper's parameter list — you cannot store P.args alone or reorder them. Both ParamSpec and TypeVarTuple need Python 3.10+/3.11+ (or typing_extensions on older versions).

2. textwrap, shutil & tempfile 15 days overdue rep 0 → 1

textwrap reflows strings (dedent, fill, shorten, indent). shutil does high-level file operations (copy/copytree, move, rmtree, which, disk_usage). tempfile creates auto-cleaning temp files and directories (TemporaryDirectory, NamedTemporaryFile).

Key Pattern
import textwrap, shutil, tempfile, os

print(textwrap.shorten("one two three four", width=12))  # one [...]
print(textwrap.indent("a\nb", "> "))                  # "> a\n> b"
print(shutil.which("python3") is not None)        # True if on PATH

with tempfile.TemporaryDirectory() as d:
    p = os.path.join(d, "x.txt")
    open(p, "w").close()
    print(os.path.exists(p))   # True
print(os.path.exists(d))       # False -- directory auto-removed
Challenge

Take a multi-line indented docstring, dedent it, then re-wrap each paragraph to 60 columns with fill. Separately, write a file atomically: write to a NamedTemporaryFile then shutil.move it into place so readers never see a half-written file.

Pitfall

shutil.rmtree deletes an entire directory tree with no recycle bin — validate the path first. A NamedTemporaryFile may not be re-openable by name on Windows while still open; prefer TemporaryDirectory for cross-platform temp paths.

3. Advanced typing 13 days overdue rep 1 → 2

Build generic classes with Generic[T] and a TypeVar; constrain it with bound= (a subtype ceiling) or a value set (TypeVar("T", int, str)). Final forbids reassignment/override; ClassVar marks class-level (non-instance) attributes for checkers and dataclasses.

Key Pattern
from typing import TypeVar, Generic, Final, ClassVar

T = TypeVar("T")            # unbounded: accepts anything
N = TypeVar("N", int, float)  # constrained: int OR float, nothing else

class Stack(Generic[T]):
    count: ClassVar[int] = 0      # shared across instances
    def __init__(self) -> None:
        self._items: list[T] = []
    def push(self, x: T) -> None:
        self._items.append(x); Stack.count += 1
    def pop(self) -> T:
        return self._items.pop()

s: Stack[int] = Stack()
s.push(1)
print(s.pop(), Stack.count)   # 1 1

MAX: Final = 100             # reassigning MAX later is a type error
Challenge

Write a generic Pair[K, V] class with two type parameters and a swap() method annotated to return Pair[V, K]. Add a ClassVar counter that tracks how many pairs were created, and a Final module constant for the max allowed.

Pitfall

An unbounded TypeVar lets the checker assume nothing about T, so calling methods on it is an error — use bound= when you need behaviour (e.g. T must support <). A constrained TypeVar("T", int, str) means "exactly one of these types", not "a subclass of either".

4. functools.cached_property & total_ordering 13 days overdue rep 0 → 1

@cached_property computes a value once on first access and stores it in the instance __dict__ (so later reads are free). @total_ordering fills in the missing rich-comparison operators from just __eq__ plus one of __lt__/__le__/__gt__/__ge__.

Key Pattern
from functools import cached_property, total_ordering

class Data:
    def __init__(self, nums): self.nums = nums
    @cached_property
    def total(self):
        print("computing")
        return sum(self.nums)

d = Data([1, 2, 3])
print(d.total)   # computing -> 6
print(d.total)   # 6  (cached: no recompute)

@total_ordering
class Ver:
    def __init__(self, n): self.n = n
    def __eq__(self, o): return self.n == o.n
    def __lt__(self, o): return self.n < o.n

print(Ver(1) < Ver(2), Ver(2) >= Ver(2))   # True True
Challenge

Give a Circle class a cached_property area that prints "computing" only once even when read many times; confirm it recomputes after del obj.area. Then make the class fully orderable by radius using @total_ordering and only __eq__ + __lt__.

Pitfall

cached_property needs a writable instance __dict__, so it breaks on classes whose __slots__ omit that attribute. @total_ordering is slightly slower than hand-written comparisons and still requires you to define __eq__ yourself.

5. dis Module & Bytecode Inspection 13 days overdue rep 0 → 1

dis.dis(obj) disassembles a function or code object into CPython bytecode instructions. It reveals what the interpreter actually does — the LOAD_FAST (local) vs LOAD_GLOBAL gap, how comprehensions compile into nested code objects, and where attribute lookups cost you.

Key Pattern
import dis

def f(x):
    return x + 1

dis.dis(f)
# LOAD_FAST    0 (x)      <- locals are fast
# LOAD_CONST   1 (1)
# BINARY_OP    0 (+)      <- 3.11+ merged numeric ops
# RETURN_VALUE

# inspect instructions programmatically:
ops = [i.opname for i in dis.get_instructions(f)]
print("LOAD_FAST" in ops)   # True
Challenge

Disassemble one function that reads a global and one that reads a local, and spot the LOAD_GLOBAL vs LOAD_FAST difference. Then use dis.get_instructions to count how many BINARY_OP (or BINARY_ADD, pre-3.11) instructions an expression-heavy function emits.

Pitfall

Bytecode is a CPython implementation detail and changes between versions (3.11 introduced BINARY_OP and adaptive specialization) — never write program logic that depends on specific opcodes. Use dis only for understanding and profiling, not control flow.

6. socket Programming Basics 13 days overdue rep 0 → 1

A socket is an endpoint for network I/O. For TCP: socket(AF_INET, SOCK_STREAM), then bind/listen/accept on the server and connect on the client. send/recv move bytes, not str, so encode/decode explicitly and loop recv because it returns whatever has arrived so far.

Key Pattern
import socket

# --- server ---
srv = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
srv.setsockopt(socket.SOL_SOCKET, socket.SO_REUSEADDR, 1)
srv.bind(("127.0.0.1", 0))   # 0 = let the OS pick a free port
srv.listen()
host, port = srv.getsockname()

# --- client ---
cli = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
cli.connect((host, port))
conn, addr = srv.accept()
cli.sendall(b"hello")
print(conn.recv(1024))   # b'hello'
for s in (conn, cli, srv): s.close()
Challenge

Write an echo server using a context manager (with socket.socket() as s:) that accepts one connection, reads a message, and sends it back uppercased — plus the matching client. Use sendall, not send, so partial writes are handled for you.

Pitfall

recv(n) returns up to n bytes — possibly fewer — and an empty b"" means the peer closed. Never assume one send maps to one recv (TCP is a byte stream); frame messages with a length prefix or delimiter, and remember sockets carry bytes so you must encode/decode.

New Concepts — Numbers & Math (3 concepts)

The standard library's numeric toolkit: the math module's exact integer helpers and float-safe comparisons, the powerful built-ins hiding in plain sight (divmod, three-argument pow, round), and Python's first-class complex numbers.

Numeric

Concept 1: The math Module Essentials

Why learn this? Floats can't be compared with ==, and rolling your own GCD or integer square root is slow and bug-prone. math gives you exact integer helpers (gcd, lcm, isqrt, comb), float-safe comparison (isclose), and the special values inf/nan that show up constantly in algorithms and data.
Example 1 — Constants and the special values inf / nan
import math

print(math.pi, math.e)        # 3.141592653589793 2.718281828459045
print(math.inf, -math.inf)    # inf -inf  (bigger/smaller than any number)
print(math.nan == math.nan)   # False -- NaN never equals itself
print(math.isnan(float("nan")))  # True -- the only safe NaN test
Output
3.141592653589793 2.718281828459045 inf -inf False True
Example 2 — Exact integer helpers
import math

print(math.gcd(12, 18))     # 6
print(math.lcm(4, 6))       # 12
print(math.isqrt(50))        # 7  (floor sqrt, EXACT for ints -- no float error)
print(math.factorial(5))    # 120
print(math.comb(5, 2), math.perm(5, 2))  # 10 20
Output
6 12 7 120 10 20
Example 3 — Floats: rounding direction, isclose, distances
import math

print(math.floor(2.7), math.ceil(2.1), math.trunc(-2.7))  # 2 3 -2
print(0.1 + 0.2 == 0.3)              # False -- binary float error
print(math.isclose(0.1 + 0.2, 0.3))  # True  -- compare floats this way
print(math.dist((0, 0), (3, 4)))     # 5.0  (Euclidean distance)
print(math.hypot(3, 4))            # 5.0
Output
2 3 -2 False True 5.0 5.0
Example 4 — Logs and powers (and why ** stays exact)
import math

print(math.log(math.e))          # 1.0  (natural log)
print(math.log2(8))              # 3.0
print(math.log10(1000))          # 3.0
print(math.pow(2, 10))           # 1024.0  -- math.pow ALWAYS returns float
print(type(2 ** 10).__name__)   # int  -- ** keeps integers exact
Output
1.0 3.0 3.0 1024.0 int

Common Mistakes

✗ Using math.pow for integer powers — it returns a float and loses exactness for large results

✓ Use ** for integer exponents (exact, arbitrary precision); reserve math.pow for floats


✗ Comparing floats with == — 0.1 + 0.2 == 0.3 is False

✓ Use math.isclose(a, b) (tune with rel_tol/abs_tol) for float equality

Key Rules

  • math.inf/-math.inf and math.nan; nan != nan, so test with math.isnan
  • Integer helpers return exact ints: gcd, lcm, isqrt, comb, perm, factorial
  • isclose for float equality; dist/hypot for Euclidean distance; trig/log functions return float

Exercises

  1. Write is_perfect_square(n) using math.isqrt only (no float sqrt), and prove it is exact for n = 10**30.
  2. Use math.comb to compute the number of unique paths in an m × n grid (moving only right/down).
  3. Given two lists of floats, return True if they are element-wise near-equal using math.isclose.
Connections: pairs with statistics and with decimal/fractions when you need exactness math can't give; math.inf is the classic "no value yet" sentinel in graph/DP algorithms; feeds the format spec mini-language when you display results.
Numeric

Concept 2: Numeric Built-ins — divmod, pow, round & int methods

Why learn this? Several heavily-used numeric tools are plain built-ins, no import needed: divmod gets quotient and remainder at once, three-argument pow does fast modular exponentiation (and modular inverse), and int objects carry handy methods for bases, bit lengths, and byte conversion. These show up constantly in interview problems and low-level code.
Example 1 — divmod: quotient and remainder together
print(divmod(17, 5))     # (3, 2)  ==  (17 // 5, 17 % 5)
print(divmod(-17, 5))    # (-4, 3) -- follows FLOORED division

# perfect for splitting a total into units:
secs = 3725
h, rem = divmod(secs, 3600)
m, s = divmod(rem, 60)
print(h, m, s)            # 1 2 5  ->  1h 2m 5s
Output
(3, 2) (-4, 3) 1 2 5
Example 2 — Three-argument pow: modular exponentiation
print(pow(2, 10))         # 1024
print(pow(2, 10, 1000))   # 24   ==  2**10 % 1000, but computed efficiently
print(pow(3, 100, 7))     # 4    -- no giant intermediate number

# modular inverse (Python 3.8+): pow(a, -1, m)
print(pow(3, -1, 7))      # 5  because (3 * 5) % 7 == 1
Output
1024 24 4 5
Example 3 — round: banker's rounding and negative ndigits
print(round(2.5), round(3.5))   # 2 4  -- round HALF TO EVEN!
print(round(2.675, 2))         # 2.67 -- float repr surprise
print(round(12345, -2))         # 12300 -- negative ndigits rounds tens/hundreds
print(round(3.14159, 2))       # 3.14
Output
2 4 2.67 12300 3.14
Example 4 — int methods: bases, bits, bytes
print(int("ff", 16), int("1010", 2))   # 255 10  (parse in any base)
print((255).bit_length())                 # 8  (bits needed)
print((0b1011).bit_count())               # 3  (popcount, Python 3.10+)
print((255).to_bytes(2, "big"))           # b'\x00\xff'
print(int.from_bytes(b"\x00\xff", "big"))  # 255
Output
255 10 8 3 b'\x00\xff' 255

Common Mistakes

✗ Assuming round() rounds .5 up — Python uses banker's rounding, so round(0.5) == 0 and round(2.5) == 2

✓ Expect round-half-to-even; for money use decimal.Decimal with an explicit rounding mode


✗ Computing (a ** b) % m for huge numbers — it builds a giant intermediate integer first

✓ Use three-argument pow(a, b, m) — efficient and built in; pow(a, -1, m) gives the modular inverse

Key Rules

  • divmod(a, b) == (a // b, a % b) in one call, following floored division for negatives
  • pow(b, e, m) is fast modular exponentiation; pow(a, -1, m) is the modular inverse (3.8+)
  • round is round-half-to-even; negative ndigits rounds to tens/hundreds. int has bit_length, bit_count (3.10+), to_bytes/from_bytes

Exercises

  1. Convert a total number of seconds into an "Hh Mm Ss" string using two divmod calls.
  2. Compute (a**b) % m for large a, b, m two ways and confirm pow(a, b, m) matches but avoids the huge intermediate.
  3. Write to_base(n, b) returning the digit string of n in base b (2..16) using divmod in a loop.
Connections: three-argument pow underpins modular arithmetic in hashlib/secrets and competitive problems; to_bytes/from_bytes ties to struct and the bytes concept; round connects to decimal and the format spec mini-language.
Numeric

Concept 3: complex Numbers & cmath

Why learn this? Python has complex numbers as a first-class built-in type — written with a j suffix. They make 2D geometry, signal processing, and any math that "goes imaginary" clean, and the cmath module mirrors math for the complex domain so sqrt(-1) works instead of crashing.
Example 1 — Literals and the parts of a complex number
z = 3 + 4j
print(z.real, z.imag)     # 3.0 4.0  (both are floats)
print(abs(z))             # 5.0  -- magnitude (sqrt(real^2 + imag^2))
print(z.conjugate())      # (3-4j)
print(type(z).__name__)   # complex
Output
3.0 4.0 5.0 (3-4j) complex
Example 2 — Arithmetic just works
a = 1 + 2j
b = 2 - 1j
print(a + b)      # (3+1j)
print(a * b)      # (4+3j)
print(a / b)      # 1j
print((1j) ** 2)  # (-1+0j)  -- i squared is -1
Output
(3+1j) (4+3j) 1j (-1+0j)
Example 3 — cmath for complex-valued functions
import cmath

print(cmath.sqrt(-1))            # 1j
print(cmath.phase(1 + 1j))       # 0.7853981633974483  (pi/4 radians)
print(cmath.polar(1 + 1j))       # (1.4142135623730951, 0.7853981633974483)
print(cmath.exp(cmath.pi * 1j))   # (-1+1.2e-16j)  -- Euler's identity ~ -1
Output
1j 0.7853981633974483 (1.4142135623730951, 0.7853981633974483) (-1+1.2246467991473532e-16j)
Example 4 — math raises, cmath stays in the complex domain
import math, cmath

try:
    math.sqrt(-1)
except ValueError as e:
    print("math:", e)         # math: math domain error
print("cmath:", cmath.sqrt(-1))  # cmath: 1j
print(cmath.rect(2, 0))         # (2+0j)  -- rebuild from polar form
Output
math: math domain error cmath: 1j (2+0j)

Common Mistakes

✗ Calling math.sqrt/math.log on a negative number — raises ValueError: math domain error

✓ Use cmath for any computation that can legitimately produce a complex result


✗ Comparing complex numbers with </> — complex has no ordering and raises TypeError

✓ Compare magnitudes via abs(z), or compare .real/.imag explicitly

Key Rules

  • The j (or J) suffix makes an imaginary literal; complex has .real, .imag, .conjugate()
  • abs(z) is the magnitude; complex supports + - * / ** but NOT </>
  • cmath mirrors math for the complex domain: sqrt, exp, log, phase, polar, rect

Exercises

  1. Verify Euler's identity numerically: show cmath.exp(1j * cmath.pi) is within 1e-9 of -1 using math.isclose on the real and imaginary parts.
  2. Given two 2D points encoded as complex numbers, compute the distance between them with abs(p2 - p1).
  3. Write polar_str(z) that returns "r∠θ°" using cmath.polar and math.degrees.
Connections: builds on the math module (Concept 1) and shares the abs built-in for magnitude; phase/polar relate to trigonometry; encoding 2D points as complex numbers is a slick trick for geometry-heavy interview problems.

New Concepts — Collections & Timing (2 concepts)

The dict variant that still earns its place after dicts became ordered, and the clocks you actually want for benchmarking and timeouts (hint: not time.time()).

Collections

Concept 4: OrderedDict — Order-Aware Mappings

Why learn this? Since Python 3.7 a regular dict keeps insertion order, so why use OrderedDict? It adds order-sensitive equality, move_to_end, and popitem(last=...) — exactly the toolkit for building an LRU cache or any structure where you reorder and evict by position.
Example 1 — move_to_end reorders in place
from collections import OrderedDict

od = OrderedDict([("a", 1), ("b", 2), ("c", 3)])
od.move_to_end("a")               # send 'a' to the end
print(list(od))                  # ['b', 'c', 'a']
od.move_to_end("c", last=False)    # send 'c' to the front
print(list(od))                  # ['c', 'b', 'a']
Output
['b', 'c', 'a'] ['c', 'b', 'a']
Example 2 — popitem(last=...) is LIFO or FIFO
from collections import OrderedDict

od = OrderedDict(a=1, b=2, c=3)
print(od.popitem())            # ('c', 3)  -- last by default (LIFO)
print(od.popitem(last=False))  # ('a', 1)  -- first (FIFO)
print(dict(od))               # {'b': 2}
Output
('c', 3) ('a', 1) {'b': 2}
Example 3 — Equality is order-sensitive (the key difference)
from collections import OrderedDict

o1 = OrderedDict([("a", 1), ("b", 2)])
o2 = OrderedDict([("b", 2), ("a", 1)])
print(o1 == o2)                # False -- order matters for OrderedDict
print(dict(o1) == dict(o2))    # True  -- plain dicts ignore order
Output
False True
Example 4 — A tiny LRU cache
from collections import OrderedDict

class LRU:
    def __init__(self, cap):
        self.cap, self.data = cap, OrderedDict()
    def get(self, key):
        if key not in self.data:
            return -1
        self.data.move_to_end(key)          # mark most-recently used
        return self.data[key]
    def put(self, key, val):
        if key in self.data:
            self.data.move_to_end(key)
        self.data[key] = val
        if len(self.data) > self.cap:
            self.data.popitem(last=False)   # evict least-recently used

c = LRU(2)
c.put(1, "a"); c.put(2, "b")
print(c.get(1))   # a  (key 1 now most-recent)
c.put(3, "c")      # evicts key 2
print(c.get(2))   # -1 (evicted)
Output
a -1

Common Mistakes

✗ Assuming plain dict equality respects order — it does not; {'a':1,'b':2} == {'b':2,'a':1} is True

✓ Use OrderedDict when equality must respect order, or compare list(d.items())


✗ Reaching for OrderedDict just to preserve insertion order on 3.7+ — a plain dict already does that

✓ Use OrderedDict for its extra API: move_to_end, popitem(last=...), order-sensitive ==

Key Rules

  • Plain dict preserves insertion order (3.7+); OrderedDict adds move_to_end & popitem(last=...)
  • OrderedDict equality is order-sensitive; dict equality is not
  • popitem(last=False) pops the oldest entry (FIFO) — ideal for LRU eviction

Exercises

  1. Write same_order(o1, o2) returning True only if two OrderedDicts have the same items in the same order.
  2. Build a "most recent N keys" tracker using move_to_end and popitem(last=False) to drop the oldest.
  3. Produce an order-preserving unique list from a list with duplicates using OrderedDict.fromkeys; compare with dict.fromkeys.
Connections: contrasts with defaultdict/Counter and UserDict from earlier collections guides; the LRU pattern here is essentially what functools.lru_cache maintains internally.
Timing

Concept 5: The time Module — perf_counter, monotonic & sleep

Why learn this? Wall-clock time can jump — NTP corrections and DST can move it backward, ruining a duration measurement. The time module gives you monotonic clocks that never go backward (perf_counter, monotonic) for benchmarking and timeouts, plus sleep for pacing.
Example 1 — Timing a block with perf_counter
import time

print(type(time.time()).__name__)     # float -- seconds since the epoch

t0 = time.perf_counter()
total = sum(range(1_000_000))
elapsed = time.perf_counter() - t0
print(elapsed > 0)                 # True (a small positive duration)
Output
float True
Example 2 — The three clocks compared
import time

print(time.monotonic() > 0)       # True -- arbitrary start, never goes backward
print(time.perf_counter() > 0)    # True -- highest-resolution timer
print(time.process_time() >= 0)   # True -- CPU time only (excludes sleep)
# time.time() can move backward if the system clock is adjusted; these cannot.
Output
True True True
Example 3 — sleep blocks; process_time barely moves
import time

start = time.perf_counter()
time.sleep(0.05)                  # block for ~50ms
print(time.perf_counter() - start >= 0.05)  # True

c0 = time.process_time()
time.sleep(0.05)
print(time.process_time() - c0 < 0.05)     # True -- no CPU used while sleeping
Output
True True
Example 4 — Formatting times and nanosecond precision
import time

t = time.gmtime(0)                          # struct_time for the epoch (UTC)
print(time.strftime("%Y-%m-%d", t))          # 1970-01-01
print(time.strftime("%H:%M:%S", time.gmtime(3661)))  # 01:01:01
print(type(time.perf_counter_ns()).__name__)  # int -- integer nanoseconds, no float rounding
Output
1970-01-01 01:01:01 int

Common Mistakes

✗ Using time.time() to measure elapsed time — it can jump backward/forward when the system clock is corrected

✓ Use time.perf_counter() (or perf_counter_ns()) for durations and time.monotonic() for timeouts


✗ Calling time.sleep() inside async code — it blocks the whole event loop

✓ Use await asyncio.sleep() in coroutines; reserve time.sleep() for synchronous code

Key Rules

  • time.time(): wall clock (epoch seconds) — for "what time is it", not for measuring intervals
  • time.perf_counter()/monotonic(): never go backward — use for benchmarks and timeouts
  • time.process_time(): CPU time only (excludes sleep); the *_ns() variants return integer nanoseconds

Exercises

  1. Write a timer(func, *args) that runs func and returns (result, seconds) using perf_counter.
  2. Show that process_time barely changes across a time.sleep(0.1) while perf_counter advances about 0.1s.
  3. Build a rate limiter that allows one action per 0.5s using time.monotonic() as the reference clock.
