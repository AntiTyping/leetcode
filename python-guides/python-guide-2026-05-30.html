Function Decorators & @wraps Decorators 1 days since review
A decorator is a function that takes a function and returns a modified version. @wraps(func) from functools preserves the original function’s __name__, __doc__, and __module__.
Python
from functools import wraps

def timer(func):
    @wraps(func)
    def wrapper(*args, **kwargs):
        import time
        start = time.perf_counter()
        result = func(*args, **kwargs)
        print(f"{func.__name__}: {time.perf_counter()-start:.4f}s")
        return result
    return wrapper

@timer
def slow():
    import time; time.sleep(0.1)

Stacking & Parameterized Decorators Decorators 1 days since review
Parameterized decorators use a three-level nesting: the outer function takes parameters, returns the actual decorator, which returns the wrapper. Stacking applies multiple decorators bottom-up.
Python
def repeat(n):
    def decorator(func):
        @wraps(func)
        def wrapper(*args, **kwargs):
            for _ in range(n):
                result = func(*args, **kwargs)
            return result
        return wrapper
    return decorator

@repeat(3)
def greet(name):
    print(f"Hello {name}")
Challenge:
!Write a @validate_types parameterized decorator that checks argument types at runtime. Usage: @validate_types(int, str) on def greet(age, name) should raise TypeError if wrong types are passed. Handle *args only (not kwargs).
#1 Pitfall: When stacking decorators, order matters — @A @B def f means A(B(f)). The bottom decorator runs first (closest to def). Reversing the order can change behavior entirely.
List, Dict & Set Comprehensions Comprehensions 1 days since review
Comprehensions are concise syntax for building collections: [expr for x in iter if cond] for lists, {k: v for ...} for dicts, {expr for ...} for sets. They can nest (outer loop first) and use the walrus operator := to capture intermediate values.
Python
# Dict comp with conditional
word_lengths = {w: len(w) for w in words if len(w) > 3}

# Nested: flatten a matrix
flat = [x for row in matrix for x in row]

# Walrus: compute once, filter, and use
results = [y for x in data if (y := expensive(x)) > threshold]

Closures & Lambdas

Understanding the mechanism behind decorators, and mastering anonymous functions for concise code.

Closures & nonlocal Functions

Closures are the mechanism that makes decorators work. They’re also used to create factory functions, callback handlers, and lightweight stateful objects without writing a class.

A closure is a nested function that “remembers” variables from its enclosing scope even after that scope has finished executing. When you return an inner function from an outer function, the inner function carries a snapshot of the variables it references — this bundle of function + captured environment is the closure.

Python stores these captured variables in special cell objects accessible via the function’s __closure__ attribute. By default you can read closed-over variables freely, but if you need to modify them you must declare them with the nonlocal keyword.

Python
# Simplest closure — a function that remembers a variable from its enclosing scope
def make_greeter(greeting):
    def greet(name):
        return f"{greeting}, {name}!"
    return greet

hello = make_greeter("Hello")
print(hello("Alice"))
print(hello("Bob"))

howdy = make_greeter("Howdy")
print(howdy("Alice"))
Hello, Alice! Hello, Bob! Howdy, Alice!

greet “closes over” the greeting variable. Each call to make_greeter creates a new closure with its own independent copy of greeting.

Python
# Inspecting the closure — the __closure__ attribute
hello = make_greeter("Hello")

print(hello.__closure__)                    # tuple of cell objects
print(hello.__closure__[0].cell_contents)   # the captured value
(<cell at 0x...: str object at 0x...>,) Hello

Every closure carries a __closure__ tuple of cell objects. Each cell holds one captured variable, and .cell_contents reveals its current value.

Python
# The nonlocal keyword — modifying a closed-over variable
def make_counter():
    count = 0
    def increment():
        nonlocal count
        count += 1
        return count
    return increment

counter = make_counter()
print(counter())
print(counter())
print(counter())
1 2 3

Without nonlocal, writing count += 1 would make Python treat count as a new local variable. Since it hasn’t been assigned yet in the local scope, you’d get an UnboundLocalError. The nonlocal declaration tells Python: “this name refers to the variable in the enclosing scope — modify it there.”

Python
# Factory functions — closures as lightweight alternatives to classes
def make_validator(min_val, max_val):
    def validate(x):
        if not min_val <= x <= max_val:
            raise ValueError(f"{x} not in [{min_val}, {max_val}]")
        return x
    return validate

check_age = make_validator(0, 150)
check_percentage = make_validator(0, 100)

print(check_age(25))
print(check_percentage(95))
# check_age(200)  # raises ValueError: 200 not in [0, 150]
25 95

This pattern produces specialized functions without the ceremony of defining a class. Each validator “remembers” its own min_val and max_val.

Python
# Classic gotcha — late binding in loops

# WRONG: all functions reference the same variable i
funcs = []
for i in range(3):
    funcs.append(lambda: i)
print([f() for f in funcs])   # all see i=2!

# FIX: capture the current value with a default argument
funcs = []
for i in range(3):
    funcs.append(lambda i=i: i)
print([f() for f in funcs])   # each captured its own value
[2, 2, 2] [0, 1, 2]

Closures capture variables, not values. By the time you call the lambdas, the loop has finished and i is stuck at its final value (2). The default-argument trick (lambda i=i: i) evaluates i at function-creation time, freezing the current value.

Common Mistakes:
WRONG: Trying to modify a closed-over variable without nonlocal → UnboundLocalError.
RIGHT: Declare nonlocal varname before modifying the variable.
WRONG: Assuming closures in loops capture the current value (late-binding trap).
RIGHT: Use a default argument lambda i=i: i to capture the current value at creation time.
Key Rules:
  • 2026-07-08 (Day 38)
  • Use closures when you need a function with “memory” but a full class is overkill.
  • Use nonlocal only when you need to modify the closed-over variable — reading works without it.
  • Prefer closures over mutable default arguments for maintaining state.

Test with: check = make_password_checker(8, True), check("hello") → False (too short), check("hellowor") → False (no digit), check("hellow0r") → True.

Closures are the mechanism behind decorators (Day 2). Understanding closures makes parameterized decorators (@decorator(arg) with 3 levels of nesting) intuitive — each level is just another closure.
Lambda Functions Functions

Lambdas are anonymous one-expression functions. They’re most useful as throwaway functions for sorted(), filter(), map(), and callbacks — anywhere a named function would be overkill.

A lambda expression creates a small anonymous function inline. The syntax is lambda args: expression. It can take any number of arguments but is limited to a single expression — no statements, no assignments, no multi-line bodies. The expression’s result is automatically returned.

Lambdas are most valuable as short-lived key= functions for sorting and as lightweight callbacks. If you find yourself naming a lambda or stuffing complex logic into one, switch to a regular def.

Python
# Basic lambda syntax
square = lambda x: x ** 2
print(square(5))

add = lambda a, b: a + b
print(add(3, 4))
25 7

Note: assigning a lambda to a variable like this is fine for quick demos, but PEP 8 recommends using def for named functions (rule E731). Lambdas are meant to be anonymous.

Python
# Where lambdas shine — sort keys
students = [("Alice", 85), ("Bob", 92), ("Charlie", 78)]

# Sort by grade (second element)
by_grade = sorted(students, key=lambda s: s[1])
print(by_grade)

# Sort by name length
by_name_len = sorted(students, key=lambda s: len(s[0]))
print(by_name_len)
[('Charlie', 78), ('Alice', 85), ('Bob', 92)] [('Bob', 92), ('Alice', 85), ('Charlie', 78)]

This is the canonical use case for lambdas. The key= parameter expects a callable, and a lambda lets you define it right where it’s used without jumping away to a separate def.

Python
# With filter() and map()
nums = [1, 2, 3, 4, 5, 6, 7, 8]

evens = list(filter(lambda x: x % 2 == 0, nums))
print(evens)

squares = list(map(lambda x: x ** 2, nums))
print(squares)
[2, 4, 6, 8] [1, 4, 9, 16, 25, 36, 49, 64]

These work, but comprehensions are usually more Pythonic and readable: [x for x in nums if x % 2 == 0] and [x ** 2 for x in nums]. Reserve map/filter with lambdas for cases where the functional style genuinely reads better.

Python
# Lambdas as callbacks / dict dispatch
ops = {
    '+': lambda a, b: a + b,
    '-': lambda a, b: a - b,
    '*': lambda a, b: a * b,
    '/': lambda a, b: a / b,
}

print(ops['*'](6, 7))
print(ops['+'](10, 5))
print(ops['-'](100, 58))
42 15 42

A dispatch dictionary replaces long if/elif chains. Each key maps to a lambda that performs the operation. This pattern scales cleanly and is easy to extend.

Python
# Multi-key sorting with tuples
data = [("Alice", "B", 85), ("Bob", "A", 92), ("Charlie", "A", 78)]

# Sort by letter grade first (ascending), then by score (descending)
result = sorted(data, key=lambda x: (x[1], -x[2]))
print(result)
[('Bob', 'A', 92), ('Charlie', 'A', 78), ('Alice', 'B', 85)]

Returning a tuple from the key function gives you multi-level sorting. Python compares tuples element by element. Negating a numeric field (-x[2]) reverses the sort order for that field while keeping the others ascending.

Common Mistakes:
WRONG: Assigning a lambda to a variable for reuse: double = lambda x: x * 2. PEP 8 says if you’re naming it, use def.
RIGHT: def double(x): return x * 2
WRONG: Using map(lambda ...) / filter(lambda ...) when a comprehension is clearer.
RIGHT: [x**2 for x in nums] instead of list(map(lambda x: x**2, nums)).
WRONG: Cramming multi-line logic into a single lambda expression.
RIGHT: Use a named def function for anything complex.
Key Rules:
  • Use lambdas for short, throwaway functions — especially key= arguments to sorted(), min(), max().
  • If you’re naming the lambda, convert it to def — PEP 8 rule E731.
  • Prefer comprehensions over map(lambda ...) and filter(lambda ...) in most cases.
Practice Exercises:
1 Given words = ["apple", "Banana", "cherry", "Date"], sort them case-insensitively using sorted() with a lambda key. Expected: ['apple', 'Banana', 'cherry', 'Date'].
2 Given a list of dicts people = [{"name": "Alice", "age": 30}, {"name": "Bob", "age": 25}, {"name": "Charlie", "age": 35}], use sorted() with a lambda to sort by age descending. Then use max() with a lambda to find the oldest person.
3 Create a simple calculator dict that maps operation strings ("add", "sub", "mul") to lambda functions. Write a calculate(op, a, b) function that looks up and calls the right lambda.
Lambdas are closures too — they close over variables from their enclosing scope (which is why the loop late-binding trap from the closures section applies). Tomorrow we’ll see operator.itemgetter and operator.attrgetter which often replace lambdas in sort keys more readably.

Functional Programming Toolkit

The standard library modules that make functional programming in Python practical: itertools for lazy iteration, functools for higher-order functions, and operator for clean sort keys and reductions.

itertools — Iterator Building Blocks Iteration

itertools provides memory-efficient building blocks for working with iterators. These functions are lazy (produce items one at a time), composable (chain them together), and fast (implemented in C). Essential for processing large datasets, generating combinations, and elegant loop patterns.

The itertools module is Python’s toolkit for working with iterators in a functional style. Every function in this module returns an iterator — meaning it produces values on demand rather than building an entire list in memory. This makes them ideal for large datasets and infinite sequences.

The most commonly used tools fall into three categories: combining iterables (chain, product), filtering/slicing iterables (islice, groupby), and generating combinatorics (combinations, permutations, accumulate).

Python
# chain — concatenate iterables without copying
from itertools import chain

list1 = [1, 2, 3]
list2 = [4, 5, 6]
list3 = [7, 8, 9]
combined = list(chain(list1, list2, list3))
print(combined)

# chain.from_iterable for a list of lists
nested = [[1, 2], [3, 4], [5, 6]]
flat = list(chain.from_iterable(nested))
print(flat)
[1, 2, 3, 4, 5, 6, 7, 8, 9] [1, 2, 3, 4, 5, 6]

chain takes explicit iterables as arguments. chain.from_iterable takes a single iterable of iterables — use it when flattening a dynamic list of lists.

Python
# islice — slice any iterator (no negative indices)
from itertools import islice

# Get first 5 items from a generator
squares = (x**2 for x in range(100))
first_five = list(islice(squares, 5))
print(first_five)

# Skip first 2, take next 4
data = range(10)
sliced = list(islice(data, 2, 6))
print(sliced)
[0, 1, 4, 9, 16] [2, 3, 4, 5]

Unlike regular slicing (seq[2:6]), islice works on any iterator — including generators that don’t support indexing. It consumes items lazily, so it’s memory-efficient even on huge or infinite iterators.

Python
# groupby — group consecutive elements by a key
from itertools import groupby

# IMPORTANT: data must be sorted by the key first!
data = [("A", 1), ("A", 2), ("B", 3), ("B", 4), ("A", 5)]
data.sort(key=lambda x: x[0])  # Sort first!
for key, group in groupby(data, key=lambda x: x[0]):
    print(key, list(group))
A [('A', 1), ('A', 2), ('A', 5)] B [('B', 3), ('B', 4)]

groupby only groups consecutive elements with the same key. If your data isn’t sorted by the key, you’ll get multiple separate groups for the same key value. Always sort first with the same key function.

Python
# product — Cartesian product (nested loops in one line)
from itertools import product

colors = ['red', 'blue']
sizes = ['S', 'M', 'L']
combos = list(product(colors, sizes))
print(combos)

# Equivalent to:
# for c in colors:
#     for s in sizes:
#         (c, s)
[('red', 'S'), ('red', 'M'), ('red', 'L'), ('blue', 'S'), ('blue', 'M'), ('blue', 'L')]

product replaces deeply nested for loops with a single clean call. It generates tuples of every combination — one element from each input iterable.

Python
# combinations and permutations
from itertools import combinations, permutations

# All 2-element subsets (order doesn't matter)
print(list(combinations([1, 2, 3, 4], 2)))

# All 2-element orderings (order matters)
print(list(permutations([1, 2, 3], 2)))
[(1, 2), (1, 3), (1, 4), (2, 3), (2, 4), (3, 4)] [(1, 2), (1, 3), (2, 1), (2, 3), (3, 1), (3, 2)]

combinations gives you subsets (order doesn’t matter, no repeats). permutations gives you arrangements (order matters). For combinations with replacement, use combinations_with_replacement.

Python
# accumulate — running reductions
from itertools import accumulate
import operator

nums = [1, 2, 3, 4, 5]
running_sum = list(accumulate(nums))
print(running_sum)

running_max = list(accumulate(nums, max))
print(running_max)

running_product = list(accumulate(nums, operator.mul))
print(running_product)
[1, 3, 6, 10, 15] [1, 2, 3, 4, 5] [1, 2, 6, 24, 120]

accumulate produces every intermediate result of a fold. The default operation is addition, but you can pass any two-argument function. Think of it as a “running” version of reduce.

Common Mistakes:
WRONG: Using groupby on unsorted data — it only groups CONSECUTIVE elements.
RIGHT: Always sort(key=...) before groupby(key=...) with the same key function.
WRONG: Iterating over a groupby group after advancing to the next key — the group iterator is consumed.
RIGHT: Materialize each group immediately with list(group).
WRONG: list(chain([1,2], [3,4])) to flatten a dynamic list of lists.
RIGHT: list(chain.from_iterable(list_of_lists)) when the input is a single iterable of iterables.
Key Rules:
  • All itertools functions return iterators — they’re lazy and memory-efficient.
  • groupby requires data sorted by the grouping key.
  • Use product instead of deeply nested loops for cleaner Cartesian products.
Practice Exercises:
1 Given data = [[1, 2, 3], [4, 5], [6, 7, 8, 9]], flatten it to a single list using chain.from_iterable. Then get only the first 5 elements using islice. Expected: [1, 2, 3, 4, 5].
2 Use combinations to find all pairs from [10, 20, 30, 40, 50] that sum to 60. Expected: [(10, 50), (20, 40)].
3 Given a sorted list of transactions [("groceries", 50), ("groceries", 30), ("gas", 40), ("gas", 25), ("dining", 60)], use groupby to compute the total spending per category. Expected: {'dining': 60, 'gas': 65, 'groceries': 80}.
chain.from_iterable is the lazy version of the list comprehension flatten [x for row in matrix for x in row] from Day 1. accumulate pairs naturally with generator expressions for streaming data processing. combinations and permutations are essential for backtracking problems in LeetCode.
functools — Higher-Order Function Tools Functions

functools provides higher-order functions and tools for working with callables. lru_cache gives you free memoization, partial creates specialized versions of functions, and reduce folds sequences. These are building blocks for writing clean, efficient Python.

The functools module is Python’s toolbox for working with functions as first-class objects. Its utilities transform, combine, and optimize callables. The most impactful tools are: lru_cache for automatic memoization (turning exponential recursion into linear), partial for freezing arguments to create specialized functions, and reduce for folding a sequence down to a single value.

These tools build directly on the closures and decorators you’ve already learned — lru_cache is a decorator, partial works like a closure, and reduce is the classic functional fold operation.

Python
# lru_cache / cache — automatic memoization
from functools import lru_cache

@lru_cache(maxsize=128)
def fibonacci(n):
    if n < 2:
        return n
    return fibonacci(n - 1) + fibonacci(n - 2)

print(fibonacci(50))
print(fibonacci.cache_info())
12586269025 CacheInfo(hits=48, misses=51, maxsize=128, currsize=51)

Without caching, fibonacci(50) would require over a trillion recursive calls. With @lru_cache, each unique argument is computed only once and the result is stored. cache (Python 3.9+) is shorthand for lru_cache(maxsize=None) — an unbounded cache.

Python
# partial — freeze some arguments
from functools import partial

def power(base, exponent):
    return base ** exponent

square = partial(power, exponent=2)
cube = partial(power, exponent=3)

print(square(5))
print(cube(3))

# Real-world: pre-configure a debug printer
debug = partial(print, "[DEBUG]")
debug("starting process")
25 27 [DEBUG] starting process

partial creates a new callable with some arguments pre-filled. Unlike a closure, partial objects expose their frozen arguments via .func, .args, and .keywords attributes — making them easier to inspect and debug.

Python
# reduce — fold a sequence to a single value
from functools import reduce

nums = [1, 2, 3, 4, 5]
product = reduce(lambda a, b: a * b, nums)
print(product)

# With initial value
total = reduce(lambda a, b: a + b, nums, 100)
print(total)

# Flatten nested lists
nested = [[1, 2], [3, 4], [5, 6]]
flat = reduce(lambda a, b: a + b, nested)
print(flat)
120 115 [1, 2, 3, 4, 5, 6]

reduce applies a two-argument function cumulatively: reduce(f, [a, b, c, d]) computes f(f(f(a, b), c), d). The optional third argument is an initial value placed before all elements. Always provide an initial value when the sequence might be empty.

Python
# total_ordering — fill in comparison methods automatically
from functools import total_ordering

@total_ordering
class Student:
    def __init__(self, name, grade):
        self.name = name
        self.grade = grade

    def __eq__(self, other):
        return self.grade == other.grade

    def __lt__(self, other):
        return self.grade < other.grade

s1 = Student("Alice", 85)
s2 = Student("Bob", 92)
print(s1 < s2)
print(s1 >= s2)
print(s1 != s2)
True False True

Define just __eq__ and one ordering method (__lt__, __le__, __gt__, or __ge__), and @total_ordering auto-generates the rest. This saves boilerplate when making sortable objects.

Python
# singledispatch — function overloading by type
from functools import singledispatch

@singledispatch
def process(data):
    raise TypeError(f"Unsupported type: {type(data)}")

@process.register(str)
def _(data):
    return data.upper()

@process.register(list)
def _(data):
    return [x * 2 for x in data]

@process.register(int)
def _(data):
    return data ** 2

print(process("hello"))
print(process([1, 2, 3]))
print(process(5))
HELLO [2, 4, 6] 25

@singledispatch gives Python a form of function overloading. The base function handles the default case. Register specialized implementations for specific types using @funcname.register(type). Dispatch is based on the type of the first argument only.

Common Mistakes:
WRONG: Using @lru_cache on functions with mutable arguments (lists, dicts) — they’re unhashable.
RIGHT: Convert to tuples: @lru_cache + def f(items: tuple) instead of def f(items: list).
WRONG: Forgetting that reduce with an empty sequence and no initial value raises TypeError.
RIGHT: Always provide an initial value: reduce(func, seq, initial).
WRONG: Using partial when a lambda is simpler for one-off use.
RIGHT: Use partial when you’ll reuse the partially-applied function multiple times.
Key Rules:
  • @lru_cache arguments must be hashable — convert lists to tuples, dicts to frozensets of items.
  • Prefer sum(), max(), min() over reduce() for simple aggregations — reduce is for complex folds.
  • @total_ordering saves boilerplate but adds slight overhead — define all comparison methods manually for performance-critical code.
Practice Exercises:
1 Write a recursive @lru_cache-decorated function count_paths(m, n) that counts the number of unique paths from top-left to bottom-right of an m×n grid (can only move right or down). Test with count_paths(3, 7) → 28.
2 Use partial to create three specialized string formatters: format_usd(amount) → "$1,234.56", format_eur(amount) → "EUR 1,234.56", format_pct(value) → "85.2%". Hint: create a base format_value(template, value) function.

@lru_cache is a specialized decorator — you now understand why it works (Day 2 decorators + today’s closures). partial is related to closures — both “freeze” some data into a callable. reduce + accumulate (from itertools) are two sides of the same coin: reduce gives you the final value, accumulate gives you every intermediate value.
operator Module — Functions for Operators Functions

The operator module provides function versions of Python’s operators and attribute/item accessors. These are faster and more readable than equivalent lambdas — especially as key= arguments to sorted(), min(), max(), and groupby().

Every Python operator (+, *, [], .) has a corresponding function in the operator module. Why does this matter? Because many higher-order functions like sorted(), reduce(), and map() accept a callable — and operator functions are implemented in C, making them faster than equivalent lambdas.

The three most useful tools are: itemgetter for accessing items by index or dict key, attrgetter for accessing object attributes, and methodcaller for calling methods on objects. These replace the lambda x: x[key], lambda x: x.attr, and lambda x: x.method() patterns you’ve been writing.

Python
# itemgetter — access items by index/key
from operator import itemgetter

# Sort tuples by second element
students = [("Alice", 85), ("Bob", 92), ("Charlie", 78)]
by_grade = sorted(students, key=itemgetter(1))
print(by_grade)

# Multiple keys — returns a tuple
first_and_last = itemgetter(0, -1)
print(first_and_last([10, 20, 30, 40]))
[('Charlie', 78), ('Alice', 85), ('Bob', 92)] (10, 40)

itemgetter(1) creates a callable equivalent to lambda x: x[1], but faster. When given multiple arguments, it returns a tuple of the requested items — handy for extracting several fields at once.

Python
# itemgetter with dicts
from operator import itemgetter

people = [
    {"name": "Alice", "age": 30},
    {"name": "Bob", "age": 25},
    {"name": "Charlie", "age": 35},
]
by_age = sorted(people, key=itemgetter("age"))
print([p["name"] for p in by_age])

# Extract multiple fields
get_info = itemgetter("name", "age")
print(get_info(people[0]))
['Bob', 'Alice', 'Charlie'] ('Alice', 30)

itemgetter works with any object that supports [] access — lists, tuples, dicts, strings. For dicts, pass the key name as a string instead of a numeric index.

Python
# attrgetter — access object attributes
from operator import attrgetter
from collections import namedtuple

Student = namedtuple("Student", ["name", "grade", "age"])
students = [
    Student("Alice", "A", 20),
    Student("Bob", "B", 22),
    Student("Charlie", "A", 19),
]
by_age = sorted(students, key=attrgetter("age"))
print(by_age)

# Multi-key sort: by grade then age
by_grade_age = sorted(students, key=attrgetter("grade", "age"))
print(by_grade_age)
[Student(name='Charlie', grade='A', age=19), Student(name='Alice', grade='A', age=20), Student(name='Bob', grade='B', age=22)] [Student(name='Charlie', grade='A', age=19), Student(name='Alice', grade='A', age=20), Student(name='Bob', grade='B', age=22)]

attrgetter("age") is equivalent to lambda x: x.age. With multiple arguments, it returns a tuple — perfect for multi-key sorting of namedtuples and objects. It also supports nested access: attrgetter("address.city") is equivalent to lambda x: x.address.city.

Python
# methodcaller — call a method on objects
from operator import methodcaller

words = ["  hello  ", "  world  ", "  python  "]
stripped = list(map(methodcaller("strip"), words))
print(stripped)

# With arguments
upper_words = list(map(methodcaller("upper"), ["hello", "world"]))
print(upper_words)
['hello', 'world', 'python'] ['HELLO', 'WORLD']

methodcaller("strip") creates a callable equivalent to lambda x: x.strip(). You can also pass arguments: methodcaller("replace", "a", "b") is equivalent to lambda x: x.replace("a", "b").

Python
# Operator functions — replace lambdas for arithmetic
from operator import add, mul, neg
from functools import reduce

nums = [1, 2, 3, 4, 5]
product = reduce(mul, nums)
print(product)

# Compare readability:
# reduce(lambda a, b: a * b, nums)  — lambda version
# reduce(mul, nums)                  — operator version
120

The operator module provides add, sub, mul, truediv, mod, pow, neg, and_, or_, not_, and more. Each is a direct C-level function — faster and more readable than an equivalent lambda in functional pipelines.

Python
# Combining with groupby — the full functional toolkit
from itertools import groupby
from operator import itemgetter

sales = [
    ("Electronics", 500), ("Clothing", 200),
    ("Electronics", 300), ("Clothing", 150),
    ("Food", 80),
]
sales.sort(key=itemgetter(0))
for category, items in groupby(sales, key=itemgetter(0)):
    total = sum(price for _, price in items)
    print(f"{category}: ${total}")
Clothing: $350 Electronics: $800 Food: $80

This example brings together all three modules: groupby from itertools, itemgetter from operator, and a generator expression for the sum. Notice how itemgetter(0) replaces lambda x: x[0] in both the sort key and the groupby key — cleaner and consistent.

Common Mistakes:
WRONG: Writing key=lambda x: x[1] when key=itemgetter(1) is clearer and faster.
RIGHT: Use itemgetter for index/key access in sort keys.
WRONG: Using attrgetter on dicts — it accesses attributes, not keys.
RIGHT: Use itemgetter for dict keys, attrgetter for object attributes.
WRONG: Confusing attrgetter("x.y") — it does nested attribute access (obj.x.y), not dict nesting.
RIGHT: attrgetter("x.y") is equivalent to lambda obj: obj.x.y.
Key Rules:
  • Use itemgetter instead of lambda x: x[key] — it’s faster (C implementation) and more readable.
  • Use attrgetter for sorting objects by attribute — especially with namedtuples and dataclasses.
  • operator functions (add, mul, etc.) are ideal replacements for lambdas in reduce() and map().

Use itemgetter (hint: you’ll need a lambda wrapping itemgetter, or sort twice using sort stability).

3 Rewrite this code to use no lambdas — only operator module functions: from functools import reduce; result = reduce(lambda a, b: a + b, map(lambda x: x ** 2, range(1, 6))). Expected result: 55.
