ChainMap

Collections • 9 days overdue • rep 2

Quick Reminder

collections.ChainMap overlays multiple dictionaries into a single logical mapping. Lookups search the maps in order — first match wins. This makes it ideal for layered configuration where user settings override defaults without copying or merging dicts.

Key Pattern

from collections import ChainMap

defaults = {"color": "blue", "size": "medium", "debug": False}
user_config = {"color": "red", "debug": True}

config = ChainMap(user_config, defaults)
print(config["color"])   # red   (from user_config — first map wins)
print(config["size"])    # medium (falls through to defaults)
print(config["debug"])   # True  (from user_config)

# Iterate sees all unique keys
print(list(config.keys()))

# .maps gives you the underlying list of dicts
print(len(config.maps))  # 2
red medium True ['color', 'debug', 'size'] 2
Challenge

Implement a 3-layer config system: environment variables (highest priority) > config file (dict) > hard-coded defaults. Use ChainMap with os.environ as the first map. Look up keys "APP_HOST" and "APP_PORT". Set an env var with os.environ["APP_HOST"] = "0.0.0.0" before building the chain and verify it overrides the file config.

Pitfall: Mutations (assignments, deletions) only affect the first mapping in the chain. Writing config["size"] = "large" stores the value in user_config, not defaults. And del config["size"] raises KeyError because "size" lives in defaults, not the first map.

Closures & nonlocal

Functions • 9 days overdue • rep 2

Quick Reminder

An inner function captures variables from its enclosing scope, forming a closure. The inner function retains access to those variables even after the outer function returns. Use nonlocal to mutate captured variables — without it, assignment creates a new local variable instead of modifying the captured one.

Key Pattern

def make_counter(start=0):
    count = start
    def counter():
        nonlocal count
        count += 1
        return count
    return counter

c = make_counter(10)
print(c())  # 11
print(c())  # 12
print(c())  # 13

# Each call to make_counter creates a separate closure
d = make_counter(100)
print(d())  # 101
print(c())  # 14 — c's state is independent
11 12 13 101 14
Challenge

Write a rate_limiter(max_calls, period_seconds) closure that returns a function allow() -> bool. It returns True if fewer than max_calls have been made in the last period_seconds, False otherwise. Track timestamps with a list. Use time.time() for the current time. Prune old timestamps on each call.

Pitfall: Late binding — closures capture the variable, not its value at the time of creation. Classic trap: funcs = [lambda: i for i in range(3)] — all three lambdas return 2 because they all reference the same i variable, which is 2 after the loop. Fix with a default argument: lambda i=i: i.

Context Managers

Resource Mgmt • 9 days overdue • rep 1

Quick Reminder

The with statement guarantees cleanup via __enter__/__exit__ (class-based) or @contextmanager from contextlib (generator-based). Use them for file handles, locks, database connections, timers, temporary state changes — anything that needs deterministic cleanup.

Key Pattern

from contextlib import contextmanager
import time

@contextmanager
def timer(label):
    start = time.perf_counter()
    try:
        yield start  # value available as 'as' target
    finally:
        elapsed = time.perf_counter() - start
        print(f"{label}: {elapsed:.4f}s")

with timer("sort") as t:
    data = sorted(range(100_000, 0, -1))
    print(f"Sorted {len(data)} items")
Sorted 100000 items sort: 0.0054s
Challenge

Write a @contextmanager called db_transaction(conn) that calls conn.begin() on entry, yields the connection, calls conn.commit() on success, or conn.rollback() on any exception. The exception must be re-raised after rollback so callers see it. Test with a mock connection object.

Pitfall: Forgetting to yield inside a @contextmanager function. Without yield, the function is a regular generator that never pauses for the with block body to execute. The with block appears to do nothing.

Lambda Functions

Functions • 9 days overdue • rep 2

Quick Reminder

Anonymous single-expression functions: lambda args: expression. They return the value of the expression automatically. Best used as short, throwaway callbacks for sorted(), map(), filter(), and similar higher-order functions.

Key Pattern

students = [
    {"name": "Alice", "age": 22, "gpa": 3.8},
    {"name": "Bob", "age": 19, "gpa": 3.5},
    {"name": "Carol", "age": 25, "gpa": 3.9},
]

# Sort by age
by_age = sorted(students, key=lambda s: s["age"])
print("By age:", [s["name"] for s in by_age])

# Sort by GPA descending, then name ascending
by_gpa = sorted(students, key=lambda s: (-s["gpa"], s["name"]))
print("By GPA:", [s["name"] for s in by_gpa])

# Filter with lambda
adults = list(filter(lambda s: s["age"] >= 21, students))
print("Adults:", [s["name"] for s in adults])
By age: ['Bob', 'Alice', 'Carol'] By GPA: ['Carol', 'Alice', 'Bob'] Adults: ['Alice', 'Carol']
Challenge

Using only map(), filter(), and functools.reduce() with lambdas, take a list of integers [1, 2, 3, 4, 5, 6, 7] and: (1) square each value, (2) keep only values greater than 10, (3) sum the remaining values. Write it as one chained expression without intermediate variables.

Pitfall: Assigning a lambda to a variable violates PEP 8. Instead of double = lambda x: x * 2, write def double(x): return x * 2. Named functions have better debugging output (double vs <lambda> in tracebacks) and support docstrings.

functools

Functional • 9 days overdue • rep 2

Quick Reminder

Key tools: partial (freeze some arguments), reduce (fold a sequence), lru_cache (memoize function calls), total_ordering (auto-generate comparison methods from __eq__ + one other), singledispatch (function overloading by argument type).

Key Pattern

from functools import lru_cache

@lru_cache(maxsize=256)
def fibonacci(n):
    if n < 2:
        return n
    return fibonacci(n - 1) + fibonacci(n - 2)

print(fibonacci(50))
info = fibonacci.cache_info()
print(f"Hits: {info.hits}, Misses: {info.misses}, Size: {info.currsize}")
12586269025 Hits: 48, Misses: 51, Size: 51
Challenge

Implement memoized Fibonacci with @lru_cache. Compute fib(100), then print the cache stats showing hits vs. misses. Also use functools.partial to create a fib_mod function that computes fibonacci(n) % m for a fixed modulus.

Pitfall: Passing unhashable arguments (lists, dicts, sets) to an @lru_cache-decorated function raises TypeError. Convert to tuples or frozensets before calling. Example: f(tuple(my_list)) instead of f(my_list).

Import System

Modules • 9 days overdue • rep 0

Quick Reminder

Python’s import system searches sys.path for modules. import foo and from foo import bar are the basic forms. Use importlib.import_module("foo.bar") for dynamic/lazy imports (e.g., plugin systems). __all__ controls what from pkg import * exports. Modules are cached in sys.modules after first import.

Key Pattern

import importlib
import sys

# Dynamic import — useful for plugin systems
def load_module(module_name):
    """Lazily import a module by name."""
    if module_name in sys.modules:
        print(f"  {module_name} already cached")
        return sys.modules[module_name]
    print(f"  Loading {module_name}...")
    return importlib.import_module(module_name)

json_mod = load_module("json")
parser = json_mod.loads
data = parser('{"key": "value", "count": 42}')
print(data)

# Second import hits the cache
json_again = load_module("json")
print(f"Same module? {json_mod is json_again}")
Loading json... {'key': 'value', 'count': 42} json already cached Same module? True
Challenge

Write a plugin loader: given a package name (e.g., "plugins") and a base class, use importlib.import_module() combined with inspect.getmembers() and inspect.isclass() to discover all subclasses of the base class defined in that package’s modules. Return a dict mapping class names to class objects.

Pitfall: Circular imports — module A imports from B, which imports from A. The import partially completes, leading to ImportError or AttributeError for names that haven’t been defined yet. Fix: (1) move the import inside the function that needs it, (2) restructure so shared code lives in a third module, or (3) use lazy imports.

Advanced Standard Library II

weakref — Weak References

Memory GC

Why It Matters

Every normal variable assignment in Python creates a strong reference that keeps the object alive. This is usually what you want, but in three common patterns it causes memory leaks:

  • 2026-07-08 (Day 38)
  • Caches — a cache dict keeps objects alive even after every other part of the program has forgotten them.
  • Observer patterns — a subject holds references to listeners, preventing their garbage collection when they are no longer needed.
  • Parent-child cycles — child objects reference their parent, creating reference cycles that defeat simple reference counting.

The weakref module provides references that do not prevent garbage collection. When the last strong reference is deleted, the object is collected and any weak references to it return None.

Core API at a Glance

Function / ClassPurpose
weakref.ref(obj, callback=None)Create a weak reference; call it to get obj or None
weakref.proxy(obj)Transparent proxy; raises ReferenceError on dead access
weakref.WeakValueDictionary()Dict whose values are weak refs (auto-evict on GC)
weakref.WeakKeyDictionary()Dict whose keys are weak refs
weakref.WeakSet()Set of weak references
weakref.finalize(obj, func, *args)Call func when obj is collected (more robust than callbacks)

Example 1 — Basic weakref.ref() and Callback on Deletion

import weakref

class Sensor:
    def __init__(self, name):
        self.name = name
    def __repr__(self):
        return f"Sensor({self.name!r})"

def on_finalize(ref):
    """Called when the referent is about to be collected."""
    print(f"Callback fired! ref() -> {ref()}")

# Create a sensor and a weak reference to it
sensor = Sensor("temperature")
weak = weakref.ref(sensor, on_finalize)

# Access the object through the weak reference by CALLING it
print(f"weak() = {weak()}")
print(f"Same object? {weak() is sensor}")

# Check reference count
import sys
print(f"Strong ref count: {sys.getrefcount(sensor) - 1}")  # subtract 1 for getrefcount's own ref

# Delete the only strong reference
del sensor
# The callback fires BEFORE the ref becomes None
print(f"After del: weak() = {weak()}")
weak() = Sensor('temperature') Same object? True Strong ref count: 1 Callback fired! ref() -> None After del: weak() = None

Key points: (1) you call the weak reference with () to get the object — it returns None if collected; (2) the callback receives the dead ref as its argument, so ref() is already None inside the callback.

Example 2 — WeakValueDictionary for Caches That Don’t Prevent GC

A regular dict keeps its values alive forever. A WeakValueDictionary holds weak references as values — entries disappear automatically when the value object is garbage-collected elsewhere. This is perfect for caches that should not be the reason an object stays in memory.

import weakref

class ExpensiveResource:
    _id_counter = 0

    def __init__(self, name):
        ExpensiveResource._id_counter += 1
        self.id = ExpensiveResource._id_counter
        self.name = name
        self.data = list(range(1000))  # simulate large payload
        print(f"  Created {self!r}")

    def __repr__(self):
        return f"Resource(id={self.id}, name={self.name!r})"

    def __del__(self):
        print(f"  GC'd {self!r}")

# Build a cache with weak values
cache = weakref.WeakValueDictionary()

def get_resource(name):
    """Fetch from cache or create new."""
    resource = cache.get(name)
    if resource is not None:
        print(f"  Cache HIT for {name!r}")
        return resource
    print(f"  Cache MISS for {name!r}")
    resource = ExpensiveResource(name)
    cache[name] = resource
    return resource

# First load — cache miss
r1 = get_resource("database")
r2 = get_resource("database")  # cache hit
print(f"Same object? {r1 is r2}")
print(f"Cache keys: {list(cache.keys())}")

# Hold a strong ref, then release it
r3 = get_resource("network")
print(f"Cache keys: {list(cache.keys())}")

del r3  # network resource has no more strong refs
print(f"Cache keys after del r3: {list(cache.keys())}")

# database is still alive because r1 and r2 hold strong refs
del r1, r2
print(f"Cache keys after del r1, r2: {list(cache.keys())}")
Cache MISS for 'database' Created Resource(id=1, name='database') Cache HIT for 'database' Same object? True Cache keys: ['database'] Cache MISS for 'network' Created Resource(id=2, name='network') Cache keys: ['database', 'network'] GC'd Resource(id=2, name='network') Cache keys after del r3: ['database'] GC'd Resource(id=1, name='database') Cache keys after del r1, r2: []

Example 3 — Observer Pattern with Weak References (Real-World)

In event-driven systems, the publisher (subject) holds references to subscribers (observers). If those references are strong, deleting a subscriber elsewhere does not actually free it — the publisher keeps it alive. Weak references fix this by letting subscribers be collected when they are no longer needed.

import weakref

class EventBus:
    """Publish-subscribe hub with automatic dead-observer cleanup."""

    def __init__(self):
        self._subscribers = {}  # event_name -> list of (weakref, method_name)

    def subscribe(self, event, observer, method_name):
        """Subscribe an observer's method to an event using a weak reference."""
        ref = weakref.ref(observer, lambda dead_ref: self._prune(event))
        self._subscribers.setdefault(event, []).append((ref, method_name))
        print(f"  Subscribed {type(observer).__name__}.{method_name} to '{event}'")

    def _prune(self, event):
        """Remove dead weak references for a given event."""
        if event in self._subscribers:
            before = len(self._subscribers[event])
            self._subscribers[event] = [
                (r, m) for r, m in self._subscribers[event] if r() is not None
            ]
            after = len(self._subscribers[event])
            if before != after:
                print(f"  Pruned {before - after} dead subscriber(s) from '{event}'")

    def publish(self, event, *args, **kwargs):
        """Notify all living subscribers."""
        live = 0
        for ref, method_name in self._subscribers.get(event, []):
            obj = ref()
            if obj is not None:
                getattr(obj, method_name)(*args, **kwargs)
                live += 1
        print(f"  Published '{event}' to {live} subscriber(s)")

class FileLogger:
    def __init__(self, path):
        self.path = path
    def handle_event(self, msg):
        print(f"    [FileLogger:{self.path}] {msg}")

class MetricsCollector:
    def __init__(self, name):
        self.name = name
    def handle_event(self, msg):
        print(f"    [Metrics:{self.name}] {msg}")

# Set up the event bus
bus = EventBus()

# Create observers
file_log = FileLogger("/var/log/app.log")
metrics = MetricsCollector("prometheus")

# Subscribe
bus.subscribe("user.signup", file_log, "handle_event")
bus.subscribe("user.signup", metrics, "handle_event")

# Publish — both receive
print("\n--- Both alive ---")
bus.publish("user.signup", "User alice@example.com signed up")

# Delete the file logger
del file_log
print("\n--- After deleting FileLogger ---")
bus.publish("user.signup", "User bob@example.com signed up")

# Delete the metrics collector
del metrics
print("\n--- After deleting MetricsCollector ---")
bus.publish("user.signup", "User carol@example.com signed up")
Subscribed FileLogger.handle_event to 'user.signup' Subscribed MetricsCollector.handle_event to 'user.signup' --- Both alive --- [FileLogger:/var/log/app.log] User alice@example.com signed up [Metrics:prometheus] User alice@example.com signed up Published 'user.signup' to 2 subscriber(s) --- After deleting FileLogger --- Pruned 1 dead subscriber(s) from 'user.signup' [Metrics:prometheus] User bob@example.com signed up Published 'user.signup' to 1 subscriber(s) --- After deleting MetricsCollector --- Pruned 1 dead subscriber(s) from 'user.signup' Published 'user.signup' to 0 subscriber(s)
Key Insight: The EventBus never explicitly unsubscribes anyone. Subscribers are automatically removed when they are garbage-collected. This eliminates an entire class of bugs where forgotten unsubscribe calls cause memory leaks or stale callbacks.

Common Mistakes

Wrong — weakref to built-in immutable types
import weakref

# These all FAIL with TypeError:
# weakref.ref(42)         # int
# weakref.ref("hello")    # str
# weakref.ref((1, 2, 3))  # tuple
# weakref.ref(None)       # NoneType

# Most built-in types don't support
# weak references to save memory
Right — subclass or wrap in a user class
import weakref

class MyInt(int):
    """Subclass adds __weakref__ slot."""
    pass

x = MyInt(42)
ref = weakref.ref(x)
print(ref())   # 42
print(ref() + 8)  # 50 — still behaves like int
Wrong — using ref() result inside callback
def on_death(ref):
    # ref() is ALWAYS None here — the object
    # is already dead when the callback fires!
    obj = ref()
    print(obj.name)  # AttributeError: NoneType
Right — capture info before the object dies
def make_callback(name):
    """Capture data we need before the ref dies."""
    def on_death(ref):
        print(f"Object '{name}' was garbage collected")
    return on_death

sensor = Sensor("temp")
ref = weakref.ref(sensor, make_callback(sensor.name))

Key Rules

  • Use WeakValueDictionary for caches where you want automatic eviction when objects leave scope
  • Use weakref.ref() or WeakSet for observer/listener patterns to prevent subjects from keeping observers alive
  • Use weakref.finalize(obj, func, *args) for guaranteed cleanup actions — it is more robust than ref callbacks because it survives even if the finalizer itself is not referenced
  • You cannot create weak references to most built-in types: int, str, tuple, list, dict, set, bytes, NoneType
  • Don’t use weak references for simple ownership — use normal (strong) references when you want the object to stay alive
  • Always check if ref() returns None before using the object

functools.cached_property

Descriptors Performance

Why It Matters

Sometimes an attribute is expensive to compute — a database query, file parsing, matrix multiplication, or API call — but its value never changes for the lifetime of the object. Recomputing it on every access wastes resources. functools.cached_property computes the value once on first access and stores it directly in the instance’s __dict__. Subsequent accesses find the cached value in __dict__ without invoking the descriptor at all. Zero boilerplate memoization for attributes.

How It Works Under the Hood

cached_property is a non-data descriptor (it only defines __get__, not __set__). On first access:

  1. Python doesn’t find the name in the instance __dict__.
  2. It falls through to the class and finds the cached_property descriptor.
  3. The descriptor’s __get__ calls your function, stores the result in instance.__dict__[name].
  4. On subsequent accesses, Python finds the name in __dict__ first (instance dict takes priority over non-data descriptors), so the descriptor is never called again.

Example 1 — Basic cached_property vs property

from functools import cached_property
import time

class DataAnalyzer:
    def __init__(self, data):
        self._data = data

    @property
    def mean_property(self):
        """Recomputed on EVERY access."""
        print("  [property] Computing mean...")
        return sum(self._data) / len(self._data)

    @cached_property
    def mean_cached(self):
        """Computed ONCE, then cached in __dict__."""
        print("  [cached_property] Computing mean...")
        return sum(self._data) / len(self._data)

analyzer = DataAnalyzer(range(1_000_000))

print("=== @property (accessed 3 times) ===")
for _ in range(3):
    val = analyzer.mean_property  # prints each time!
print(f"  Result: {val}")

print("\n=== @cached_property (accessed 3 times) ===")
for _ in range(3):
    val = analyzer.mean_cached  # prints only ONCE
print(f"  Result: {val}")

# Proof: the value is now in __dict__
print(f"\n'mean_cached' in __dict__: {'mean_cached' in analyzer.__dict__}")
print(f"'mean_property' in __dict__: {'mean_property' in analyzer.__dict__}")
=== @property (accessed 3 times) === [property] Computing mean... [property] Computing mean... [property] Computing mean... Result: 499999.5 === @cached_property (accessed 3 times) === [cached_property] Computing mean... Result: 499999.5 'mean_cached' in __dict__: True 'mean_property' in __dict__: False

Example 2 — Using with dataclasses

from dataclasses import dataclass, field
from functools import cached_property
from typing import List
import hashlib

@dataclass
class Document:
    title: str
    content: str
    tags: List[str] = field(default_factory=list)

    @cached_property
    def word_count(self):
        """Expensive-ish computation done once."""
        print(f"  Computing word count for '{self.title}'...")
        return len(self.content.split())

    @cached_property
    def content_hash(self):
        """SHA-256 hash of content, computed once."""
        print(f"  Computing hash for '{self.title}'...")
        return hashlib.sha256(self.content.encode()).hexdigest()[:16]

    @cached_property
    def summary(self):
        """First 50 chars + word count."""
        return f"{self.content[:50]}... ({self.word_count} words)"

doc = Document(
    title="My Article",
    content="Python is a versatile programming language " * 100,
    tags=["python", "tutorial"]
)

print(f"Word count: {doc.word_count}")
print(f"Word count: {doc.word_count}")  # no recomputation
print(f"Hash: {doc.content_hash}")
print(f"Summary: {doc.summary}")  # word_count already cached

Example 3 — Lazy-Loaded Configuration (Real-World Pattern)

from functools import cached_property

class AppConfig:
    """
    Lazy-loading config: nothing is read until first attribute access.
    Supports invalidation for config reload.
    """

    def __init__(self, env="production"):
        self._env = env
        self._load_count = 0

    @cached_property
    def _raw(self):
        """Simulate expensive config load (file I/O, API call, etc.)."""
        self._load_count += 1
        print(f"  [{self._load_count}] Loading config for env='{self._env}'...")
        configs = {
            "production": {
                "host": "prod.db.internal",
                "port": 5432,
                "pool_size": 20,
                "debug": False,
                "log_level": "WARNING",
            },
            "development": {
                "host": "localhost",
                "port": 5432,
                "pool_size": 5,
                "debug": True,
                "log_level": "DEBUG",
            },
        }
        return configs.get(self._env, {})

    @cached_property
    def db_host(self):
        return self._raw["host"]

    @cached_property
    def db_port(self):
        return self._raw["port"]

    @cached_property
    def pool_size(self):
        return self._raw["pool_size"]

    @cached_property
    def debug(self):
        return self._raw["debug"]

    def invalidate(self):
        """Clear all cached properties to force a fresh load on next access."""
        for attr in ("_raw", "db_host", "db_port", "pool_size", "debug"):
            self.__dict__.pop(attr, None)
        print("  Config cache invalidated")

# Create config — nothing loaded yet
app = AppConfig("development")
print("Config created, nothing loaded yet")
print(f"Load count: {app._load_count}\n")

# First access triggers the load
print(f"Host: {app.db_host}")
print(f"Port: {app.db_port}")
print(f"Pool: {app.pool_size}")
print(f"Debug: {app.debug}")
print(f"Load count: {app._load_count}\n")

# All subsequent accesses are free
print(f"Host again: {app.db_host}")
print(f"Load count: {app._load_count}\n")

# Invalidate and reload
app.invalidate()
print(f"Host after invalidate: {app.db_host}")
print(f"Load count: {app._load_count}")

Wrong — using with __slots__ (no __dict__)

    def expensive(self):

# Python < 3.12: two threads can call the
# cached_property simultaneously, leading to
# double computation or race conditions

Right — add locking for thread safety

from functools import cached_property

    _lock = threading.Lock()

    @cached_property
    def value(self):
        with self._lock:
            # Double-check: another thread may have
            # cached it while we waited for the lock
            if "value" in self.__dict__:
                return self.__dict__["value"]
            return expensive_computation()

Key Rules

  • Use @cached_property for expensive, immutable computed attributes that should be calculated once
  • Use @property when the underlying data can change and the value must stay fresh
  • The cache lives in the instance’s __dict__ — it will not work with __slots__ unless you include "__dict__"
  • Invalidate the cache by deleting the attribute: del obj.attr_name — next access recomputes
  • Thread-safe in Python 3.12+ (uses a per-instance lock internally), but not in earlier versions
  • cached_property replaces the descriptor with the value in __dict__ — there is no setter or deleter like @property

property Setters & Deleters

OOP Descriptors

Why It Matters

Properties let you add validation, computation, logging, or side effects to attribute access without changing the caller’s interface. Code that previously did obj.x = value still works, but now the assignment can validate input, transform data, or trigger actions. This is Python’s answer to Java-style getters/setters — but with a much cleaner syntax.

Core Idea

The @property decorator creates a getter. You chain @name.setter to intercept assignment and @name.deleter to intercept del. The actual data is stored in a “backing attribute” (conventionally prefixed with _). Under the hood, property is a data descriptor (it defines both __get__ and __set__), which means it takes priority over instance __dict__ entries.

Example 1 — Basic @property Getter (Read-Only)

import math

class Circle:
    def __init__(self, radius):
        self._radius = radius  # backing attribute

    @property
    def radius(self):
        """Read-only access to the radius."""
        return self._radius

    @property
    def diameter(self):
        """Computed property: always 2x radius."""
        return self._radius * 2

    @property
    def area(self):
        """Computed property: always up to date."""
        return math.pi * self._radius ** 2

    @property
    def circumference(self):
        return 2 * math.pi * self._radius

c = Circle(5)
print(f"Radius: {c.radius}")
print(f"Diameter: {c.diameter}")
print(f"Area: {c.area:.4f}")
print(f"Circumference: {c.circumference:.4f}")

# Attempting to set a read-only property
try:
    c.radius = 10
except AttributeError as e:
    print(f"\nAttributeError: {e}")

# But we can still set _radius directly (no protection)
c._radius = 10
print(f"\nAfter _radius = 10:")
print(f"Radius: {c.radius}, Diameter: {c.diameter}, Area: {c.area:.4f}")

class CachedFile:
    """Lazy-loading file with explicit cache management via deleter."""

    def __init__(self, path):
        self._path = path
        self._content = None
        self._access_count = 0

    @property
    def content(self):
        self._access_count += 1
        if self._content is None:
            print(f"  [READ] Loading {self._path} (access #{self._access_count})")
            # Simulate file read
            self._content = f"=== Contents of {self._path} ==="
        else:
            print(f"  [CACHE HIT] (access #{self._access_count})")
        return self._content

    @content.setter
    def content(self, value):
        print(f"  [WRITE] Updating content of {self._path}")
        self._content = value

    @content.deleter
    def content(self):
        print(f"  [INVALIDATE] Clearing cache for {self._path}")
        self._content = None

f = CachedFile("/etc/config.yaml")

# First access — triggers load
print(f.content)

# Second access — cache hit
print(f.content)

# Overwrite content
f.content = "new configuration data"
print(f.content)

# Invalidate cache
del f.content

# Next access reloads
print(f.content)

Example 4 — Temperature Class with Celsius/Fahrenheit/Kelvin

This real-world example shows how multiple properties can share a single backing attribute, providing different “views” of the same data with automatic conversion:

class Temperature:
    """Temperature with automatic unit conversion via properties."""

    ABSOLUTE_ZERO_C = -273.15

    def __init__(self, celsius=0.0):
        self.celsius = celsius  # goes through the setter!

    # --- Celsius (primary storage) ---
    @property
    def celsius(self):
        return self._celsius

    @celsius.setter
    def celsius(self, value):
        if value < self.ABSOLUTE_ZERO_C:
            raise ValueError(
                f"Temperature {value}C is below absolute zero ({self.ABSOLUTE_ZERO_C}C)"
            )
        self._celsius = round(float(value), 2)

    # --- Fahrenheit (computed from celsius) ---
    @property
    def fahrenheit(self):
        return round(self._celsius * 9 / 5 + 32, 2)

    @fahrenheit.setter
    def fahrenheit(self, value):
        self.celsius = (value - 32) * 5 / 9  # delegates to celsius setter

    # --- Kelvin (computed from celsius) ---
    @property
    def kelvin(self):
        return round(self._celsius + 273.15, 2)

    @kelvin.setter
    def kelvin(self, value):
        self.celsius = value - 273.15  # delegates to celsius setter

    def __repr__(self):
        return f"Temperature({self._celsius}C | {self.fahrenheit}F | {self.kelvin}K)"

    def __eq__(self, other):
        if isinstance(other, Temperature):
            return self._celsius == other._celsius
        return NotImplemented

# Create using Celsius

print(f"Water boils at: {water_boil}")

# Create using Fahrenheit

body_temp.fahrenheit = 98.6
print(f"Body temperature: {body_temp}")

# Create using Kelvin
absolute_zero = Temperature()
absolute_zero.kelvin = 0
print(f"Absolute zero: {absolute_zero}")

# Freezing point

print(f"Water freezes at: {freezing}")
print(f"0C == 32F? {freezing.fahrenheit == 32.0}")

    cold = Temperature()

    print(f"  Kelvin setter: {e}")

    bad = Temperature(-300)  # below absolute zero
except ValueError as e:
    print(f"  Celsius constructor: {e}")

Common Mistakes

Wrong — infinite recursion (most common bug)
class Bad:
    @property
    def x(self):
        return self.x  # calls the getter AGAIN!

    @x.setter
    def x(self, value):
        self.x = value  # calls the setter AGAIN!

# b = Bad()
# b.x = 5  # RecursionError: maximum recursion depth
Right — use underscore-prefixed backing attr
class Good:
    @property
    def x(self):
        return self._x  # different name!

    @x.setter
    def x(self, value):
        self._x = value  # different name!
Wrong — __init__ bypasses validation
class Account:
    def __init__(self, balance):
        self._balance = balance  # BYPASSES setter!

    @property
    def balance(self):
        return self._balance

    @balance.setter
    def balance(self, value):
        if value < 0:
            raise ValueError("Balance cannot be negative!")
        self._balance = value

a = Account(-1000)  # No error! Validation skipped!
Right — __init__ goes through the setter
class Account:
    def __init__(self, balance):
        self.balance = balance  # USES setter! Validates!

    @property
    def balance(self):
        return self._balance

    @balance.setter
    def balance(self, value):
        if value < 0:
            raise ValueError("Balance cannot be negative!")
        self._balance = value

# a = Account(-1000)  # ValueError! Caught!
Wrong — property for everything
class OverEngineered:
    @property
    def name(self):
        return self._name

    @name.setter
    def name(self, value):
        self._name = value  # no validation, no logic!

# Pointless! A plain attribute does the same thing.
Right — plain attrs when no logic needed
class Simple:
    def __init__(self, name):
        self.name = name  # plain attribute — fast, simple

# You can always add a property LATER if you need
# validation — Python's uniform access principle.

Key Rules

  • Use @property when you need validation, computation, logging, or side effects on attribute access
  • Prefer plain attributes when no logic is needed — don’t wrap everything in properties (YAGNI)
  • Always use a backing attribute with a different name (conventionally _name) to avoid infinite recursion
  • Route __init__ assignments through the setter (self.name = value) to get validation on construction
  • Use @name.deleter sparingly — mainly for cache invalidation or resource cleanup
  • Properties are data descriptors, so they override instance __dict__ — unlike cached_property

Example 3 — Custom Error Reporter with Context (Real-World)

Production services need structured error reports with context — request IDs, user info, timestamps. Here is a reusable error reporter:

import traceback
import sys
from datetime import datetime

class ErrorReporter:
    """
    Collects structured error reports for a service.
    In production, these would be sent to Sentry/Datadog/etc.
    """

    def __init__(self, service_name):
        self.service_name = service_name
        self.errors = []

    def capture(self, context=None):
        """Call inside an except block to capture the current exception."""
        exc_type, exc_value, exc_tb = sys.exc_info()
        if exc_type is None:
            return None

        frames = traceback.extract_tb(exc_tb)

        error_record = {
            "timestamp": datetime.now().isoformat(),
            "service": self.service_name,
            "error_type": exc_type.__name__,
            "error_module": exc_type.__module__,
            "message": str(exc_value),
            "context": context or {},
            "stack_depth": len(frames),
            "stack": [
                {
                    "file": f.filename,
                    "line": f.lineno,
                    "function": f.name,
                    "code": f.line,
                }
                for f in frames
            ],
            "origin": {
                "function": frames[-1].name if frames else "unknown",
                "file": frames[-1].filename if frames else "unknown",
                "line": frames[-1].lineno if frames else -1,
            },
        }
        self.errors.append(error_record)
        return error_record

    def summary(self):
        """Group errors by type."""
        by_type = {}
        for err in self.errors:
            t = err["error_type"]
            by_type[t] = by_type.get(t, 0) + 1
        return {
            "total": len(self.errors),
            "by_type": by_type,
            "service": self.service_name,
        }

Common Mistakes

Wrong — printing traceback to stdout in production

    risky_operation()

    # Goes to stdout — gets mixed with
    # normal program output!
    print(traceback.format_exc())

    # Or worse: silently swallowing it
    pass  # NO!
Right — use logging.exception()
import logging
logger = logging.getLogger(__name__)

    risky_operation()
except Exception:
    # .exception() automatically includes
    # the full traceback in the log record
    logger.exception("Unexpected error in risky_operation")
    # Goes to stderr/file/aggregator, with level ERROR
Wrong — losing the exception chain
try:
    result = parse_config(path)
except ValueError:
    # The original traceback is LOST
    msg = str(sys.exc_info()[1])
    raise RuntimeError(msg)
    # Caller only sees RuntimeError, not the
    # original ValueError and where it happened
Right — preserve the chain with raise...from
try:
    result = parse_config(path)
except ValueError as original:
    # Both tracebacks are preserved!
    raise RuntimeError(
        f"Config error in {path}"
    ) from original
    # Caller sees: "RuntimeError ... caused by ValueError ..."
Wrong — bare except that hides bugs
try:
    complex_operation()
except:  # catches EVERYTHING including KeyboardInterrupt!
    traceback.print_exc()
    # Might swallow SystemExit, KeyboardInterrupt...

try:
    complex_operation()

    raise  # re-raise after logging

Key Rules

  • Use traceback.format_exc() to get the current exception traceback as a string for logging
  • Use traceback.extract_tb(tb) for programmatic access to individual stack frames
  • Use logging.exception(msg) instead of print() for production error reporting
  • Never silently swallow tracebacks — at minimum, log them at WARNING or ERROR level
  • Use raise NewError(...) from original_error to preserve exception chains
  • Use traceback.format_exception(exc) (Python 3.10+) when you have the exception object but not sys.exc_info()

Example 1 — atexit.register() for Cleanup on Exit

atexit functions run in LIFO (last registered, first called) order when the Python interpreter exits normally — via sys.exit(), end of main(), or unhandled exception (but NOT SIGKILL or os._exit()).

import atexit
import tempfile
import os

# Simulate resources that need cleanup
_temp_files = []
_open_connections = []

def cleanup_temp_files():
    """Remove all temporary files we created."""
    print(f"[atexit] Cleaning up {len(_temp_files)} temp files...")
    for path in _temp_files:
        if os.path.exists(path):
            os.unlink(path)
            print(f"  Deleted {path}")
    _temp_files.clear()

def cleanup_connections():
    """Close all open connections."""
    print(f"[atexit] Closing {len(_open_connections)} connections...")
    for conn in _open_connections:
        print(f"  Closed {conn}")
    _open_connections.clear()

# Register cleanup functions — LIFO order
atexit.register(cleanup_temp_files)    # runs second
atexit.register(cleanup_connections)   # runs first (LIFO)

# Simulate app work
_open_connections.append("postgres://localhost/mydb")
_open_connections.append("redis://localhost:6379")
_temp_files.append("/tmp/session_abc123.json")
_temp_files.append("/tmp/upload_xyz789.dat")

print("Application running...")
print(f"Open connections: {_open_connections}")
print(f"Temp files: {_temp_files}")
print("Application shutting down...")

# When the script ends, atexit handlers fire automatically.
# Simulated output shows what happens at exit:

You can also use atexit.register as a decorator for functions that take no arguments:

import atexit

@atexit.register
def say_goodbye():
    print("[atexit] Goodbye! Application exiting cleanly.")

# Can also register with arguments:
def log_shutdown(service_name, code):
    print(f"[atexit] Service '{service_name}' exiting with code {code}")

atexit.register(log_shutdown, "api-gateway", 0)

print("Working...")

Example 2 — signal.signal() for Ctrl+C and SIGTERM Handling

Signal handlers let you intercept OS signals before they kill your process. The handler receives two arguments: the signal number and the current stack frame.

import signal
import sys

class GracefulShutdown:
    """
    Intercept SIGINT (Ctrl+C) and SIGTERM (kill)
    to perform graceful shutdown.
    """

    def __init__(self):
        self.shutdown_requested = False
        self.request_count = 0

        # Install signal handlers
        signal.signal(signal.SIGINT, self._handle_signal)
        signal.signal(signal.SIGTERM, self._handle_signal)

    def _handle_signal(self, signum, frame):
        """Called when SIGINT or SIGTERM is received."""
        self.request_count += 1
        sig_name = signal.Signals(signum).name
        print(f"\n[SIGNAL] Received {sig_name} (#{self.request_count})")

        if self.request_count == 1:
            print("[SIGNAL] Graceful shutdown initiated. Finishing current work...")
            self.shutdown_requested = True
        else:
            print("[SIGNAL] Second signal! Forcing immediate exit.")
            sys.exit(1)

# Demo
shutdown = GracefulShutdown()
print("Server ready. Press Ctrl+C to initiate graceful shutdown.")
print("(Simulating signal delivery for demo)\n")

# Simulate work loop
tasks = ["process_order_101", "send_email_202", "update_cache_303"]

for task in tasks:
    if shutdown.shutdown_requested:
        print(f"  Skipping {task} — shutting down")
        continue
    print(f"  Completed: {task}")

    # Simulate receiving SIGTERM after the first task
    if task == tasks[0]:
        print("\n  --- SIGTERM arrives here ---")
        shutdown._handle_signal(signal.SIGTERM, None)

print("\nAll in-flight work complete.")
print(f"Shutdown was requested: {shutdown.shutdown_requested}")

Example 3 — signal.alarm() for Timeouts (Unix Only)

signal.alarm(seconds) schedules a SIGALRM to be delivered after the specified time. By installing a handler that raises an exception, you can enforce timeouts on any synchronous operation:

import signal

class OperationTimeout(Exception):
    """Raised when an operation exceeds its time limit."""
    pass

def timeout_handler(signum, frame):
    raise OperationTimeout("Operation exceeded time limit!")

# Install the handler
signal.signal(signal.SIGALRM, timeout_handler)

def slow_computation():
    """Simulate a function that might run too long."""
    total = 0
    for i in range(50_000_000):
        total += i
    return total

def fast_computation():
    """A quick function that finishes in time."""
    return sum(range(1_000_000))

# Test 1: Timeout fires
signal.alarm(1)  # 1-second deadline
try:
    print("Starting slow computation (1s timeout)...")
    result = slow_computation()
    signal.alarm(0)  # cancel alarm on success
    print(f"Result: {result}")
except OperationTimeout as e:
    print(f"TIMEOUT: {e}")

# Test 2: Finishes in time
signal.alarm(10)  # generous 10-second deadline
try:
    print("\nStarting fast computation (10s timeout)...")
    result = fast_computation()
    signal.alarm(0)  # cancel alarm
    print(f"Result: {result}")
except OperationTimeout:
    print("TIMEOUT: unexpected!")

print("\nBoth tests complete.")
Starting slow computation (1s timeout)... TIMEOUT: Operation exceeded time limit! Starting fast computation (10s timeout)... Result: 499999500000 Both tests complete.
Important: Always call signal.alarm(0) to cancel the alarm after the operation succeeds! Otherwise the SIGALRM will fire later and interrupt whatever code is running at that point.

Example 4 — Graceful Server Shutdown (Production Pattern)

Real services combine atexit (normal exit cleanup) with signal handlers (external termination) for robust lifecycle management:

import signal
import atexit
import time
import sys

class ProductionServer:
    """
    Demonstrates the full lifecycle pattern used in production services:
    1. atexit handles normal shutdown cleanup
    2. signal handlers handle external termination (Ctrl+C, kill)
    3. Double-signal forces immediate exit
    """

    def __init__(self, name, port):
        self.name = name
        self.port = port
        self.running = False
        self._clients = []
        self._request_count = 0
        self._shutdown_count = 0
        self._setup_lifecycle()

    def _setup_lifecycle(self):
        """Register all lifecycle hooks."""
        # atexit: runs on normal exit (sys.exit, end of script)
        atexit.register(self._final_cleanup)

        # signals: run on external termination
        signal.signal(signal.SIGINT, self._on_signal)
        signal.signal(signal.SIGTERM, self._on_signal)

    def _on_signal(self, signum, frame):
        """Handle termination signals."""
        self._shutdown_count += 1
        sig_name = signal.Signals(signum).name
        print(f"\n[{self.name}] Received {sig_name} (attempt #{self._shutdown_count})")

        if self._shutdown_count >= 2:
            print(f"[{self.name}] Forced shutdown!")
            sys.exit(1)

        self.stop()

    def _final_cleanup(self):
        """Last-resort cleanup — runs even if stop() wasn't called."""
        if self._clients:
            print(f"[{self.name}] FINAL CLEANUP: {len(self._clients)} orphaned clients")
            self._clients.clear()
        print(f"[{self.name}] Total requests served: {self._request_count}")
        print(f"[{self.name}] Goodbye.")

    def start(self):
        """Start the server."""
        print(f"[{self.name}] Starting on port {self.port}...")
        self.running = True
        # Simulate accepting connections
        self._clients = ["client-alpha", "client-beta", "client-gamma"]
        print(f"[{self.name}] Ready. Serving {len(self._clients)} clients.")

    def handle_request(self, request_id):
        """Simulate handling a request."""
        self._request_count += 1
        print(f"[{self.name}] Handled request #{request_id}")

    def stop(self):
        """Graceful shutdown: drain connections, save state."""
        if not self.running:
            print(f"[{self.name}] Already stopped.")
            return

        print(f"[{self.name}] Initiating graceful shutdown...")
        print(f"[{self.name}] Draining {len(self._clients)} client connections...")

        for client in self._clients:
            print(f"  Gracefully closing {client}...")
            time.sleep(0.01)  # simulate drain time

        self._clients.clear()
        self.running = False
        print(f"[{self.name}] Shutdown complete.")

# --- Demo ---
server = ProductionServer("api-server", 8080)
server.start()

# Handle some requests
for i in range(1, 4):
    server.handle_request(i)

# Simulate receiving SIGTERM from Kubernetes
print("\n--- Kubernetes sends SIGTERM ---")
server._on_signal(signal.SIGTERM, None)
print("\n--- Script exit (atexit handlers run) ---")

Common Mistakes

Wrong — signal handlers in non-main threads
import signal
import threading

def worker():
    # FAILS! Signals can only be registered
    # in the main thread.
    signal.signal(signal.SIGINT, my_handler)
    # ValueError: signal only works in main thread
Right — register in main thread, share via Event
import signal
import threading

shutdown_event = threading.Event()

# Register in main thread
def handler(signum, frame):
    print("Signal received, setting shutdown event")
    shutdown_event.set()

signal.signal(signal.SIGINT, handler)

# Workers check the event
def worker():
    while not shutdown_event.is_set():
        do_work()
        shutdown_event.wait(timeout=1.0)
    print("Worker exiting gracefully")
Wrong — assuming atexit runs on kill -9
import atexit

def save_critical_data():
    write_to_disk(important_data)

atexit.register(save_critical_data)

# atexit does NOT run on:
#   - SIGKILL (kill -9) — cannot be caught!
#   - os._exit() — bypasses all cleanup
#   - Fatal Python errors (segfault)
# Data may be LOST!
Right — combine signal + atexit + periodic saves
import signal, atexit

def save_critical_data():
    write_to_disk(important_data)

# atexit: covers normal exit
atexit.register(save_critical_data)

# signal: covers SIGTERM/SIGINT
def graceful_exit(signum, frame):
    save_critical_data()
    sys.exit(0)  # this WILL trigger atexit

signal.signal(signal.SIGTERM, graceful_exit)
signal.signal(signal.SIGINT, graceful_exit)

# For kill -9 protection: periodic checkpoint saves
# save_critical_data() every N minutes in the main loop
Wrong — complex work in signal handler
def handler(signum, frame):
    # BAD: signal handlers should be fast!
    # This could deadlock or cause issues.
    save_to_database(data)  # I/O in handler!
    send_email("shutting down")  # network in handler!
    cleanup_all_resources()  # too much work!
Right — set a flag, do work in main loop
shutdown_requested = False

def handler(signum, frame):
    global shutdown_requested
    shutdown_requested = True  # just set a flag!

# Main loop does the actual work
while not shutdown_requested:
    process_next_request()

# After loop: safe to do cleanup
save_to_database(data)
send_email("shutting down")

Key Rules

  • Use atexit.register() for cleanup on normal program exit: temp file deletion, connection closing, state saving
  • Use signal.signal() to intercept OS signals: SIGINT (Ctrl+C), SIGTERM (kill), SIGHUP (config reload)
  • Signal handlers can only be registered in the main thread — use threading.Event to communicate with worker threads
  • atexit does NOT run on SIGKILL (kill -9) or os._exit() — nothing can catch SIGKILL
  • Keep signal handlers minimal — set a flag or event, do actual cleanup in the main thread
  • For production services, combine all three: signal handler sets flag, main loop checks flag and cleans up, atexit handles the final edge cases
  • signal.alarm() is Unix-only; for cross-platform timeouts, use threading.Timer or concurrent.futures with timeout
