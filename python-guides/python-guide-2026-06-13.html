Generator Expressions

Generators • 6 days overdue • rep 2

Quick Reminder

Generator expressions use () instead of [] and produce items lazily — one at a time, without storing the whole sequence in memory. They are single-use iterators that can be passed directly to consuming functions without extra parentheses.

Key Pattern

# Genexp vs listcomp
squares_gen  = (x**2 for x in range(10))   # lazy, single-use
squares_list = [x**2 for x in range(10)]   # eager, reusable

# Pass directly to functions (no extra parens needed)
total = sum(x**2 for x in range(10))
print(total)  # 285
285
Challenge

Write a single expression using a generator that finds the first filename in a list that ends with .py and contains more than 20 characters. Use next() with a default of None. Do not convert to a list.

#1 Pitfall: A generator is exhausted after one pass — iterating it a second time yields nothing silently. If you assign a genexp to a variable and iterate it twice, the second loop produces zero items with no error.

namedtuple

Collections • 6 days overdue • rep 2

Quick Reminder

namedtuple creates lightweight, immutable tuple subclasses with named fields. They support indexing, unpacking, and iteration like regular tuples, plus provide _asdict(), _replace(), and _make() helper methods.

Key Pattern

from collections import namedtuple

Point = namedtuple('Point', ['x', 'y'])
p = Point(3, 4)

print(p.x, p.y)          # named access
print(p[0], p[1])         # index access
print(p._asdict())        # OrderedDict
print(p._replace(x=10))   # new instance with x=10
3 4 3 4 {'x': 3, 'y': 4} Point(x=10, y=4)
Challenge

Create a Color namedtuple with fields r, g, b and a default of 0 for all. Write a function that takes a list of Color instances and returns the one with the highest luminance using the formula 0.299*r + 0.587*g + 0.114*b.

#1 Pitfall: _replace() returns a new instance — it does not mutate in place. Writing p._replace(x=10) without assigning the result is a no-op. namedtuples are immutable; you cannot do p.x = 10.

Type Hints — Basic Annotations

Type System • due today • rep 2

Quick Reminder

Type hints annotate function parameters and return types without affecting runtime behavior. Since Python 3.9+, built-in types like list[int] and dict[str, int] can be used directly. Use Optional[X] (or X | None in 3.10+) for nullable types, and tuple[int, ...] for variable-length homogeneous tuples.

Key Pattern

def greet(name: str) -> str:
    return f"Hello, {name}!"

def process(items: list[int], lookup: dict[str, int]) -> tuple[int, ...]:
    return tuple(lookup.get(str(i), 0) for i in items)

# Type hints are metadata only - no runtime enforcement
result: int = "not an int"  # runs fine, mypy catches it
Challenge

Write a fully type-annotated function merge_dicts(a: dict[str, list[int]], b: dict[str, list[int]]) -> dict[str, list[int]] that merges two dictionaries, concatenating lists for shared keys. Include a return type and annotate any local variables.

#1 Pitfall: Type hints are not enforced at runtime. Passing "hello" to a function annotated x: int raises no error. You need a static checker like mypy or pyright to catch violations.

Classes — __init__, __repr__, __str__

OOP • due today • rep 2

Quick Reminder

__repr__ targets developers (unambiguous, ideally eval()-able), while __str__ targets end users (readable, human-friendly). If only __repr__ is defined, str() falls back to it. Always implement __repr__; add __str__ only when a different user-facing format is needed.

Key Pattern

class Temperature:
    def __init__(self, celsius: float):
        self.celsius = celsius

    def __repr__(self) -> str:
        return f"Temperature(celsius={self.celsius!r})"

    def __str__(self) -> str:
        return f"{self.celsius:.1f}°C"

t = Temperature(36.6)
print(repr(t))  # Temperature(celsius=36.6)
print(str(t))   # 36.6°C
print(t)        # 36.6°C  (print calls __str__)
Temperature(celsius=36.6) 36.6°C 36.6°C
Challenge

Create a Duration class that stores seconds internally. __repr__ should return Duration(seconds=3661). __str__ should return "1h 1m 1s". Handle zero components gracefully (omit 0h, 0m when appropriate).

#1 Pitfall: Implementing __str__ but not __repr__. In the REPL and debuggers, Python uses __repr__, so you get the unhelpful default <Temperature object at 0x...>. Always define __repr__ first.

copy: Shallow vs Deep Copy

Advanced stdlib • due today • rep 0

Quick Reminder

copy.copy() creates a new container but shares references to nested objects. copy.deepcopy() recursively clones everything, including nested mutables. Use shallow copy for flat structures with immutable elements; use deep copy when nested mutables must be independent.

Key Pattern

import copy

original = {"users": [{"name": "Alice"}, {"name": "Bob"}]}
shallow = copy.copy(original)
deep    = copy.deepcopy(original)

original["users"][0]["name"] = "CHANGED"
print(shallow["users"][0]["name"])  # CHANGED (shared inner ref)
print(deep["users"][0]["name"])     # Alice   (fully independent)
CHANGED Alice
Challenge

Create a class Config with nested dict settings. Implement __copy__ and __deepcopy__ manually. Demonstrate that shallow-copying shares the nested dict while deep-copying does not.

#1 Pitfall: dict.copy() and list.copy() are shallow copies, not deep. Developers often assume my_dict.copy() is safe for nested structures, but modifications to nested values propagate to the original.

Mocking with unittest.mock

Testing • due today • rep 1

Quick Reminder

@patch('module.ClassName') replaces an object with a MagicMock for the duration of a test. Use return_value to set what the mock returns, side_effect for exceptions or multiple return values, and assert_called_once_with() to verify call arguments.

Key Pattern

from unittest.mock import patch, MagicMock

# Patch where the object is USED, not where it's defined
@patch('myapp.service.requests.get')
def test_fetch_data(mock_get):
    mock_get.return_value = MagicMock(
        status_code=200,
        json=lambda: {"key": "value"}
    )
    # ... call function under test ...
    mock_get.assert_called_once_with("https://api.example.com/data")
Challenge

Write a test for a function send_email(to, subject, body) that calls smtplib.SMTP. Mock the SMTP connection so no real email is sent. Use side_effect to simulate a connection timeout on the second call. Verify the first call succeeded and the second raised.

#1 Pitfall: Patching at the wrong location. @patch('smtplib.SMTP') patches the definition site, but if your module does from smtplib import SMTP, you must patch 'mymodule.SMTP' instead — patch where it's looked up, not where it's defined.

asyncio Internals: Event Loop, Tasks & Futures

Advanced asyncio • concept_id: asyncio_internals

The asyncio event loop is a single-threaded scheduler that multiplexes I/O using OS-level selectors (epoll on Linux, kqueue on macOS). It continuously calls _run_once() in a loop: polling for I/O readiness, executing callbacks, and advancing coroutines. Understanding the distinction between coroutines, Tasks, and Futures is essential for writing efficient async code.

Example 1 — Coroutine vs Task vs Future

import asyncio

async def compute(x: int) -> int:
    await asyncio.sleep(0.1)
    return x * x

async def main():
    # A coroutine object: not scheduled until awaited or wrapped in a Task
    coro = compute(5)
    print(f"Coroutine: {coro}")

    # A Task: wraps a coroutine and schedules it on the event loop
    task = asyncio.create_task(compute(7))
    print(f"Task: {task}")

    # A Future: low-level promise of a result
    loop = asyncio.get_running_loop()
    future = loop.create_future()
    future.set_result(42)
    print(f"Future result: {future.result()}")

    # Await the task
    result = await task
    print(f"Task result: {result}")

    # Clean up unawaited coroutine
    await coro

asyncio.run(main())
Coroutine: <coroutine object compute at 0x...> Task: <Task pending name='Task-2' coro=<compute() running at ...>> Future result: 42 Task result: 49

Example 2 — TaskGroup for Structured Concurrency (3.11+)

import asyncio

async def fetch(name: str, delay: float) -> str:
    await asyncio.sleep(delay)
    return f"{name} done"

async def main():
    results = []
    async with asyncio.TaskGroup() as tg:
        # All tasks must complete (or one raises, cancelling the rest)
        t1 = tg.create_task(fetch("API-1", 0.2))
        t2 = tg.create_task(fetch("API-2", 0.1))
        t3 = tg.create_task(fetch("DB",    0.15))

    # After the block, all tasks are done
    print(t1.result())
    print(t2.result())
    print(t3.result())

asyncio.run(main())
API-1 done API-2 done DB done

Example 3 — asyncio.to_thread: Bridging Sync Code

import asyncio
import time

def blocking_io(name: str) -> str:
    """Simulate blocking I/O (e.g., file read, legacy library call)."""
    time.sleep(0.5)  # blocks the thread, NOT the event loop
    return f"{name}: finished blocking work"

async def main():
    start = time.perf_counter()

    # Run blocking functions in separate threads concurrently
    results = await asyncio.gather(
        asyncio.to_thread(blocking_io, "Task-A"),
        asyncio.to_thread(blocking_io, "Task-B"),
        asyncio.to_thread(blocking_io, "Task-C"),
    )

    elapsed = time.perf_counter() - start
    for r in results:
        print(r)
    print(f"Total: {elapsed:.2f}s (not 1.5s!)")

asyncio.run(main())
Task-A: finished blocking work Task-B: finished blocking work Task-C: finished blocking work Total: 0.50s (not 1.5s!)

Example 4 — Custom Event Loop Inspection

import asyncio

async def show_loop_internals():

Example 5 — Error Handling with TaskGroup

import asyncio

async def risky(name: str, fail: bool) -> str:
    await asyncio.sleep(0.1)
    if fail:
        raise ValueError(f"{name} failed!")
    return f"{name} OK"

async def main():
    try:
        async with asyncio.TaskGroup() as tg:
            tg.create_task(risky("A", fail=False))
            tg.create_task(risky("B", fail=True))   # will fail
            tg.create_task(risky("C", fail=False))
    except* ValueError as eg:
        # ExceptionGroup (3.11+) catches multiple errors
        for exc in eg.exceptions:
            print(f"Caught: {exc}")

asyncio.run(main())
Caught: B failed!
DO
# Use TaskGroup for structured concurrency
async with asyncio.TaskGroup() as tg:
    tg.create_task(coro1())
    tg.create_task(coro2())
# All tasks guaranteed done here
DON'T
# Fire-and-forget tasks (easy to lose errors)
asyncio.create_task(coro1())
asyncio.create_task(coro2())
# Tasks may still be running at shutdown

Typing: Protocol, NewType & TypeAlias

Advanced Type System • concept_id: typing_runtime_protocol

Python's typing module provides advanced tools for precise type specifications beyond basic annotations. Protocol enables structural subtyping (duck typing with static checking), NewType creates distinct types at zero runtime cost, TypeAlias makes alias declarations explicit, and TypedDict types dictionaries with specific key-value schemas.

Example 1 — @runtime_checkable Protocol

from typing import Protocol, runtime_checkable

@runtime_checkable
class Renderable(Protocol):
    def render(self) -> str: ...

class HTMLWidget:
    def render(self) -> str:
        return "<div>Widget</div>"

class JSONResponse:
    def render(self) -> str:
        return '{"status": "ok"}'

class PlainText:
    pass  # no render method

def display(item: Renderable) -> None:
    print(item.render())

# Structural typing: no inheritance needed
display(HTMLWidget())
display(JSONResponse())

# Runtime isinstance checks work with @runtime_checkable
print(isinstance(HTMLWidget(), Renderable))   # True
print(isinstance(PlainText(), Renderable))    # False
<div>Widget</div> {"status": "ok"} True False

Example 2 — NewType for Type-Safe IDs

from typing import NewType

UserId = NewType('UserId', int)
OrderId = NewType('OrderId', int)

def get_user(uid: UserId) -> str:
    return f"User#{uid}"

def get_order(oid: OrderId) -> str:
    return f"Order#{oid}"

# At runtime, NewType is just the base type
uid = UserId(42)
oid = OrderId(42)

print(get_user(uid))     # User#42
print(type(uid))         # <class 'int'> (zero runtime overhead)

# mypy would catch this mistake:
# get_user(oid)  # error: Argument 1 has incompatible type "OrderId"

# NewType values work like their base type
print(uid + 1)           # 43 (still an int at runtime)
User#42 <class 'int'> 43

Example 3 — TypeAlias (3.10+)

from typing import TypeAlias

# Explicit type alias declaration
Vector: TypeAlias = list[float]
Matrix: TypeAlias = list[Vector]
Callback: TypeAlias = callable  # simplified

# Without TypeAlias, this looks like a regular assignment
# and static checkers might treat it differently
UserScores: TypeAlias = dict[str, list[int]]

def normalize(matrix: Matrix) -> Matrix:
    max_val = max(val for row in matrix for val in row)
    return [[v / max_val for v in row] for row in matrix]

m: Matrix = [[1.0, 2.0], [3.0, 4.0]]
print(normalize(m))

scores: UserScores = {"alice": [90, 85], "bob": [78, 92]}
print(scores)
[[0.25, 0.5], [0.75, 1.0]] {'alice': [90, 85], 'bob': [78, 92]}

Example 4 — TypedDict for Structured Dictionaries

from typing import TypedDict, Required, NotRequired

class UserProfile(TypedDict):
    name: str
    email: str
    age: int
    bio: NotRequired[str]  # optional key (3.11+)

class APIResponse(TypedDict, total=False):
    # total=False makes all keys optional by default
    data: Required[dict]    # but this one is required
    error: str
    status_code: int

def greet_user(user: UserProfile) -> str:
    bio = user.get("bio", "No bio")
    return f"Hello {user['name']} ({user['email']}), age {user['age']}. Bio: {bio}"

user: UserProfile = {"name": "Alice", "email": "alice@test.com", "age": 30}
print(greet_user(user))

# TypedDicts are regular dicts at runtime
print(type(user))
print(isinstance(user, dict))
Hello Alice (alice@test.com), age 30. Bio: No bio <class 'dict'> True

Example 5 — Combining Protocol with Generic

from typing import Protocol, TypeVar, runtime_checkable

T = TypeVar('T')

@runtime_checkable
class Comparable(Protocol):
    def __lt__(self, other: object) -> bool: ...
    def __eq__(self, other: object) -> bool: ...

CT = TypeVar('CT', bound=Comparable)

def min_max(items: list[CT]) -> tuple[CT, CT]:
    """Works with any type that supports < and ==."""
    return min(items), max(items)

# Works with ints, strings, or any Comparable
print(min_max([3, 1, 4, 1, 5]))
print(min_max(["banana", "apple", "cherry"]))
print(isinstance(42, Comparable))      # True (int has __lt__ and __eq__)
print(isinstance("hi", Comparable))    # True
(1, 5) ('apple', 'cherry') True True

DON'T

class Saveable(ABC):

gc Module: Garbage Collection & Reference Cycles

Python Internals • concept_id: gc_module

CPython uses reference counting as its primary memory management strategy: each object has a count of references to it, and when that count reaches zero, the object is freed immediately. However, reference counting cannot detect reference cycles (A references B, B references A). The gc module provides a cyclic garbage collector that periodically detects and collects such cycles.

Example 1 — Reference Counting in Action

import sys

a = [1, 2, 3]
print(f"refcount after creation: {sys.getrefcount(a) - 1}")  # -1 for getrefcount's own ref

b = a  # another reference
print(f"refcount after b = a:   {sys.getrefcount(a) - 1}")

c = [a, a, a]  # three more references
print(f"refcount after c = [...]: {sys.getrefcount(a) - 1}")

del b
print(f"refcount after del b:   {sys.getrefcount(a) - 1}")

del c
print(f"refcount after del c:   {sys.getrefcount(a) - 1}")
refcount after creation: 1 refcount after b = a: 2 refcount after c = [...]: 5 refcount after del b: 4 refcount after del c: 1

Example 2 — Detecting Reference Cycles

import gc

class Node:
    def __init__(self, name):
        self.name = name
        self.ref = None
    def __repr__(self):
        return f"Node({self.name!r})"

# Disable GC to observe cycles manually
gc.disable()

# Create a reference cycle
a = Node("A")
b = Node("B")
a.ref = b
b.ref = a  # cycle: A -> B -> A

# Delete external references
del a, b
# Objects still exist in memory (cycle prevents refcount from reaching 0)

# Manually collect
collected = gc.collect()
print(f"Objects collected: {collected}")

gc.enable()  # re-enable
Objects collected: 2

Example 3 — Inspecting the GC Generations

import gc

# GC uses three generations (0, 1, 2)
# Objects start in gen 0; survivors get promoted
thresholds = gc.get_threshold()
print(f"Thresholds (gen0, gen1, gen2): {thresholds}")

counts = gc.get_count()
print(f"Current counts (gen0, gen1, gen2): {counts}")

# Get stats for each generation
stats = gc.get_stats()
for i, s in enumerate(stats):
    print(f"Gen {i}: collections={s['collections']}, "
          f"collected={s['collected']}, uncollectable={s['uncollectable']}")
Thresholds (gen0, gen1, gen2): (700, 10, 10) Current counts (gen0, gen1, gen2): (87, 1, 1) Gen 0: collections=15, collected=120, uncollectable=0 Gen 1: collections=1, collected=0, uncollectable=0 Gen 2: collections=0, collected=0, uncollectable=0

Example 4 — Finding Memory Leaks with gc.get_referrers

import gc

class LeakyCache:
    _cache = {}  # class-level dict holds references forever

    def store(self, key, value):
        LeakyCache._cache[key] = value

cache = LeakyCache()
big_data = [0] * 1_000_000
cache.store("data", big_data)

# Find what's holding a reference to big_data
referrers = gc.get_referrers(big_data)
for ref in referrers:
    if isinstance(ref, dict):
        # Check if it's a class dict or instance dict
        for key, val in ref.items():
            if val is big_data:
                print(f"Found reference in dict with key: {key!r}")
                break

# How many objects does GC track?
all_objects = gc.get_objects()
print(f"Total tracked objects: {len(all_objects)}")

# Find specific types
node_count = sum(1 for obj in all_objects if isinstance(obj, LeakyCache))
print(f"LeakyCache instances: {node_count}")
Found reference in dict with key: 'data' Total tracked objects: 8432 LeakyCache instances: 1

Example 5 — __del__ Pitfalls and weakref Connection

import gc
import weakref

class Resource:
    def __init__(self, name):
        self.name = name
        print(f"Resource({name}) created")

    def __del__(self):
        print(f"Resource({self.name}) finalized")

# Normal cleanup
r = Resource("A")
del r  # __del__ called immediately (refcount -> 0)

# With cycles, __del__ makes objects uncollectable in Python < 3.4
# Python 3.4+ handles it, but order of __del__ calls is undefined

# Use weakref to avoid cycles entirely
class Parent:
    def __init__(self):
        self.child = None

class Child:
    def __init__(self, parent):
        self.parent = weakref.ref(parent)  # weak ref: doesn't increment refcount

p = Parent()
c = Child(p)
p.child = c

print(f"Parent alive: {c.parent() is not None}")
del p
print(f"Parent alive: {c.parent() is not None}")
Resource(A) created Resource(A) finalized Parent alive: True Parent alive: False
DO
# Use weakref for back-references
import weakref
class Child:
    def __init__(self, parent):
        self._parent = weakref.ref(parent)
DON'T
# Create strong reference cycles
class Child:
    def __init__(self, parent):
        self._parent = parent  # strong ref
        parent.child = self    # cycle!

ast Module: Parsing & Transforming Python Code

Python Internals • concept_id: ast_module

The ast module parses Python source code into an Abstract Syntax Tree — a tree of node objects representing the syntactic structure. You can walk, analyze, and even transform the tree before compiling it back to executable code. This is the foundation for linters, code formatters, security analyzers, and metaprogramming tools.

Example 1 — Parsing and Inspecting an AST

import ast

source = """
def greet(name):
    return f"Hello, {name}!"

x = greet("World")
"""

tree = ast.parse(source)
print(ast.dump(tree, indent=2)[:300])  # truncated for readability
print()

# Top-level structure
for node in ast.iter_child_nodes(tree):
    print(f"{type(node).__name__}: line {node.lineno}")
Module( body=[ FunctionDef( name='greet', args=arguments( posonlyargs=[], args=[ arg(arg='name')], ... ), body=[ Return( value=JoinedStr(... FunctionDef: line 2 Assign: line 5

Example 2 — Walking the AST to Find All Function Calls

import ast

source = """
import os
result = os.path.join("a", "b")
length = len(result)
print(f"Path: {result}, length: {length}")
"""

tree = ast.parse(source)

class CallFinder(ast.NodeVisitor):
    def visit_Call(self, node):
        # Extract the function name from the call
        if isinstance(node.func, ast.Name):
            print(f"Line {node.lineno}: {node.func.id}({len(node.args)} args)")
        elif isinstance(node.func, ast.Attribute):
            print(f"Line {node.lineno}: ...{node.func.attr}({len(node.args)} args)")
        self.generic_visit(node)  # continue walking child nodes

Example 3 — Transforming the AST with NodeTransformer

import ast

source = "result = 2 + 3 * 4"

class ConstantFolder(ast.NodeTransformer):
    """Replace constant arithmetic expressions with their results."""
    def visit_BinOp(self, node):
        self.generic_visit(node)  # transform children first

        if isinstance(node.left, ast.Constant) and isinstance(node.right, ast.Constant):
            # Evaluate the constant expression
            code = compile(ast.Expression(body=node), "<fold>", "eval")
            value = eval(code)
            return ast.Constant(value=value)
        return node

tree = ast.parse(source)

ast.fix_missing_locations(transformed)

Example 4 — Security: Detecting Dangerous Calls

import ast

source = """
import os
user_input = input("Enter command: ")
os.system(user_input)        # DANGEROUS
eval(user_input)             # DANGEROUS
subprocess.run(user_input)   # DANGEROUS
len(user_input)              # safe
"""

DANGEROUS_FUNCTIONS = {"eval", "exec", "compile", "system", "popen", "run"}

class SecurityChecker(ast.NodeVisitor):
    def __init__(self):
        self.warnings = []

    def visit_Call(self, node):
        name = None
        if isinstance(node.func, ast.Name):
            name = node.func.id
        elif isinstance(node.func, ast.Attribute):
            name = node.func.attr

        if name in DANGEROUS_FUNCTIONS:
            self.warnings.append(
                f"Line {node.lineno}: potentially dangerous call to '{name}'"
            )
        self.generic_visit(node)

checker = SecurityChecker()
checker.visit(ast.parse(source))
for w in checker.warnings:
    print(f"  WARNING: {w}")
WARNING: Line 4: potentially dangerous call to 'system' WARNING: Line 5: potentially dangerous call to 'eval' WARNING: Line 6: potentially dangerous call to 'run'

inspect Module: Introspecting Live Objects

Python Internals • concept_id: inspect_module

The inspect module provides tools to examine live Python objects: functions, classes, modules, frames, and code objects. It can retrieve source code, function signatures, call stacks, and class hierarchies at runtime. This is indispensable for debugging tools, documentation generators, and advanced decorators.

Example 1 — Function Signature Introspection

Example 4 — Call Stack Inspection

import inspect

def inner_function():
    """Print the call stack from inside."""
    stack = inspect.stack()
    print(f"Call stack depth: {len(stack)}")
    for i, frame_info in enumerate(stack[:4]):  # show top 4 frames
        print(f"  Frame {i}: {frame_info.function} "
              f"(line {frame_info.lineno})")
    print()

    # Get the caller's local variables
    caller_frame = inspect.currentframe().f_back
    print(f"Caller's locals: {list(caller_frame.f_locals.keys())}")
    # IMPORTANT: always delete frame references to avoid leaks
    del caller_frame

def middle_function():
    secret = 42
    message = "hello"
    inner_function()

def outer_function():
    middle_function()

outer_function()

Example 5 — Building a Decorator with inspect

import inspect
import functools
import time

def auto_log(func):
    """Decorator that auto-logs function calls with argument names."""
    sig = inspect.signature(func)

    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        # Bind arguments to parameter names
        bound = sig.bind(*args, **kwargs)
        bound.apply_defaults()

        params_str = ", ".join(
            f"{k}={v!r}" for k, v in bound.arguments.items()
        )
        print(f"CALL: {func.__name__}({params_str})")

        start = time.perf_counter()
        result = func(*args, **kwargs)
        elapsed = time.perf_counter() - start

        print(f"RETURN: {func.__name__} -> {result!r} [{elapsed:.4f}s]")
        return result

    return wrapper

@auto_log
def calculate(x: int, y: int, operation: str = "add") -> int:
    if operation == "add":
        return x + y
    return x * y

calculate(3, 7)
print()
calculate(5, 4, operation="multiply")
CALL: calculate(x=3, y=7, operation='add') RETURN: calculate -> 10 [0.0000s] CALL: calculate(x=5, y=4, operation='multiply') RETURN: calculate -> 20 [0.0000s]
DO
# Clean up frame references
frame = inspect.currentframe()
try:
    # use frame...
    pass
finally:
    del frame  # prevent reference cycles
DON'T
# Hold frame references indefinitely
frames = inspect.stack()
self.saved_frames = frames  # memory leak!
