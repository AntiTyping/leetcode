1. Thread-Safe Queues 16 days overdue rep 1 → 2

The queue module gives you thread-safe FIFO (Queue), LIFO (LifoQueue), and PriorityQueue containers — the safe way to hand work between producer and consumer threads. put/get lock internally; task_done() plus join() let the main thread wait until every queued item has been processed. 17 days since last review.

Key Pattern
import queue, threading

q = queue.Queue()

def worker():
    while True:
        item = q.get()             # blocks until an item is available
        if item is None:            # sentinel = shut down
            q.task_done()
            break
        q.task_done()              # mark this item handled

t = threading.Thread(target=worker); t.start()
for i in range(3):
    q.put(i)
q.put(None)                       # tell the worker to stop
q.join()                           # wait until every task_done() fired
t.join()
print("all done")                 # all done
Challenge

Build a producer/consumer pair: one thread puts ten numbers, one or more workers get them, square each, and store the result; use task_done()/join() so the main thread blocks until all work finishes. Then swap in a PriorityQueue and show items come out in priority order regardless of insertion order.

Pitfall

Every get() must be paired with exactly one task_done(), or join() hangs forever (too few) or raises ValueError (too many). A plain list is NOT thread-safe for cross-thread hand-off — use queue. And a bare get() blocks; pass timeout= or use get_nowait() if you must not block.

2. Standard Library HTTP Tools 16 days overdue rep 1 → 2

urllib.request.urlopen fetches a URL and returns a file-like response (.read(), .status, .headers); wrap the URL in a Request to set method, headers, or a body. urllib.parse handles the pieces around the request: urlencode a query dict, urlparse a URL into components, and parse_qs a query string back into a dict. 17 days since last review.

Key Pattern
from urllib.parse import urlencode, urlparse, parse_qs

qs = urlencode({"q": "python", "page": 2})
print(qs)                              # q=python&page=2

u = urlparse("https://ex.com/search?q=py&n=3")
print(u.scheme, u.netloc, u.path)      # https ex.com /search
print(parse_qs(u.query))               # {'q': ['py'], 'n': ['3']}

# from urllib.request import Request, urlopen
# req = Request("https://ex.com", headers={"User-Agent": "demo"})
# with urlopen(req) as resp: body = resp.read()
Challenge

Without touching the network, urlencode a params dict (including a list value — try doseq=True), then urlparse a full URL and pull out scheme, netloc, path, and query; feed the query into parse_qs. Then sketch (in comments) a Request with a custom User-Agent header opened via urlopen as a context manager.

Pitfall

urlopen raises HTTPError (a subclass of URLError) on 4xx/5xx — catch it, and always use the response as a context manager so the connection closes. parse_qs returns lists of values (a key can repeat); reach for parse_qsl if you want ordered pairs. Never build query strings by hand — urlencode does the percent-escaping for you.

3. I/O Multiplexing with selectors 16 days overdue rep 1 → 2

selectors lets one thread watch many sockets/files at once: register each with an interest mask (EVENT_READ/EVENT_WRITE) plus an arbitrary data payload, then call select() to get back only the objects that are ready. DefaultSelector picks the best backend for your OS (epoll/kqueue/select). It's the foundation under single-threaded servers and event loops. 17 days since last review.

Key Pattern
import selectors, socket

sel = selectors.DefaultSelector()   # best backend for this OS
srv = socket.socket()
srv.bind(("localhost", 0)); srv.listen()
srv.setblocking(False)

sel.register(srv, selectors.EVENT_READ, data="accept")   # watch for readability

for key, mask in sel.select(timeout=0):   # only ready objects come back
    print(key.data, key.fileobj)         # dispatch on the stored data tag

sel.close()
Challenge

Register a listening socket with DefaultSelector and EVENT_READ, then loop on select(timeout=...); on a ready listener, accept() the connection and register the new client socket with its own data tag so a single thread services many connections. Explain how key.data lets you route each ready event to the right handler.

Pitfall

Sockets must be non-blocking (setblocking(False)) or a "ready" fd can still stall you. select() with no timeout blocks until something is ready; pass timeout=0 to poll. You must unregister (or close) a socket before discarding it, and re-register to change the interest mask — a closed-but-registered fd corrupts the selector.

4. unittest Framework 16 days overdue rep 1 → 2

unittest is the batteries-included xUnit test framework: subclass TestCase, write test_* methods, and assert with assertEqual, assertIn, assertRaises, etc. setUp/tearDown run before/after each test; setUpClass once per class. Run with unittest.main() or the python -m unittest discovery CLI. 17 days since last review.

Key Pattern
import unittest

class TestMath(unittest.TestCase):
    def setUp(self):
        self.x = 10                 # fresh fixture per test

    def test_add(self):
        self.assertEqual(self.x + 5, 15)

    def test_raises(self):
        with self.assertRaises(ZeroDivisionError):
            _ = 1 / 0

# unittest.main() when run as a script, or build a suite:
suite = unittest.TestLoader().loadTestsFromTestCase(TestMath)
result = unittest.TextTestRunner(verbosity=0).run(suite)
print(result.wasSuccessful())      # True
Challenge

Write a TestCase with a setUp that builds a fixture, one test using assertEqual, one using assertIn, and one using assertRaises as a context manager. Add a @unittest.skipIf to a test and a subTest loop that checks several inputs in one method. Explain how setUp differs from setUpClass.

Pitfall

Test methods MUST start with test or the runner silently skips them. Assert with the framework methods (assertEqual), not bare assert — you lose the rich failure diff (and assert vanishes under python -O). setUp runs before EVERY test (isolation); setUpClass runs once — don't mutate shared class state between tests.

5. Advanced Enum: IntEnum, Flag, StrEnum, auto() 16 days overdue rep 1 → 2

auto() assigns member values for you. IntEnum/StrEnum members ARE ints/strs, so they compare and interoperate with those types. Flag (and IntFlag) members combine with bitwise |, &, ~ — perfect for permission/option sets. 17 days since last review.

Key Pattern
from enum import IntEnum, Flag, auto

class Perm(Flag):
    R = auto(); W = auto(); X = auto()   # 1, 2, 4

p = Perm.R | Perm.W
print(p)                     # Perm.R|Perm.W
print(Perm.W in p)            # True -- membership test

class Size(IntEnum):
    S = 1; M = 2; L = 3

print(Size.M < Size.L)       # True -- compares as int
print(Size.M + 0)           # 2  -- IS an int
Challenge

Define a Flag enum Perm(R, W, X) with auto(), combine members with |, test membership with in, and strip a flag with & ~Perm.W. Then make an IntEnum and show a member is == to the underlying int and sorts as one. Explain when IntEnum is a footgun (it leaks into arithmetic and JSON as a plain int).

Pitfall

auto() in a Flag yields powers of two (1, 2, 4…) — do NOT hand-assign 3, which silently means R|W. IntEnum members compare equal to bare ints, which is convenient but leaks type safety (they serialize as ints and mix into math). Plain Enum is NOT ordered — only IntEnum supports </>.

6. Metaclasses 14 days overdue rep 1 → 2

A metaclass is the "class of a class" — type is the default. Override __new__/__init__ on a subclass of type to inspect or rewrite a class as it is created (inject attributes, register subclasses, validate the namespace). Powerful, but for most jobs a class decorator or __init_subclass__ is simpler. 15 days since last review.

Key Pattern
class Meta(type):
    def __new__(mcs, name, bases, ns):
        ns["created_by"] = "Meta"       # inject into the class namespace
        print(f"building {name}")        # runs at class-definition time
        return super().__new__(mcs, name, bases, ns)

class Widget(metaclass=Meta):     # prints: building Widget
    pass

print(Widget.created_by)             # Meta
print(type(Widget) is Meta)          # True
Challenge

Write a metaclass whose __new__ injects a class attribute and keeps a registry of every class it builds (append name to a list on the metaclass). Create two classes using it and print the registry. Then re-implement the registry with __init_subclass__ instead, and explain why that is usually the better tool.

Pitfall

Metaclass __new__ runs once per class DEFINITION, not per instance — don't confuse it with __call__/instance creation. Two bases with different (non-related) metaclasses raise a "metaclass conflict" TypeError. Reach for a class decorator or __init_subclass__ first; a metaclass is the heavyweight option, justified only when you must control class creation itself.

New Concepts — Text Encoding & Escaping (2 concepts)

Two modules for moving text safely across boundaries. codecs is the machinery behind str.encode/bytes.decode — a registry of codecs, tunable error handlers, and streaming incremental coders for multibyte data split across chunks. html handles the web's escaping rules: turn text into safe markup (and back), and parse tags out of an HTML document.

Text & Encoding

Concept 1: codecs — Encodings, Error Handlers & Incremental Coders

Why learn this? str.encode/bytes.decode are the everyday face of the codecs registry, but the module gives you more: text↔text transforms (rot_13, hex), fine-grained error handlers for messy real-world bytes, and incremental decoders that correctly stitch a multibyte character split across two network chunks.
Example 1 — the codec registry: transforms beyond utf-8
import codecs

print(codecs.encode("hello", "rot_13"))   # text -> text cipher
print(codecs.decode("uryyb", "rot_13"))   # reverses it

print(codecs.encode(b"AB", "hex"))       # bytes -> hex bytes
print(codecs.decode(b"4142", "hex"))     # hex bytes -> bytes
Output
uryyb hello b'4142' b'AB'
Example 2 — error handlers on a bad decode
# b'\xe9' is latin-1 for 'e-acute' -- invalid on its own in UTF-8
raw = b"caf\xe9"

print(raw.decode("utf-8", errors="replace"))  # bad byte -> U+FFFD
print(raw.decode("utf-8", errors="ignore"))   # bad byte dropped
print(raw.decode("latin-1"))                 # right codec: clean decode
Output
caf� caf café
Example 3 — incremental decoder stitches a split character
import codecs

# the euro sign is 3 UTF-8 bytes: e2 82 ac -- arriving split
dec = codecs.getincrementaldecoder("utf-8")()
print(repr(dec.decode(b"\xe2\x82")))   # '' -- incomplete, buffered
print(repr(dec.decode(b"\xac")))       # the completed character
Output
'' '€'

Common Mistakes

✗ Decoding each network chunk with bytes.decode independently — a multibyte char split across chunks raises UnicodeDecodeError or corrupts

✓ Feed chunks to one getincrementaldecoder(...)(), which buffers the partial trailing bytes until the next chunk


✗ Reaching for errors="ignore" to make a decode "just work" — it silently deletes data

✓ Prefer the correct codec; use replace (visible U+FFFD) for display and reserve ignore for truly disposable bytes

Key Rules

  • codecs.encode/decode(obj, name) reaches the whole registry — including text↔text (rot_13) and bytes↔bytes (hex, zlib) codecs, not just utf-8
  • Error handlers on a decode: strict (default, raises), replace (U+FFFD / ?), ignore (drops), backslashreplace; pass via errors=
  • For streamed data use getincrementaldecoder(name)() and feed it chunks — it buffers incomplete trailing bytes so multibyte characters survive a split

Exercises

  1. Round-trip a string through the rot_13 codec with codecs.encode/decode, and a bytes object through the hex codec.
  2. Take b"caf\xe9" and decode it three ways: utf-8 with errors="replace", utf-8 with errors="ignore", and latin-1; compare the results.
  3. Create a UTF-8 incremental decoder and feed it the euro sign's bytes b"\xe2\x82\xac" two bytes then one; show the first call returns '' and the second returns the character.
Connections: codecs is the layer under the str/bytes boundary and unicodedata you have studied; its bytes↔bytes codecs overlap with base64/binascii; and incremental decoding is exactly what io.TextIOWrapper and async stream readers do for you.
Web & Text

Concept 2: html — Escaping & Parsing Markup

Why learn this? Dropping user text straight into HTML invites broken pages and XSS. html.escape converts the dangerous characters (< > & " ') into entities; html.unescape reverses named and numeric references. And html.parser.HTMLParser gives you an event-driven way to pull data out of a document without a third-party library.
Example 1 — escape then unescape (round trip)
import html

s = '<a href="?x=1&y=2">Tom & Jerry</a>'
esc = html.escape(s)              # quote=True by default: also escapes " and '
print(esc)
print(html.unescape(esc) == s)   # True -- reversible
Output
&lt;a href=&quot;?x=1&amp;y=2&quot;&gt;Tom &amp; Jerry&lt;/a&gt; True
Example 2 — unescape named & numeric references
import html

print(html.unescape("caf&eacute;"))   # named entity  -> accented e
print(html.unescape("&#233;"))       # decimal ref   -> accented e
print(html.unescape("&#x1F600;"))    # hex ref       -> emoji
Output
café é 😀
Example 3 — subclass HTMLParser to extract links
from html.parser import HTMLParser

class LinkExtractor(HTMLParser):
    def __init__(self):
        super().__init__()
        self.links = []

    def handle_starttag(self, tag, attrs):
        if tag == "a":
            for name, value in attrs:   # attrs is a list of (name, value)
                if name == "href":
                    self.links.append(value)

p = LinkExtractor()
p.feed('<p>See <a href="/a">A</a> and <a href="/b">B</a></p>')
print(p.links)                       # ['/a', '/b']
Output
['/a', '/b']

Common Mistakes

✗ Interpolating user text into an HTML attribute without escaping quotes — a stray " breaks out of the attribute (classic XSS)

✓ html.escape(value) keeps quote=True so " and ' become entities and can't close the attribute


✗ Reaching for re to parse real HTML — nesting, attributes, and entities defeat regexes fast

✓ Subclass HTMLParser (or use a real parser) and handle handle_starttag/handle_data events

Key Rules

  • html.escape(s, quote=True) converts & < > " ' to entities; keep quote=True whenever the value lands inside an attribute
  • html.unescape(s) resolves named (&eacute;) and numeric (&#233;, &#x1F600;) references back to characters
  • HTMLParser is event-driven: override handle_starttag/handle_endtag/handle_data and drive it with feed() — do not parse HTML with regexes

Exercises

  1. Escape a string containing <, &, and a double quote, then confirm html.unescape returns the exact original.
  2. Unescape a string mixing a named entity, a decimal reference, and a hex reference; verify each becomes the expected character.
  3. Subclass HTMLParser to collect the text inside every <li> tag (override handle_data and track whether you're inside an li), and run it over a small list markup snippet.
Connections: html.escape's job mirrors shlex.quote (shell) and parameterized SQL — escape at the boundary, never by hand; unescape resolves the same numeric references you saw in unicodedata; and the event-driven HTMLParser parallels xml.etree's streaming parse.

New Concepts — Parsing & Scheduling (2 concepts)

Two very different tools that both hand you fine control over a stream. tokenize is CPython's own lexer — the exact token stream the interpreter sees, available to your code for source analysis and rewriting. sched is a tiny, dependency-free event scheduler that runs callbacks at chosen times, ordered by time then priority.

Source Tooling

Concept 3: tokenize — Lexing Python Source

Why learn this? When you need to analyze or transform source code — a linter, a formatter, a macro, a "strip comments" tool — tokenize gives you the same token stream CPython's parser consumes: names, numbers, strings, operators, and the invisible INDENT/DEDENT/NEWLINE tokens that encode block structure. It sits one level below ast.
Example 1 — tokenize an expression
import tokenize, io

src = "x = 10 + 2\n"
for tok in tokenize.generate_tokens(io.StringIO(src).readline):
    print(tokenize.tok_name[tok.type], repr(tok.string))
Output
NAME 'x' OP '=' NUMBER '10' OP '+' NUMBER '2' NEWLINE '\n' ENDMARKER ''
Example 2 — pull out every identifier (keywords included)
import tokenize, io

code = "def add(a, b):\n    return a + b\n"
names = [t.string
         for t in tokenize.generate_tokens(io.StringIO(code).readline)
         if t.type == tokenize.NAME]      # tokenize does NOT split out keywords
print(names)
Output
['def', 'add', 'a', 'b', 'return', 'a', 'b']
Example 3 — INDENT / DEDENT encode block structure
import tokenize, io

code = "if x:\n    y\n"
for t in tokenize.generate_tokens(io.StringIO(code).readline):
    print(tokenize.tok_name[t.type])
Output
NAME NAME OP NEWLINE INDENT NAME NEWLINE DEDENT ENDMARKER

Common Mistakes

✗ Expecting keywords as their own token type — def, if, return all arrive as NAME; and forgetting tokenize.tokenize (bytes) emits a leading ENCODING token

✓ Use generate_tokens for str input (no ENCODING token) and check membership against keyword.kwlist when you need to tell keywords apart


✗ Passing a plain string to generate_tokens — it wants a readline callable, not the text

✓ Wrap the source: io.StringIO(src).readline (or an open file's .readline)

Key Rules

  • tokenize.generate_tokens(readline) takes a readline callable over str and yields TokenInfo(type, string, start, end, line) named tuples
  • Map a numeric type to its name with tokenize.tok_name[t.type]; block structure shows up as INDENT/DEDENT/NEWLINE tokens, ending with ENDMARKER
  • tokenize.tokenize(fp.readline) works on bytes and emits a leading ENCODING token; untokenize can rebuild source from a token stream

Exercises

  1. Tokenize "total = price * 2\n" and print each token's tok_name and string; identify which tokens are NAME, OP, and NUMBER.
  2. Count how many NUMBER tokens appear in a multi-line snippet using a generator expression over generate_tokens.
  3. Write a function that returns the set of identifiers in a snippet that are NOT Python keywords (combine tokenize with keyword.iskeyword).
Connections: tokenize is the lexing layer beneath ast (which gives you the parsed tree) and dis (which shows compiled bytecode); it reads the same keyword.kwlist you can query directly; and it pairs with io.StringIO for the required readline source.
Scheduling

Concept 4: sched — The Event Scheduler

Why learn this? When you need to run callbacks at specific delays or times — a timed simulation, a retry-with-backoff, a lightweight cron — sched.scheduler gives you an ordered event queue with no external dependencies. Because you supply the timefunc and delayfunc, you can drive it with real time or a virtual clock for deterministic tests.
Example 1 — events run in time order (virtual clock)
import sched

events = []
clock = [0.0]
def now(): return clock[0]
def advance(seconds): clock[0] += seconds   # virtual sleep -- never blocks

s = sched.scheduler(now, advance)
s.enter(3, 1, lambda: events.append("three"))   # enter(delay, priority, action)
s.enter(1, 1, lambda: events.append("one"))
s.enter(2, 1, lambda: events.append("two"))
s.run()                                    # runs earliest-first
print(events)                            # ['one', 'two', 'three']
print(clock[0])                          # 3.0 -- advanced to the last event
Output
['one', 'two', 'three'] 3.0
Example 2 — priority breaks ties at the same time
import sched

order = []
s = sched.scheduler(lambda: 0.0, lambda t: None)   # all fire at t=0
s.enter(0, 2, lambda: order.append("low prio"))
s.enter(0, 1, lambda: order.append("high prio"))   # smaller number = higher priority
s.run()
print(order)                            # ['high prio', 'low prio']
Output
['high prio', 'low prio']
Example 3 — enterabs & cancel
import sched

hits = []
clock = [0.0]
s = sched.scheduler(lambda: clock[0],
                    lambda t: clock.__setitem__(0, clock[0] + t))
e1 = s.enterabs(5, 1, lambda: hits.append("5s"))   # absolute time, not delay
e2 = s.enterabs(10, 1, lambda: hits.append("10s"))
s.cancel(e2)                            # drop the second event before running
print(len(s.queue))                     # 1 -- one event remains
s.run()
print(hits)                             # ['5s']
Output
1 ['5s']

Common Mistakes

✗ Expecting run() to return immediately — by default it BLOCKS (calling your delayfunc, usually time.sleep) until every event has fired

✓ Pass run(blocking=False) to run only what is already due and return the delay until the next event, or supply a non-blocking delayfunc


✗ Assuming a bigger priority number runs first — it's the reverse

✓ Lower priority numbers run first when two events share the same time; use it to order simultaneous callbacks

Key Rules

  • sched.scheduler(timefunc, delayfunc) is time-source agnostic — pass time.monotonic, time.sleep for real time, or fakes for deterministic tests
  • enter(delay, priority, action, argument=(), kwargs={}) schedules relative to now; enterabs(time, priority, ...) at an absolute time; both return an event handle for cancel
  • run() executes due events in (time, priority) order and blocks by default; run(blocking=False) polls; inspect pending events via s.queue

Exercises

  1. With a virtual clock (a timefunc/delayfunc pair over a mutable list), schedule three events at delays 2, 1, 3 and show they run in the order 1, 2, 3.
  2. Schedule two events at the same time with different priorities and confirm the lower-priority-number one runs first.
  3. Use enterabs to schedule two events, cancel one before run(), and verify via len(s.queue) and the recorded results that only the surviving event fired.
Connections: sched is the callback-timing cousin of threading.Timer and asyncio.call_later; its pluggable timefunc pairs with the time module (monotonic/perf_counter) you studied; and the virtual-clock trick is the same dependency-injection idea behind mocking.

New Concepts — Structured Messages (1 concept)

One module for the format that quietly underpins email, HTTP-ish headers, and MIME attachments. The modern email.message.EmailMessage API builds, reads, and multipart-assembles messages with a clean, dict-like header interface — no manual header wrangling.

Data Formats

Concept 5: email.message — Building & Parsing Email

Why learn this? Whenever you send mail (via smtplib), read an .eml file, or handle MIME parts, you need a structured message rather than hand-glued header strings. The modern EmailMessage class exposes headers like a dict, sets the body with correct MIME metadata via set_content, and assembles multipart alternatives — the right replacement for the older MIMEText machinery.
Example 1 — build a message, read its metadata
from email.message import EmailMessage

msg = EmailMessage()
msg["From"] = "alice@example.com"   # headers behave like a dict
msg["To"] = "bob@example.com"
msg["Subject"] = "Hi"
msg.set_content("Hello Bob!\n")       # body + correct MIME headers

print(msg["Subject"])              # Hi
print(msg.get_content_type())     # text/plain
Output
Hi text/plain
Example 2 — parse a raw message with the modern policy
from email import message_from_string
from email.policy import default

raw = "From: a@x.com\nSubject: Report\n\nBody text here\n"
msg = message_from_string(raw, policy=default)   # default -> EmailMessage

print(msg["From"])                    # a@x.com
print(msg["Subject"])                 # Report
print(msg.get_content().rstrip())   # Body text here
Output
a@x.com Report Body text here
Example 3 — a multipart alternative (plain + HTML)
from email.message import EmailMessage

msg = EmailMessage()
msg["Subject"] = "Newsletter"
msg.set_content("plain body")               # the text/plain fallback
msg.add_alternative("<b>rich</b>", subtype="html")  # the text/html version

print(msg.is_multipart())                    # True
print([p.get_content_type() for p in msg.iter_parts()])
Output
True ['text/plain', 'text/html']

Common Mistakes

✗ Parsing with plain message_from_string(raw) and expecting the modern API — without a policy you get the legacy Message, whose get_payload() won't decode content for you

✓ Pass policy=default to get an EmailMessage with get_content()/set_content()


✗ Setting the body by hand (msg.set_payload(text)) and forgetting the Content-Type/charset headers — recipients see mojibake

✓ Use set_content(...) / add_alternative(...), which set the MIME type, charset, and transfer-encoding for you

Key Rules

  • EmailMessage headers act like a case-insensitive dict: msg["Subject"] = ... to set, msg["Subject"] to read, msg.items() to iterate
  • set_content(body) sets a single-part body with correct MIME headers; add_alternative(body, subtype="html") turns it into a multipart/alternative
  • To parse, use email.message_from_string/bytes(raw, policy=default) for the modern EmailMessage; read the body with get_content(), walk parts with iter_parts()/walk()

Exercises

  1. Build an EmailMessage with From/To/Subject and a plain-text body via set_content; print the subject and get_content_type().
  2. Parse a raw header+body string with policy=default and print the From header and the decoded body from get_content().
  3. Create a multipart-alternative message (plain + HTML), then assert is_multipart() is true and list the content types of its parts via iter_parts().
Connections: email.message is the payload you would hand to smtplib for delivery; its dict-like headers echo configparser and HTTP header handling; and MIME type detection pairs naturally with the mimetypes module you studied for attachments.
