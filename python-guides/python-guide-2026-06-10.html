argparse for CLI Tools

Error Handling & I/O • 5 days overdue • rep 0

Quick Reminder

ArgumentParser handles CLI arg parsing. add_argument defines positional and optional args with type=, choices=, default=, and help=. Subparsers (add_subparsers) enable git-style subcommands. Use action='store_true' for boolean flags.

Key Pattern — Subcommands with add_subparsers

import argparse

parser = argparse.ArgumentParser(description="Task manager")
subparsers = parser.add_subparsers(dest="command", required=True)

# "add" subcommand
add_parser = subparsers.add_parser("add", help="Add a task")
add_parser.add_argument("title", help="Task title")
add_parser.add_argument("--priority", type=int, choices=[1,2,3], default=2)

# "list" subcommand
list_parser = subparsers.add_parser("list", help="List tasks")
list_parser.add_argument("--all", action="store_true", help="Include done")

# Parse
args = parser.parse_args(["add", "Buy groceries", "--priority", "1"])
print(args)
# Namespace(command='add', title='Buy groceries', priority=1)
Challenge

Build a CLI tool with two subcommands: add (takes a title and optional --priority) and list (takes an optional --all flag). Print the parsed arguments for each subcommand.

#1 Pitfall: type=bool doesn't work as expected — bool("False") is True because any non-empty string is truthy. Use action='store_true' for boolean flags instead.

async/await Basics

Concurrency • 5 days overdue • rep 0

Quick Reminder

async def defines a coroutine. await suspends until the awaitable completes. asyncio.gather(*coros) runs coroutines concurrently. asyncio.run(main()) starts the event loop. Always await coroutine calls — forgetting produces a bare coroutine object that never executes.

Key Pattern — Concurrent Fetching with Timeout

import asyncio

async def fetch(url: str, delay: float) -> str:
    print(f"  Fetching {url}...")
    await asyncio.sleep(delay)  # simulate network I/O
    return f"Data from {url}"

async def main():
    urls = [("api/users", 0.5), ("api/posts", 0.3), ("api/comments", 0.7)]
    tasks = [fetch(url, d) for url, d in urls]

    # Run all concurrently with a 1-second timeout
    results = await asyncio.wait_for(
        asyncio.gather(*tasks),
        timeout=1.0
    )
    for r in results:
        print(f"  Got: {r}")

asyncio.run(main())
Challenge

Write an async program that fetches 3 URLs concurrently (simulated with asyncio.sleep) and applies a 1-second timeout to the entire batch using asyncio.wait_for.

#1 Pitfall: Forgetting to await an async call gives you a coroutine object instead of the result. The coroutine never runs, and Python may warn: RuntimeWarning: coroutine was never awaited.

threading vs multiprocessing vs asyncio

Concurrency • 5 days overdue • rep 0

Quick Reminder

threading — shared memory, limited by GIL for CPU work, good for I/O-bound tasks. multiprocessing — separate processes, bypasses GIL, good for CPU-bound work, but data must be pickled. asyncio — single-threaded event loop, best for high-concurrency I/O (thousands of connections).

Key Pattern — Decision Matrix

# I/O-bound, few tasks     -> threading
# I/O-bound, many tasks    -> asyncio
# CPU-bound                 -> multiprocessing

import threading
import multiprocessing
import asyncio
import time

# Threading: I/O-bound
def io_task(name):
    time.sleep(0.5)
    return f"{name} done"

threads = [threading.Thread(target=io_task, args=(f"t{i}",)) for i in range(4)]
for t in threads: t.start()
for t in threads: t.join()
# All 4 finish in ~0.5s total (concurrent I/O)

# Multiprocessing: CPU-bound
def cpu_task(n):
    return sum(i*i for i in range(n))

with multiprocessing.Pool(4) as pool:
    results = pool.map(cpu_task, [10**6]*4)
# Uses all 4 CPU cores
Challenge

Benchmark an I/O-bound task (sleeping 0.5s) run 4 times using: (1) sequential, (2) threading, (3) asyncio. Compare total wall-clock time for each approach.

#1 Pitfall: Using threading for CPU-bound work. The GIL prevents true parallelism for CPU tasks in threads — 4 threads computing squares will take roughly the same time as sequential. Use multiprocessing for CPU-bound work.

concurrent.futures

Concurrency • 5 days overdue • rep 0

Quick Reminder

ThreadPoolExecutor for I/O-bound, ProcessPoolExecutor for CPU-bound. executor.submit(fn, *args) returns a Future. executor.map(fn, iterable) returns results in order. as_completed(futures) yields futures as they finish (not in order).

Key Pattern — submit + as_completed

from concurrent.futures import ProcessPoolExecutor, as_completed

def process_file(path: str) -> dict:
    # Simulate CPU-heavy file processing
    import hashlib
    data = path.encode() * 10000
    return {"file": path, "hash": hashlib.sha256(data).hexdigest()[:12]}

files = ["data_01.csv", "data_02.csv", "data_03.csv", "data_04.csv"]

with ProcessPoolExecutor(max_workers=4) as exe:
    futures = {exe.submit(process_file, f): f for f in files}
    for future in as_completed(futures):
        result = future.result()
        print(f"  {result['file']} -> {result['hash']}")
Challenge

Process a list of 6 files in parallel using ProcessPoolExecutor. Use submit + as_completed to print results as they arrive. Handle exceptions from individual tasks with future.exception().

#1 Pitfall: Passing unpicklable objects (lambdas, open file handles, database connections) to ProcessPoolExecutor. Data crosses process boundaries via pickle — if it can't be pickled, you get PicklingError. Use ThreadPoolExecutor for unpicklable data.

Profiling with timeit & cProfile

Concurrency • 5 days overdue • rep 0

Quick Reminder

timeit.timeit(stmt, number=N) for micro-benchmarks. cProfile.run(stmt) for function-level profiling — shows ncalls, tottime, cumtime. line_profiler (third-party) for line-by-line analysis. Always profile before optimizing — measure, don't guess.

Key Pattern — Compare and Profile

import timeit
import cProfile

# Micro-benchmark: list comp vs map
setup = "data = list(range(10000))"

t1 = timeit.timeit("[x**2 for x in data]", setup=setup, number=1000)
t2 = timeit.timeit("list(map(lambda x: x**2, data))", setup=setup, number=1000)
print(f"List comp: {t1:.3f}s")
print(f"map+lambda: {t2:.3f}s")

# cProfile a sorting function
def sort_demo():
    import random
    data = [random.randint(0, 10000) for _ in range(50000)]
    return sorted(data)

cProfile.run("sort_demo()")
# Shows call count, time per function
Challenge

Profile a sorting function using cProfile and identify the most time-consuming function call. Then use timeit to compare sorted(data) vs data.sort() for a list of 100,000 random integers.

#1 Pitfall: Micro-benchmarking in the REPL vs a script gives different results. The REPL has overhead from input processing and display. Always use timeit.timeit() with a proper number parameter for reliable measurements.

Class Methods & Static Methods

OOP • 5 days overdue • rep 1

Quick Reminder

@classmethod receives cls as first arg — use for factory methods and alternative constructors. @staticmethod receives no implicit arg — use for utility functions that belong to the class namespace. Always use cls() inside classmethods, never ClassName(), so subclasses work correctly.

Key Pattern — Factory Method with cls

class Date:
    def __init__(self, year: int, month: int, day: int):
        self.year = year
        self.month = month
        self.day = day

    @classmethod
    def from_string(cls, s: str) -> "Date":
        y, m, d = map(int, s.split("-"))
        return cls(y, m, d)  # uses cls, not Date

    @staticmethod
    def is_leap_year(year: int) -> bool:
        return year % 4 == 0 and (year % 100 != 0 or year % 400 == 0)

    def __repr__(self):
        return f"Date({self.year}, {self.month}, {self.day})"

d = Date.from_string("2026-06-10")
print(d)                         # Date(2026, 6, 10)
print(Date.is_leap_year(2024))   # True
Challenge

Create a Date class with a from_string("YYYY-MM-DD") classmethod factory. Add a subclass USDate that formats as MM/DD/YYYY. Verify that USDate.from_string("2026-06-10") returns a USDate instance (not a plain Date).

#1 Pitfall: Using Date() instead of cls() inside a classmethod. If USDate inherits from_string and it calls Date(), the result is a Date instead of a USDate — breaking inheritance.

subprocess Module

System Integration • New

Why It Matters

Python often needs to call external programs — build tools, system commands, data pipelines. subprocess is the modern replacement for os.system() and the old commands module. It gives you fine-grained control over stdin/stdout/stderr, timeouts, and error handling.

Progressive Explanation

Basic subprocess.run() — capture output:

import subprocess

# Run a command and capture output
result = subprocess.run(
    ["echo", "Hello from subprocess"],
    capture_output=True,
    text=True          # decode bytes to str
)

print(result.stdout)       # Hello from subprocess\n
print(result.returncode)   # 0 (success)

check=True — raise on non-zero exit:

import subprocess

# check=True raises CalledProcessError on failure
try:
    result = subprocess.run(
        ["ls", "/nonexistent"],
        capture_output=True,
        text=True,
        check=True
    )
except subprocess.CalledProcessError as e:
    print(f"Command failed with code {e.returncode}")
    print(f"stderr: {e.stderr.strip()}")

Timeout — kill long-running processes:

import subprocess

try:
    result = subprocess.run(
        ["sleep", "10"],
        timeout=2      # kill after 2 seconds
    )
except subprocess.TimeoutExpired:
    print("Process timed out after 2 seconds")

Piping between processes (replacing shell pipes):

import subprocess

# Equivalent of: echo "hello world" | tr ' ' '\n' | sort
p1 = subprocess.Popen(
    ["echo", "cherry banana apple"],
    stdout=subprocess.PIPE
)
p2 = subprocess.Popen(
    ["tr", " ", "\n"],
    stdin=p1.stdout,
    stdout=subprocess.PIPE
)
p1.stdout.close()  # Allow p1 to receive SIGPIPE if p2 exits

p3 = subprocess.Popen(
    ["sort"],
    stdin=p2.stdout,
    stdout=subprocess.PIPE,
    text=True
)
p2.stdout.close()

output, _ = p3.communicate()
print(output)
# apple
# banana
# cherry

Sending input to stdin:

import subprocess

# Pass data to stdin
result = subprocess.run(
    ["python3", "-c", "import sys; print(sys.stdin.read().upper())"],
    input="hello from stdin",
    capture_output=True,
    text=True
)
print(result.stdout)  # HELLO FROM STDIN

Common Mistakes

Don't
# DANGER: shell=True with user input
user_input = "; rm -rf /"
subprocess.run(
    f"echo {user_input}",
    shell=True  # shell injection!
)
Do
# Safe: pass args as a list
user_input = "; rm -rf /"
subprocess.run(
    ["echo", user_input],  # treated as literal
    capture_output=True,
    text=True
)
Don't
# Forgetting to handle errors
result = subprocess.run(["gcc", "bad.c"])
# result.returncode might be non-zero
# but you never checked!
Do
# Always check or use check=True
result = subprocess.run(
    ["gcc", "bad.c"],
    capture_output=True,
    text=True,
    check=True  # raises on failure
)

Key Rules

  • Use subprocess.run() for simple commands. Use Popen only when you need streaming or pipe chains.
  • Always pass commands as a list of strings, not a single string (avoids shell=True).
  • Never use shell=True with untrusted input — it enables shell injection attacks.
  • Use check=True to automatically raise on non-zero exit codes.
  • Use timeout= to prevent hanging on unresponsive processes.
  • Use text=True (or encoding="utf-8") to get strings instead of bytes.

Exercise 2

Typing: ParamSpec & TypeVarTuple

Type System • New

Why It Matters

Decorators that wrap functions often lose their type signatures. ParamSpec preserves the decorated function's parameter types, while TypeVarTuple enables variadic generics — generic classes/functions that accept an arbitrary number of type parameters.

Progressive Explanation

The problem — decorators lose type info:

from functools import wraps
from typing import Any, Callable

# Without ParamSpec, the wrapper's signature is lost
def log_calls(func: Callable[..., Any]) -> Callable[..., Any]:
    @wraps(func)
    def wrapper(*args: Any, **kwargs: Any) -> Any:
        print(f"Calling {func.__name__}")
        return func(*args, **kwargs)
    return wrapper

@log_calls
def greet(name: str, excited: bool = False) -> str:
    return f"Hello, {name}{'!' if excited else '.'}"

# Type checker sees wrapper(*args, **kwargs) -> Any
# Lost: name is str, excited is bool, returns str

ParamSpec preserves signatures:

from functools import wraps
from typing import Callable, ParamSpec, TypeVar

P = ParamSpec("P")       # captures *args and **kwargs types
R = TypeVar("R")          # captures return type

def log_calls(func: Callable[P, R]) -> Callable[P, R]:
    @wraps(func)
    def wrapper(*args: P.args, **kwargs: P.kwargs) -> R:
        print(f"Calling {func.__name__}")
        return func(*args, **kwargs)
    return wrapper

@log_calls
def greet(name: str, excited: bool = False) -> str:
    return f"Hello, {name}{'!' if excited else '.'}"

# Type checker now knows:
# greet(name: str, excited: bool = False) -> str
result = greet("Alice", excited=True)
print(result)  # Hello, Alice!

Concatenate — adding parameters to a signature:

from typing import Callable, Concatenate, ParamSpec, TypeVar

P = ParamSpec("P")
R = TypeVar("R")

def with_logging(
    func: Callable[Concatenate[str, P], R]
) -> Callable[Concatenate[str, P], R]:
    """Decorator for functions whose first arg is a log prefix."""
    @wraps(func)
    def wrapper(prefix: str, *args: P.args, **kwargs: P.kwargs) -> R:
        print(f"[{prefix}] Calling {func.__name__}")
        return func(prefix, *args, **kwargs)
    return wrapper

@with_logging
def process(prefix: str, data: list[int]) -> int:
    return sum(data)

# Type checker knows: process(prefix: str, data: list[int]) -> int
result = process("INFO", [1, 2, 3])
print(result)  # 6

TypeVarTuple for variadic generics (Python 3.11+):

from typing import TypeVarTuple, Unpack

Ts = TypeVarTuple("Ts")

# A generic tuple that preserves element types
class TypedRow:
    """Demonstrates the concept of variadic generics."""

    @staticmethod
    def make_pair(a: int, b: str) -> tuple[int, str]:
        return (a, b)

    @staticmethod
    def make_triple(a: int, b: str, c: float) -> tuple[int, str, float]:
        return (a, b, c)

# TypeVarTuple lets you generalize over the number of type params:
# def process(*args: Unpack[Ts]) -> tuple[Unpack[Ts]]: ...
# This preserves the exact types and count of arguments

pair = TypedRow.make_pair(1, "hello")
triple = TypedRow.make_triple(1, "hello", 3.14)
print(pair)    # (1, 'hello')
print(triple)  # (1, 'hello', 3.14)

Real-world: typed retry decorator:

import time
from functools import wraps
from typing import Callable, ParamSpec, TypeVar

P = ParamSpec("P")
R = TypeVar("R")

def retry(max_attempts: int = 3, delay: float = 0.1):
    """Retry decorator that preserves the wrapped function's type signature."""
    def decorator(func: Callable[P, R]) -> Callable[P, R]:
        @wraps(func)
        def wrapper(*args: P.args, **kwargs: P.kwargs) -> R:
            last_error: Exception | None = None
            for attempt in range(1, max_attempts + 1):
                try:
                    return func(*args, **kwargs)
                except Exception as e:
                    last_error = e
                    print(f"  Attempt {attempt} failed: {e}")
                    if attempt < max_attempts:
                        time.sleep(delay)
            raise last_error  # type: ignore[misc]
        return wrapper
    return decorator

@retry(max_attempts=3, delay=0.05)
def fetch_data(url: str, timeout: int = 30) -> dict:
    # Type checker knows: fetch_data(url: str, timeout: int = 30) -> dict
    if url == "bad":
        raise ConnectionError("Connection refused")
    return {"url": url, "status": 200}

# Correct usage -- type checker validates args
result = fetch_data("https://api.example.com", timeout=10)
print(result)  # {'url': 'https://api.example.com', 'status': 200}

Common Mistakes

Don't
from typing import Any, Callable

# Loses all parameter type information
def deco(fn: Callable[..., Any]):
    def wrapper(*a, **kw):
        return fn(*a, **kw)
    return wrapper
Do
from typing import Callable, ParamSpec, TypeVar

P = ParamSpec("P")
R = TypeVar("R")

def deco(fn: Callable[P, R]) -> Callable[P, R]:
    def wrapper(*a: P.args, **kw: P.kwargs) -> R:
        return fn(*a, **kw)
    return wrapper

Key Rules

  • Use ParamSpec whenever you write a decorator that wraps a function — it preserves the exact parameter signature.
  • P.args and P.kwargs must always be used together (not P.args alone).
  • Use Concatenate[ExtraArg, P] when your decorator adds parameters to the wrapped function.
  • TypeVarTuple is for advanced library code (ORMs, tensor libraries) — most application code won't need it.
  • Requires Python 3.10+ for ParamSpec, 3.11+ for TypeVarTuple. Use typing_extensions for older versions.

Exercises

Exercise 1

Exercise 2

Connections: Builds on decorators from Day 6 and type hints from Day 10. ParamSpec solves the main pain point of decorator typing. Relates to functools.wraps (reviewed in Day 5) which preserves runtime metadata while ParamSpec preserves static type metadata.

collections.abc & Custom Collections

Data Structures • New

Why It Matters

collections.abc provides abstract base classes that define the interface for container types. Implementing them guarantees your custom collections work with for loops, in checks, len(), and other Python protocols. They also serve as type-check targets via isinstance().

Progressive Explanation

Why use ABCs — guaranteed interface:

from collections.abc import Sequence, Mapping, MutableSet

# ABCs tell you what methods are required
print(Sequence.__abstractmethods__)
# frozenset({'__getitem__', '__len__'})

print(Mapping.__abstractmethods__)
# frozenset({'__getitem__', '__len__', '__iter__'})

print(MutableSet.__abstractmethods__)
# frozenset({'__contains__', '__iter__', '__len__', 'add', 'discard'})

Custom Sequence — sorted list:

from collections.abc import Sequence
import bisect

class SortedList(Sequence):
    """A list that maintains sorted order."""

    def __init__(self, iterable=()):
        self._data = sorted(iterable)

    def __getitem__(self, index):
        return self._data[index]

    def __len__(self):
        return len(self._data)

    def add(self, value):
        bisect.insort(self._data, value)

    def __repr__(self):
        return f"SortedList({self._data})"

sl = SortedList([3, 1, 4, 1, 5])
print(sl)           # SortedList([1, 1, 3, 4, 5])
print(len(sl))      # 5
print(sl[2])        # 3
print(4 in sl)      # True  (free from Sequence!)
sl.add(2)
print(sl)           # SortedList([1, 1, 2, 3, 4, 5])

# Iteration works automatically
for item in sl:
    print(item, end=" ")
# 1 1 2 3 4 5

Custom Mapping — case-insensitive dict:

from collections.abc import MutableMapping

class CaseInsensitiveDict(MutableMapping):
    """Dict with case-insensitive keys."""

    def __init__(self, *args, **kwargs):
        self._store = {}    # lowercase key -> (original_key, value)
        self.update(dict(*args, **kwargs))

    def __getitem__(self, key):
        return self._store[key.lower()][1]

    def __setitem__(self, key, value):
        self._store[key.lower()] = (key, value)

    def __delitem__(self, key):
        del self._store[key.lower()]

    def __iter__(self):
        return (original for original, _ in self._store.values())

    def __len__(self):
        return len(self._store)

    def __repr__(self):
        items = ", ".join(f"{k!r}: {v!r}" for k, v in self.items())
        return f"CaseInsensitiveDict({{{items}}})"

d = CaseInsensitiveDict({"Content-Type": "application/json"})
print(d["content-type"])   # application/json
print(d["CONTENT-TYPE"])   # application/json
d["Accept"] = "text/html"
print(len(d))              # 2
print(list(d.keys()))      # ['Content-Type', 'Accept']

Custom MutableSet — bounded set:

from collections.abc import MutableSet

class BoundedSet(MutableSet):
    """A set with a maximum size. Oldest items are evicted."""

    def __init__(self, maxsize: int, iterable=()):
        self._maxsize = maxsize
        self._data: list = []
        for item in iterable:
            self.add(item)

    def __contains__(self, item):
        return item in self._data

    def __iter__(self):
        return iter(self._data)

    def __len__(self):
        return len(self._data)

    def add(self, item):
        if item not in self._data:
            if len(self._data) >= self._maxsize:
                self._data.pop(0)  # evict oldest
            self._data.append(item)

    def discard(self, item):
        try:
            self._data.remove(item)
        except ValueError:
            pass

    def __repr__(self):
        return f"BoundedSet(max={self._maxsize}, items={self._data})"

bs = BoundedSet(3, [1, 2, 3])
print(bs)          # BoundedSet(max=3, items=[1, 2, 3])
bs.add(4)          # evicts 1
print(bs)          # BoundedSet(max=3, items=[2, 3, 4])
print(3 in bs)     # True
print(1 in bs)     # False

isinstance checks with ABCs:

from collections.abc import Sequence, Mapping, Iterable

# Built-in types satisfy ABCs
print(isinstance([1, 2], Sequence))     # True
print(isinstance((1, 2), Sequence))     # True
print(isinstance("abc", Sequence))      # True
print(isinstance({1, 2}, Sequence))     # False (sets aren't sequences)

print(isinstance({"a": 1}, Mapping))    # True
print(isinstance(range(10), Iterable))  # True

# Better than checking for specific types:
def process(data):
    if isinstance(data, Mapping):
        return list(data.values())
    elif isinstance(data, Sequence):
        return list(data)
    else:
        raise TypeError(f"Unsupported: {type(data)}")

Common Mistakes

Don't
from collections.abc import Sequence

class MyList(Sequence):
    def __getitem__(self, i):
        return self._data[i]
    # Missing __len__!
# TypeError: Can't instantiate abstract class
# MyList with abstract method __len__
Do
from collections.abc import Sequence

class MyList(Sequence):
    def __init__(self):
        self._data = []
    def __getitem__(self, i):
        return self._data[i]
    def __len__(self):           # required!
        return len(self._data)

Key Rules

  • Check ClassName.__abstractmethods__ to see which methods you must implement.
  • Implementing Sequence requires only __getitem__ and __len__ — you get __contains__, __iter__, __reversed__, index, and count for free.
  • Implementing MutableMapping requires __getitem__, __setitem__, __delitem__, __iter__, __len__ — gives you keys, values, items, get, update, pop, etc.
  • Use isinstance(obj, Mapping) instead of isinstance(obj, dict) for more flexible type checking.
  • ABCs cannot be instantiated directly — they serve as interfaces/contracts.

Exercise 2

dataclasses Advanced

Data Modeling • New

Why It Matters

Basic @dataclass covers simple cases, but real code needs mutable defaults, post-init validation, frozen (immutable) instances, memory optimization with slots, and inheritance patterns. These features make dataclasses a viable replacement for hand-written classes in most scenarios.

Progressive Explanation

field() with default_factory — mutable defaults:

from dataclasses import dataclass, field

@dataclass
class Config:
    name: str
    tags: list[str] = field(default_factory=list)
    metadata: dict[str, str] = field(default_factory=dict)

c1 = Config("prod")
c2 = Config("dev")
c1.tags.append("deployed")

print(c1.tags)  # ['deployed']
print(c2.tags)  # []  -- separate list, not shared!

__post_init__ for validation and computed fields:

from dataclasses import dataclass, field

@dataclass
class Rectangle:
    width: float
    height: float
    area: float = field(init=False)  # computed, not in __init__

    def __post_init__(self):
        if self.width <= 0 or self.height <= 0:
            raise ValueError("Dimensions must be positive")
        self.area = self.width * self.height

r = Rectangle(4, 5)
print(r)        # Rectangle(width=4, height=5, area=20)
print(r.area)   # 20

try:
    Rectangle(-1, 5)
except ValueError as e:
    print(e)    # Dimensions must be positive

frozen=True — immutable dataclasses:

from dataclasses import dataclass

@dataclass(frozen=True)
class Point:
    x: float
    y: float

p = Point(3.0, 4.0)
print(p)         # Point(x=3.0, y=4.0)

try:
    p.x = 10    # Can't modify frozen dataclass
except AttributeError as e:
    print(e)     # cannot assign to field 'x'

# Frozen dataclasses are hashable (can be dict keys / set members)
points = {Point(0, 0), Point(1, 1), Point(0, 0)}
print(len(points))  # 2 (duplicate removed)

slots=True (Python 3.10+) — memory optimization:

from dataclasses import dataclass
import sys

@dataclass
class RegularPoint:
    x: float
    y: float

@dataclass(slots=True)
class SlottedPoint:
    x: float
    y: float

r = RegularPoint(1.0, 2.0)
s = SlottedPoint(1.0, 2.0)

print(sys.getsizeof(r) + sys.getsizeof(r.__dict__))  # ~200 bytes
print(sys.getsizeof(s))                                # ~56 bytes
print(hasattr(s, '__dict__'))                           # False

KW_ONLY (Python 3.10+) — keyword-only fields:

from dataclasses import dataclass, KW_ONLY

@dataclass
class Request:
    url: str
    method: str = "GET"
    _: KW_ONLY             # everything after this is keyword-only
    timeout: int = 30
    verify_ssl: bool = True
    headers: dict = field(default_factory=dict)

# Must use keyword args for timeout, verify_ssl, headers
req = Request("/api/users", "POST", timeout=10, verify_ssl=False)
print(req)
# Request(url='/api/users', method='POST', timeout=10,
#         verify_ssl=False, headers={})

Dataclass inheritance:

from dataclasses import dataclass, field

@dataclass
class Base:
    id: int
    name: str

@dataclass
class Employee(Base):
    department: str = "Engineering"
    salary: float = 0.0

e = Employee(id=1, name="Alice", department="Research", salary=95000)
print(e)
# Employee(id=1, name='Alice', department='Research', salary=95000.0)

# Fields are ordered: parent fields first, then child fields

asdict and astuple for serialization:

from dataclasses import dataclass, asdict, astuple
import json

@dataclass
class User:
    name: str
    email: str
    age: int

u = User("Alice", "alice@test.com", 30)

# Convert to dict (deep copy, recursive)
d = asdict(u)
print(d)      # {'name': 'Alice', 'email': 'alice@test.com', 'age': 30}
print(json.dumps(d))  # easy JSON serialization

# Convert to tuple
t = astuple(u)
print(t)      # ('Alice', 'alice@test.com', 30)

Common Mistakes

Don't
@dataclass
class Bad:
    name: str
    tags: list = []  # shared mutable default!
# ValueError: mutable default for field tags
# is not allowed: use default_factory
Do
@dataclass
class Good:
    name: str
    tags: list = field(default_factory=list)
Don't
@dataclass
class Parent:
    name: str
    value: int = 0

@dataclass
class Child(Parent):
    extra: str  # non-default after default!
# TypeError: non-default argument 'extra'
# follows default argument
Do
@dataclass
class Parent:
    name: str
    value: int = 0

@dataclass
class Child(Parent):
    extra: str = ""  # must have default too

Key Rules

  • Always use field(default_factory=list) for mutable defaults — never = [] or = {}.
  • Use __post_init__ for validation and computed fields (mark computed fields with init=False).
  • Use frozen=True when instances should be immutable (enables hashing for free).
  • Use slots=True for memory-critical classes with many instances.
  • In inheritance, fields with defaults must come after fields without defaults across the entire hierarchy.
  • asdict() performs a deep copy — it can be slow for large nested dataclasses.

textwrap, shutil & tempfile

Standard Library Utilities • New

Why It Matters

Three standard library modules that solve common scripting needs: textwrap for formatting text output, shutil for high-level file/directory operations, and tempfile for safe temporary file handling. Together they cover most file manipulation and text formatting tasks without external dependencies.

Progressive Explanation

textwrap — clean text formatting:

import textwrap

# dedent: remove common leading whitespace
code = """
    def hello():
        print("world")
"""
print(textwrap.dedent(code))
# def hello():
#     print("world")

# fill: wrap text to a given width
long_text = "Python is a versatile programming language that supports multiple paradigms including object-oriented, functional, and procedural programming."
print(textwrap.fill(long_text, width=50))
# Python is a versatile programming language
# that supports multiple paradigms including
# object-oriented, functional, and procedural
# programming.

# wrap: returns a list of lines
lines = textwrap.wrap(long_text, width=40)
for i, line in enumerate(lines):
    print(f"  {i}: {line}")

# indent: add a prefix to each line
text = "Line one\nLine two\nLine three"
print(textwrap.indent(text, ">>> "))
# >>> Line one
# >>> Line two
# >>> Line three

# shorten: truncate with placeholder
short = textwrap.shorten(long_text, width=50, placeholder="...")
print(short)
# Python is a versatile programming language...

tempfile — safe temporary files and directories:

import tempfile
import os

# NamedTemporaryFile -- auto-deleted when closed
with tempfile.NamedTemporaryFile(mode="w", suffix=".txt", delete=True) as f:
    f.write("temporary data\n")
    f.flush()
    print(f"Temp file: {f.name}")
    print(f"Exists: {os.path.exists(f.name)}")  # True
# File is automatically deleted after the with block

# TemporaryDirectory -- auto-deleted when done
with tempfile.TemporaryDirectory(prefix="myapp_") as tmpdir:
    print(f"Temp dir: {tmpdir}")

    # Create files inside the temp directory
    filepath = os.path.join(tmpdir, "data.txt")
    with open(filepath, "w") as f:
        f.write("temporary content")

    print(f"File exists: {os.path.exists(filepath)}")  # True
# Entire directory tree is deleted after the with block

# mkdtemp -- manual cleanup required
tmpdir = tempfile.mkdtemp()
print(f"Manual temp dir: {tmpdir}")
# ... use it ...
import shutil
shutil.rmtree(tmpdir)  # must clean up yourself

# gettempdir -- find the system temp directory
print(f"System temp dir: {tempfile.gettempdir()}")

Combining all three — practical script:

import textwrap
import shutil
import tempfile
from pathlib import Path

def generate_report(data: dict, output_path: str):
    """Generate a report in a temp dir, then move to final location."""

    with tempfile.TemporaryDirectory() as tmpdir:
        tmp_file = Path(tmpdir) / "report.txt"

        # Build formatted content
        header = textwrap.dedent(f"""\
            ===========================
            Report: {data['title']}
            Generated: {data['date']}
            ===========================
        """)

        body = textwrap.fill(data["summary"], width=60)
        body = textwrap.indent(body, "  ")

        tmp_file.write_text(header + "\n" + body + "\n")

        # Atomically move to final location
        shutil.move(str(tmp_file), output_path)

    print(f"Report written to {output_path}")

# Example usage
generate_report(
    {"title": "Q2 Analysis", "date": "2026-06-10",
     "summary": "Revenue increased by 15% quarter over quarter driven by strong performance in the enterprise segment and expanding international markets."},
    "/tmp/q2_report.txt"
)

Common Mistakes

Don't
# Manual temp file -- easy to forget cleanup
f = open("/tmp/myapp_data.txt", "w")
f.write("data")
f.close()
# Leaked file if exception occurs before close
# Name collision with other processes
Do
import tempfile

# Auto-cleanup, unique name, safe
with tempfile.NamedTemporaryFile(
    mode="w", suffix=".txt"
) as f:
    f.write("data")
    f.flush()
    # process f.name
# Automatically deleted
Don't
import shutil

# rmtree with no safeguard
path = user_input  # could be "/"!
shutil.rmtree(path)  # DANGEROUS
Do
import shutil
from pathlib import Path

path = Path(user_input).resolve()
# Validate before deleting
assert str(path).startswith("/tmp/"), "unsafe"
if path.exists() and path.is_dir():
    shutil.rmtree(path)

Key Rules

  • Use textwrap.dedent() to clean up indented multi-line strings in source code.
  • Use textwrap.fill() for terminal-width text wrapping and shorten() for truncation.
  • Prefer shutil.copy2() over shutil.copy() to preserve file metadata (timestamps).
  • Always use tempfile instead of hardcoding paths in /tmp — it handles unique naming and cross-platform compatibility.
  • Use context managers (with) for temp files/dirs to guarantee cleanup even on exceptions.
  • shutil.rmtree() is irreversible — validate paths before calling it.
