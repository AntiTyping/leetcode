String Methods Deep Dive

Strings • 3 days since last review

Quick Reminder

partition() splits into (before, sep, after) tuple. removeprefix()/removesuffix() cleanly strip known prefixes/suffixes (3.9+). str.translate() + str.maketrans() for character-level substitution.

Key Pattern

before, sep, after = line.partition("=")
# Config parsing: "host=localhost" -> ("host", "=", "localhost")
Challenge

Write a function that takes a string of comma-separated "key=value" pairs and returns a dict. Use partition for splitting on = and handle keys without values (default to empty string).

#1 Pitfall: Using split("=") instead of partition("=") when values can contain =. "key=a=b".split("=") gives ["key", "a", "b"] but partition("=") gives ("key", "=", "a=b").

defaultdict & deque

Collections • 3 days since last review

Quick Reminder

defaultdict(factory) auto-creates missing keys. deque(maxlen=N) for fixed-size buffers. deque.appendleft()/popleft() are O(1).

Key Pattern

from collections import defaultdict

graph = defaultdict(list)
graph["a"].append("b")
graph["a"].append("c")
print(dict(graph))
# Output: {'a': ['b', 'c']}
Challenge

Implement a "sliding window maximum" that maintains a deque of indices to track the current window's maximum efficiently. Given a list and window size k, return the max of each window.

#1 Pitfall: Using defaultdict(list) then checking if key in d — the key won't auto-create on in check, but d[key] will. Be careful not to accidentally create empty entries by accessing keys.

Generator Expressions

Generators • 2 days since last review

Quick Reminder

(expr for x in iterable if cond) — lazy, single-pass, memory-efficient. Can be passed directly to functions like sum(), any(), all().

Key Pattern

total = sum(x**2 for x in range(1000000))
# No intermediate list -- processes one element at a time
Challenge

Write a pipeline using generator expressions that reads lines from a list of strings, strips whitespace, filters out blank lines, and converts remaining lines to uppercase. Chain 3+ generators.

#1 Pitfall: Generators are exhausted after one iteration — can't reuse them. Store the result in a list if you need multiple passes.

namedtuple

Collections • 2 days since last review

Quick Reminder

Point = namedtuple('Point', ['x', 'y']) creates immutable named tuples. Access via .x or [0]. Supports _replace(), _asdict(), _fields.

Key Pattern

from collections import namedtuple

Point = namedtuple('Point', 'x y')
p = Point(3, 4)
print(p.x)  # 3
Challenge

Create a Color namedtuple with r, g, b fields. Write a function that blends two colors by averaging their components, returning a new Color. Use _replace() to create a "darkened" version (halve all values).

#1 Pitfall: Forgetting that namedtuples are immutable — _replace() returns a NEW tuple, it does not modify in place.

Lambda Functions

Functions • 2 days since last review • first review

Quick Reminder

lambda args: expression — anonymous single-expression functions. Best for short callbacks in sorted(), map(), filter(). Don't assign to variables (use def instead).

Key Pattern

students = [("Alice", 92), ("Bob", 85), ("Charlie", 98)]
ranked = sorted(students, key=lambda s: s[1], reverse=True)
print(ranked)
# Output: [('Charlie', 98), ('Alice', 92), ('Bob', 85)]
Challenge

Given a list of dicts like [{"name": "Alice", "scores": [85, 92, 78]}, ...], sort by average score (descending) using sorted() with a lambda key.

#1 Pitfall: Assigning lambda to a variable (f = lambda x: x+1) defeats the purpose. Use def instead for named functions.

functools

Functions • 2 days since last review • first review

Quick Reminder

partial(func, *args) for pre-filling arguments. reduce(func, iterable) for cumulative operations. @lru_cache/@cache for memoization. @total_ordering for comparison methods.

Key Pattern

from functools import cache

@cache
def fib(n):
    return n if n < 2 else fib(n - 1) + fib(n - 2)

print(fib(30))
# Output: 832040
Challenge

Use functools.reduce to implement a compose function that takes multiple functions and returns their composition (right to left). E.g., compose(str, abs, int)(-3.7) should give "3".

#1 Pitfall: @lru_cache on methods without maxsize — can cause memory leaks since self is part of the cache key and keeps instances alive.

Class Attributes & Methods

Properties & @property

OOP • New

Why It Matters

Properties let you control attribute access with getter/setter/deleter logic while keeping the clean obj.attr syntax. You can add validation, compute values on the fly, and maintain backward compatibility — all without changing how callers use your class.

Progressive Explanation

Simple @property getter — a computed attribute:

class Person:
    def __init__(self, first: str, last: str):
        self.first = first
        self.last = last

    @property
    def full_name(self) -> str:
        return f"{self.first} {self.last}"

p = Person("Ada", "Lovelace")
print(p.full_name)
# Output: Ada Lovelace

Property with setter and validation — reject invalid values:

class Temperature:
    def __init__(self, kelvin: float):
        self.kelvin = kelvin   # uses the setter below

    @property
    def kelvin(self) -> float:
        return self._kelvin

    @kelvin.setter
    def kelvin(self, value: float):
        if value < 0:
            raise ValueError("Temperature cannot be below absolute zero")
        self._kelvin = value

t = Temperature(300)
print(t.kelvin)    # 300
t.kelvin = 0       # OK -- absolute zero
# t.kelvin = -10   # ValueError!

Read-only property — no setter defined:

class Square:
    def __init__(self, side: float):
        self._side = side

    @property
    def area(self) -> float:
        return self._side ** 2

s = Square(5)
print(s.area)     # 25
# s.area = 100    # AttributeError: property 'area' has no setter

Property for backward compatibility — rename without breaking callers:

class User:
    def __init__(self, username: str):
        self._username = username

    @property
    def name(self) -> str:
        """Old attribute name, kept for backward compat."""
        return self._username

    @property
    def username(self) -> str:
        return self._username

u = User("alice")
print(u.name)      # alice (old API still works)
print(u.username)   # alice (new API)

Example 1: Basic getter — computed area

class Rectangle:
    def __init__(self, width: float, height: float):
        self.width = width
        self.height = height

    @property
    def area(self) -> float:
        return self.width * self.height

    @property
    def perimeter(self) -> float:
        return 2 * (self.width + self.height)

r = Rectangle(4, 5)

print(r.area)       # 50  (recomputed automatically)

Example 2: Getter + setter with validation

class BankAccount:
    def __init__(self, owner: str, balance: float = 0):
        self.owner = owner
        self.balance = balance   # triggers setter

    @property
    def balance(self) -> float:
        return self._balance

    @balance.setter
    def balance(self, value: float):
        if value < 0:
            raise ValueError(f"Balance cannot be negative: {value}")
        self._balance = value

acct = BankAccount("Alice", 100)

acct.balance = 50    # OK
print(acct.balance)  # 50

try:
    acct.balance = -20
except ValueError as e:
    print(e)
# Output: Balance cannot be negative: -20

Example 3: Deleter

class CachedPage:
    def __init__(self, url: str):
        self.url = url
        self._content = None

    @property
    def content(self) -> str:
        if self._content is None:
            self._content = f"[Fetched content from {self.url}]"
        return self._content

    @content.deleter
    def content(self):
        self._content = None
        print("Cache cleared")

page = CachedPage("https://example.com")
print(page.content)   # [Fetched content from https://example.com]
print(page.content)   # [Fetched content from https://example.com]  (cached)
del page.content       # Cache cleared
print(page.content)   # [Fetched content from https://example.com]  (re-fetched)

Example 4: Using property() directly (alternative syntax)

class Circle:
    def __init__(self, radius: float):
        self._radius = radius

    def _get_radius(self) -> float:
        return self._radius

    def _set_radius(self, value: float):
        if value < 0:
            raise ValueError("Radius must be non-negative")
        self._radius = value

    radius = property(_get_radius, _set_radius, doc="Circle radius")

c = Circle(5)
print(c.radius)     # 5
c.radius = 10
print(c.radius)     # 10

Example 5: cached_property for expensive one-time computation

import functools
import time

class DataAnalysis:
    def __init__(self, data: list[int]):
        self.data = data

    @functools.cached_property
    def summary(self) -> dict:
        """Expensive computation, cached after first access."""
        time.sleep(0.01)  # simulate expensive work
        return {
            "min": min(self.data),
            "max": max(self.data),
            "mean": sum(self.data) / len(self.data),
        }

analysis = DataAnalysis([10, 20, 30, 40, 50])
print(analysis.summary)
# Output: {'min': 10, 'max': 50, 'mean': 30.0}
print(analysis.summary)  # instant -- cached

Common Mistakes

Don't
class User:
    @property
    def name(self):
        return self.name  # infinite recursion!

    @name.setter
    def name(self, value):
        self.name = value  # calls setter again!
Do
class User:
    @property
    def name(self):
        return self._name  # private attribute

    @name.setter
    def name(self, value):
        self._name = value  # store in _name
Don't
class Report:
    @property
    def data(self):
        # Runs expensive query EVERY access!
        return db.query("SELECT * FROM ...")
Do
import functools

class Report:
    @functools.cached_property
    def data(self):
        # Computed once, then cached
        return db.query("SELECT * FROM ...")

Key Rules

  • Use properties for computed attributes, validation on set, and API backward compatibility.
  • Don't use properties for simple get/set without logic — just use regular attributes.
  • Always store the underlying value in a private attribute (e.g., self._name) to avoid infinite recursion.
  • Use @functools.cached_property for expensive one-time computations.

__slots__

OOP • New

Why It Matters

__slots__ restricts instance attributes to a fixed set, reducing memory usage by ~40% and slightly speeding up attribute access. Essential for classes with millions of instances (data pipelines, game entities, scientific computing).

Progressive Explanation

Basic __slots__ declaration:

class Point:
    __slots__ = ('x', 'y')

    def __init__(self, x: float, y: float):
        self.x = x
        self.y = y

p = Point(3.0, 4.0)
print(p.x, p.y)    # 3.0 4.0
# p.z = 5.0        # AttributeError: 'Point' has no attribute 'z'

Memory comparison — with vs without slots:

import sys

class RegularPoint:
    def __init__(self, x, y):
        self.x = x
        self.y = y

class SlottedPoint:
    __slots__ = ('x', 'y')
    def __init__(self, x, y):
        self.x = x
        self.y = y

r = RegularPoint(1, 2)
s = SlottedPoint(1, 2)
print(sys.getsizeof(r) + sys.getsizeof(r.__dict__))  # ~200 bytes
print(sys.getsizeof(s))                                # ~56 bytes

Slots with inheritance (gotcha):

class Base:
    __slots__ = ('x',)

class Child(Base):
    __slots__ = ('y',)  # only NEW attributes here

    def __init__(self, x, y):
        self.x = x  # inherited from Base's slots
        self.y = y

c = Child(1, 2)
print(c.x, c.y)  # 1 2

Slots + dataclasses (Python 3.10+):

from dataclasses import dataclass

@dataclass(slots=True)
class Pixel:
    x: int
    y: int
    r: int = 0
    g: int = 0
    b: int = 0

px = Pixel(10, 20, 255, 128, 0)
print(px)
# Output: Pixel(x=10, y=20, r=255, g=128, b=0)

s = Slotted("key", 42)
print(s.name, s.value)  # key 42

Example 2: Memory savings at scale

import sys

class Regular:
    def __init__(self, a, b, c):
        self.a = a
        self.b = b
        self.c = c

class Slotted:
    __slots__ = ('a', 'b', 'c')
    def __init__(self, a, b, c):
        self.a = a
        self.b = b
        self.c = c

regulars = [Regular(i, i+1, i+2) for i in range(100000)]
slotteds = [Slotted(i, i+1, i+2) for i in range(100000)]

# Approximate per-instance size
r_size = sys.getsizeof(regulars[0]) + sys.getsizeof(regulars[0].__dict__)
s_size = sys.getsizeof(slotteds[0])
print(f"Regular: ~{r_size} bytes/instance")
print(f"Slotted: ~{s_size} bytes/instance")
print(f"Savings: ~{(1 - s_size/r_size)*100:.0f}%")
# Output (approx):
# Regular: ~200 bytes/instance
# Slotted: ~56 bytes/instance
# Savings: ~72%

Example 3: Slots with default values

class Config:
    __slots__ = ('debug', 'verbose', 'timeout')

    def __init__(self, debug=False, verbose=True, timeout=30):
        self.debug = debug
        self.verbose = verbose
        self.timeout = timeout

Example 4: Empty slots in mixin classes

class SerializeMixin:
    __slots__ = ()  # empty -- no new attributes

    def to_dict(self):
        return {slot: getattr(self, slot)
                for cls in type(self).__mro__
                for slot in getattr(cls, '__slots__', ())}

class Record(SerializeMixin):
    __slots__ = ('id', 'name', 'value')

    def __init__(self, id, name, value):
        self.id = id
        self.name = name
        self.value = value

r = Record(1, "test", 42)
print(r.to_dict())
# Output: {'id': 1, 'name': 'test', 'value': 42}

Common Mistakes

Don't
class Incomplete:
    __slots__ = ('x',)

    def __init__(self, x, y):
        self.x = x
        self.y = y  # AttributeError!
# Forgot to include 'y' in __slots__
Do
class Complete:
    __slots__ = ('x', 'y')

    def __init__(self, x, y):
        self.x = x
        self.y = y  # works fine
Don't
class Parent:
    __slots__ = ('x', 'y')

class Child(Parent):
    __slots__ = ('x', 'y', 'z')  # duplicates x, y!
# Wastes memory with duplicate slot descriptors
Do
class Parent:
    __slots__ = ('x', 'y')

class Child(Parent):
    __slots__ = ('z',)  # only NEW attributes
# Inherits x, y slots from Parent

Key Rules

  • Use __slots__ for data-heavy classes with many instances where memory matters.
  • Don't use when you need dynamic attributes or __dict__ (e.g., for **kwargs storage).
  • List ALL instance attributes in __slots__, including those set in __init__.
  • Use empty __slots__ = () in mixin classes to avoid conflicts.
  • Python 3.10+ @dataclass(slots=True) is the easiest way to use slots.

Class Methods & Static Methods

OOP • New

Why It Matters

@classmethod receives the class as first arg — perfect for alternative constructors. @staticmethod receives no implicit arg — for utility functions that belong to the class namespace but don't need instance or class state.

Progressive Explanation

Basic @classmethod as alternative constructor:

import json

class User:
    def __init__(self, name: str, email: str):
        self.name = name
        self.email = email

    @classmethod
    def from_json(cls, data: str) -> "User":
        d = json.loads(data)
        return cls(d["name"], d["email"])

    @classmethod
    def from_dict(cls, d: dict) -> "User":
        return cls(d["name"], d["email"])

    def __repr__(self):
        return f"User({self.name!r}, {self.email!r})"

u = User.from_json('{"name": "Alice", "email": "alice@test.com"}')
print(u)
# Output: User('Alice', 'alice@test.com')

@classmethod with inheritance — cls preserves subclass:

class Animal:
    def __init__(self, name: str, sound: str):
        self.name = name
        self.sound = sound

    @classmethod
    def create_default(cls):
        return cls("Unknown", "...")

    def __repr__(self):
        return f"{type(self).__name__}({self.name!r}, {self.sound!r})"

class Dog(Animal):
    def __init__(self, name: str, sound: str = "Woof"):
        super().__init__(name, sound)

d = Dog.create_default()
print(d)            # Dog('Unknown', '...')
print(type(d))      # <class 'Dog'>  -- NOT Animal!

Basic @staticmethod for validation/utility:

class MathUtils:
    @staticmethod
    def is_prime(n: int) -> bool:
        if n < 2:
            return False
        for i in range(2, int(n**0.5) + 1):
            if n % i == 0:
                return False
        return True

print(MathUtils.is_prime(17))   # True
print(MathUtils.is_prime(24))   # False

When to use which:

# Regular method: needs instance state (self)
# Class method:   needs class state (cls), alt constructors
# Static method:  needs neither, utility/validation

class Date:
    def __init__(self, year: int, month: int, day: int):
        self.year = year
        self.month = month
        self.day = day

    def __repr__(self):
        return f"Date({self.year}, {self.month}, {self.day})"

    # Regular method -- uses self
    def format(self) -> str:
        return f"{self.year:04d}-{self.month:02d}-{self.day:02d}"

    # Class method -- alt constructor, uses cls
    @classmethod
    def from_string(cls, s: str) -> "Date":
        y, m, d = map(int, s.split("-"))
        return cls(y, m, d)

    # Static method -- utility, no self/cls needed
    @staticmethod
    def is_leap_year(year: int) -> bool:
        return year % 4 == 0 and (year % 100 != 0 or year % 400 == 0)

d = Date.from_string("2026-06-01")
print(d.format())                   # 2026-06-01
print(Date.is_leap_year(2024))      # True
print(Date.is_leap_year(2026))      # False

Example 1: Multiple alternative constructors

class Color:
    def __init__(self, r: int, g: int, b: int):
        self.r = r
        self.g = g
        self.b = b

    @classmethod
    def from_hex(cls, hex_str: str) -> "Color":
        hex_str = hex_str.lstrip("#")
        r = int(hex_str[0:2], 16)
        g = int(hex_str[2:4], 16)
        b = int(hex_str[4:6], 16)
        return cls(r, g, b)

    @classmethod
    def from_name(cls, name: str) -> "Color":
        colors = {"red": (255, 0, 0), "green": (0, 255, 0), "blue": (0, 0, 255)}
        r, g, b = colors[name.lower()]
        return cls(r, g, b)

    def __repr__(self):
        return f"Color({self.r}, {self.g}, {self.b})"

print(Color.from_hex("#FF8C00"))   # Color(255, 140, 0)
print(Color.from_name("blue"))     # Color(0, 0, 255)

Example 3: @staticmethod for validation

class Password:
    MIN_LENGTH = 8

    def __init__(self, value: str):
        if not Password.is_strong(value):
            raise ValueError("Password too weak")
        self._value = value

    @staticmethod
    def is_strong(password: str) -> bool:
        if len(password) < 8:
            return False
        has_upper = any(c.isupper() for c in password)
        has_lower = any(c.islower() for c in password)
        has_digit = any(c.isdigit() for c in password)
        return has_upper and has_lower and has_digit

print(Password.is_strong("Abc12345"))    # True
print(Password.is_strong("weak"))         # False

p = Password("Str0ngP@ss")
print("Password created successfully")
# Output: Password created successfully

Example 4: Comparison table in action

class Demo:
    class_var = "shared"

    def __init__(self, instance_var):
        self.instance_var = instance_var

    def regular_method(self):
        # Has access to self (instance) and self.__class__
        return f"instance={self.instance_var}, class={self.class_var}"

    @classmethod
    def class_method(cls):
        # Has access to cls (class) but NOT a specific instance
        return f"class={cls.class_var}, name={cls.__name__}"

    @staticmethod
    def static_method():
        # Has access to NOTHING implicitly
        return "I'm just a function in the class namespace"

d = Demo("hello")
print(d.regular_method())     # instance=hello, class=shared
print(Demo.class_method())    # class=shared, name=Demo
print(Demo.static_method())   # I'm just a function in the class namespace

Common Mistakes

Don't
class Shape:
    @staticmethod
    def create(kind):
        if kind == "circle":
            return Shape(...)  # hardcoded!
# Won't work with subclasses
Do
class Shape:
    @classmethod
    def create(cls, kind):
        return cls(...)  # uses cls!
# Works correctly with subclasses
Don't
class Converter:
    @classmethod
    def celsius_to_fahrenheit(cls, c):
        return c * 9/5 + 32
# cls is never used -- wasteful
Do
class Converter:
    @staticmethod
    def celsius_to_fahrenheit(c):
        return c * 9/5 + 32
# No implicit arg needed

Key Rules

  • Use @classmethod for alternative constructors and factory methods that need to instantiate the class.
  • Use @staticmethod for utility functions that don't need class or instance state.
  • Always use cls() inside classmethods, never ClassName() — preserves subclass behavior.
  • When in doubt between classmethod and staticmethod, prefer classmethod — it's more flexible.

Inheritance & Method Resolution

Inheritance & MRO

OOP • New

Why It Matters

Inheritance enables code reuse and polymorphism. MRO (Method Resolution Order) determines which method is called in diamond inheritance — essential for understanding Python's class system and avoiding subtle bugs.

Progressive Explanation

Simple single inheritance:

class Animal:
    def __init__(self, name: str):
        self.name = name

    def speak(self) -> str:
        return "..."

class Dog(Animal):
    def speak(self) -> str:
        return f"{self.name} says Woof!"

class Cat(Animal):
    def speak(self) -> str:
        return f"{self.name} says Meow!"

animals = [Dog("Rex"), Cat("Whiskers"), Dog("Buddy")]
for a in animals:
    print(a.speak())
# Output:
# Rex says Woof!
# Whiskers says Meow!
# Buddy says Woof!

Method overriding and calling parent with super():

class Shape:
    def __init__(self, color: str = "black"):
        self.color = color

    def describe(self) -> str:
        return f"A {self.color} shape"

class Circle(Shape):
    def __init__(self, radius: float, color: str = "black"):
        super().__init__(color)
        self.radius = radius

    def describe(self) -> str:
        base = super().describe()
        return f"{base} (circle, radius={self.radius})"

c = Circle(5, "red")
print(c.describe())
# Output: A red shape (circle, radius=5)

Multiple inheritance — diamond pattern:

class A:
    def greet(self):
        return "Hello from A"

class B(A):
    def greet(self):
        return "Hello from B"

class C(A):
    def greet(self):
        return "Hello from C"

    pass  # inherits greet -- but from B or C?

# Because MRO is: D -> B -> C -> A

# B
# C
# A
# object

Mixin pattern — adding cross-cutting functionality:

import json

class JsonMixin:
    """Adds JSON serialization to any class with a __dict__."""
    def to_json(self) -> str:
        return json.dumps(self.__dict__, default=str)

    @classmethod
    def from_json(cls, data: str):
        return cls(**json.loads(data))

class LoggingMixin:
    """Adds simple logging to any class."""
    def log(self, message: str):
        print(f"[{type(self).__name__}] {message}")

class User(JsonMixin, LoggingMixin):
    def __init__(self, name: str, email: str):
        self.name = name
        self.email = email

u = User("Alice", "alice@test.com")
u.log("User created")
print(u.to_json())
# Output:
# [User] User created
# {"name": "Alice", "email": "alice@test.com"}

Example 4: Practical mixin — comparison mixin

from functools import total_ordering

@total_ordering
class SizeMixin:
    """Mixin that provides comparison based on a 'size' attribute."""
    def __eq__(self, other):
        if not isinstance(other, SizeMixin):
            return NotImplemented
        return self.size == other.size

    def __lt__(self, other):
        if not isinstance(other, SizeMixin):
            return NotImplemented
        return self.size < other.size

class File(SizeMixin):
    def __init__(self, name: str, size: int):
        self.name = name
        self.size = size

    def __repr__(self):
        return f"File({self.name!r}, {self.size})"

files = [File("a.txt", 100), File("b.txt", 50), File("c.txt", 200)]
print(sorted(files))
# Output: [File('b.txt', 50), File('a.txt', 100), File('c.txt', 200)]
print(File("x", 100) == File("y", 100))  # True (same size)

Common Mistakes

Don't
# Deep inheritance hierarchies
class A: ...
class B(A): ...
class C(B): ...
class D(C): ...
class E(D): ...  # 5 levels deep!
# Hard to understand, debug, and maintain
Do
# Prefer composition + shallow hierarchies
class Engine: ...
class Car:
    def __init__(self):
        self.engine = Engine()  # composition
# Or use mixins for cross-cutting concerns
Don't
class Parent:
    def __init__(self, x):
        self.x = x

class Child(Parent):
    def __init__(self, x, y):
        self.x = x  # bypasses parent init!
        self.y = y
Do
class Parent:
    def __init__(self, x):
        self.x = x

class Child(Parent):
    def __init__(self, x, y):
        super().__init__(x)  # delegate to parent
        self.y = y

Key Rules

  • Keep hierarchies shallow (3 levels max). Prefer composition over deep inheritance.
  • Use mixins for cross-cutting concerns (serialization, logging, comparison).
  • Always call super().__init__() in subclass constructors.
  • Check ClassName.__mro__ when debugging method resolution in multiple inheritance.
  • Use isinstance() for type checks, not type() ==.

super() Deep Dive

OOP • New

Why It Matters

super() doesn't just call the parent — it follows the MRO. Understanding this is critical for cooperative multiple inheritance, where method calls chain through the entire hierarchy correctly.

Progressive Explanation

Basic super().__init__() in single inheritance:

class Animal:
    def __init__(self, name: str):
        self.name = name

class Dog(Animal):
    def __init__(self, name: str, breed: str):
        super().__init__(name)    # calls Animal.__init__
        self.breed = breed

d = Dog("Rex", "Labrador")
print(d.name, d.breed)
# Output: Rex Labrador

super() with keyword arguments pattern:

class Base:
    def __init__(self, **kwargs):
        # Absorbs any remaining kwargs
        pass

class Named(Base):
    def __init__(self, name: str = "", **kwargs):
        super().__init__(**kwargs)   # pass remaining kwargs up
        self.name = name

class Aged(Base):
    def __init__(self, age: int = 0, **kwargs):
        super().__init__(**kwargs)   # pass remaining kwargs up
        self.age = age

class Person(Named, Aged):
    def __init__(self, **kwargs):
        super().__init__(**kwargs)

p = Person(name="Alice", age=30)
print(p.name, p.age)
# Output: Alice 30

super() in diamond inheritance — follows MRO, not parent:

class A:
    def method(self):
        print("A.method")

class B(A):
    def method(self):
        print("B.method")
        super().method()    # calls C.method (next in MRO), NOT A.method!

class C(A):
    def method(self):
        print("C.method")
        super().method()    # calls A.method

class D(B, C):
    def method(self):
        print("D.method")
        super().method()    # calls B.method

d = D()
d.method()
# Output:
# D.method
# B.method
# C.method      <-- super() in B goes to C, not A!
# A.method

print([c.__name__ for c in D.__mro__])
# Output: ['D', 'B', 'C', 'A', 'object']

Explicit form — super(ClassName, self):

class A:
    def greet(self):
        return "Hello from A"

class B(A):
    def greet(self):
        return "Hello from B"

class C(A):
    def greet(self):
        return "Hello from C"

class D(B, C):
    def greet(self):
        # super() is equivalent to super(D, self) -- starts after D in MRO
        print(super().greet())           # Hello from B

        # Explicit: start lookup after B in MRO
        print(super(B, self).greet())    # Hello from C

        # Explicit: start lookup after C in MRO
        print(super(C, self).greet())    # Hello from A

d = D()
d.greet()
# Output:
# Hello from B
# Hello from C
# Hello from A

When NOT to use super():

class Logger:
    def log(self, msg):
        print(f"LOG: {msg}")

class FileLogger:
    def log(self, msg):
        print(f"FILE: {msg}")

class DualLogger(Logger, FileLogger):
    def log(self, msg):
        # Want BOTH loggers, not just the next in MRO
        Logger.log(self, msg)       # explicit call
        FileLogger.log(self, msg)   # explicit call

dl = DualLogger()
dl.log("test")
# Output:
# LOG: test
# FILE: test

class Base:
    def __init__(self, **kwargs):
        if kwargs:
            print(f"Warning: unused kwargs: {kwargs}")

class Positionable(Base):
    def __init__(self, x: int = 0, y: int = 0, **kwargs):
        super().__init__(**kwargs)
        self.x = x
        self.y = y

    def __init__(self, color: str = "white", **kwargs):
        super().__init__(**kwargs)
        self.color = color

class A:
    def process(self, data):
        print(f"A processes: {data}")
        return data.upper()

class B(A):
    def process(self, data):
        print(f"B processes: {data}")
        data = data.strip()
        return super().process(data)

class C(A):
    def process(self, data):
        print(f"C processes: {data}")
        data = data.replace("!", "")
        return super().process(data)

class D(B, C):
    def process(self, data):
        print(f"D processes: {data}")
        return super().process(data)

# D processes:   hello!
# B processes:   hello!
# C processes: hello!
# A processes: hello
# Result: HELLO

Example 4: Real-world cooperative pattern — middleware chain

class Handler:
    def handle(self, request: dict) -> dict:
        return {"status": "ok", **request}

class AuthMiddleware(Handler):
    def handle(self, request: dict) -> dict:
        if "token" not in request:
            return {"status": "unauthorized"}
        print("  Auth: token verified")
        return super().handle(request)

class LogMiddleware(Handler):
    def handle(self, request: dict) -> dict:
        print(f"  Log: processing {request.get('path', '/')}")
        return super().handle(request)

class App(AuthMiddleware, LogMiddleware):
    pass

app = App()

print("Valid request:")
result = app.handle({"path": "/api/data", "token": "abc123"})
print(f"  Result: {result}")
# Output:
# Valid request:
#   Auth: token verified
#   Log: processing /api/data
#   Result: {'status': 'ok', 'path': '/api/data', 'token': 'abc123'}

print()
print("Missing token:")
result = app.handle({"path": "/api/data"})
print(f"  Result: {result}")
# Output:
# Missing token:
#   Result: {'status': 'unauthorized'}

Common Mistakes

Don't
class Parent:
    def __init__(self, x, y):
        self.x = x
        self.y = y

class Child(Parent):
    def __init__(self, x, y, z):
        super().__init__(x, y)
        self.z = z

class Other(Parent):
    def __init__(self, x, y, w):
        super().__init__(x, y)
        self.w = w

class Multi(Child, Other):
    def __init__(self):
        super().__init__(1, 2, 3)
# Breaks! Positional args don't cooperate
Do
class Parent:
    def __init__(self, **kwargs):
        pass  # absorb remaining

class Child(Parent):
    def __init__(self, z=0, **kwargs):
        super().__init__(**kwargs)
        self.z = z

class Other(Parent):
    def __init__(self, w=0, **kwargs):
        super().__init__(**kwargs)
        self.w = w

class Multi(Child, Other):
    def __init__(self, **kwargs):
        super().__init__(**kwargs)

m = Multi(z=3, w=4)  # works!
Don't
# Mixing super() and direct parent calls
class A:
    def setup(self): print("A")

class B(A):
    def setup(self):
        super().setup()   # uses MRO
        print("B")

class C(A):
    def setup(self):
        A.setup(self)     # skips MRO!
        print("C")
# Inconsistent -- A.setup may run twice
Do
# Consistent super() throughout
class A:
    def setup(self): print("A")

class B(A):
    def setup(self):
        super().setup()
        print("B")

class C(A):
    def setup(self):
        super().setup()   # consistent!
        print("C")

Key Rules

  • Always use super() (not explicit parent calls) for consistent MRO behavior.
  • Use **kwargs for cooperative __init__ in multiple inheritance hierarchies.
  • super() calls the NEXT class in the MRO, not necessarily the direct parent.
  • Be consistent: use super() throughout the entire hierarchy, or don't use it at all.
  • Every class in a cooperative hierarchy should call super().__init__(**kwargs).
