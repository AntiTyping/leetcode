Review Section (6 items)

1. Advanced typing 21 days overdue rep 1 → 2

TypeVar names a type placeholder; Generic[T] makes a class parametric; a constrained TypeVar("N", int, float) allows only those types, while bound= allows any subtype. ParamSpec captures a callable's whole signature for decorators, and @overload declares multiple precise signatures. 22 days since last review.

Key Pattern
from typing import TypeVar, Generic, ParamSpec, Callable, overload

T = TypeVar("T")
class Box(Generic[T]):            # parametric container
    def __init__(self, item: T) -> None:
        self.item = item
    def get(self) -> T:
        return self.item

b: Box[int] = Box(5)
print(b.get())                    # 5

N = TypeVar("N", int, float)      # constrained: int OR float only
def double(x: N) -> N:
    return x + x

P = ParamSpec("P")                # capture a callable's full signature
def logged(fn: Callable[P, T]) -> Callable[P, T]:
    def wrap(*a: P.args, **k: P.kwargs) -> T:
        return fn(*a, **k)
    return wrap
Challenge

Build a generic Stack[T] with push/pop typed by T. Write an @overload-ed parse(x) that a type-checker reads as returning int for a str argument and list[int] for a bytes argument. Then explain the difference between a constrained TypeVar("N", int, float) and a bound TypeVar("N", bound=float).

Pitfall

A constrained TypeVar permits ONLY the listed types (each call picks one); bound= permits any subtype of the bound. TypeVar is invariant by default, so Box[bool] is not a Box[int]. @overload stubs need a single real (non-overloaded) implementation below them, and ParamSpec must be used with Callable plus .args/.kwargs.

2. copy: Shallow vs Deep 20 days overdue rep 1 → 2

copy.copy makes a shallow copy — a new outer object that still shares the inner objects; copy.deepcopy recursively clones everything, so the copy is fully independent. Plain assignment copies nothing at all. 21 days since last review.

Key Pattern
import copy

original = {"tags": ["a", "b"], "n": 1}
shallow = copy.copy(original)      # new dict, SAME inner list
deep = copy.deepcopy(original)     # fully independent clone

shallow["tags"].append("c")        # mutates the shared list!
print(original["tags"])            # ['a', 'b', 'c']  -- leaked
print(deep["tags"])                # ['a', 'b']       -- isolated
Challenge

Given a nested config dict whose values are lists, produce an edited copy that leaves the original untouched, and show why copy.copy is not enough. Implement __deepcopy__ on a class that holds a cache you want dropped in the clone (reset it to empty). Explain how deepcopy survives a cyclic reference using its memo dict.

Pitfall

copy.copy is shallow: nested mutable objects are shared, so mutating them corrupts the original. deepcopy is recursive and slower but uses a memo dict to handle shared and cyclic references correctly. Remember b = a copies nothing — both names bind the same object.

3. venv & pyproject.toml 18 days overdue rep 1 → 2

A virtual environment isolates a project's dependencies from the system Python. pyproject.toml (PEP 517/518/621) is the modern, declarative project manifest: a [build-system] table names the build backend and a [project] table holds metadata, dependencies, and entry points. 19 days since last review.

Key Pattern
# create & activate an isolated environment
python -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate
python -m pip install -e .          # editable install of THIS project

# --- pyproject.toml ---
[build-system]
requires = ["setuptools>=61"]
build-backend = "setuptools.build_meta"

[project]
name = "myapp"
version = "0.1.0"
dependencies = ["requests>=2.31"]

[project.scripts]
myapp = "myapp.cli:main"            # console entry point
Challenge

Write a minimal pyproject.toml with a [build-system] table, a [project] table declaring one dependency, and a console-script entry point. Then create a venv, activate it, and editable-install the project. Explain what pip install -e . does differently from a regular install, and why a per-project venv beats installing packages into the system Python.

Pitfall

Activating a venv only affects the CURRENT shell — new terminals and detached subprocesses need re-activation (or an absolute path to .venv/bin/python). A build fails without a [build-system] table; the [project] table (PEP 621) is the modern replacement for setup.py metadata. Never pip install straight into the system interpreter.

4. Design Patterns in Python 18 days overdue rep 1 → 2

Many classic OO patterns collapse in Python because functions are first-class. Strategy becomes "pass a function"; Observer is a list of callbacks; Factory is a dict mapping keys to constructors; Singleton is just a module-level object (imported once, cached in sys.modules). 19 days since last review.

Key Pattern
# Strategy -- first-class functions replace strategy classes
strategies = {"flat": lambda w: 5.0, "by_weight": lambda w: w * 0.5}
print(strategies["by_weight"](10))     # 5.0

# Observer -- a list of callbacks
class Button:
    def __init__(self):
        self._subs = []
    def on_click(self, fn):             # subscribe
        self._subs.append(fn)
    def click(self):
        for fn in self._subs:
            fn()

b = Button()
b.on_click(lambda: print("clicked!"))
b.click()                              # clicked!
Challenge

Implement Strategy two ways — a class hierarchy with an execute() method, and a plain function passed as an argument — and argue why the function version is more Pythonic. Add an Observer in which a Timer notifies multiple subscribers on each tick(). Then explain how a module already gives you a Singleton for free.

Pitfall

Don't transliterate Java-style patterns verbatim — Strategy, Command, and Factory usually reduce to functions, dicts, and closures in Python. A hand-rolled Singleton (metaclass or __new__ tricks) is rarely worth it: a module-level instance is already a shared, lazily-imported single object.

5. Metaprogramming: __init_subclass__ & __class_getitem__ 18 days overdue rep 1 → 2

__init_subclass__ is an implicit classmethod that runs on the base class every time it is subclassed — perfect for auto-registration and subclass validation without a metaclass. __class_getitem__ makes MyType[...] subscription work (the machinery behind list[int]). 19 days since last review.

Key Pattern
class Plugin:
    registry = {}
    def __init_subclass__(cls, /, key, **kwargs):   # runs when subclassed
        super().__init_subclass__(**kwargs)
        Plugin.registry[key] = cls

class JsonPlugin(Plugin, key="json"):
    pass

print(Plugin.registry)          # {'json': <class '__main__.JsonPlugin'>}

class Column:
    def __class_getitem__(cls, item):   # powers Column[int] subscripting
        return f"Column of {item.__name__}"

print(Column[int])              # Column of int
Challenge

Build a base class whose __init_subclass__ auto-registers each subclass in a dict keyed by a class-keyword argument, and raises TypeError if a subclass forgets to define a required method. Implement __class_getitem__ so MyType[int] returns a parametrized helper. Explain when __init_subclass__ is a lighter alternative to writing a full metaclass.

Pitfall

__init_subclass__ is implicitly a classmethod — don't add @classmethod, and always call super().__init_subclass__(**kwargs) so cooperative chains keep working. It fires for subclasses, NOT for the defining base class itself. Class keyword arguments (class X(Base, key="a")) are delivered to it.

6. Import System 16 days overdue rep 1 → 2

import x resolves x on sys.path, runs its top-level code ONCE, and caches the module object in sys.modules. importlib.import_module("name") imports by string (for plugins/config), and importlib.reload re-executes a module in place. 17 days since last review.

Key Pattern
import importlib, sys

# import a module named by a string (e.g. from a config file)
math_mod = importlib.import_module("math")
print(math_mod.sqrt(9))             # 3.0

# every imported module is cached here
print("math" in sys.modules)        # True

# dynamic attribute lookup on a module object
fn = getattr(math_mod, "factorial")
print(fn(5))                     # 120
Challenge

Write a plugin loader that reads a list of module-name strings and imports each with importlib.import_module, collecting them into a registry. Show that a second import of the same module is a cheap sys.modules cache hit, not a re-execution (add a top-level print and observe it fires once). Explain the roles of sys.path, sys.modules, and a module's __spec__/loader when resolving import x.

Pitfall

Top-level module code runs only ONCE; later imports return the cached object from sys.modules, so mutating a module's globals is a program-wide side effect. Mutating sys.path at runtime is fragile — prefer proper packaging. importlib.reload re-executes but keeps the SAME module object, so names already bound from an old import can go stale.

New Concepts — Compression & Integrity (2 concepts)

Two standard-library toolkits for working with bytes safely. The gzip/bz2/lzma codecs shrink single streams with one shared API — you trade speed for compression ratio. And hmac produces keyed hashes that authenticate a message: unlike a bare checksum, only holders of the secret key can forge or verify the tag.

Compression

Concept 1: gzip, bz2 & lzma — Stream Compression Codecs

Why learn this? These three modules compress a single byte stream (logs, backups, payloads) with the same API shape — compress/decompress for one-shot bytes and open() for file-like streaming. They differ only in the speed/ratio trade-off, so you can swap codecs by changing one import.
Example 1 — one-shot compress and round-trip
import gzip

data = b"log line: request ok\n" * 50   # 1050 bytes, very repetitive
comp = gzip.compress(data)
print(len(data))                      # 1050
print(comp[:2] == b"\x1f\x8b")         # True -- gzip magic number
print(gzip.decompress(comp) == data)   # True -- lossless round trip
print(len(comp) < len(data))          # True -- repetition packs down hard
Output
1050 True True True
Example 2 — open() reads/writes compressed text transparently
import gzip

with gzip.open("app.log.gz", "wt", encoding="utf-8") as f:
    f.write("first line\n")
    f.write("second line\n")            # compressed on the way out

with gzip.open("app.log.gz", "rt", encoding="utf-8") as f:
    print(f.read().splitlines())        # decompressed on the way in
Output
['first line', 'second line']
Example 3 — same API, three codecs, distinct file signatures
import gzip, bz2, lzma

data = b"abcdefgh" * 200                # 1600 bytes
for name, mod in [("gzip", gzip), ("bz2", bz2), ("lzma", lzma)]:
    comp = mod.compress(data)
    print(f"{name:5} magic={comp[:3]!r} smaller={len(comp) < len(data)}")
Output
gzip magic=b'\x1f\x8b\x08' smaller=True bz2 magic=b'BZh' smaller=True lzma magic=b'\xfd7z' smaller=True

Common Mistakes

✗ Passing str to gzip.compress (TypeError) or opening a compressed file in binary mode and expecting str back

✓ compress/decompress work on bytes; use gzip.open(path, "rt"/"wt", encoding=...) for text


✗ Loading a multi-gigabyte file fully into memory just to compress it

✓ Stream it: shutil.copyfileobj(src, gzip.open(dst, "wb")) flows fixed-size chunks with constant memory

Key Rules

  • All three share an API: module-level compress/decompress for bytes, and open() returning a file object for streaming
  • Rough trade-off: gzip is fast/ubiquitous, bz2 compresses more, lzma (.xz) usually smallest but slowest — benchmark on YOUR data
  • Tune with compresslevel= (or lzma preset=); the format is self-identifying via magic bytes, so decompression needs no size hint

Exercises

  1. Compress b"x" * 10_000 with all three codecs and print each compressed length; confirm every one round-trips back to the original with ==.
  2. Write a text file, then use gzip.open(..., "wt") to write a .gz copy and gzip.open(..., "rt") to read it back into a list of lines.
  3. Stream-compress a large binary file into a .gz without loading it into memory, using shutil.copyfileobj between an open source file and gzip.open(dst, "wb").
Connections: these codecs sit alongside zipfile/tarfile archives (single stream vs container of files) from earlier days; the open() streaming pattern mirrors io.StringIO/BytesIO file-likes; and the encoding argument reuses the Unicode str/bytes boundary you have already studied.
Security & Cryptography

Concept 2: hmac — Keyed-Hash Message Authentication

Why learn this? A plain hash proves a message was not accidentally corrupted, but anyone can recompute it. HMAC mixes a secret key into the hash, so only key-holders can produce or verify the tag — the standard way to sign webhooks, API requests, and session cookies. Always verify with hmac.compare_digest to resist timing attacks.
Example 1 — produce an authentication tag
import hmac, hashlib

key = b"s3cret-key"
msg = b"amount=100&to=bob"
tag = hmac.new(key, msg, hashlib.sha256).hexdigest()
print(len(tag))                     # 64 -- sha256 hex digest is 64 chars
print(tag.isalnum())                # True -- hex string of the tag
Output
64 True
Example 2 — verify in constant time with compare_digest
import hmac, hashlib

def sign(key, msg):
    return hmac.new(key, msg, hashlib.sha256).digest()

key = b"key"
good = sign(key, b"hello")
print(hmac.compare_digest(good, sign(key, b"hello")))   # True  -- matches
print(hmac.compare_digest(good, sign(key, b"HELLO")))   # False -- tampered
Output
True False
Example 3 — the webhook signature pattern
import hmac, hashlib

SECRET = b"whsec_abc"

def verify(payload: bytes, header_sig: str) -> bool:
    expected = hmac.new(SECRET, payload, hashlib.sha256).hexdigest()
    return hmac.compare_digest(expected, header_sig)

body = b'{"event":"paid"}'
sig = hmac.new(SECRET, body, hashlib.sha256).hexdigest()
print(verify(body, sig))            # True  -- signature checks out
print(verify(body, "deadbeef"))     # False -- forged/wrong signature
Output
True False

Common Mistakes

✗ Comparing digests with == — it short-circuits on the first differing byte, leaking timing information

✓ Use hmac.compare_digest(a, b), which runs in constant time regardless of where they differ


✗ Using a plain hashlib.sha256(msg) and thinking it authenticates the sender

✓ A bare hash only checks integrity; HMAC needs a shared secret key so tags cannot be forged by an outsider

Key Rules

  • hmac.new(key, msg, digestmod) takes bytes key and message; read the tag with .hexdigest() (text) or .digest() (raw bytes)
  • Always verify with hmac.compare_digest — never == — to avoid timing side-channels
  • Pick a strong digest (sha256+), keep the key secret and high-entropy, and compare like-for-like (hex vs hex, bytes vs bytes)

Exercises

  1. Write sign(key, msg) returning a hex HMAC-SHA256 tag and verify(key, msg, tag) returning a bool via compare_digest; show it accepts a good tag and rejects a tampered message.
  2. Given a payload and a secret, compute a signature, flip one byte of the payload, and confirm verification now fails.
  3. Explain in a comment why hmac.compare_digest is used instead of ==, and demonstrate that swapping in == still gives the right boolean (the difference is timing, not correctness).
Connections: hmac builds directly on hashlib digests (Security topic) and complements secrets for generating the keys; the constant-time comparison idea generalizes to any token/password check; and the byte-oriented API reuses the str/bytes discipline from Concept 1.

New Concepts — Networking & Web (2 concepts)

Two small modules that replace fragile hand-rolled logic. ipaddress parses, validates, and reasons about IP addresses and CIDR networks — membership, host iteration, private-vs-global — without regex. mimetypes maps filenames to content types (and back) so you set correct HTTP headers and handle uploads without hardcoding a lookup table.

Networking

Concept 3: ipaddress — IP Addresses & Networks

Why learn this? Parsing IPs with string tricks or regex is error-prone and ignores IPv6. ipaddress gives you real address and network objects that compare numerically, iterate hosts, test membership, and classify (private, loopback, global) — ideal for allowlists, firewall rules, and log analysis.
Example 1 — parse and classify an address
import ipaddress

ip = ipaddress.ip_address("192.168.1.10")
print(ip.version)                 # 4
print(ip.is_private)              # True
print(int(ip))                    # 3232235786 -- addresses are integers
print(ipaddress.ip_address("::1").is_loopback)   # True (IPv6)
Output
4 True 3232235786 True
Example 2 — networks, host iteration, membership
import ipaddress

net = ipaddress.ip_network("10.0.0.0/29")
print(net.num_addresses)                    # 8
print(list(net.hosts())[:2])                # [IPv4Address('10.0.0.1'), IPv4Address('10.0.0.2')]
print(ipaddress.ip_address("10.0.0.5") in net)   # True
print(ipaddress.ip_address("10.0.1.1") in net)   # False
Output
8 [IPv4Address('10.0.0.1'), IPv4Address('10.0.0.2')] True False
Example 3 — strict=False snaps to the network address
import ipaddress

# an address with host bits set is fine when strict=False
net = ipaddress.ip_network("192.168.1.130/26", strict=False)
print(net)                         # 192.168.1.128/26
print(net.netmask)                 # 255.255.255.192
print(net.broadcast_address)       # 192.168.1.191
Output
192.168.1.128/26 255.255.255.192 192.168.1.191

Common Mistakes

✗ Calling ip_network("192.168.1.130/26") with host bits set — it raises ValueError: has host bits set

✓ Pass strict=False to snap to the network address, or hand it a clean network like 192.168.1.128/26


✗ Sorting or comparing IPs as strings — "10.0.0.9" > "10.0.0.10" lexically, which is wrong

✓ Compare/sort ip_address objects; they order by their integer value, so .9 < .10 as expected

Key Rules

  • ip_address for one host, ip_network for a subnet, ip_interface for host-plus-prefix; all handle IPv4 and IPv6
  • Objects are hashable, comparable, and sortable (great as dict keys / in sets); membership is addr in net
  • hosts() excludes the network and broadcast addresses; use .is_private/.is_global/.is_loopback instead of hand-written ranges

Exercises

  1. Given a list of IP strings, print only the ones inside 10.0.0.0/8 using ip_address(...) in ip_network("10.0.0.0/8").
  2. For a /28 network, print num_addresses, the first and last usable host from hosts(), and the broadcast address.
  3. Sort a list of IPv4 strings correctly by converting to ip_address objects; show the result differs from a plain string sort.
Connections: ipaddress objects are hashable/comparable like the numeric and frozenset types you have studied; classification (is_private) pairs naturally with the hmac-signed allowlists above; and int(ip) ties back to base conversions and bit-level thinking.
Web & File Types

Concept 4: mimetypes — Guessing Content Types

Why learn this? When serving downloads, handling uploads, or building email attachments you need a MIME type like image/png. mimetypes maps filenames and extensions to content types (and back) from a built-in table, so you set correct Content-Type headers without hardcoding one yourself.
Example 1 — guess a type from a filename
import mimetypes

print(mimetypes.guess_type("photo.png"))       # ('image/png', None)
print(mimetypes.guess_type("archive.tar.gz"))  # ('application/x-tar', 'gzip')
print(mimetypes.guess_type("notes.txt"))       # ('text/plain', None)
print(mimetypes.guess_type("mystery.xyz"))     # (None, None) -- unknown
Output
('image/png', None) ('application/x-tar', 'gzip') ('text/plain', None) (None, None)
Example 2 — go the other way: type to extension
import mimetypes

print(mimetypes.guess_extension("application/json"))   # '.json'
print(".html" in mimetypes.guess_all_extensions("text/html"))  # True
Output
.json True
Example 3 — register a custom type
import mimetypes

mimetypes.add_type("application/x-my-format", ".myf")
print(mimetypes.guess_type("data.myf"))   # ('application/x-my-format', None)
Output
('application/x-my-format', None)

Common Mistakes

✗ Expecting guess_type to return a single string — and forgetting the second, usually-None, encoding element

✓ It returns a (type, encoding) tuple; encoding is set for compressed names like .gz/.bz2


✗ Trusting guess_type as a security check — it looks ONLY at the name, never the bytes

✓ Treat it as a hint; validate real content by sniffing the file's actual bytes when it matters

Key Rules

  • guess_type(name) returns (type, encoding) based on the extension only — no file access, no I/O
  • guess_extension/guess_all_extensions reverse the map; results can vary slightly by platform registry
  • Extend the table at runtime with add_type(type, ext), or load system files with mimetypes.init([...])

Exercises

  1. Write content_type(path) that returns the guessed MIME type or "application/octet-stream" as a fallback when guess_type yields None.
  2. Show that guess_type("report.tar.gz") reports both a base type and the gzip encoding; explain what the encoding element means.
  3. Register a custom extension with add_type, then confirm guess_type now recognizes a file using it.
Connections: the gzip encoding hint here ties straight back to Concept 1's codecs; mimetypes pairs with pathlib suffixes and the fnmatch/glob file matching from earlier days; and a fallback-on-None guard is a small case of the truthiness idioms you have practiced.

New Concepts — The Numeric Tower (1 concept)

One abstract-base-class hierarchy that makes numeric code duck-typed. numbers.Number and its subclasses let a function accept int, float, and Fraction uniformly through isinstance checks — with one famous exception (Decimal) that teaches you exactly what the tower does and does not promise.

Numbers

Concept 5: numbers — The Numeric Tower & ABCs

Why learn this? numbers defines an ABC hierarchy — Number ⊃ Complex ⊃ Real ⊃ Rational ⊃ Integral — so a single isinstance(x, numbers.Real) accepts int, float, and Fraction alike, instead of listing concrete types. It's the right tool for writing generic numeric functions and for registering your own numeric types.
Example 1 — the hierarchy in isinstance checks
import numbers
from fractions import Fraction

print(isinstance(3, numbers.Integral))            # True
print(isinstance(3, numbers.Real))                # True  -- an Integral is also Real
print(isinstance(3.5, numbers.Integral))          # False -- float is Real, not Integral
print(isinstance(Fraction(1, 2), numbers.Rational)) # True
print(isinstance(2j, numbers.Complex))            # True
Output
True True False True True
Example 2 — the famous Decimal gotcha
import numbers
from decimal import Decimal

print(isinstance(Decimal("1.5"), numbers.Real))    # False! not registered as Real
print(isinstance(Decimal("1.5"), numbers.Number))  # True  -- but it IS a Number
Output
False True
Example 3 — a duck-typed numeric function
import numbers
from fractions import Fraction

def scale(x, factor):
    if not isinstance(x, numbers.Real):
        raise TypeError("need a real number")
    return x * factor

print(scale(10, 0.5))              # 5.0
print(scale(Fraction(3, 4), 2))    # 3/2 -- stays an exact Fraction
Output
5.0 3/2

Common Mistakes

✗ Guarding with isinstance(x, (int, float)), which silently rejects Fraction and other real-number types

✓ Use isinstance(x, numbers.Real) — but remember it's the abstract contract, not a concrete-type list


✗ Assuming Decimal is a numbers.Real, or forgetting bool is an Integral (isinstance(True, numbers.Integral) is True)

✓ Decimal registers only as numbers.Number by design; special-case it if your function must also accept decimals

Key Rules

  • The tower nests: Number ⊃ Complex ⊃ Real ⊃ Rational ⊃ Integral; each level is an ABC, so check against the abstraction you need
  • bool < int < Integral; float is Real; Fraction is Rational; complex is Complex
  • Decimal is intentionally only a Number (not Real) because its arithmetic semantics differ; register custom numeric types with numbers.Real.register(MyType)

Exercises

  1. Write is_whole(x) that returns True for any numbers.Integral and for a numbers.Real whose value equals its int(); test it on 4, 4.0, and 4.5.
  2. Show that Decimal("2") fails a numbers.Real check but passes numbers.Number, and write a guard that accepts int, float, Fraction, AND Decimal.
  3. Demonstrate that isinstance(True, numbers.Integral) is True, and adjust a function that should reject booleans while accepting other integers.
Connections: the numeric tower is the numeric branch of the collections.abc / ABC world you met with abc and Protocols; it pairs with Fraction and Decimal from the numeric-precision day; and register() mirrors the virtual-subclass mechanism from the ABC review topics.
