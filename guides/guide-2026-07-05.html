#53 Maximum Subarray MediumArrayDP / Kadane~6 days since review · rep 2 → 3, interval → 15

The ProblemGiven an integer array nums, find the contiguous subarray (at least one element) with the largest sum and return that sum. Values may be negative.
Quick ReminderScan once, keeping a running sum. Whenever the running sum drops below 0 it can only hurt what follows, so reset it to 0 before adding the next element. Track the best sum seen.
Key InsightThe best subarray ending at index i is either just nums[i] or nums[i] appended to the best subarray ending at i-1 — whichever is larger. A negative running prefix is dead weight, so drop it. That is Kadane's algorithm, O(n).
Try to recall: Why is it always safe to discard a running sum the moment it goes negative? Can you reproduce Kadane in under 6 minutes?
Language challenge: You last wrote this with a manual while loop in Python. This time implement it idiomatically in Rust using an iterator fold or a for over &nums[1..].
Brute: O(n²)→Optimal: O(n) time, O(1) space
Input: nums = [-2,1,-3,4,-1,2,1,-5,4]
Output: 6
Explanation: The subarray [4,-1,2,1] sums to 6.
Input: nums = [1]
Output: 1
Input: nums = [-3,-1,-2]
Output: -1
Explanation: All-negative edge case — the answer is the largest single element, never 0.

class Solution(object):
    def maxSubArray(self, nums):
        prev_sum = nums[0]
        max_sum = nums[0]
        r = 1
        while r < len(nums):
            if prev_sum < 0:
                prev_sum = 0
            prev_sum += nums[r]
            max_sum = max(max_sum, prev_sum)
            r += 1
        return max_sum

The ProblemGiven an integer array nums where nums[i] is the maximum jump length from index i, starting at index 0, return true if you can reach the last index.
Quick ReminderTrack the furthest index reachable so far. Sweep left to right; if you ever stand on an index beyond that frontier you are stuck. Or, sweep right to left moving a "goal" post.
Key InsightYou never need to enumerate jump sequences. Greedily maintain the maximum reach: at each reachable index i, update reach = max(reach, i + nums[i]). If reach ever falls behind the current index, no sequence can bridge the gap.

        goal = len(nums) - 1

class Solution(object):
    def longestPalindromeSubseq(self, s):
        q = s[::-1]
        rows = cols = len(s) + 1
        dp = [[0 for _ in range(cols)] for _ in range(rows)]
        for r in reversed(range(rows - 1)):
            for c in reversed(range(cols - 1)):
                if s[r] == q[c]:

Why optimize? The linear scan re-examines elements the mountain shape already lets us skip — O(n) is wasteful when the structure guarantees a single turning point.

Key InsightThe brute force ignores the mountain's shape and checks every element. But at any index mid, comparing arr[mid] to its right neighbor tells you which slope you are on: still climbing means the peak is to the right, already descending means the peak is here or to the left. How to Think About It

  2. Pick mid and compare arr[mid] with arr[mid+1].

Why This WorksThe neighbor comparison partitions the array into a "still rising" prefix and a "falling" suffix, and this boundary is exactly the peak. Watch Out For

The ProblemGiven a 2D grid image, a start pixel (sr, sc), and a color: recolor the start pixel and every pixel 4-directionally connected to it that shares the start pixel's original color. Return the modified image. Constraints: grid up to a few hundred per side; colors are small integers.
Brute Force — O((mn)²) time

def floodFill(image, sr, sc, color):
    old = image[sr][sc]
    if old == color:
        return image
    image[sr][sc] = color
    m, n = len(image), len(image[0])
    changed = True
    while changed:
        changed = False
        for i in range(m):
            for j in range(n):
                if image[i][j] != old:
                    continue
                for di, dj in ((1,0),(-1,0),(0,1),(0,-1)):
                    ni, nj = i + di, j + dj
                    if 0 <= ni < m and 0 <= nj < n and image[ni][nj] == color:
                        image[i][j] = color
                        changed = True
                        break
    return image

Key InsightBrute force rediscovers the connected region by repeated global sweeps. But connectivity is transitive: once you stand on a matching pixel, its matching neighbors are reachable directly. How to Think About It
  1. Save the original color; if it already equals the target, do nothing (this prevents a fill that never stops).
  2. Start from the source pixel and consider its four neighbors.
  3. A neighbor belongs to the region only if it currently holds the original color.
  4. Recolor as you arrive at each pixel so it is never processed twice, and expand outward from there.
Why This WorksEvery pixel of the target region is connected to the source by a path of original-colored pixels. Key InsightInstead of each cell searching outward for a zero, let all zeros search outward simultaneously. Seed a BFS with every zero at distance 0; the wavefront expands one ring at a time, so the first time a cell is reached is guaranteed to be via its nearest zero.
How to Think About It

  • Using DFS here can produce non-minimal distances; BFS's level order is what guarantees the minimum.

The ProblemGiven an array candidates (which may contain duplicates) and a target, return all unique combinations whose elements sum to the target. Each number may be used at most once (per its occurrence in the input), and the result must contain no duplicate combinations. Order within a combination does not matter.
Brute Force — O(2ⁿ · n) time

Enumerate every subset of the input using a bitmask over the indices. For each subset whose sum equals the target, sort it to form a canonical key and store it in a set so duplicate combinations collapse to one. Because each index is chosen at most once, the “use each element once” rule is automatic.

def combinationSum2(candidates, target):

How to Think About It

#24 Swap Nodes in Pairs MediumLinked ListPointers

The ProblemGiven the head of a singly linked list, swap every two adjacent nodes and return the new head. You must swap the nodes themselves (values may not be modified in the intended solution). An odd trailing node stays in place.
Brute Force — O(n) time

Walk the list once collecting values into an array, swap each adjacent pair of values in the array, then walk the list again writing the values back. It reaches the right answer but sidesteps the pointer work by editing values.

class ListNode:

Why optimize? It is already linear time, but it uses O(n) extra memory and dodges the actual requirement: swapping nodes, not values.

Key InsightYou do not need the values in an array — you need three pointers per pair: the node before the pair, and the two nodes being swapped. Redirecting next links locally reorders the nodes in place, and a dummy sentinel before the head removes the special case for the very first swap.
How to Think About It
  1. Create a dummy node pointing at the head so the “previous” pointer always exists.
  2. While there are at least two nodes ahead, name the pair (first, second).
  3. Rewire so previous points to second, second points to first, and first points to whatever followed second.
  4. Advance previous to first (now the tail of the swapped pair) and repeat.
Why This WorksEach iteration handles one disjoint pair and leaves the list fully connected: the three pointer reassignments exactly reverse two adjacent nodes while preserving the link into the rest of the list. Because previous moves two nodes forward each time, every pair is processed once and a lone final node is naturally left untouched.
Watch Out For

  • Return dummy.next, not the original head, since the head changes.
  • Odd-length lists and empty/single-node lists must terminate cleanly.

#328 Odd Even Linked List MediumLinked ListPointers

The ProblemGiven a singly linked list, group all nodes at odd positions together followed by all nodes at even positions (positions are 1-indexed by node order, not by value), preserving relative order within each group. Return the reordered head. The intended solution uses O(1) extra space.
Brute Force — O(n) time

Collect the node references (or their values) in order, split them into odd-position and even-position buckets, concatenate the buckets, and relink them into one chain. Correct, but it allocates arrays instead of splicing in place.

Python
def oddEvenList(head):
    if not head:
        return head
    nodes = []
    node = head
    while node:
        nodes.append(node)
        node = node.next
    odd = [nodes[i] for i in range(0, len(nodes), 2)]
    even = [nodes[i] for i in range(1, len(nodes), 2)]
    order = odd + even
    for i in range(len(order) - 1):
        order[i].next = order[i + 1]
    order[-1].next = None
    return order[0]

Key InsightYou can weave the two groups apart with just two moving pointers: one tracing the odd chain, one tracing the even chain. How to Think About It
  1. Keep an odd pointer starting at the head and an even pointer starting at the second node; remember the even head to splice on later.
  2. Repeatedly set odd.next to the node after even (the next odd) and advance odd.
  3. Set even.next to the node after the new odd (the next even) and advance even.
  4. When you run out of nodes, attach the saved even head to the tail of the odd chain.
Why This WorksSkipping one node each time follows exactly the odd or even subsequence in original order, so relative order is preserved. Watch Out For
  • Save the even head before you start rewiring, or you cannot reconnect the groups.
  • Loop-guard on both even and even.next to handle odd-length lists without dereferencing null.
  • Lists of length 0 or 1 should be returned unchanged.

#338 Counting Bits EasyBit ManipulationDP

The ProblemGiven a non-negative integer n, return an array ans of length n + 1 where ans[i] is the number of set bits (1s) in the binary representation of i, for every i from 0 to n.
Brute Force — O(n log n) time

For each number from 0 to n, count its set bits independently by repeatedly testing the lowest bit and shifting right. Simple and correct, but it recomputes bit counts from scratch for every value.

def countBits(n):
    res = []
    for i in range(n + 1):

        x = i
        while x:
            c += x & 1
            x >>= 1
        res.append(c)
    return res

Key InsightThe bit count of i relates directly to a smaller number you already solved. Drop i's lowest bit (shift right by one) and you get a number whose count you have stored; add back whether that lowest bit was set. One table lookup plus one bit test replaces the whole counting loop.
How to Think About It
  1. Answer for 0 is 0; build the array upward.
  2. For each i, look at i shifted right by one — a strictly smaller, already-computed index.
  3. Its stored count is i's count minus the lowest bit.
  4. Add the lowest bit of i back to finish in constant time per number.
Why This WorksShifting right by one removes exactly the least significant bit, so i and i >> 1 have the same set bits except that one. Since i >> 1 < i, its answer is already finalized when you reach i, making the recurrence well-founded and exact for every value.
Watch Out For
  • Off-by-one on array size: you need n + 1 entries including index 0.
  • The recurrence reads a lower index, so you must fill the table in increasing order.
  • Alternative i & (i - 1) also works but pairs with a different stored index — do not mix the two recurrences.

#371 Sum of Two Integers MediumBit ManipulationMath

The ProblemGiven two integers a and b, return their sum a + b without using the + or - operators. Values fit in 32 bits (roughly -1000 to 1000 in the test constraints, results within 32-bit range).
Brute Force — O(|b|) time

Simulate addition by moving one step at a time: increment a once for every unit in b (or decrement when b is negative). The single-step increment/decrement is itself built from bitwise operations, so no + or - operator ever appears. It is correct but loops |b| times.

Python
def getSum(a, b):
    MASK = 0xFFFFFFFF
    def inc(x):                      # (x + 1) using only bitwise ops
        y = 1
        while y:
            c = ((x & y) << 1) & MASK
            x = (x ^ y) & MASK
            y = c
        return x
    def dec(x):                      # (x - 1) using only bitwise ops
        y = 1
        while y:
            borrow = ((~x & y) << 1) & MASK
            x = (x ^ y) & MASK
            y = borrow
        return x
    r = a & MASK
    steps = abs(b)                   # loop |b| times
    while steps != 0:
        r = inc(r) if b > 0 else dec(r)
        steps = dec(steps)
    return r if r < 0x80000000 else r | ~MASK

Why optimize? Looping once per unit of b is linear in the value's magnitude — wasteful when you could add all the bits of both numbers at once.

Key InsightBrute force adds b one unit at a time. But binary addition already tells you how to combine two full numbers in one shot: XOR gives the sum ignoring carries, and AND-then-shift gives exactly the carry bits. Feed the carry back in and repeat until there is no carry — a handful of iterations instead of |b|.
How to Think About It
  1. Compute the “sum without carry” of the two numbers with XOR.
  2. Compute where carries occur with AND, then shift them left by one into their true position.
  3. Now the problem is: add that partial sum to those carries — the same subproblem again.
  4. Repeat until the carry becomes zero; the remaining XOR result is the answer.
Why This WorksColumn by column, XOR reproduces the schoolbook “digit” of binary addition and AND marks every column that generates a carry into the next. Since each round pushes carries strictly leftward, within a fixed 32-bit width the carry must eventually vanish, at which point no columns interact and the XOR alone is the exact sum — including negatives via two's-complement wraparound.
Watch Out For
  • In fixed-width languages, rely on natural 32-bit wraparound; in Python you must mask to 32 bits or the carry propagates forever on negatives.
  • After masking, reinterpret a value with the top bit set as a negative number.
  • Negation for “-b” must also avoid the - operator (use bitwise NOT plus a carry step).
