78. Subsets MediumBacktracking · due today · last reviewed 2026-06-08 · rep 3, interval 15

The Problem

Given an array of unique integers nums, return all possible subsets (the power set). The solution set must not contain duplicate subsets. You may return the answer in any order.

Quick Reminder

Backtracking with an include/exclude choice at each index. Collect the current path into the result at every node, not just at the leaves.

Key InsightEvery element has exactly 2 choices: include it or exclude it. Build choices incrementally with a start index, appending a copy of the current path to the result at each step. The number of subsets is exactly 2n.

Python
class Solution(object):
    def subsets(self, nums):
        out = []
        def backtrack(curr, i):
            if i > len(nums):
                return
            out.append(curr[:])
            for k in range(i, len(nums)):
                curr.append(nums[k])
                backtrack(curr, k + 1)
                curr.pop()
        backtrack([], 0)
        return out

Solid backtracking. The if i > len(nums) guard never actually triggers since the for loop bounds k — the recursion naturally stops when the range is empty. The curr[:] copy is essential: without it every entry would alias the same mutating list.

The Problem

There are n houses in a row, each to be painted with one of k colors. No two adjacent houses may share a color. The cost of painting house i with color j is costs[i][j]. Return the minimum total cost to paint all houses.

Quick Reminder

Track the minimum and second-minimum cost from the previous row. For each color in the current row, if the previous row's best came from this same color, you must use the second-minimum; otherwise use the minimum.

Key InsightThe naive DP is O(n·k²): for each house and color you scan all k previous colors. But you only ever need the two cheapest values from the previous row — the cheapest for any color that differs, and the second-cheapest for the one color that matches. That drops it to O(n·k).

The Problem

Given the head of a singly linked list, return true if it is a palindrome (reads the same forward and backward), otherwise false.

The Problem

Given an array of distinct integers candidates and a target, return all unique combinations of candidates that sum to target. The same number may be chosen unlimited times. Two combinations are unique if the multiset of chosen numbers differs.

The Problem

Given an integer array nums, in one operation you pick any nums[i] and earn nums[i] points — but you must then delete every element equal to nums[i] - 1 and every element equal to nums[i] + 1. Repeat to maximize total points.

                    visited.remove(nxt)

Why optimize? DFS explores exponentially many transformation paths. BFS visits each reachable word once and the first time it reaches endWord is guaranteed shortest.

Key InsightBFS on an implicit word graph. Each word is a node; two words are adjacent if they differ by exactly one letter. BFS layer by layer finds the shortest transformation length.
How to Think About It
  1. If endWord isn't in the word list, return 0 immediately.
  2. BFS from beginWord, tracking the current level (distance).
  3. For each word, try every one of 26 letters at each position to generate neighbors.
  4. If a generated word is in the list and unvisited, mark it and enqueue. Return the level when endWord is dequeued.
Why This WorksBFS explores all words reachable in k steps before any reachable in k+1, so the first time endWord appears it is at minimum distance. Marking words visited prevents re-processing and keeps the work linear in the explored graph.

  • Generating neighbors is O(26·M) per word; with M the word length this is the dominant cost.
  • Mark a word visited when you enqueue it, not when you dequeue it, to avoid duplicates in the same layer.
  • The answer counts words (nodes), not edges — start the level at 1 for beginWord.

The Problem

Brute Force — level-order with explicit null markers

Key InsightTranspose, then reverse each row. Transposing swaps matrix[i][j] with matrix[j][i]; reversing each row afterward completes a 90° clockwise rotation — all in place.
How to Think About It
  1. Transpose the matrix: for every i < j, swap matrix[i][j] with matrix[j][i].
  2. Reverse each row in place.
  3. The combination maps each element to its 90°-clockwise position without extra storage.
Why This WorksTransposing reflects the matrix across its main diagonal, turning rows into columns. Reversing each row then flips horizontally. The composition of those two reflections is exactly a 90° clockwise rotation.
Watch Out For
  • Transpose only the upper triangle (i < j); swapping the full matrix double-swaps back to the original.
  • Clockwise = transpose + reverse rows; counter-clockwise = transpose + reverse columns. Don't mix them up.
  • This must be in place — don't allocate a second matrix.
