Quick ReminderEvery palindrome has a center. Treat each index (odd length) and each gap (even length) as a center and expand outward while the two ends match, tracking the longest span.

class Solution:
    def maxProfit(self, prices: List[int]) -> int:
        @lru_cache
        def dp(i, holding, cooldown):
            if i == len(prices):
                return 0
            do_nothing = dp(i + 1, holding, 0)
            do_something = 0
            if holding:
                do_something = prices[i] + dp(i + 1, 0, 1)
            else:

                return do_nothing

How to Think About It

  2. For each bar i, while the stack top is taller than heights[i], pop it — that popped bar's rectangle ends here.
  3. On pop, height = heights[popped]; right boundary = i; left boundary = new stack top (or −1 if empty), so width = i - stack_top - 1.
  4. Push i after popping.
  5. Append a sentinel height of 0 at the end (or drain the stack afterward) to flush all remaining bars.
Why This WorksThe stack is always increasing, so when bar i forces a pop, everything between the popped bar and i was taller than it (they were pushed and not yet popped), meaning the popped bar spans that whole range at its full height. The new stack top is the nearest shorter bar to the left. Both boundaries are exact, so each bar's maximal rectangle is measured exactly once — totaling O(n) since each index is pushed and popped at most once.
Watch Out For

  • Forgetting the final flush — append a 0 sentinel or drain the stack with right boundary = n.
  • Handling equal heights: use strict comparison consistently so you don't double-count or lose width.

  5. Alternative: do an in-order traversal and verify each value exceeds the previous.

Key InsightDFS from each cell, matching one letter at a time and backtracking on mismatch. The two crucial ideas: (1) mark the current cell as visited before recursing and restore it after (so the same path can't reuse a cell but other paths can), and (2) prune immediately when board[r][c] != word[idx], so dead paths die at the first wrong letter.
How to Think About It
  1. Try starting the search at every cell.
  2. DFS with index idx into the word; if board[r][c] != word[idx], return false.
  3. If idx is the last letter and it matched, return true.
  4. Temporarily blank out the current cell (e.g. set to '#'), recurse into the four neighbors, then restore it.
  5. Return true if any neighbor branch succeeds.
Why This WorksMarking-then-restoring enforces "each cell used once per path" while allowing full reuse across independent paths — exactly the constraint. Because a branch returns as soon as a letter mismatches, only genuinely viable prefixes are extended, so the search correctly explores every valid path and no invalid one persists past its first wrong step.
Watch Out For
  • Forgetting to restore the cell after recursion — later paths would wrongly see it as blocked.
  • Not bounds-checking neighbors before indexing, causing out-of-range access.
  • Mutating the board but returning early without undo on the success path is fine, but on failure you must restore.

board[r][c] = tmp                     # restore
for dr, dc in ((1,0),(-1,0),(0,1),(0,-1)):  # 4 directions

#90 Subsets II MediumBacktrackingDedup

The ProblemGiven an integer array nums that may contain duplicates, return all possible subsets (the power set) without duplicate subsets, in any order.
Brute Force — generate all, dedup with a set

Generate all 2ⁿ subsets ignoring duplicates, sort each subset to a canonical form, and drop repeats using a set. Works but wastes effort building and hashing duplicate subsets.

Python
def subsetsWithDup(nums):
    from itertools import combinations
    seen = set()
    res = []
    nums.sort()
    for k in range(len(nums) + 1):
        for combo in combinations(nums, k):
            if combo not in seen:
                seen.add(combo); res.append(list(combo))
    return res

// The backtracking skip below avoids ever creating duplicates.

// Works but redundant; skip duplicates during backtracking instead.

Why optimize? The set-dedup approach builds many duplicate subsets only to throw them away, plus pays hashing cost. Sorting first and skipping duplicates during backtracking avoids generating them at all.

Key InsightSort nums so equal values are adjacent. During backtracking, when choosing the next element to add at a given depth, skip any value equal to the one just tried at this same level. This guarantees each distinct multiset is generated exactly once — duplicates never arise, so no dedup structure is needed.
How to Think About It
  1. Sort nums to group duplicates.
  2. Backtrack from a start index, recording the current subset at every node (all subsets, not just leaves).
  3. Loop i from start to end; if i > start and nums[i] == nums[i-1], continue to skip the duplicate branch.
  4. Include nums[i], recurse with start = i + 1, then remove it (backtrack).
  5. The skip condition uses i > start, not i > 0 — duplicates are allowed within one subset, just not as sibling choices.
Why This WorksSorting makes equal elements adjacent, so the first time a value appears at a level it spawns all subsets that include it there; a second, equal value at the same level would only re-spawn identical subsets, so skipping it removes exactly the duplicates. Using i > start still permits picking a repeated value at a deeper level, preserving multisets like [1,1,2].
Watch Out For
  • Skip condition i > start vs i > 0: using i > 0 wrongly drops valid subsets containing the duplicate value together.
  • Forgetting to sort first — the adjacency the skip relies on disappears.
  • Appending a reference to the working list instead of a copy — later mutations corrupt stored subsets.

#973 K Closest Points to Origin MediumHeapQuickSelect

The ProblemGiven an array of points on the plane and an integer k, return the k points closest to the origin (0,0), by Euclidean distance. The answer may be in any order.
Brute Force — O(n log n) full sort

Compute each point's squared distance, sort all points by it, and take the first k. Correct and simple, but sorts everything when you only need the k smallest.

Python
def kClosest(points, k):
    points.sort(key=lambda p: p[0]*p[0] + p[1]*p[1])
    return points[:k]
Go
func kClosest(points [][]int, k int) [][]int {
    sort.Slice(points, func(i, j int) bool {
        return points[i][0]*points[i][0]+points[i][1]*points[i][1] <
               points[j][0]*points[j][0]+points[j][1]*points[j][1]
    })
    return points[:k]
}
TypeScript
function kClosest(points: number[][], k: number): number[][] {
    return points
        .sort((a, b) => (a[0]**2 + a[1]**2) - (b[0]**2 + b[1]**2))
        .slice(0, k);
}
Rust
impl Solution {
    pub fn k_closest(mut points: Vec<Vec<i32>>, k: i32) -> Vec<Vec<i32>> {
        points.sort_by_key(|p| p[0]*p[0] + p[1]*p[1]);
        points.truncate(k as usize);
        points
    }
}
C++
vector<vector<int>> kClosest(vector<vector<int>>& points, int k) {
    sort(points.begin(), points.end(), [](auto& a, auto& b){
        return a[0]*a[0]+a[1]*a[1] < b[0]*b[0]+b[1]*b[1];
    });
    return {points.begin(), points.begin() + k};
}

Why optimize? Sorting is O(n log n) but you only need k points. A size-k max-heap gives O(n log k), and QuickSelect gives O(n) average — meaningful when k « n.

Key InsightTo keep the k smallest distances, maintain a max-heap of size k keyed by distance. Push each point; whenever the heap exceeds k, pop the largest. What remains is the k closest, in O(n log k). (Compare squared distances — no need for sqrt.) For O(n) average, QuickSelect partitions around the k-th distance.
How to Think About It
  1. Use squared distance x² + y² as the key — monotonic in true distance, avoids floats.
  2. Maintain a max-heap of at most k points.
  3. Push each point; if the heap size passes k, pop the farthest (the max).
  4. After processing all points, the heap holds the k closest.
  5. QuickSelect alternative: partition points around a pivot distance until the k-th is in place, then take the left k.
Why This WorksA size-k max-heap always evicts the current farthest of the retained points, so at every moment it holds the k smallest distances seen so far. After all points pass through, that invariant makes it exactly the global k closest. Squared distance preserves ordering because sqrt is monotonic, so comparisons are identical without its cost.
Watch Out For
  • Using a min-heap of all n points and popping k works but is O(n + k log n) with O(n) space — the size-k max-heap is tighter; don't confuse the two directions.
  • Languages with only min-heaps (Python heapq) need negated keys to simulate a max-heap.
  • Computing actual Euclidean distance with sqrt — unnecessary and introduces float error.

#136 Single Number EasyBit ManipulationXOR

The ProblemGiven a non-empty array where every element appears exactly twice except for one that appears once, find that single element. Do it in O(n) time and O(1) extra space.
Brute Force — O(n) time, O(n) space hash count

Count occurrences with a hash map (or set toggle), then return the element with count 1. Correct and O(n) time, but uses O(n) extra space — violating the O(1)-space requirement.

Python
def singleNumber(nums):
    from collections import Counter
    for x, c in Counter(nums).items():
        if c == 1:
            return x
Go
func singleNumber(nums []int) int {
    count := map[int]int{}
    for _, x := range nums { count[x]++ }
    for x, c := range count { if c == 1 { return x } }
    return 0
}
TypeScript
function singleNumber(nums: number[]): number {
    const count = new Map<number, number>();
    for (const x of nums) count.set(x, (count.get(x) ?? 0) + 1);
    for (const [x, c] of count) if (c === 1) return x;
    return 0;
}
Rust
use std::collections::HashMap;
impl Solution {
    pub fn single_number(nums: Vec<i32>) -> i32 {
        let mut count = HashMap::new();
        for x in nums { *count.entry(x).or_insert(0) += 1; }
        *count.iter().find(|(_, &c)| c == 1).unwrap().0
    }
}
C++
int singleNumber(vector<int>& nums) {
    unordered_map<int,int> count;
    for (int x : nums) count[x]++;
    for (auto& [x, c] : count) if (c == 1) return x;
    return 0;
}

Why optimize? The hash count is O(n) space. XOR solves it in O(1) space with a single accumulator — no auxiliary structure at all.

Key InsightXOR has two magic properties: x ^ x = 0 (a value cancels itself) and x ^ 0 = x (identity), and it's commutative/associative. XOR-ing every element together cancels all the paired values to 0, leaving exactly the lone element — O(n) time, one integer of space.
How to Think About It
  1. Start an accumulator at 0.
  2. XOR every element of the array into it.
  3. All values appearing twice XOR to 0 and vanish.
  4. The one unpaired value XORs with 0 to remain.
  5. Return the accumulator.
Why This WorksBecause XOR is associative and commutative, the order of operations doesn't matter — you can mentally regroup the XOR of all elements so each duplicated pair sits together. Every pair reduces to 0, and 0 XOR the single element is the single element. So the final accumulator is provably that unique value.
Watch Out For
  • This trick relies on the exact "every other element appears twice" guarantee — it doesn't generalize to triples (that's #137, needing bit-count mod 3).
  • Initializing the accumulator to something other than 0 — the identity element for XOR is 0.

#152 Maximum Product Subarray MediumDPKadane

The ProblemGiven an integer array nums, find the contiguous non-empty subarray with the largest product and return that product. The array may contain negatives and zeros.
Brute Force — O(n²) all subarrays

Try every start index, extend the subarray one element at a time multiplying into a running product, and track the maximum seen. Correct but quadratic.

Python
def maxProduct(nums):
    best = nums[0]
    for i in range(len(nums)):
        prod = 1
        for j in range(i, len(nums)):
            prod *= nums[j]
            best = max(best, prod)
    return best
Go
func maxProduct(nums []int) int {
    best := nums[0]
    for i := range nums {
        prod := 1
        for j := i; j < len(nums); j++ {
            prod *= nums[j]
            if prod > best { best = prod }
        }
    }
    return best
}

Why optimize? With n up to 2×10⁴, O(n²) is ~4×10⁸ operations. A one-pass DP tracking running max and min gets it to O(n).

Key InsightUnlike sums, a product flips sign when multiplied by a negative — so the minimum product ending here can become the maximum after one more negative. Track both curMax and curMin ending at each index; on a negative number they swap roles. A zero resets both to that element. This one-pass DP is Kadane's algorithm adapted for products.
How to Think About It
  1. Maintain curMax and curMin = best/worst product of a subarray ending at the current index.
  2. For each new value x, the three candidates are x, curMax * x, and curMin * x.
  3. Set curMax = max of the three, curMin = min of the three (compute both from the old values).
  4. Update a global best with curMax.
  5. When x is negative, curMax and curMin effectively swap — the formula handles it automatically.
Why This WorksAny subarray ending at index i either is [x] alone or extends the best/worst subarray ending at i−1. Because a negative multiplier turns the smallest (most negative) product into the largest, carrying both extremes guarantees the true maximum is always among the candidates. Taking the running max over all positions covers every possible ending, so the global maximum is found.
Watch Out For
  • Computing the new curMax then using it to compute curMin — you must use the old curMax/curMin for both. Snapshot before updating.
  • Initializing best to 0 — use nums[0], since an all-negative array has a negative answer.
  • Forgetting that a single element is a valid subarray (the x alone candidate handles zeros resetting the run).
