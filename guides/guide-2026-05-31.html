#286 · Walls and Gates

Medium Graph BFS 382 days since last review (solved 2025-05-13)

The Problem: Given an m×n grid where 0 = gate, -1 = wall, INF (231-1) = empty room, fill each empty room with its distance to the nearest gate. If impossible to reach a gate, leave as INF.

Quick Reminder: Multi-source BFS — start from ALL gates simultaneously, expand level by level. Instead of BFS from each empty room to find nearest gate (O(m·n per room)), reverse the search: BFS outward from all gates at once. Each cell is visited exactly once, giving O(m·n) total.
Challenge: Can you implement multi-source BFS in under 10 minutes? Previously solved in Python — try it in Go or Rust this time.

Example 1:

Input: rooms = [[INF,-1,0,INF],[INF,INF,INF,-1],[INF,-1,INF,-1],[0,-1,INF,INF]]

Output: [[3,-1,0,1],[2,2,1,-1],[1,-1,2,-1],[0,-1,3,4]]

Edge: Single gate [[0]] → [[0]]

Edge: No gates [[INF,-1],[INF,INF]] → stays as INF

Time: O(m·n)  |  Space: O(m·n)

        v = set()

        rows, cols = len(rooms), len(rooms[0])

                if rooms[r][c] == 0:

                    v.add((r, c))

        while q:
            for i in range(len(q)):
                row, col = q.popleft()
                rooms[row][col] = d
                directions = [[0, 1], [1, 0], [0, -1], [-1, 0]]
                for dr, dc in directions:
                    r, c = row + dr, col + dc
                    if r > -1 and r < rows and c > -1 and c < cols and rooms[r][c] != -1 and (r, c) not in v:
                        q.append((r, c))
                        v.add((r, c))
            d += 1

The Problem: Given an integer array nums and an integer k, return the total number of subarrays whose sum equals k.

Quick Reminder: Prefix sum + hash map counting. Sum of subarray nums[i..j] = prefix[j] - prefix[i-1]. For each j, count how many previous prefix sums equal prefix[j] - k.
Challenge: This uses the same prefix sum technique as #523 and #525. Can you write it in under 5 minutes? Previously solved in Python — try it in TypeScript or C++.

Example 1: nums = [1,1,1], k = 2 → Output: 2

Example 2: nums = [1,2,3], k = 3 → Output: 2 (subarrays [1,2] and [3])

Edge: nums = [1,-1,0], k = 0 → Output: 3

Time: O(n)  |  Space: O(n)

class Solution(object):
    def subarraySum(self, nums, k):

        n = 0
        for i in range(len(nums)):
            sum += nums[i]
            if sum - k in sums:

#78 · Subsets

The Problem: Given an integer array nums of unique elements, return all possible subsets (the power set). The solution must not contain duplicate subsets.

Quick Reminder: Subsets is the foundation of backtracking. At each index, you have two choices: include the current element or skip it. This creates a binary decision tree of depth n, generating all 2n subsets.
Challenge: Just solved yesterday. Can you implement both backtracking and iterative approaches? Previously solved in Python — try it in TypeScript or C++.

Time: O(n·2n)  |  Space: O(n·2n)
Previous Solution (Python) 
class Solution(object):
    def subsets(self, nums):
        ans = []
        def backtrack(curr, i):
            if i >= len(nums):
                ans.append(curr[:])
                return
            curr.append(nums[i])
            backtrack(curr, i+1)
            curr.pop()
            backtrack(curr, i+1)
        backtrack([], 0)
        return list(ans)

Thinking Steps

  1. lo = 0, hi = len(nums) - 1
  2. While lo < hi: compute mid

  4. Else: min is in left half (including mid) → hi = mid
  5. Return nums[lo]

Watch Out For:
  • Use hi = mid (not mid - 1) because mid itself could be the minimum
  • Use lo < hi (not lo <= hi) as the loop condition — they converge to the answer
  • Compare with nums[hi], not nums[lo] — comparing with lo has edge cases with fully sorted arrays

Key Insight: The brute force rescans forward for every element. A monotonic decreasing stack resolves ALL pending "next warmer" queries in one pass. When you encounter a warmer temperature, it's the answer for all colder days still on the stack.

Thinking Steps

  1. Initialize answer = [0] * n and empty stack (stores indices)
  2. Iterate through temperatures left to right
  3. While stack is non-empty AND current temp > temp at stack top: pop index, set answer[popped] = current_index - popped
  4. Push current index onto stack
  5. Remaining indices in stack have no warmer day (answer stays 0)
Why This Works: The stack maintains a decreasing sequence of temperatures (by index). A new warmer temperature resolves all pending queries for cooler temps that were waiting for their "next warmer." Each index is pushed and popped at most once, giving O(n) amortized.
Watch Out For:
  • Store INDICES on the stack, not temperatures (you need indices to compute the distance)
  • The stack should be monotonically decreasing (top is the most recent unresolved day)
  • This is the classic "next greater element" pattern — also used in #496, #503, #84

Key Insight: This is cycle detection in a directed graph. If there's a cycle, you can't complete all courses. Use DFS with three states: unvisited, in-progress (on the current path), and completed. A back edge to an in-progress node means a cycle exists.
How to Think About It
  1. Build an adjacency list from prerequisites.
  2. Track node state: 0=unvisited, 1=in-progress, 2=completed.
  3. For each unvisited node, run DFS.
  4. Mark node as in-progress (1) on entry.
  5. For each neighbor: if in-progress → cycle found (return false); if unvisited → recurse.
  6. Mark node as completed (2) on exit.
  7. If all nodes complete without cycle → return true.
Why This Works: The three-state DFS detects back edges (edges pointing to an ancestor in the DFS tree). A back edge in a directed graph means a cycle. If no cycles exist, a topological ordering exists and all courses can be completed.
Watch Out For:
  • Disconnected graph: Must start DFS from every unvisited node.
  • Edge direction: prerequisites[i] = [a, b] means edge FROM b TO a (b is prerequisite of a).
  • Alternative: Kahn's algorithm (BFS-based topological sort) using in-degree counting.
Alternative Approach — Kahn's Algorithm (BFS): Count in-degrees, start from nodes with in-degree 0, process level by level. If all nodes processed → no cycle.

The Problem: Given a reference to a node in a connected undirected graph, return a deep copy. Each node has a val (int) and neighbors (list of Node). Graph nodes are labeled 1 to n.

Key Insight: The challenge is handling cycles and shared references. Use a hash map from original node → cloned node. Before cloning a neighbor, check if it's already been cloned (exists in the map). This prevents infinite recursion in cycles and ensures each node is cloned exactly once.
How to Think About It
  1. Create a hash map: old_node → new_node.
  2. Clone the starting node, add to map.
  3. Use BFS (or DFS): for each node, iterate over its neighbors.
  4. If neighbor not in map: clone it, add to map, add to queue.
  5. Link the cloned node's neighbor list to the cloned neighbors.
  6. Return the clone of the starting node.
Why This Works: The map serves as both a visited set (preventing cycles) and a lookup table (connecting cloned nodes). Every original node maps to exactly one clone, preserving the graph structure.
Watch Out For:
  • Empty graph: Must handle null input → null output.
  • Duplicate clones: Don't create duplicate clones — always check the map first.
  • Undirected edges: The graph is undirected, so A→B and B→A both exist. The map prevents re-cloning.

How to Think About It (DFS approach)
  1. Same three-state DFS as #207.
  2. After a node is marked completed (all descendants processed), append it to a result list.
  3. The result list is in reverse topological order — reverse it at the end.
  4. If a cycle is detected, return [].
How to Think About It (Kahn's / BFS approach)
  1. Compute in-degree for each node.
  2. Add all nodes with in-degree 0 to a queue.
  3. Process queue: for each node, add to result, decrement neighbors' in-degrees.
  4. If a neighbor's in-degree becomes 0, add to queue.
  5. If result has all nodes → return result; otherwise cycle exists → return [].
Why This Works: Topological sort works because a DAG always has at least one node with in-degree 0. Removing it (and its edges) reveals the next layer. This continues until all nodes are processed (success) or none have in-degree 0 (cycle).
Watch Out For:
  • DFS gives reverse order: DFS gives reverse topological order — don't forget to reverse.
  • BFS gives forward order: Kahn's gives forward order directly — no reversal needed.
  • Cycle detection: Check len(result) == numCourses to detect cycles.
  • Return value: Return [] for cycles, not an error.

How to Think About It
  1. addWord is identical to Trie insert from #208.
  2. search uses DFS with a helper function dfs(node, index).
  3. If character is a letter: follow that child (if it exists).
  4. If character is '.': recursively try ALL children — return true if any path matches.
  5. At end of word: check node.is_end.
Why This Works: The trie constrains the search space. Without dots, it's O(m). With dots, it branches but only into existing paths — much fewer than brute-force checking every word.
Watch Out For:
  • Wildcard branching: '.' must try ALL 26 possible children (or all keys in the dict).
  • Short-circuit: Return true as soon as any branch succeeds.
  • Worst case: All dots ("...") still has to explore the full trie — but that's bounded by total characters stored.

How to Think About It
  1. Initialize count = 0.
  2. While n != 0: n = n & (n - 1) — clears the lowest set bit.
  3. Increment count each iteration.
  4. Return count.
Why This Works: n - 1 flips the lowest set bit and all bits below it. ANDing with n clears exactly that lowest set bit while preserving all higher bits. Each iteration removes one 1-bit, so we iterate exactly k times where k is the number of 1 bits.
Watch Out For:
  • Performance: The n & (n-1) trick is faster than checking each bit: O(k) where k = number of 1 bits vs O(32).
  • Python integers: In Python, integers can be arbitrarily large — but the problem constrains to 32-bit.
  • Alternative: Check each bit with n & 1 then right-shift n >>= 1 (O(32) always).

bitset<32>(n).count();

int count = 0;
while (n) {

}
