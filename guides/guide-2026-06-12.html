#5 · Longest Palindromic Substring

The Problem: Given a string s, return the longest palindromic substring in s.

Quick Reminder: Expand around every center (odd and even). O(n²) time, O(1) space. For each index i, expand outward while characters match. Track the longest palindrome found.

Example 2: Input: s = "cbbd" → Output: "bb"

Example 3: Input: s = "a" → Output: "a"

class Solution:
    def maxProfit(self, prices: List[int]) -> int:
        @lru_cache
        def dp(i, holding, cooldown):
            if i == len(prices): return 0
            do_nothing = dp(i + 1, holding, 0)
            do_something = 0
            if holding:
                do_something = prices[i] + dp(i + 1, 0, 1)
            else:
                do_something = -prices[i] + dp(i + 1, 1, 0)

            else: return max(do_nothing, do_something)
        return dp(0, 0, 0)

Kadane's Algorithm

Kadane's algorithm is a fundamental greedy/DP technique for finding the maximum subarray sum in O(n) time. The core idea: at each position, decide whether to extend the current subarray or start fresh.

#53 · Maximum Subarray

Medium Array DP Divide and Conquer

The Problem: Given an integer array nums, find the contiguous subarray (containing at least one number) which has the largest sum, and return its sum.

Example 1: Input: nums = [-2,1,-3,4,-1,2,1,-5,4] → Output: 6 (subarray [4,-1,2,1])

Example 2: Input: nums = [1] → Output: 1

Example 3: Input: nums = [5,4,-1,7,8] → Output: 23 (entire array)

Key Insight: Kadane’s algorithm — at each position, either extend the current subarray or start fresh. current_sum = max(nums[i], current_sum + nums[i]). Track the global maximum. If the running sum dips below the current element, the prefix is hurting us — discard it.
Brute Force Approach

Check all possible subarrays with two nested loops. For each starting index i, accumulate the sum to every ending index j, tracking the maximum sum seen. This is O(n²) time, O(1) space.

Medium Linked List Two Pointers

The Problem: Given the head of a linked list, return the node where the cycle begins. If there is no cycle, return null.

        for (int v : vals) { n->val = v; n = n->next; }
        return head;

Step-by-Step Thinking

  1. Base case: if head is null or head.next is null, return head (already sorted).
  2. Find the middle node using slow/fast pointers. Slow stops at the node before the midpoint.
  3. Split: set mid = slow.next, then slow.next = null to cut the list.
  4. Recursively sort the left half (head to slow) and the right half (mid to end).
  5. Merge the two sorted halves using a dummy head node, comparing values and linking nodes.
Why This Works: Merge sort is naturally suited for linked lists because splitting at the middle costs O(n) with slow/fast, and merging two sorted lists costs O(n). The divide-and-conquer approach gives O(n log n) total. Unlike arrays, no extra space is needed for merging (just rewire pointers).
Common Pitfalls: Off-by-one when splitting (must actually cut the list by setting slow.next = null, not just find mid). For even-length lists, pick the left middle for correct splitting. Not handling the remaining tail after one list is exhausted in merge.

#110 · Balanced Binary Tree

Easy Tree DFS

The Problem: Given a binary tree, determine if it is height-balanced. A height-balanced binary tree is one in which the depth of the two subtrees of every node never differs by more than one.

Example 1: Input: root = [3,9,20,null,null,15,7] → Output: true

Example 2: Input: root = [1,2,2,3,3,null,null,4,4] → Output: false

Example 3: Input: root = [] → Output: true

Key Insight: DFS returning height. If any subtree is unbalanced, return -1 as a sentinel. At each node, check if |left_height - right_height| > 1. This way each node is visited exactly once.
Brute Force Approach

For each node, compute the height of its left and right subtrees separately, then check the balance condition. Recursively check all nodes. This recomputes heights redundantly, leading to O(n²) time.

                and self.isBalanced(root.right))

#543 · Diameter of Binary Tree

Easy Tree DFS

The Problem: Given the root of a binary tree, return the length of the diameter. The diameter is the length of the longest path between any two nodes (measured in number of edges).

Example 1: Input: root = [1,2,3,4,5] → Output: 3 (path: 4 → 2 → 1 → 3)

Example 2: Input: root = [1,2] → Output: 1

Key Insight: At each node, the longest path through it = left_depth + right_depth (in edges). Use DFS to compute depth and track a global max of (left + right). The diameter might not pass through the root.
Brute Force Approach

For each node, compute the depth of its left and right subtrees, calculate the path through that node, and track the maximum. This recomputes depths redundantly: O(n²).

Python Brute Force

class Solution:
    def diameterOfBinaryTree(self, root: Optional[TreeNode]) -> int:
        self.diameter = 0
        def depth(node):
            if not node: return 0
            return 1 + max(depth(node.left), depth(node.right))
        def dfs(node):
            if not node: return
            d = depth(node.left) + depth(node.right)
            self.diameter = max(self.diameter, d)
            dfs(node.left)
            dfs(node.right)
        dfs(root)
        return self.diameter

#337 · House Robber III

Medium Tree DP

The Problem: Houses are arranged in a binary tree. A thief cannot rob two directly-linked houses (parent-child). Return the maximum amount the thief can rob.

Example 1: Input: root = [3,2,3,null,3,null,1] → Output: 7 (rob 3 + 3 + 1)

Example 2: Input: root = [3,4,5,1,3,null,1] → Output: 9 (rob 4 + 5)

Key Insight: Tree DP: each node returns a pair (rob_this, skip_this). rob_this = node.val + skip_left + skip_right. skip_this = max(rob_left, skip_left) + max(rob_right, skip_right). This captures the constraint elegantly.
Brute Force Approach

For each node, recursively compute: (a) rob it + rob grandchildren, or (b) skip it + rob children. Without memoization this recomputes subtrees exponentially: O(2^n).

Python Brute Force

class Solution:
    def rob(self, root: Optional[TreeNode]) -> int:
        if not root: return 0
        rob_this = root.val
        if root.left:
            rob_this += self.rob(root.left.left) + self.rob(root.left.right)
        if root.right:
            rob_this += self.rob(root.right.left) + self.rob(root.right.right)
        skip_this = self.rob(root.left) + self.rob(root.right)
        return max(rob_this, skip_this)

Medium Array Backtracking

The Problem: Given a candidate array (may contain duplicates) and a target, find all unique combinations where each number is used at most once and the candidates sum to target.

Example 1: Input: candidates = [10,1,2,7,6,1,5], target = 8 → Output: [[1,1,6],[1,2,5],[1,7],[2,6]]

Example 2: Input: candidates = [2,5,2,1,2], target = 5 → Output: [[1,2,2],[5]]

Key Insight: Sort first. In backtracking, skip candidates[i] if candidates[i] == candidates[i-1] at the same recursion level (i.e., i > start). This avoids generating duplicate combinations while still allowing the same value from different positions.
Brute Force Approach

The brute force is essentially the same backtracking with the duplicate-skip optimization, since generating all 2^n subsets and filtering is too slow. The code below already includes the dedup logic, which is the standard approach for this problem.

Python Brute Force

class Solution:
    def combinationSum2(self, candidates: List[int], target: int) -> List[List[int]]:
        candidates.sort()
        result = []
        def backtrack(start, path, remaining):
            if remaining == 0:
                result.append(path[:])
                return
            for i in range(start, len(candidates)):
                if candidates[i] > remaining: break
                if i > start and candidates[i] == candidates[i-1]: continue
                path.append(candidates[i])
                backtrack(i + 1, path, remaining - candidates[i])
                path.pop()
        backtrack(0, [], target)
        return result

Step-by-Step Thinking

  1. Sort candidates to group duplicates together.
  2. Backtrack starting from index start with remaining sum.
  3. For each candidates[i] from start, skip if candidates[i] == candidates[i-1] and i > start.
  4. Add candidates[i] to path, recurse with i+1 and reduced remaining.
  5. Backtrack by removing the last element from path.
  6. Early termination: if candidates[i] > remaining, break (sorted, so all subsequent are larger).
Why This Works: Sorting ensures duplicates are adjacent. The skip condition (i > start && same as prev) prevents using the same value at the same decision level, eliminating duplicate combinations while allowing the same value to appear from different array positions in one combination.
Common Pitfalls: Forgetting to sort first. Using i > 0 instead of i > start (too aggressive — prevents valid uses of duplicates at deeper levels). Not breaking early when candidate exceeds remaining.

The Problem: Given an integer array nums that may contain duplicates, return all possible unique subsets (the power set). The solution set must not contain duplicate subsets.

Example 1: Input: nums = [1,2,2] → Output: [[],[1],[1,2],[1,2,2],[2],[2,2]]

#131 · Palindrome Partitioning

Medium String Backtracking DP

The Problem: Given a string s, partition it such that every substring of the partition is a palindrome. Return all possible palindrome partitionings.

Example 1: Input: s = "aab" → Output: [["a","a","b"],["aa","b"]]

Example 2: Input: s = "a" → Output: [["a"]]

Key Insight: Backtrack by trying every prefix s[start:end]. If the prefix is a palindrome, recurse on the remainder s[end:]. When start == len(s), all parts are palindromes — add the path to results. Optional optimization: precompute a palindrome lookup table.
Brute Force Approach

Try every possible first cut. If the prefix is a palindrome, recurse on the rest. The palindrome check is O(n) per call, giving O(n × 2^n) overall. Can be optimized with a precomputed DP palindrome table.

Python Brute Force

class Solution:
    def partition(self, s: str) -> List[List[str]]:
        result = []
        def is_palindrome(sub):
            return sub == sub[::-1]
        def backtrack(start, path):
            if start == len(s):
                result.append(path[:])
                return
            for end in range(start + 1, len(s) + 1):
                sub = s[start:end]
                if is_palindrome(sub):
                    path.append(sub)
                    backtrack(end, path)
                    path.pop()
        backtrack(0, [])
        return result

#130 · Surrounded Regions

Medium Graph DFS BFS Matrix

The Problem: Given an m × n board containing 'X' and 'O', capture all regions that are completely surrounded by 'X'. A region is captured by flipping all 'O's to 'X's. An 'O' on the border (or connected to a border 'O') is NOT surrounded.

Example: Input: board = [["X","X","X","X"],["X","O","O","X"],["X","X","O","X"],["X","O","X","X"]] → Flip the inner O’s to X, but keep border-connected O’s.

Key Insight: Reverse thinking: instead of finding surrounded O’s, find UNsurrounded O’s. DFS/BFS from all border O’s, mark them as safe (e.g., ‘S’). Then flip remaining O’s to X, and S’s back to O.
Brute Force Approach

For each O cell, run a DFS/BFS to check if it can reach a border. If it can’t, it’s surrounded. This is O(m²n²) since each cell might trigger a full traversal.

        fn dfs(board: &mut Vec<Vec<char>>, r: i32, c: i32, m: i32, n: i32) {
            if r < 0 || r >= m || c < 0 || c >= n { return; }
            if board[r as usize][c as usize] != 'O' { return; }
            board[r as usize][c as usize] = 'S';
            for &(dr, dc) in &[(0,1),(0,-1),(1,0),(-1,0)] {
                dfs(board, r+dr, c+dc, m, n);
            }
        }
        let (mi, ni) = (m as i32, n as i32);
        for i in 0..m { dfs(board, i as i32, 0, mi, ni); dfs(board, i as i32, ni-1, mi, ni); }
        for j in 0..n { dfs(board, 0, j as i32, mi, ni); dfs(board, mi-1, j as i32, mi, ni); }
        for i in 0..m {
            for j in 0..n {
                if board[i][j] == 'O' { board[i][j] = 'X'; }
                else if board[i][j] == 'S' { board[i][j] = 'O'; }
            }
        }
    }
}

#127 · Word Ladder

The Problem: Given beginWord, endWord, and a wordList, find the length of the shortest transformation sequence from beginWord to endWord. Each step changes exactly one letter, and each intermediate word must be in wordList. Return 0 if no such sequence exists.

Example 1: beginWord = "hit", endWord = "cog", wordList = ["hot","dot","dog","lot","log","cog"] → Output: 5 (hit → hot → dot → dog → cog)

Example 2: beginWord = "hit", endWord = "cog", wordList = ["hot","dot","dog","lot","log"] → Output: 0

Key Insight: BFS level by level. For each word, try changing each character to a-z. If the new word is in the word set, add to the next level. Track visited words. Return the level count when endWord is reached. BFS guarantees shortest path in an unweighted graph.
Brute Force Approach

The BFS approach below is the standard solution. The “brute force” is BFS without optimizations (no bidirectional BFS, no wildcard pattern grouping). Each word generates 26 × word_length neighbors.

Python Brute Force

class Solution:

Step-by-Step Thinking

  1. Initialize first = infinity, second = infinity.
  2. For each num in nums:
  3. If num <= first, update first = num (found a smaller first element).
  4. Else if num <= second, update second = num (found a smaller second element that’s greater than some first).
  5. Else return true (num > second > some previous first).
  6. If loop completes, return false.
Why This Works: first and second track the smallest pair (a, b) where a < b seen so far. Even if first gets updated to a smaller value after second was set, the original “first” that justified second still exists earlier in the array. Finding anything greater than second completes the triplet.
Common Pitfalls: Thinking that updating first invalidates second. It does not — second’s original first value still exists at an earlier index. Using >= instead of > in comparisons (we need strictly increasing).

Stack

Stack problems simulate processes where elements interact with previous elements. The asteroid collision problem is a classic example of using a stack to manage pairwise interactions efficiently.

#735 · Asteroid Collision

Medium Stack Array

The Problem: Asteroids are represented as integers in an array. The absolute value represents size; the sign represents direction (positive = right, negative = left). All asteroids move at the same speed. When two asteroids meet, the smaller one explodes. If equal, both explode. Find out the state of the asteroids after all collisions.

Example 1: Input: asteroids = [5,10,-5] → Output: [5,10] (10 and -5 collide, -5 destroyed)

Example 2: Input: asteroids = [8,-8] → Output: [] (both destroyed)

Example 3: Input: asteroids = [-2,-1,1,2] → Output: [-2,-1,1,2] (no collision, moving apart)

Key Insight: Use a stack. Process left to right. Positive asteroids: push. Negative asteroids: compare with the top of the stack (if positive). Pop smaller positives. If equal, pop and skip the negative. If the stack is empty or the top is negative, push the negative asteroid.
Brute Force Approach

Simulate collisions iteratively: scan for adjacent right-left pairs, resolve them, repeat until no more collisions. O(n²) worst case.

                    i += 1
            asteroids = new_list
        return asteroids

Step-by-Step Thinking

  1. Initialize an empty stack.
  2. For each asteroid in the array:
  3. If positive, push to stack.
  4. If negative, while the stack top is positive AND smaller than |asteroid|, pop it (it’s destroyed).
  5. If the stack top is positive and equal to |asteroid|, pop both (both destroyed), and do NOT push the negative.
  6. If the stack is empty or the top is negative, push the negative asteroid (it survives).
  7. Return the stack as the result array.
Why This Works: The stack maintains the current state of surviving asteroids. Collisions only happen at the boundary between right-moving (positive, on stack) and left-moving (negative, incoming). Once a negative asteroid resolves all its collisions, no further collisions occur for it. Each asteroid is pushed and popped at most once, giving O(n) total.
Common Pitfalls: Not handling the case where a large negative asteroid destroys multiple positives (need a while loop, not just one comparison). Forgetting the equal-size case (both destroyed). Not pushing the negative asteroid when all opposing positives are destroyed.
