#20 · Valid Parentheses

Easy Stack

The Problem

Given a string containing only (), {}, and [], determine if the input string is valid. Every open bracket must be closed by the same type in the correct order.

Key Insight

Push every opening bracket onto a stack. When you encounter a closing bracket, pop from the stack and check that it matches. The stack naturally enforces correct nesting order — the most recently opened bracket must be closed first (LIFO = nesting).

How to Think About It

  1. Create a mapping from closing brackets to their corresponding opening brackets: )→(, }→{, ]→[.
  2. Initialize an empty stack (list).
  3. Iterate through each character. If it's an opening bracket, push it onto the stack.
  4. If it's a closing bracket, check: is the stack empty? If yes, return False. Otherwise, pop and verify the popped bracket matches.
  5. After processing all characters, return whether the stack is empty — leftover opening brackets mean invalid.

Why This Works

Valid nesting means the most recent unmatched opener must match the current closer. A stack's LIFO property guarantees exactly this: pop always returns the most recent opener. If it doesn't match, the string is invalid. If the stack is empty when we try to pop, there's a closer without an opener. If the stack isn't empty at the end, there are unmatched openers.

⚠ Most Dangerous Mistake

Forgetting to check if the stack is empty before popping when you encounter a closing bracket. Strings like ")" or "](){" will cause an IndexError (Python) or panic (Go) if you pop from an empty stack.

  • Not checking stack is empty at the end — "(((" would incorrectly return True
  • Mapping in the wrong direction (opening→closing instead of closing→opening)

mapping = {')': '(', '}': '{', ']': '['}

# list as stack — append pushes, pop removes last, [-1] peeks
stack = []
stack.append('(')
top = stack[-1]    # peek without removing
stack.pop()        # remove and return last element

if char in mapping:
    # it's a closing bracket

#155 · Min Stack

Medium Stack Design

The Problem

Design a stack that supports push, pop, top, and retrieving the minimum element — all in O(1) time.

Key Insight

Maintain a second "min stack" alongside the main stack. Every time you push a value, also push the current minimum (min(val, current_min)) onto the min stack. When you pop, pop from both. The top of the min stack always holds the minimum of all elements currently in the main stack.

How to Think About It

  1. Use two stacks: stack for values and min_stack for running minimums.
  2. Push: push value to stack. Push min(val, min_stack[-1]) to min_stack (or just val if empty).
  3. Pop: pop from both stacks.
  4. Top: return stack[-1].
  5. GetMin: return min_stack[-1].

Why This Works

The min stack records a "snapshot" of the minimum at each depth level. When elements are popped, the min stack shrinks in sync, revealing the previous minimum. Because every push/pop keeps both stacks in lockstep, the min stack's top is always the minimum of all elements currently in the main stack.

⚠ Most Dangerous Mistake

Only pushing to the min stack when the new value is less than the current minimum. If you push [2, 1, 1] and then pop once, the min stack would be wrong if you skipped the duplicate 1.

Builds on #20's stack fundamentals — now the stack tracks additional state (the running minimum) alongside its values.
Python 
# two lists as parallel stacks
self.stack = []
self.min_stack = []

# min() for comparing current value with running minimum
self.min_stack.append(min(val, self.min_stack[-1]))

# alternatively, store tuples: (value, current_min)
self.stack.append((val, min(val, self.stack[-1][1])))
Go 
// two slices as parallel stacks
type MinStack struct {
    stack    []int
    minStack []int
}

// push to both stacks
func (s *MinStack) Push(val int) {
    s.stack = append(s.stack, val)
    minVal := val
    if len(s.minStack) > 0 && s.minStack[len(s.minStack)-1] < val {
        minVal = s.minStack[len(s.minStack)-1]
    }
    s.minStack = append(s.minStack, minVal)
}
Time: O(1) all operations
Space: O(n)

#739 · Daily Temperatures

Medium Stack Monotonic Stack

The Problem

Given an array of daily temperatures, return an array where answer[i] is the number of days you have to wait until a warmer temperature. If no future warmer day exists, put 0.

Key Insight

Use a monotonic decreasing stack of indices. When the current day's temperature is warmer than the stack's top, pop the top — the current day is the answer for that popped day. The distance is current_index - popped_index.

How to Think About It

  1. Initialize result = [0] * n and an empty stack.
  2. Iterate through temperatures with index. The stack holds indices of unresolved days.
  3. While the stack is non-empty AND the current temp is greater than the temp at the stack's top index: pop the index, set result[popped] = i - popped.
  4. Push the current index onto the stack.
  5. Any indices remaining after iteration keep their default 0.

Why This Works

The stack maintains days in decreasing temperature order. When a warmer day arrives, it resolves all consecutive cooler days on the stack. Each day is pushed once and popped at most once, giving O(n). The monotonic property guarantees the first warmer day found is the nearest one.

⚠ Most Dangerous Mistake

Storing temperature values instead of indices on the stack. You need indices to compute the distance and to set the result at the correct position. Always push i, access the temperature via temperatures[stack[-1]].

Introduces the monotonic stack pattern — a specialized use of the stack from #20 and #155 where elements are kept in sorted order by popping when the invariant would be violated.

    result[idx] = i - idx

# enumerate for index + value together
for i, temp in enumerate(temperatures):
    ...

#150 · Evaluate Reverse Polish Notation

Medium Stack

The Problem

Evaluate an arithmetic expression in Reverse Polish Notation. Valid operators are +, -, *, /. Division truncates toward zero.

Key Insight

Push numbers onto a stack. When you encounter an operator, pop two operands (b first, then a), compute a op b, and push the result back. The final stack value is the answer. RPN's postfix order eliminates precedence rules.

How to Think About It

  1. Initialize an empty stack.
  2. Iterate through tokens. If the token is a number (can be negative like "-11"), push it as an integer.
  3. If the token is an operator: pop b (second operand), pop a (first operand). Compute a op b. Push the result.
  4. The order matters: a - b and a / b, where a was pushed first (deeper in stack).
  5. For division, truncate toward zero: use int(a / b) in Python (not //).
  6. After all tokens, the stack has exactly one element — the answer.

Why This Works

RPN encodes the evaluation order of an expression without parentheses. Each operator implicitly binds to its nearest two preceding values. The stack accumulates intermediate results in exactly the right order because operands are pushed before their operator appears.

⚠ Most Dangerous Mistake

Python's // (floor division) truncates toward negative infinity, not toward zero. -7 // 2 = -4 in Python, but the problem requires -3. Use int(a / b) instead.

  • Popping operands in wrong order — b is popped first, a second, compute a op b
  • Forgetting that tokens like "-11" are negative numbers, not a minus followed by 11
Python 
# truncate toward zero (NOT floor division)
result = int(a / b)   # correct: int(-7/2) = -3
result = a // b       # wrong:   -7//2 = -4

# check if token is an operator
if token in {'+', '-', '*', '/'}:
    b, a = stack.pop(), stack.pop()

# tokens like "-11" parse fine with int()
stack.append(int(token))
Go 
// strconv.Atoi for string-to-int conversion
num, err := strconv.Atoi(token)
if err == nil {
    stack = append(stack, num)
}

// switch for operator dispatch
switch token {
case "+": stack = append(stack, a+b)
case "-": stack = append(stack, a-b)
case "*": stack = append(stack, a*b)
case "/": stack = append(stack, a/b) // Go int division truncates toward zero
}

#74 · Search a 2D Matrix

Medium Binary Search Matrix

The Problem

Given an m×n matrix where each row is sorted and the first element of each row is greater than the last element of the previous row, determine if a target value exists.

Key Insight

Treat the entire 2D matrix as a single sorted 1D array. A virtual index mid in range [0, m*n - 1] maps to matrix[mid // n][mid % n]. Now apply standard binary search on this virtual array.

How to Think About It

  1. Compute total elements: m * n. Set left = 0, right = m*n - 1.
  2. Standard binary search: compute mid = (left + right) // 2.
  3. Convert mid to 2D coordinates: row = mid // n, col = mid % n.
  4. Compare matrix[row][col] with target. Adjust left/right accordingly.
  5. This is exactly the #704 template with one extra step (index conversion).

Why This Works

Because each row's first element exceeds the previous row's last element, the matrix is one long sorted array in row-major order. The // n and % n conversion is just the inverse of row-major flattening. Binary search's O(log n) guarantee applies to the entire flattened array.

⚠ Most Dangerous Mistake

Using m (rows) instead of n (columns) in the index conversion. mid // n gives the row and mid % n gives the column. Swapping m and n produces wrong coordinates.

Direct extension of #704 Binary Search — same template, just add coordinate conversion.
Python 
# divmod for simultaneous division and modulo
row, col = divmod(mid, n)

# standard binary search bounds
left, right = 0, m * n - 1

# mid calculation (overflow-safe in Python — arbitrary precision)
mid = (left + right) // 2
Go 
// index conversion
row, col := mid/n, mid%n

// standard binary search
left, right := 0, m*n-1
for left <= right {
    mid := left + (right-left)/2  // overflow-safe
    val := matrix[mid/n][mid%n]
}
Time: O(log(m·n))
Space: O(1)

#33 · Search in Rotated Sorted Array

Medium Binary Search

The Problem

Given a sorted array rotated at some unknown pivot (e.g., [4,5,6,7,0,1,2]), search for a target value. All values are unique. Return the index or -1.

Key Insight

At any mid point, at least one half of the array is always sorted. Identify which half is sorted by comparing nums[left] with nums[mid]. Once you know which half is sorted, check if the target falls within that sorted range to decide which half to search.

How to Think About It

  1. Set left = 0, right = n - 1. Standard binary search loop.
  2. Compute mid. If nums[mid] == target, return mid.
  3. Check if the left half is sorted: nums[left] <= nums[mid].
  4. If left half is sorted: check if target is in [nums[left], nums[mid]). If yes, search left. Otherwise search right.
  5. If right half is sorted: check if target is in (nums[mid], nums[right]]. If yes, search right. Otherwise search left.
  6. The <= in step 3 handles the case where left == mid (two-element subarray).

Why This Works

A rotation splits the sorted array into two sorted halves. At any mid point, the rotation break can be in at most one half — the other is guaranteed sorted. By identifying the sorted half, you can determine if the target could be in it (simple range check). If not, it must be in the other half. This preserves the O(log n) elimination guarantee.

⚠ Most Dangerous Mistake

Using < instead of <= when checking nums[left] <= nums[mid]. When left == mid (subarray of size 2), < would incorrectly classify the left half as unsorted.

Extends the #704 template with one additional decision: "which half is sorted?"
Python 
# chained comparison for range check
if nums[left] <= target < nums[mid]:
    right = mid - 1

# standard binary search skeleton
while left <= right:
    mid = (left + right) // 2
Go 
// range check requires two comparisons (no chaining)
if nums[left] <= target && target < nums[mid] {
    right = mid - 1
}

// overflow-safe mid
mid := left + (right-left)/2
Time: O(log n)
Space: O(1)

#153 · Find Minimum in Rotated Sorted Array

Medium Binary Search

The Problem

Given a sorted array rotated at some pivot, find the minimum element. All values are unique.

Key Insight

Compare nums[mid] with nums[right]. If nums[mid] > nums[right], the minimum is in the right half (the rotation break is there). Otherwise, the minimum is in the left half including mid. Always compare against the right endpoint.

How to Think About It

  1. Set left = 0, right = n - 1.
  2. While left < right (strict inequality, not <=).
  3. If nums[mid] > nums[right]: the rotation break is in (mid, right], so left = mid + 1.
  4. Otherwise: mid itself might be the minimum, so right = mid (not mid - 1).
  5. When left == right, that's the minimum.

Why This Works

The minimum is the point where the sorted order "breaks." Comparing nums[mid] with nums[right] tells you which side the break is on: if mid is greater than right, the break must be between mid and right. If mid is less than or equal to right, that half is properly sorted and the break is on the other side (or at mid itself).

⚠ Most Dangerous Mistake

Comparing with nums[left] instead of nums[right]. When the array is not rotated at all (e.g., [1,2,3,4,5]), comparing with left doesn't work correctly.

Same "which half is sorted?" logic from #33, but searching for the rotation point instead of a target.
Python 
# note: while left < right (not <=)
while left < right:
    mid = (left + right) // 2
    if nums[mid] > nums[right]:
        left = mid + 1   # min is in right half
    else:
        right = mid       # mid could be the min
return nums[left]

#104 · Maximum Depth of Binary Tree

EasyTreeDFS

The Problem

Given the root of a binary tree, return its maximum depth — the number of nodes along the longest path from root to a leaf.

Key Insight

depth(node) = 1 + max(depth(left), depth(right)). Base case: depth(None) = 0. This one-line recurrence captures the entire algorithm.

How to Think About It

  1. Base case: if node is None, return 0.
  2. Recursively compute the depth of the left subtree.
  3. Recursively compute the depth of the right subtree.
  4. Return 1 + max(left_depth, right_depth).

Why This Works

Every node adds exactly 1 to the depth. The longest root-to-leaf path goes through whichever subtree is deeper. By taking the max at each node, we propagate the longest path upward.

  • Off-by-one: returning 1 for None instead of 0 makes every depth one too high
Python 
# recursive one-liner
def maxDepth(self, root):
    if not root: return 0
    return 1 + max(self.maxDepth(root.left), self.maxDepth(root.right))

# "not root" is idiomatic Python for None check
Go 
func maxDepth(root *TreeNode) int {
    if root == nil { return 0 }
    left := maxDepth(root.Left)
    right := maxDepth(root.Right)
    return 1 + max(left, right) // Go 1.21+ built-in max
}

type TreeNode struct {
    Val   int
    Left  *TreeNode
    Right *TreeNode
}
Time: O(n)
Space: O(h)

#226 · Invert Binary Tree

EasyTreeDFS

The Problem

Invert a binary tree — swap every left child with its right child at every node.

Key Insight

Swap left and right children at each node, then recursively invert both subtrees. Swapping at every level mirrors the entire tree.

How to Think About It

  1. Base case: if node is None, return None.
  2. Swap: node.left, node.right = node.right, node.left.
  3. Recursively invert left and right subtrees.
  4. Return the node.

Why This Works

Inverting means mirroring. Swapping children at each node achieves this locally. Since we recurse into both subtrees, the mirror is applied all the way down.

Same recursive tree traversal as #104, but modifying structure instead of computing a value.
Python 
# tuple swap — no temp variable needed
root.left, root.right = root.right, root.left
self.invertTree(root.left)
self.invertTree(root.right)
return root
Go 
func invertTree(root *TreeNode) *TreeNode {
    if root == nil { return nil }
    root.Left, root.Right = root.Right, root.Left
    invertTree(root.Left)
    invertTree(root.Right)
    return root
}
Time: O(n)
Space: O(h)

#98 · Validate Binary Search Tree

MediumTreeBST

The Problem

Determine if a binary tree is a valid BST: for every node, all values in its left subtree are strictly less, and all values in its right subtree are strictly greater.

Key Insight

Pass a valid range (low, high) down during recursion. The root's range is (-∞, +∞). When going left, the upper bound tightens to the parent's value. When going right, the lower bound tightens. Each node must satisfy low < node.val < high.

How to Think About It

  1. Define a helper validate(node, low, high) starting with (-inf, inf).
  2. Base case: None is valid.
  3. If node.val <= low or node.val >= high, return False.
  4. Recurse left with validate(node.left, low, node.val).
  5. Recurse right with validate(node.right, node.val, high).

Why This Works

The BST property is transitive: a node deep in the left subtree must be less than every ancestor it's a left descendant of. By propagating bounds downward, each node's constraint incorporates all ancestor constraints.

⚠ Most Dangerous Mistake

Only checking against the immediate parent. A tree like [5, 1, 4, null, null, 3, 6] has 3 in the right subtree of 5's right child 4 — valid locally but 3 < 5 makes it invalid globally.

Alternative: In-order traversal should produce a strictly increasing sequence.
Python 
def validate(node, low=float('-inf'), high=float('inf')):
    if not node: return True
    if not (low < node.val < high): return False  # chained comparison
    return validate(node.left, low, node.val) and \
           validate(node.right, node.val, high)
Go 
import "math"

func isValidBST(root *TreeNode) bool {
    return validate(root, math.MinInt64, math.MaxInt64)
}
func validate(node *TreeNode, low, high int) bool {
    if node == nil { return true }
    if node.Val <= low || node.Val >= high { return false }
    return validate(node.Left, low, node.Val) &&
           validate(node.Right, node.Val, high)
}
Time: O(n)
Space: O(h)

#230 · Kth Smallest Element in a BST

MediumTreeBSTIn-Order

The Problem

Given the root of a BST and integer k, return the kth smallest value (1-indexed).

Key Insight

In-order traversal of a BST visits nodes in ascending sorted order. Perform in-order traversal and stop at the kth node. Use iterative traversal with an explicit stack for clean early termination.

How to Think About It

  1. Use iterative in-order: push all left children onto a stack until reaching None.
  2. Pop a node — this is the next smallest element.
  3. Decrement k. When k reaches 0, return current value.
  4. Move to the popped node's right child and repeat.

Why This Works

The BST property guarantees in-order traversal = sorted order. The kth value popped is the kth smallest. The iterative approach allows stopping as soon as we've found it.

⚠ Most Dangerous Mistake

Off-by-one: k is 1-indexed. Decrement k after visiting, check k == 0.

Leverages the BST property validated in #98. In-order traversal is the "read" operation for a BST.
Python 
# iterative in-order with stack
stack = []
current = root
while stack or current:
    while current:          # go as far left as possible
        stack.append(current)
        current = current.left
    current = stack.pop()   # visit
    k -= 1
    if k == 0: return current.val
    current = current.right # move right
Go

        stack = append(stack, current)
        current = current.Left
    }

    stack = stack[:len(stack)-1]

    current = current.Right
}

#236 · Lowest Common Ancestor of a Binary Tree

MediumTreeDFS

The Problem

Given a binary tree and two nodes p and q (guaranteed to exist), find their lowest common ancestor.

Key Insight

DFS returns the node itself if it matches p or q, and propagates non-None results upward. If a node gets non-None from both subtrees, p and q are split — that node is the LCA. If only one side returns non-None, propagate it up.

How to Think About It

  1. Base case: if node is None, return None. If node is p or q, return node.
  2. Recurse left and right.
  3. If both return non-None: current node is the LCA.
  4. Otherwise, return whichever is non-None.

Why This Works

The LCA is the deepest node where paths to p and q diverge. The DFS finds p and q and bubbles them upward. The first node to receive signals from both subtrees is the divergence point. If p is an ancestor of q, p is returned before q is found — making p itself the LCA.

⚠ Most Dangerous Mistake

Not handling the case where p is an ancestor of q. The base case if node == p or node == q: return node handles this correctly.

Python 
def lowestCommonAncestor(self, root, p, q):
    if not root or root == p or root == q:
        return root
    left = self.lowestCommonAncestor(root.left, p, q)
    right = self.lowestCommonAncestor(root.right, p, q)
    if left and right: return root   # split = LCA
    return left or right             # propagate non-None

    left := lowestCommonAncestor(root.Left, p, q)

MediumGraphDijkstraHeap

The Problem

Key Insight

Dijkstra's shortest path: find the shortest distance from k to every node, then return the maximum (the last node to receive the signal determines the answer). Use a min-heap. Skip "stale" heap entries.

How to Think About It

  1. Build a weighted adjacency list.
  2. Initialize distances to infinity. Set dist[k] = 0.
  3. Push (0, k) to a min-heap.
  4. Pop minimum: (d, u). If d > dist[u], skip (stale entry).
  5. For each neighbor: if dist[u] + w < dist[v], update and push.
  6. If any node is still infinity, return -1. Otherwise return max(dist).

Why This Works

Dijkstra's invariant: when a node is popped from the min-heap, its distance is finalized (since all weights are non-negative). The heap processes nodes in order of increasing distance. The "stale entry skip" handles multiple heap entries for the same node.

⚠ Most Dangerous Mistake

Not skipping stale heap entries. When you find a shorter path and push a new entry, the old entry stays. Processing it later re-relaxes edges with a longer distance.

  1. Initialize parent[i] = i and rank[i] = 0.

Alternative — DFS/BFS: Run DFS from each unvisited node, count launches.
Python 
def find(x):
    if parent[x] != x:
        parent[x] = find(parent[x])  # path compression
    return parent[x]

    rx, ry = find(x), find(y)
    if rx == ry: return
    if rank[rx] < rank[ry]: rx, ry = ry, rx
    parent[ry] = rx
    if rank[rx] == rank[ry]: rank[rx] += 1

#124 · Binary Tree Maximum Path Sum

HardTreeDFS

The Problem

Given a binary tree, find the maximum path sum. A path is any sequence of connected nodes (doesn't need to pass through root). Node values can be negative.

Key Insight

The DFS computes two different things simultaneously: (1) the best single-branch gain going downward (returned to parent), and (2) the best path THROUGH the node using both subtrees (updates global max). A node can use both branches for a through-path, but only one branch to extend upward to its parent.

How to Think About It

  1. Initialize max_sum = -infinity (handles all-negative trees).
  2. DFS returns max downward gain: node.val + max(left_gain, right_gain).
  3. Use max(gain, 0) for subtrees — never follow a negative branch.
  4. At each node, compute through-path: node.val + left_gain + right_gain. Update global max.
  5. Return to parent: only the better single branch.
  6. Base case: None returns 0.

Why This Works

Every max path has a "peak" node — the highest point. At each node, we compute the best path that peaks there (using both subtrees) and compare to the global max. The return value only includes one branch because a path can't fork three ways. By visiting every node as a potential peak, we consider all possible paths.

⚠ Most Dangerous Mistake

Returning left_gain + node.val + right_gain to the parent. This is a V-shaped path the parent can't extend. Only return node.val + max(left_gain, right_gain).

  • Initializing max_sum = 0 instead of -inf — fails with all-negative trees
  • Not applying max(gain, 0) — negative subtrees should contribute 0
Same recursive DFS as #104 and #236, but with two-quantity pattern: return one thing, update global with another.
Python 
max_sum = float('-inf')
def dfs(node):
    nonlocal max_sum
    if not node: return 0
    left = max(dfs(node.left), 0)   # discard negative
    right = max(dfs(node.right), 0)
    max_sum = max(max_sum, node.val + left + right)  # through-path
    return node.val + max(left, right)  # single branch to parent

# nonlocal lets nested function modify enclosing scope
# float('-inf') handles all-negative trees
Go 
import "math"

func maxPathSum(root *TreeNode) int {
    maxSum := math.MinInt32
    var dfs func(*TreeNode) int
    dfs = func(node *TreeNode) int {
        if node == nil { return 0 }
        left := max(dfs(node.Left), 0)
        right := max(dfs(node.Right), 0)
        candidate := node.Val + left + right
        if candidate > maxSum { maxSum = candidate }
        return node.Val + max(left, right)
    }
    dfs(root)
    return maxSum
}
