                if (visited[node]) continue;
                visited[node] = true;
                for (int next : graph[node]) stk.push(next);

    Step-by-Step Thinking

    2. Track node states: 0 = unvisited, 1 = in-progress (on current DFS path), 2 = done.
    3. For each unvisited node, run DFS.
    4. In DFS: mark the node as in-progress, visit all its neighbors. If a neighbor is in-progress, a cycle exists (return false).
    5. After visiting all neighbors, mark the node as done.
    6. If no cycle is found across all starting points, return true.

    #684 — Redundant Connection

    Medium Union-Find Graph

    A tree of n nodes (1-indexed) has one extra edge added, forming exactly one cycle. Given the edges, find the edge that can be removed to restore the tree. If multiple answers exist, return the one that occurs last in the input.

    Brute Force Approach — O(n^2)

    For each edge (starting from the last), try removing it and check if the remaining graph is still connected. The first removal that keeps the graph connected is the answer.

    Pythonclass Solution:
    def findRedundantConnection(self, edges: List[List[int]]) -> List[int]:
        n = len(edges)
        for skip in range(n - 1, -1, -1):
            graph = [[] for _ in range(n + 1)]
            for i, (u, v) in enumerate(edges):
                if i == skip: continue
                graph[u].append(v)
                graph[v].append(u)
            visited = set()
            stack = [1]
            while stack:
                node = stack.pop()
                if node in visited: continue
                visited.add(node)
                for nei in graph[node]: stack.append(nei)
            if len(visited) == n:
                return edges[skip]
        return edges[-1]

            if k == key:
                self.cache.pop(i)
                self.cache.append([key, value])
                return
        if len(self.cache) >= self.capacity:
            self.cache.pop(0)

        for i in range(1, n):
            for j in range(i):
                if intervals[j][1] <= intervals[i][0]:
                    dp[i] = max(dp[i], dp[j] + 1)
        return n - max(dp)

        for (int i = 1; i < n; i++)
            for (int j = 0; j < i; j++)
                if (intervals[j][1] <= intervals[i][0])
                    dp[i] = max(dp[i], dp[j] + 1);
        return n - *max_element(dp.begin(), dp.end());
    }
};
    Key Insight: Sort by end time. Greedily keep intervals that do not overlap with the last kept interval. Count the number of removals. This is the classic "activity selection" problem: choosing the activity that finishes earliest maximizes the number of non-overlapping activities.

    Step-by-Step Thinking

    1. Sort intervals by end time.
    2. Track the end time of the last kept interval (initialized to negative infinity).
    3. For each interval: if start >= last_end, keep it (update last_end to this interval's end). Otherwise, it overlaps — increment the removal count.
    4. Return the removal count.
    Why It Works: Choosing intervals with the earliest end times maximizes the number of non-overlapping intervals you can keep. This is a classic greedy algorithm with a proof by exchange argument: any other choice can be improved by swapping in the earlier-ending interval.
    Watch Out: Sort by END time, not start time. Equal endpoints (e.g., [1,2] and [2,3]) do NOT overlap — use >= (not >) for the non-overlap check.

    #452 — Minimum Number of Arrows to Burst Balloons

    Medium Greedy Sorting

    Balloons are represented as intervals [x_start, x_end]. An arrow shot at position x bursts all balloons where x_start <= x <= x_end. Return the minimum number of arrows needed to burst all balloons.

    Greedy Approach — O(n log n)

    Sort by end position. Shoot at each balloon's rightmost point. Skip subsequent balloons that overlap. This is equivalent to the activity selection problem.
