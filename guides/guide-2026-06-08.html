#84 · Largest Rectangle in Histogram

Hard Monotonic Stack 6 days since last review · last 2026-06-02

The Problem: Given an array of integers heights representing the histogram’s bar height where the width of each bar is 1, return the area of the largest rectangle in the histogram.

Example 1: heights = [2,1,5,6,2,3] → 10 (rectangle of height 5, width 2)

Example 2: heights = [2,4] → 4

Quick Reminder: Use a monotonic increasing stack storing indices. When a shorter bar is found, pop and calculate areas. The stack stores indices so you can compute widths.
Key Insight: When you pop index i, the width extends from the current stack top + 1 to the current position - 1. The popped bar’s height is the limiting height of a rectangle whose left boundary is one past the new stack top and whose right boundary is one before the current index.
Try to recall: Implement the monotonic stack approach. Pay attention to: (1) what happens when the stack is empty after a pop (width extends to index 0), (2) processing remaining elements after the main loop, (3) using -1 as a sentinel on the stack.
O(n) time, O(n) space

#146 · LRU Cache

Medium Design Hash Map Doubly Linked List 6 days since last review · last 2026-06-02

The Problem: Design a data structure that follows the constraints of a Least Recently Used (LRU) cache. Implement get(key) and put(key, value) in O(1) time.

Example: cache = LRUCache(2), put(1,1), put(2,2), get(1) → 1, put(3,3), get(2) → -1 (evicted)

Quick Reminder: Hash map + doubly linked list. The map stores key → node. The DLL tracks access order with most-recent at head.
Key Insight: On access, move the node to the head of the list (mark as most recently used). On eviction when capacity is exceeded, remove the tail node (least recently used) and delete its key from the map.

O(1) time for get and put, O(capacity) space

#191 · Number of 1 Bits

Easy Bit Manipulation 6 days since last review · last 2026-06-02

The Problem: Write a function that takes the binary representation of a positive integer and returns the number of set bits (1-bits) it has (also known as the Hamming weight).

Example 1: n = 11 (binary: 1011) → 3

Example 2: n = 128 (binary: 10000000) → 1

Quick Reminder: n & (n - 1) clears the lowest set bit. Count iterations until n becomes 0.
Key Insight: Brian Kernighan’s trick — each n & (n - 1) operation removes exactly one 1-bit (the lowest). The number of iterations equals the number of set bits. This is more efficient than checking all 32 bits.
Try to recall: Write the loop using Brian Kernighan’s trick. Also consider: can you solve it with n & 1 plus right-shift? What are the trade-offs?
O(k) time where k = number of set bits, O(1) space

#496 · Next Greater Element I

Easy Monotonic Stack 6 days since last review · last 2026-06-02

The Problem: Given nums1 (a subset of nums2 with no duplicates), for each element in nums1, find the next greater element in nums2. The next greater element of x in nums2 is the first element to the right of x in nums2 that is greater than x. Return -1 if it does not exist.

Example: nums1 = [4,1,2], nums2 = [1,3,4,2] → [-1, 3, -1]

Quick Reminder: Process nums2 right-to-left with a monotonic decreasing stack. Build a map of element → next_greater. Then look up each element from nums1 in the map.
Key Insight: The stack holds candidates for “next greater.” Pop all elements smaller than or equal to the current — they can never be the next greater for anything to the left. The stack top (if non-empty) is the next greater for the current element.
Try to recall: Implement the monotonic stack approach processing right-to-left. Alternative: process left-to-right and when you pop from the stack, the current element is the next greater for the popped element.
O(n + m) time, O(n) space where n = len(nums2), m = len(nums1)

#265 · Paint House II

The Problem: There are n houses in a row, each can be painted with one of k colors. The cost of painting each house with a certain color is given by an n x k cost matrix. No two adjacent houses can have the same color. Find the minimum cost to paint all houses.

Example: costs = [[1,5,3],[2,9,4]] → 5 (house 0 color 0 cost 1 + house 1 color 2 cost 4)

#1584 · Min Cost to Connect All Points

Medium Graph MST Prim’s Kruskal’s

The Problem: You are given an array points where points[i] = [x_i, y_i] represents a point on the 2D plane. The cost of connecting two points is the Manhattan distance |x_i - x_j| + |y_i - y_j|. Return the minimum cost to make all points connected (every pair of points has exactly one simple path between them).

Example 1: points = [[0,0],[2,2],[3,10],[5,2],[7,0]] → 20

Example 2: points = [[3,12],[-2,5],[-4,1]] → 18

Example 3: points = [[0,0],[1,1],[1,0],[-1,1]] → 4

Brute Force — Try All Spanning Trees (Exponential)

Generate all possible spanning trees from n(n-1)/2 edges and find the one with minimum total weight. This is exponential in the number of edges. A more practical brute force: generate all edges, then try all subsets of size n-1 that form a spanning tree.

Python
class Solution:
    def minCostConnectPoints(self, points: list[list[int]]) -> int:
        n = len(points)
        if n <= 1:
            return 0

        # Generate all edges
        edges = []
        for i in range(n):
            for j in range(i + 1, n):
                dist = abs(points[i][0] - points[j][0]) + abs(points[i][1] - points[j][1])
                edges.append((dist, i, j))

        # Brute force: try all combinations of n-1 edges
        from itertools import combinations
        min_cost = float('inf')
        for combo in combinations(edges, n - 1):
            # Check if these edges form a spanning tree
            parent = list(range(n))
            def find(x):
                while parent[x] != x:
                    parent[x] = parent[parent[x]]
                    x = parent[x]
                return x
            cost = 0
            valid = True
            for d, u, v in combo:
                pu, pv = find(u), find(v)
                if pu == pv:
                    valid = False
                    break
                parent[pu] = pv
                cost += d
            if valid:
                # Check all connected
                root = find(0)
                if all(find(i) == root for i in range(n)):
                    min_cost = min(min_cost, cost)
        return min_cost

Key Insight (Prim’s): Start from any point. Maintain a min-heap of (distance, point_index) for all unvisited points. Greedily extract the nearest unconnected point, add it to the MST, and push distances to its neighbors. For dense graphs (all n^2 edges), Prim’s with a heap runs in O(n^2 log n).
Key Insight (Kruskal’s): Generate all n(n-1)/2 edges, sort by weight, then greedily add edges using Union-Find to avoid cycles. Stop when n-1 edges are added. Also O(n^2 log n) due to sorting.

How to Think About It

  1. This is a complete graph (every pair of points has an edge). Recognize that “connect all points with minimum total cost” = Minimum Spanning Tree.
  2. Prim’s approach: Start from node 0. Push (0, 0) into a min-heap. While the MST has fewer than n nodes: pop the minimum (cost, u); if u is already visited, skip; otherwise mark u visited, add cost to total, and push (manhattan_dist(u, v), v) for all unvisited v.
  3. Kruskal’s approach: Generate all edges (dist, i, j). Sort by dist. Initialize Union-Find. For each edge, if i and j are in different components, union them and add the cost. Stop after n-1 edges.
  4. Return the total cost.
Why This Works: MST guarantees the minimum total edge weight to connect all nodes. Prim’s greedy selection always picks the cheapest edge crossing the cut between visited and unvisited nodes. Kruskal’s greedy selection always picks the globally cheapest edge that does not form a cycle. Both are optimal by the cut property of MSTs.
Watch Out For:
  • Dense graph = n^2 edges: For n=1000, that is 500,000 edges. Kruskal’s sort is fine but Prim’s heap can get large. Consider O(n^2) Prim’s without a heap for very dense graphs.
  • Skipping visited nodes in Prim’s: The heap may contain stale entries for already-visited nodes. Always check before processing.
  • Manhattan distance, not Euclidean: |x1 - x2| + |y1 - y2|, not sqrt of squares.

import heapq

class Solution:
    def minCostConnectPoints(self, points: list[list[int]]) -> int:
        n = len(points)
        if n <= 1:
            return 0

        visited = [False] * n
        heap = [(0, 0)]  # (cost, node)
        total = 0
        edges_used = 0

        while edges_used < n:
            cost, u = heapq.heappop(heap)
            if visited[u]:
                continue
            visited[u] = True
            total += cost
            edges_used += 1

            for v in range(n):
                if not visited[v]:
                    dist = abs(points[u][0] - points[v][0]) + abs(points[u][1] - points[v][1])
                    heapq.heappush(heap, (dist, v))

        return total

function minCostConnectPoints(points: number[][]): number {
    const n = points.length;
    if (n <= 1) return 0;

    // Simple min-heap implementation
    class MinHeap {
        private data: [number, number][] = [];
        push(val: [number, number]) {
            this.data.push(val);
            this.bubbleUp(this.data.length - 1);
        }
        pop(): [number, number] {
            const top = this.data[0];
            const last = this.data.pop()!;
            if (this.data.length > 0) {
                this.data[0] = last;
                this.sinkDown(0);
            }
            return top;
        }
        get size() { return this.data.length; }
        private bubbleUp(i: number) {
            while (i > 0) {
                const parent = (i - 1) >> 1;
                if (this.data[parent][0] <= this.data[i][0]) break;
                [this.data[parent], this.data[i]] = [this.data[i], this.data[parent]];
                i = parent;
            }
        }
        private sinkDown(i: number) {
            const n = this.data.length;
            while (true) {
                let smallest = i;
                const l = 2 * i + 1, r = 2 * i + 2;
                if (l < n && this.data[l][0] < this.data[smallest][0]) smallest = l;
                if (r < n && this.data[r][0] < this.data[smallest][0]) smallest = r;
                if (smallest === i) break;
                [this.data[smallest], this.data[i]] = [this.data[i], this.data[smallest]];
                i = smallest;
            }
        }
    }

    const visited = new Array(n).fill(false);
    const heap = new MinHeap();
    heap.push([0, 0]);
    let total = 0, edgesUsed = 0;

    while (edgesUsed < n) {
        const [cost, u] = heap.pop();
        if (visited[u]) continue;
        visited[u] = true;
        total += cost;
        edgesUsed++;

        for (let v = 0; v < n; v++) {
            if (!visited[v]) {
                const d = Math.abs(points[u][0] - points[v][0]) + Math.abs(points[u][1] - points[v][1]);
                heap.push([d, v]);
            }
        }
    }
    return total;
}

Language-Specific Notes

  • Python: heapq provides a min-heap natively. Push tuples (cost, node) — Python compares tuples lexicographically.
  • Go: container/heap requires implementing the heap.Interface. Define a custom struct with Len, Less, Swap, Push, Pop.
  • TypeScript: No built-in min-heap — implement your own or use a sorted structure. The manual heap implementation above is standard for interviews.
  • Rust: BinaryHeap is a max-heap by default. Wrap elements in Reverse to get min-heap behavior.
  • C++: priority_queue<..., greater<>> for a min-heap. Structured bindings auto [cost, u] for clean extraction.
Brute: O(n^2) Prim’s without heap → Optimal: O(n^2 log n) Prim’s with heap

Connection: This is the classic MST problem on a complete graph. For dense graphs, O(n^2) Prim’s without a heap can actually be faster than O(n^2 log n) with a heap, since the heap operations dominate. Kruskal’s is an equally valid alternative — it pairs well with Union-Find (#323, #684). The choice depends on graph density: Prim’s for dense, Kruskal’s for sparse.

#1168 · Optimize Water Distribution in a Village

Hard Graph MST Kruskal’s Union-Find

The Problem: There are n houses in a village. We can supply water by building wells or laying pipes. For each house i, we can build a well with cost wells[i-1], or we can lay pipes between houses. pipes[j] = [house1, house2, cost]. Find the minimum total cost to supply water to all houses.

Key Insight: Add a virtual node 0 representing the water source. Connect it to each house i with an edge of cost wells[i]. Now the problem becomes a standard MST on the augmented graph. Building a well at house i is equivalent to connecting house i to the virtual source via an edge.

How to Think About It

  1. Create a virtual node 0. For each house i (1-indexed), add edge (0, i, wells[i-1]).
  2. Add all pipe edges to the edge list: (house1, house2, cost).
  3. Sort all edges by cost. Initialize Union-Find with n+1 nodes (0 through n).
  4. Run Kruskal’s: for each edge in sorted order, if the two endpoints are in different components, union them and add the cost. Stop after n edges (connecting n+1 nodes).
  5. Return the total cost.
Why This Works: The virtual node trick elegantly unifies two types of costs (wells and pipes) into a single graph problem. A well at house i is just an edge from the virtual source to house i. The MST on this augmented graph finds the cheapest way to connect every house to the water source, whether through wells or pipes.
Watch Out For:
  • 1-indexed houses: Houses are numbered 1 to n. The virtual node is 0. Make sure Union-Find covers indices 0 through n.
  • Forgetting the virtual node trick: Without it, you need to consider subsets of wells, making the problem exponential.
  • Number of MST edges: The MST on n+1 nodes has exactly n edges.

Key Insight: If prefix[i] % k == prefix[j] % k, then the subarray sum prefix[j] - prefix[i] is divisible by k. Use a hash map to count how many times each remainder has appeared. For each remainder r with count c, the number of valid pairs is c * (c - 1) / 2. Handle negative remainders: ((prefix % k) + k) % k.

How to Think About It

  1. Initialize a hash map count with {0: 1} (the empty prefix has sum 0, remainder 0).
  2. Iterate through nums, maintaining a running prefix sum.
  3. At each step, compute the remainder: r = ((prefix % k) + k) % k to handle negative values.
  4. Add count[r] to the result (each previous prefix with the same remainder forms a valid subarray with the current prefix).
  5. Increment count[r].
Why This Works: If two prefix sums have the same remainder when divided by k, their difference is divisible by k. By counting occurrences of each remainder, we efficiently count all such pairs. The ((prefix % k) + k) % k formula normalizes negative remainders to the range [0, k-1].
Watch Out For:
  • Negative remainders: In most languages, -1 % 5 gives -1, not 4. Use ((x % k) + k) % k to normalize.
  • Initializing count[0] = 1: The empty prefix has sum 0 (remainder 0). Without this, you miss subarrays starting from index 0.
  • Python exception: Python’s % operator already returns non-negative results for positive k, so the extra normalization is not strictly needed in Python but is good practice.

Key Insight: Sliding window with a frequency map. Expand the right pointer, adding characters to the map. When the number of distinct characters exceeds k, shrink the left pointer, decrementing frequencies and removing characters whose count hits 0. Track the maximum window size throughout.

How to Think About It

  1. Initialize two pointers left = 0, a frequency map, and best = 0.
  2. For each right from 0 to n-1: add s[right] to the frequency map.
  3. While the map has more than k distinct keys: decrement freq[s[left]]. If it becomes 0, remove the key. Increment left.
  4. Update best = max(best, right - left + 1).
  5. Return best.
Why This Works: The sliding window invariant is that the window [left, right] always contains at most k distinct characters. When the invariant is violated, we shrink from the left. Since both pointers only move forward, each element is added and removed at most once, giving O(n) total.
Watch Out For:
  • Not removing zero-count keys: If you forget to remove a key when its frequency drops to 0, the distinct count stays too high and the window shrinks unnecessarily.
  • k = 0: The answer is 0 (no characters allowed). Make sure your code handles this edge case.
  • Off-by-one with window size: The window size is right - left + 1, not right - left.

Key Insight: “Exactly k distinct” = “at most k distinct” - “at most (k-1) distinct.” Write a helper function atMost(k) that uses a sliding window to count subarrays with at most k distinct integers. The answer is atMost(k) - atMost(k-1).

How to Think About It

  1. Write atMost(t): sliding window with a frequency map. For each right, add nums[right]. While distinct > t, shrink left. Add right - left + 1 to the count (this counts all subarrays ending at right with at most t distinct).
  2. The key counting trick: for a window [left, right] with at most t distinct, the number of valid subarrays ending at right is right - left + 1 (starting at left, left+1, ..., right).
  3. Return atMost(k) - atMost(k-1).
Why This Works: The set of subarrays with “at most k” distinct integers includes all subarrays with 0, 1, 2, ..., k distinct integers. Subtracting “at most k-1” removes all subarrays with 0, 1, ..., k-1 distinct integers, leaving exactly those with k distinct. The sliding window for atMost runs in O(n), and we call it twice, so total is O(n).
Watch Out For:
  • Counting subarrays correctly: For each right, the number of valid subarrays ending at right is right - left + 1. Do not just count 1 per window position.
  • Cleaning up the frequency map: When freq[x] drops to 0, delete the key so that len(freq) / freq.size() accurately reflects the distinct count.
  • Edge case k=1: atMost(0) should return 0 (no non-empty subarray has 0 distinct integers).

Step-by-Step (Optimal — Min-Heap)

  1. Seed the heap: Push (nums1[i] + nums2[0], i, 0) for i = 0 to min(len(nums1), k) - 1. Each entry represents one “row” of candidates.
  2. Pop the minimum: The smallest sum in the heap is the next pair to add to the result.
  3. Push the next candidate: If j + 1 < len(nums2), push (nums1[i] + nums2[j+1], i, j+1) from the same row.
  4. Repeat until k pairs collected or the heap is empty.

    if (!nums1.length || !nums2.length) return [];

    const heap: [number, number, number][] = []; // [sum, i, j]

    const push = (item: [number, number, number]) => {
        heap.push(item);
        let i = heap.length - 1;
        while (i > 0) {
            const p = (i - 1) >> 1;
            if (heap[p][0] <= heap[i][0]) break;
            [heap[p], heap[i]] = [heap[i], heap[p]];
            i = p;
        }
    };

    const pop = (): [number, number, number] => {
        const top = heap[0];
        const last = heap.pop()!;
        if (heap.length > 0) {
            heap[0] = last;
            let i = 0;
            while (true) {
                let min = i;
                const l = 2 * i + 1, r = 2 * i + 2;
                if (l < heap.length && heap[l][0] < heap[min][0]) min = l;
                if (r < heap.length && heap[r][0] < heap[min][0]) min = r;
                if (min === i) break;
                [heap[i], heap[min]] = [heap[min], heap[i]];
                i = min;
            }
        }
        return top;

        push([nums1[i] + nums2[0], i, 0]);

#378 · Kth Smallest Element in a Sorted Matrix

Medium Heap Binary Search Matrix

The Problem: Given an n × n matrix where each of the rows and columns are sorted in non-decreasing order, return the kth smallest element in the matrix.

Example 1: matrix=[[1,5,9],[10,11,13],[12,13,15]], k=8 → 13

Example 2: matrix=[[-5]], k=1 → -5

Example 3: matrix=[[1,3,5],[6,7,12],[11,14,14]], k=6 → 11

Key Insight (Heap): Treat each row as a sorted list. Use a min-heap seeded with the first element of each row: (matrix[i][0], i, 0). Pop k times to get the answer. This is the same merge-k pattern as #373.

Brute Force — Flatten and Sort — O(n² log n²)

Step-by-Step (Heap Approach)

  1. Seed the heap: Push (matrix[i][0], i, 0) for each row i.
  2. Pop k times: Each pop gives the next smallest element.
  3. Push the successor: After popping (val, i, j), if j + 1 < n, push (matrix[i][j+1], i, j+1).
  4. The k-th pop is the answer.

Step-by-Step (Binary Search Approach)

  1. Set search range: lo = matrix[0][0], hi = matrix[n-1][n-1].
  2. Binary search on value: mid = (lo + hi) / 2.
  3. Count elements ≤ mid: Start from bottom-left (n-1, 0). If matrix[r][c] ≤ mid, add r + 1 to count and move right. Otherwise move up.
  4. Narrow: If count < k, lo = mid + 1. Else hi = mid.
  5. Return lo when lo == hi.

#632 · Smallest Range Covering Elements from K Lists

Hard Heap Multi-source Merge Sliding Window

The Problem: You have k lists of sorted integers in non-decreasing order. Find the smallest range [a, b] that includes at least one number from each of the k lists. The range [a, b] is smaller than [c, d] if b - a < d - c, or a < c if b - a == d - c.

Example 1: nums=[[4,10,15,24,26],[0,9,12,20],[5,18,22,30]] → [20,24] (24 from list 0, 20 from list 1, 22 from list 2)

Example 2: nums=[[1,2,3],[1,2,3],[1,2,3]] → [1,1]

Example 3: nums=[[10],[11],[13]] → [10,13]

Key Insight: Use a min-heap of (value, list_index, element_index). Track the current max across all list heads. The range is [heap_min, current_max]. Pop the minimum, push the next element from the same list, update max. Track the smallest range seen. When any list is exhausted, stop.
Brute Force — Try All Possible Ranges — O(n²)

Step-by-Step (Optimal — Min-Heap + Max Tracking)

  1. Seed the heap: Push the first element from each of the k lists: (value, list_index, 0). Track the current_max among these seeds.
  2. Compute range: Current range is [heap[0].value, current_max]. If this is smaller than the best range, update it.
  3. Advance the minimum: Pop the min element. Push the next element from the same list. Update current_max if the new element is larger.
  4. Stop when any list is exhausted: If the popped element was the last in its list, we cannot improve further because moving the minimum up would leave one list uncovered.
  5. Return the smallest range seen.

function smallestRange(nums: number[][]): number[] {
    // Min-heap implementation
    const heap: [number, number, number][] = [];
    const push = (item: [number, number, number]) => {
        heap.push(item);
        let i = heap.length - 1;
        while (i > 0) {
            const p = (i - 1) >> 1;
            if (heap[p][0] <= heap[i][0]) break;
            [heap[p], heap[i]] = [heap[i], heap[p]];
            i = p;
        }
    };
    const pop = (): [number, number, number] => {
        const top = heap[0];
        const last = heap.pop()!;
        if (heap.length > 0) {
            heap[0] = last;
            let i = 0;
            while (true) {
                let min = i;
                const l = 2 * i + 1, r = 2 * i + 2;
                if (l < heap.length && heap[l][0] < heap[min][0]) min = l;
                if (r < heap.length && heap[r][0] < heap[min][0]) min = r;
                if (min === i) break;
                [heap[i], heap[min]] = [heap[min], heap[i]];
                i = min;
            }
        }
        return top;
    };

    let curMax = -Infinity;
    for (let i = 0; i < nums.length; i++) {
        push([nums[i][0], i, 0]);
        curMax = Math.max(curMax, nums[i][0]);
    }

    let best = [-Infinity, Infinity];
    while (true) {
        const [curMin, li, ei] = pop();
        if (curMax - curMin < best[1] - best[0]) {
            best = [curMin, curMax];
        }
        if (ei + 1 === nums[li].length) break;
        const nxt = nums[li][ei + 1];
        push([nxt, li, ei + 1]);
        curMax = Math.max(curMax, nxt);
    }

#214 · Shortest Palindrome

Hard String KMP Palindrome

The Problem: You are given a string s. You can convert it to a palindrome by adding characters in front of it. Return the shortest palindrome you can find by performing this transformation.

Example 1: s="aacecaaa" → "aaacecaaa" (add "a" in front)

Example 2: s="abcd" → "dcbabcd" (add "dcb" in front)

Example 3: s="" → "" (empty string is already a palindrome)

Example 4: s="a" → "a" (single char is a palindrome)

Key Insight: Find the longest palindromic prefix of s. Characters after this prefix (reversed) must be prepended. Use KMP: build s + "#" + reverse(s), compute the failure/LPS array. The last value tells you the longest palindromic prefix length.
Why KMP Works: The KMP failure function finds the longest prefix of s that matches a suffix of reverse(s) — which is exactly the longest palindromic prefix. The # separator prevents the failure function from matching across the boundary (which would give false results longer than the original string).
Brute Force — Check Each Prefix for Palindrome — O(n²)

Try every prefix of s from longest to shortest. For each, check if it is a palindrome. Once found, prepend the reversed remaining suffix.

Step-by-Step (KMP Approach)

  1. Build the combined string: combined = s + "#" + reverse(s). The # separator prevents cross-boundary matches.
  2. Compute the KMP failure (LPS) array: For each position i, lps[i] = length of the longest proper prefix of combined[0..i] that is also a suffix.

  4. Build result: reverse(s[lps[-1]:]) + s.

#459 · Repeated Substring Pattern

Easy String KMP String Matching

The Problem: Given a string s, check if it can be constructed by taking a substring of it and appending multiple copies of the substring together. The substring must be used more than once.

Example 1: s="abab" → true ("ab" repeated twice)

Example 2: s="aba" → false

Example 3: s="abcabcabcabc" → true ("abc" repeated four times)

Example 4: s="a" → false (single char, needs more than one copy)

Key Insight (KMP): Build the KMP failure/LPS array. If lps[n-1] > 0 and n % (n - lps[n-1]) == 0, then the string has a repeating pattern of length n - lps[n-1]. The failure function tells you the longest proper prefix that is also a suffix — the remainder is the repeating unit.
Key Insight (String Trick): If s is made of a repeating pattern, then (s + s)[1:-1] (the doubled string with first and last character removed) still contains s. This works because doubling creates extra copies of the pattern, and removing just one character from each end still leaves at least one complete copy of s.
Brute Force — Try All Divisor Lengths — O(n²)

Try every possible substring length that divides n. For each, check if repeating that prefix yields s.
