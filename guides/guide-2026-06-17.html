The Problem

Brute Force — O(n²) time, O(n) space

Why optimize? With n up to 104, O(n²) = 108 operations, which is borderline. More importantly, the monotonic stack technique is essential interview knowledge.

Key Insight: The brute force rescans the array for every element. The insight is that a monotonic decreasing stack, iterated over the array twice (to simulate circularity), resolves all “next greater” queries in a single pass. When you encounter a value greater than the stack top, you’ve found the answer for that stack entry.

How to Think About It

  1. Initialize a result array of -1s and an empty stack (stores indices).
  2. Iterate i from 0 to 2*n - 1 (two full passes to handle circularity).
  3. At each step, while the stack is non-empty and nums[i % n] > nums[stack.top()], pop from the stack and set res[popped] = nums[i % n].
  4. Only push i % n onto the stack during the first pass (i < n) to avoid duplicates.
Why This Works: The monotonic stack maintains elements in decreasing order. When a larger element arrives, it “resolves” all smaller pending elements on the stack in one sweep. Iterating twice covers the circular wrap-around: an element near the end can find its next-greater among elements near the beginning.
Watch Out For:
  • Only push indices during the first pass (i < n) to avoid pushing the same index twice.
  • Use i % n for array access but push the actual modded index.
  • Elements with no greater element remain -1 (already initialized).
Brute: O(n²) time, O(n) space → Optimal: O(n) time, O(n) space

Why optimize? With up to 104 calls, worst case is O(n²) total (monotonically increasing prices). The stack gives O(1) amortized.

Key Insight: The brute force re-scans backwards every call. The insight is to store (price, span) pairs on a monotonic decreasing stack. When a new price pops entries, you absorb their spans: current_span += popped_span. Each element is pushed and popped at most once across all calls, giving O(1) amortized.

How to Think About It

  1. Maintain a stack of (price, span) tuples, kept in monotonically decreasing order of price.
  2. On each next(price) call, initialize span = 1.
  3. While the stack is non-empty and stack.top().price ≤ price, pop and add the popped span to current span.
  4. Push (price, span) onto the stack and return span.
Why This Works: When a price is popped, it will never be relevant again — the current (higher) price subsumes its entire span. By accumulating spans during pops, we compress history: the current entry on the stack effectively represents “this many consecutive days were ≤ this price.”
Watch Out For:
  • The span starts at 1 (includes today), not 0.
  • The comparison is ≤ (less than or equal), not strict less than.
  • Store spans alongside prices on the stack — indices alone won’t work since pops destroy position info.
Brute: O(n) per call worst case → Optimal: O(1) amortized per call

#856. Score of Parentheses

Medium Stack String

The Problem

Given a balanced parentheses string s, compute its score: () has score 1, (A) has score 2*A, and AB has score A+B. Example: (()(())) = 2*(1 + 2*1) = 6.

Brute Force — O(n²) time (recursive parsing)

Recursively find matching parentheses. For each top-level group, compute its score. Finding the matching close paren takes O(n) each time, leading to O(n²) for nested inputs.

Python 
def scoreOfParentheses(s):
    def score(s, l, r):
        if r - l == 2: return 1
        bal, mid = 0, l
        for i in range(l, r):
            bal += 1 if s[i] == '(' else -1
            if bal == 0: mid = i + 1; break
        inner = score(s, l+1, mid-1)
        rest = score(s, mid, r) if mid < r else 0
        return 2 * inner + rest
    return score(s, 0, len(s))

  4. After processing all characters, stack[0] is the total score.

def validPath(n, edges, source, destination):
    from collections import defaultdict, deque
    graph = defaultdict(list)
    for u, v in edges:
        graph[u].append(v)
        graph[v].append(u)
    visited = {source}
    queue = deque([source])
    while queue:
        node = queue.popleft()
        if node == destination: return True
        for nei in graph[node]:
            if nei not in visited:
                visited.add(nei)
                queue.append(nei)
    return False

The Problem

Given an undirected graph (as adjacency list), determine if it is bipartite: can you color every node with one of two colors so that no two adjacent nodes share the same color?

Brute Force — O(2n) time

def isBipartite(graph):
    n = len(graph)
    for mask in range(1 << n):
        color = [(mask >> i) & 1 for i in range(n)]
        valid = True
        for u in range(n):
            for v in graph[u]:
                if color[u] == color[v]:
                    valid = False; break
            if not valid: break
        if valid: return True
    return False

            }
        }
        if valid { return true }
    }
    return false
}

How to Think About It

  2. For each uncolored node (handles disconnected components), start a BFS/DFS.

Why This Works: BFS assigns layers alternating 0-1-0-1. If a node has a neighbor already colored the same, there’s an odd-length path between them, forming an odd cycle. Odd cycles make bipartite partitioning impossible.
Watch Out For:

Why optimize? With lists up to 1000 each, O(n×m) = 106 is fine, but the two-pointer approach is cleaner and essential interview knowledge.

Key Insight: Since both lists are sorted and disjoint, use two pointers. At each step, compute the intersection of the current pair: [max(a_start, b_start), min(a_end, b_end)]. Then advance the pointer whose interval ends first (it can’t intersect anything else).

How to Think About It

  1. Initialize pointers i=0, j=0 for lists A and B.
  2. Compute intersection: lo = max(A[i][0], B[j][0]), hi = min(A[i][1], B[j][1]).
  3. If lo ≤ hi, add [lo, hi] to result.
  4. Advance the pointer whose interval ends first: if A[i][1] < B[j][1] then i++, else j++.
Why This Works: The interval that ends earlier cannot intersect any future interval from the other list (since that list is sorted by start). So advancing the earlier-ending pointer is safe — we never miss an intersection.
Brute: O(n×m) → Optimal: O(n+m) time, O(1) extra space

  4. Return count.
Why This Works: After sorting, any interval that could cover the current one must have appeared earlier (smaller/equal start, larger end). By tracking the max end, we know the “widest reach” of all previous intervals. If the current end fits within that reach, it’s covered.
Brute: O(n²) → Optimal: O(n log n) time, O(1) space

Why optimize? The problem requires O(1) extra space. The bit manipulation approach achieves this.

Key Insight: XOR all numbers to get a ^ b (the XOR of the two unique numbers). This XOR has at least one set bit (since a ≠ b). Use any set bit (e.g., the lowest: diff & (-diff)) to partition all numbers into two groups: those with that bit set and those without. Each group contains exactly one unique number — XOR each group separately to find them.

How to Think About It

  1. XOR all numbers: result is a ^ b.
  2. Find a distinguishing bit: diff = xor & (-xor) (lowest set bit).
  3. Partition numbers into two groups based on whether num & diff is set.
  4. XOR each group separately. Each gives one of the unique numbers.
Why This Works: The set bit in a ^ b means a and b differ at that position. Partitioning by that bit puts a and b in separate groups. All duplicate numbers have both copies in the same group (same bit pattern), so they cancel via XOR. Each group’s XOR yields exactly one unique number.
Watch Out For:
  • In languages with signed integers, xor & (-xor) may need care with overflow. In Python, integers are arbitrary precision so no issue.
  • The two results can be returned in any order.
Brute: O(n) time, O(n) space → Optimal: O(n) time, O(1) space

How to Think About It

  3. For each number num: iterate j from target down to num: dp[j] = dp[j] or dp[j - num].
  4. Return dp[target].
Why This Works: dp[j] represents whether a subset summing to j exists using elements seen so far. Iterating right-to-left ensures each element is used at most once (0/1 property).   • Check total % 2 != 0 first as an early exit.

The Problem

Brute Force — O(2n) time

    n = len(points)
    min_arrows = n

        arrows = 0

                    hi = min(hi, points[j][1])
                    remaining.discard(j)
            arrows += 1
        min_arrows = min(min_arrows, arrows)
    return min_arrows
