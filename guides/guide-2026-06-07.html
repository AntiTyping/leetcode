#297 · Serialize and Deserialize Binary Tree

Hard Tree BFS DFS 2 days since last review · last 2026-06-05

The Problem: Design an algorithm to serialize a binary tree into a string and deserialize that string back into the original tree. There is no restriction on how your serialization/deserialization algorithm should work — you just need to ensure a binary tree can be serialized to a string and deserialized back to the original tree structure.

Example 1: root = [1,2,3,null,null,4,5] → serialize → "1,2,null,null,3,4,null,null,5,null,null" → deserialize → [1,2,3,null,null,4,5]

Example 2: root = [] → serialize → "" → deserialize → []

Example 3: root = [1] → serialize → "1,null,null" → deserialize → [1]

Quick Reminder: Use preorder DFS traversal for both serialize and deserialize. Record null markers ("null" or "#") so the tree structure is fully captured. On deserialization, consume tokens from an iterator/queue in the same preorder to reconstruct.
Key Insight: Preorder with null sentinels uniquely defines a binary tree. Serialize: visit node → emit value (or "null") → recurse left → recurse right. Deserialize: consume next token → if "null" return None → create node → recurse left → recurse right. The iterator naturally advances through the tokens.
Try to recall: Write the serialize and deserialize functions from scratch. Pay attention to your null sentinel format and delimiter. Can you also implement a BFS (level-order) version?
Language challenge: If you previously solved this in Python, try implementing it in Go or TypeScript this time.
O(n) time, O(n) space for both serialize and deserialize

#239 · Sliding Window Maximum

Hard Monotonic Queue Sliding Window 2 days since last review · last 2026-06-05

The Problem: Given an integer array nums and a sliding window of size k, return the maximum value in each window as it slides from left to right.

Example 1: nums = [1,3,-1,-3,5,3,6,7], k = 3 → [3,3,5,5,6,7]

Example 2: nums = [1], k = 1 → [1]

Example 3: nums = [9,11], k = 2 → [11]

Quick Reminder: Use a monotonic decreasing deque storing indices. The front of the deque is always the index of the current window maximum. Before adding a new element, pop all smaller elements from the back. Remove the front if it falls outside the window.
Key Insight: A monotonic deque maintains elements in decreasing order. When a new element enters, all smaller elements in the deque will never be the maximum again (the new element is both larger and newer), so they can be discarded. This gives O(1) amortized per element.
Try to recall: Implement the monotonic deque approach. Pay attention to: (1) storing indices vs values, (2) when to pop from front vs back, (3) when to start recording results (after the first k elements).
Language challenge: If you previously solved this in Python, try implementing it in Go using a slice-based deque.
O(n) time, O(k) space

#51 · N-Queens

Hard Backtracking 2 days since last review · last 2026-06-05

The Problem: Place n queens on an n x n chessboard such that no two queens attack each other. Return all distinct solutions, where each solution is a board configuration with 'Q' and '.' characters.

Example 1: n = 4 → [[".Q..","...Q","Q...","..Q."],["..Q.","Q...","...Q",".Q.."]]

Example 2: n = 1 → [["Q"]]

Example 3: n = 8 → 92 solutions

Quick Reminder: Backtrack row by row. Track three sets: columns used, diagonals (row - col), and anti-diagonals (row + col). If a column and both diagonals are free, place the queen and recurse to the next row.
Key Insight: Since each row must have exactly one queen, iterate column choices per row. The key constraint check uses the property that cells on the same diagonal share the same (row - col) value, and cells on the same anti-diagonal share the same (row + col) value.
Try to recall: Write the backtracking solution. Focus on the three sets for conflict detection and how you build the board string from column positions.

#72 · Edit Distance

Hard DP String 2 days since last review · last 2026-06-05

The Problem: Given two strings word1 and word2, return the minimum number of operations (insert, delete, replace a character) required to convert word1 into word2.

Example 1: word1 = "horse", word2 = "ros" → 3 (horse → rorse → rose → ros)

Example 2: word1 = "intention", word2 = "execution" → 5

Example 3: word1 = "", word2 = "abc" → 3 (three inserts)

Quick Reminder: Classic 2D DP. dp[i][j] = min ops to convert word1[:i] to word2[:j]. If characters match, dp[i][j] = dp[i-1][j-1]. Otherwise, 1 + min(dp[i-1][j], dp[i][j-1], dp[i-1][j-1]) for delete, insert, replace respectively.
Key Insight: The three operations map directly to DP transitions: replace = diagonal dp[i-1][j-1], delete = row above dp[i-1][j], insert = column left dp[i][j-1]. Base cases: dp[i][0] = i (delete all), dp[0][j] = j (insert all).
Try to recall: Write the DP table solution from scratch. Then try the space-optimized version using only two rows (or even one row with a saved diagonal variable).
Language challenge: If you solved this in Python, try Go. Focus on how Go handles 2D slices vs Python list comprehensions.
O(mn) time, O(mn) space [O(min(m,n)) with rolling array]

#696 · Count Binary Substrings

Easy String 9 days since last review · rep 2 · last 2026-05-29

The Problem: Given a binary string s, return the number of non-empty substrings that have the same number of consecutive 0s and 1s, and all the 0s and 1s are grouped consecutively (e.g., "0011" counts but "0101" does not).

Example 1: s = "00110011" → 6 (substrings: "0011", "01", "1100", "10", "0011", "01")

Example 2: s = "10101" → 4

Example 3: s = "000111" → 3 ("01", "0011", "000111")

Quick Reminder: Group consecutive identical characters and count their lengths. For adjacent groups, the number of valid substrings is the minimum of the two group lengths. Track prev and cur group lengths.
Key Insight: At each group boundary, the number of valid substrings equals min(prev_group_len, cur_group_len). You only need two variables: previous group length and current group length. One pass, O(1) space.

#104 · Maximum Depth of Binary Tree

Easy Tree DFS BFS 8 days since last review · rep 2 · last 2026-05-30

The Problem: Given the root of a binary tree, return its maximum depth. The maximum depth is the number of nodes along the longest path from the root node down to the farthest leaf node.

Example 1: root = [3,9,20,null,null,15,7] → 3

Example 2: root = [1,null,2] → 2

Example 3: root = [] → 0

Quick Reminder: Recursive DFS: if root is null return 0, else return 1 + max(depth(left), depth(right)). One line captures the essence.
Key Insight: The depth of a tree is 1 (for the root) plus the maximum depth of its two subtrees. Base case: null node has depth 0. This is the fundamental tree recursion pattern that extends to nearly every tree problem.

#19 · Remove Nth Node From End of List

Example 1: head = [1,2,3,4,5], n = 2 → [1,2,3,5]

Example 2: head = [1], n = 1 → []

Example 3: head = [1,2], n = 1 → [1]

#787 · Cheapest Flights Within K Stops

Medium Graph Shortest Path Bellman-Ford DP

The Problem: There are n cities connected by some flights. You are given the array flights where flights[i] = [from, to, price]. Given src, dst, and k, find the cheapest price from src to dst with at most k stops. If there is no such route, return -1.

Example 1: n=4, flights=[[0,1,100],[1,2,100],[2,0,100],[1,3,600],[2,3,200]], src=0, dst=3, k=1 → 700 (0→1→3)

Example 2: n=3, flights=[[0,1,100],[1,2,100],[0,2,500]], src=0, dst=2, k=1 → 200 (0→1→2)

Example 3: n=3, flights=[[0,1,100],[1,2,100],[0,2,500]], src=0, dst=2, k=0 → 500 (direct 0→2)

Example 4: n=3, flights=[[0,1,100],[1,2,100]], src=0, dst=2, k=0 → -1 (no direct flight)

Brute Force — DFS Exploring All Paths (Exponential)

Build an adjacency list and DFS from src. Track the number of stops and accumulated cost. Explore every possible path of at most k+1 edges. Prune paths that exceed the current best cost. Time: O(n^k) in the worst case.

Python
class Solution:
    def findCheapestPrice(self, n: int, flights: list[list[int]], src: int, dst: int, k: int) -> int:
        graph = [[] for _ in range(n)]
        for u, v, w in flights:
            graph[u].append((v, w))

        self.ans = float('inf')

        def dfs(node, stops, cost):
            if node == dst:
                self.ans = min(self.ans, cost)
                return
            if stops > k:
                return
            for nei, price in graph[node]:
                if cost + price < self.ans:
                    dfs(nei, stops + 1, cost + price)

        dfs(src, 0, 0)
        return self.ans if self.ans != float('inf') else -1

Key Insight: Modified Bellman-Ford — run exactly k+1 relaxation rounds. Unlike standard Bellman-Ford, you must use a copy of the prices array each round to avoid using updates from the current round. Using current-round updates creates shortcuts through more stops than allowed.

How to Think About It

  1. Initialize a prices array of size n with infinity, except prices[src] = 0.
  2. For each of k+1 rounds, create a copy temp = list(prices). For each flight (u, v, w), if prices[u] + w < temp[v], update temp[v]. Note: we read from prices (last round) and write to temp (this round).
  3. After processing all edges in this round, set prices = temp.
  4. After k+1 rounds, prices[dst] is the answer. If still infinity, return -1.
Why This Works: Each round of Bellman-Ford allows one more edge in the shortest path. After k+1 rounds, we have considered paths with at most k+1 edges, which means at most k intermediate stops. Copying the array each round prevents “cascading updates” that would effectively allow more edges than the current round permits.
Watch Out For:
  • Not copying the distance array: If you update in-place, a single round can chain multiple edges, violating the stop limit. Always read from the previous round and write to a new copy.
  • Checking prices[u] != infinity: Before relaxing, ensure the source node is reachable. Otherwise you may compare infinity + w which can overflow in some languages.
  • k stops vs k+1 edges: k stops means k+1 edges. Run the loop k+1 times.

Language-Specific Notes

  • Python: float('inf') as sentinel. list(prices) for shallow copy. No overflow concern with infinity arithmetic.
  • Go: Use math.MaxInt32 (not MaxInt64) to avoid overflow when adding weights. copy(dst, src) for slice copy.
  • TypeScript: Infinity sentinel. [...prices] spread for array copy. No integer overflow risk.
  • Rust: i32::MAX sentinel — check before adding to avoid overflow. .clone() on Vec for copy.
  • C++: INT_MAX sentinel — must check before adding to avoid signed overflow (undefined behavior). vector<int> temp(prices) for copy construction.

Alternative approach: BFS with (cost, node, stops) and pruning, or modified Dijkstra with (cost, node, stops) state.

#1514 · Path with Maximum Probability

Medium Graph Shortest Path Dijkstra Heap

The Problem: You are given an undirected weighted graph of n nodes, an array edges where edges[i] = [a, b], and corresponding succProb[i] representing the probability of success of traversing that edge. Given two nodes start and end, find the path with the maximum probability of success. If there is no path, return 0.

Example 1: n=3, edges=[[0,1],[1,2],[0,2]], succProb=[0.5,0.5,0.2], start=0, end=2 → 0.25 (0→1→2: 0.5*0.5)

Example 2: n=3, edges=[[0,1],[1,2],[0,2]], succProb=[0.5,0.5,0.3], start=0, end=2 → 0.3 (direct 0→2)

Example 3: n=3, edges=[[0,1]], succProb=[0.5], start=0, end=2 → 0.0 (no path)

Brute Force — BFS/DFS Exploring All Paths (Exponential)

Explore every possible path from start to end using DFS. Multiply probabilities along the way and track the maximum. With cycles, you need visited tracking with backtracking. Time: exponential in the worst case.

Python
from collections import defaultdict

class Solution:
    def maxProbability(self, n: int, edges: list[list[int]], succProb: list[float], start_node: int, end_node: int) -> float:
        graph = defaultdict(list)
        for (u, v), p in zip(edges, succProb):
            graph[u].append((v, p))
            graph[v].append((u, p))

        self.best = 0.0
        visited = set()

        def dfs(node, prob):
            if node == end_node:
                self.best = max(self.best, prob)
                return
            visited.add(node)
            for nei, p in graph[node]:
                if nei not in visited and prob * p > self.best:
                    dfs(nei, prob * p)
            visited.remove(node)

        dfs(start_node, 1.0)
        return self.best

func maxProbability(n int, edges [][]int, succProb []float64, start_node int, end_node int) float64 {
    graph := make([][]struct{ to int; prob float64 }, n)
    for i, e := range edges {
        graph[e[0]] = append(graph[e[0]], struct{ to int; prob float64 }{e[1], succProb[i]})
        graph[e[1]] = append(graph[e[1]], struct{ to int; prob float64 }{e[0], succProb[i]})
    }
    best := 0.0
    visited := make([]bool, n)

    var dfs func(int, float64)
    dfs = func(node int, prob float64) {
        if node == end_node {
            if prob > best { best = prob }
            return
        }
        visited[node] = true
        for _, edge := range graph[node] {
            if !visited[edge.to] && prob*edge.prob > best {
                dfs(edge.to, prob*edge.prob)
            }
        }
        visited[node] = false
    }

    dfs(start_node, 1.0)
    return best
}

Key Insight: Modified Dijkstra with a max-heap. Instead of minimizing distance, maximize probability. Push (-prob, node) into a min-heap (negating to simulate max-heap in Python). When you pop a node, if it is the destination, return the probability. If you have already visited it with a better probability, skip.

How to Think About It

  1. Build an undirected adjacency list: for each edge (u, v) with probability p, add both directions.
  2. Initialize a max_prob array of size n with 0.0, except max_prob[start] = 1.0.
  3. Push (-1.0, start) into a min-heap (we negate because Python’s heapq is a min-heap and we want max probability).
  4. Pop the highest probability node. If it is end, return the probability. For each neighbor, compute new_prob = cur_prob * edge_prob. If new_prob > max_prob[neighbor], update and push.
  5. If the heap is exhausted without reaching end, return 0.0.
Why This Works: Dijkstra works because probabilities are in [0, 1], so multiplying probabilities along a path produces non-increasing values — the same monotonicity property that makes Dijkstra correct for non-negative weights. The max-heap always expands the most promising path first. Once a node is popped, we have found the maximum probability path to it.
Watch Out For:
  • Min-heap vs max-heap: Python’s heapq is a min-heap. Negate probabilities to get max-heap behavior. In C++, priority_queue is max-heap by default, which works directly.
  • Disconnected graph: If start and end are in different components, the heap will empty without reaching end. Return 0.0 in this case.
  • Float comparison: Use strict > comparison for probability improvement, not >=, to avoid infinite loops with equal probabilities.

Key Insight: Use a min-heap of size k. Always extract the node with the smallest value, append it to the result, and push that node’s next into the heap. This maintains a running merge of k streams in O(N log k) time. In Python, push (val, index, node) tuples because ListNode is not comparable — the index breaks ties.

How to Think About It

  1. Create a min-heap. For each non-null list head, push (head.val, i, head) where i is the list index (tiebreaker).
  2. Create a dummy node and a curr pointer for building the result list.
  3. While the heap is not empty: pop the smallest (val, idx, node). Set curr.next = node, advance curr = curr.next.
  4. If node.next is not null, push (node.next.val, idx, node.next) into the heap.
  5. Return dummy.next.
Why This Works: The heap always contains at most k elements (one from each list). Each of the N total nodes is pushed and popped exactly once. Push and pop are O(log k), so total time is O(N log k). This is better than O(N log N) when k << N.
Watch Out For:
  • ListNode not comparable: In Python, if two nodes have the same value, heapq will try to compare ListNode objects and crash. Include an index as a tiebreaker: (val, index, node).
  • Null lists in input: The input array may contain null entries. Filter them out before inserting into the heap.
  • Forgetting to push node.next: After extracting a node, you must push its successor. Without this, you lose the rest of that list.

function mergeKLists(lists: Array<ListNode | null>): ListNode | null {
    // Simple min-heap using array with sift operations
    const heap: ListNode[] = [];
    const push = (node: ListNode) => {
        heap.push(node);
        let i = heap.length - 1;
        while (i > 0) {
            const parent = (i - 1) >> 1;
            if (heap[parent].val <= heap[i].val) break;
            [heap[parent], heap[i]] = [heap[i], heap[parent]];
            i = parent;
        }
    };
    const pop = (): ListNode => {
        const top = heap[0];
        const last = heap.pop()!;
        if (heap.length > 0) {
            heap[0] = last;
            let i = 0;
            while (true) {
                let smallest = i;
                const l = 2 * i + 1, r = 2 * i + 2;
                if (l < heap.length && heap[l].val < heap[smallest].val) smallest = l;
                if (r < heap.length && heap[r].val < heap[smallest].val) smallest = r;
                if (smallest === i) break;
                [heap[i], heap[smallest]] = [heap[smallest], heap[i]];
                i = smallest;
            }
        }
        return top;
    };

    for (const head of lists) {
        if (head) push(head);
    }

    const dummy = new ListNode(0);
    let curr = dummy;
    while (heap.length > 0) {
        const node = pop();
        curr.next = node;
        curr = curr.next;
        if (node.next) push(node.next);
    }
    return dummy.next;

Key Insight (O(1) space): Interleave copy nodes into the original list: insert each copy right after its original (A→A'→B→B'→...). Then set random pointers: copy.random = original.random.next (because the copy of any node is right after it). Finally, separate the interleaved list back into two independent lists.

How to Think About It

  1. Interleave: For each node, create a copy and insert it after the original. A→B→C becomes A→A'→B→B'→C→C'.
  2. Set random pointers: For each original node, original.next.random = original.random.next (if random is not null). The copy of original.random is always original.random.next.
  3. Separate lists: Restore the original list and extract the copy list by adjusting next pointers. original.next = original.next.next and copy.next = copy.next.next.
Why This Works: By interleaving, every copy node is exactly one step after its original. This gives us O(1) access from any original node to its copy (just follow .next), eliminating the need for a hash map. The random pointer of the copy can be set by following: original→random→next (which is the copy of the random target).
Watch Out For:
  • Null random pointers: Before setting copy.random = original.random.next, check that original.random is not null.
  • Not restoring the original list: After extracting the copy list, make sure the original list is restored to its initial state. Some judges check this.
  • Off-by-one in separation: When separating, handle the last node carefully — copy.next may be null if it is the last node.
Optimal Solution — Interleave (O(1) Space) — All Languages

#124 · Binary Tree Maximum Path Sum

Hard Tree DFS DP on Trees

The Problem: A path in a binary tree is a sequence of nodes where each pair of adjacent nodes has an edge. A path does not need to pass through the root. The path sum is the sum of all node values in the path. Given the root of a binary tree, return the maximum path sum of any non-empty path.

Example 1: root = [1,2,3] → 6 (path 2→1→3)

Example 2: root = [-10,9,20,null,null,15,7] → 42 (path 15→20→7)

Example 3: root = [-3] → -3 (must include at least one node)

Example 4: root = [2,-1] → 2 (take root only, ignore negative child)

Brute Force — Check All Pairs of Nodes (O(n^2))

For every pair of nodes, find the path between them and compute the sum. This requires finding the LCA of every pair which is O(n) per pair, leading to O(n^3) or O(n^2) with optimization. Instead, the “brute” approach is really the DFS solution with clear global tracking.

Python
class Solution:
    def maxPathSum(self, root: Optional[TreeNode]) -> int:
        # "Brute": try every node as the "bend point" of a path
        self.max_sum = float('-inf')

        def max_gain(node):
            if not node:
                return 0
            # Recursively get max contribution from left and right
            left = max(max_gain(node.left), 0)
            right = max(max_gain(node.right), 0)
            # Path through this node as highest point
            self.max_sum = max(self.max_sum, node.val + left + right)
            # Return max single-arm contribution
            return node.val + max(left, right)

        max_gain(root)
        return self.max_sum

Key Insight: At each node, compute the max “arm” (single path going down through one child). The best path through this node is node.val + max(0, left_arm) + max(0, right_arm). But you can only return ONE arm upward (a path cannot fork and continue up). Track the global maximum separately. Use max(0, child_sum) to ignore negative contributions.

How to Think About It

  1. Define a recursive function maxGain(node) that returns the maximum sum of a path starting at node going downward through at most one child.
  2. Base case: if node is null, return 0.
  3. Recursively compute left = max(0, maxGain(node.left)) and right = max(0, maxGain(node.right)). The max(0, ...) ignores negative subtrees.
  4. Update global max: global_max = max(global_max, node.val + left + right). This considers the path that “forks” through this node.
  5. Return node.val + max(left, right) upward. This is the best single-arm path from this node — you cannot return both arms because a path through the parent cannot fork at this node.
Why This Works: Every path has exactly one “highest” node (the node closest to the root on the path). At that node, the path goes down-left and down-right (either arm can be empty). By considering every node as the potential highest node and taking the best left + right arms, we cover all possible paths in the tree. The recursion visits every node exactly once — O(n).
Watch Out For:
  • Returning a forked path upward: You must return only one arm (not left + right) to the parent. A path is a chain, not a tree — it cannot fork and continue.
  • All-negative trees: The answer must include at least one node. Initialize max_sum to negative infinity (not 0), and always include the current node’s value.
  • Confusing path vs subtree: A path is a sequence of connected nodes. It does not include all descendants of a node.

Language-Specific Notes

  • Python: nonlocal max_sum to update a variable in the enclosing scope, or use self.max_sum as shown. float('-inf') for initialization.
  • Go: Closure captures &maxSum by reference. math.MinInt for initialization.
  • TypeScript: Outer let maxSum = -Infinity captured by the inner function. Clean closure semantics.
  • Rust: Pass &mut max_sum as a parameter since closures with mutable captures are complex with recursive functions. i32::MIN for initialization.
  • C++: Member variable maxSum or reference parameter int& maxSum. INT_MIN for initialization.
Brute: O(n^2) time (all pairs) → Optimal: O(n) time, O(h) space

Connection: This is DP on trees — the same idea as #543 (Diameter of Binary Tree), which finds the longest path. Here we find the maximum-sum path. The pattern “compute something at each node using children’s results, while tracking a global answer” is fundamental to tree DP problems.

#199 · Binary Tree Right Side View

Medium Tree BFS DFS

The Problem: Given the root of a binary tree, imagine yourself standing on the right side of it, return the values of the nodes you can see ordered from top to bottom.

Example 1: root = [1,2,3,null,5,null,4] → [1,3,4]

Example 2: root = [1,null,3] → [1,3]

Example 3: root = [] → []

Example 4: root = [1,2,3,4] → [1,3,4] (node 4 is the left child of node 2, visible because right subtree has no depth-3 node)

Standard Approach — BFS Level-Order (O(n) time)

Use BFS level-order traversal. For each level, the last element is the rightmost visible node. This is both the standard and optimal approach. Time: O(n). Space: O(w) where w = max width.

Python
from collections import deque

class Solution:
    def rightSideView(self, root: Optional[TreeNode]) -> list[int]:
        if not root:
            return []
        result = []
        queue = deque([root])
        while queue:
            level_size = len(queue)
            for i in range(level_size):
                node = queue.popleft()
                if i == level_size - 1:
                    result.append(node.val)
                if node.left:
                    queue.append(node.left)
                if node.right:
                    queue.append(node.right)
        return result

#103 · Binary Tree Zigzag Level Order Traversal

Medium Tree BFS

The Problem: Given the root of a binary tree, return the zigzag level order traversal of its nodes' values (i.e., from left to right for the first level, then right to left for the next level, and so on alternating).

Example 1: root = [3,9,20,null,null,15,7] → [[3],[20,9],[15,7]]

Example 2: root = [1] → [[1]]

Example 3: root = [] → []

Example 4: root = [1,2,3,4,5,6,7] → [[1],[3,2],[4,5,6,7]]

Standard Approach — BFS with Level Reversal (O(n))

Standard BFS level-order traversal. Collect each level’s values, then reverse every other level. The traversal order is always left-to-right; only the output order alternates. Time: O(n). Space: O(w).

Python
from collections import deque

class Solution:
    def zigzagLevelOrder(self, root: Optional[TreeNode]) -> list[list[int]]:
        if not root:
            return []
        result = []
        queue = deque([root])
        left_to_right = True
        while queue:
            level = []
            for _ in range(len(queue)):
                node = queue.popleft()
                level.append(node.val)
                if node.left:
                    queue.append(node.left)
                if node.right:
                    queue.append(node.right)
            if not left_to_right:
                level.reverse()
            result.append(level)
            left_to_right = not left_to_right
        return result

                level[i], level[j] = level[j], level[i]
            }

        leftToRight = !leftToRight
    }
    return result
}

How to Think About It

  1. Standard BFS: initialize queue with root, process level by level.
  2. Maintain a boolean left_to_right that toggles each level.
  3. After collecting all values for a level, reverse the list if left_to_right is false.
  4. Alternatively, in C++, pre-allocate the level array and place values at index = leftToRight ? i : size-1-i to avoid a separate reverse pass.

  • Off-by-one on direction: Level 0 (root) is left-to-right. Level 1 is right-to-left. Make sure your toggle starts correctly.
  • Reversing traversal vs reversing output: Do NOT change the order you add children to the queue. Always add left then right. Only reverse the collected values for display.
  • Empty tree: Return [], not [[]].

Why brute force is insufficient: O(n) linear scan doesn’t leverage the fact that the array is sorted. The problem requires O(log n).

Key Insight: Run binary search TWICE — once for the leftmost occurrence (when found, keep searching left), once for the rightmost (when found, keep searching right). Same binary search template, different tie-breaking behavior.

How to Think About It

  1. Write a helper findBound(nums, target, findFirst) that does binary search with a boolean flag controlling whether we seek the left or right boundary.
  2. For leftmost: when nums[mid] == target, set result = mid and search left (hi = mid - 1).
  3. For rightmost: when nums[mid] == target, set result = mid and search right (lo = mid + 1).
  4. Return [leftmost, rightmost] or [-1, -1] if not found.
Why This Works: Each binary search halves the search space every iteration. By continuing to search even after finding the target, we find the exact boundary. The leftmost search narrows to the first occurrence because we always try to go further left when we find a match. The rightmost search narrows to the last occurrence by always trying to go further right.
Watch Out For:
  • Returning immediately on finding target: This gives any occurrence, not specifically the first or last. You must continue searching to find the boundary.
  • Using bisect_left for right bound: You need bisect_right - 1 for the rightmost occurrence, not bisect_left.
  • Empty array: Handle the edge case where nums is empty before binary searching.

Key Insight: Binary search works because if nums[mid] < nums[mid+1], a peak MUST exist to the right (values go up, but must eventually come down since nums[n] = -∞). Similarly if nums[mid] < nums[mid-1], a peak must exist to the left. This guarantees we always move toward a peak.

How to Think About It

  1. Set lo = 0, hi = len(nums) - 1.
  2. Compute mid = lo + (hi - lo) / 2.
  3. Compare nums[mid] with nums[mid + 1].
  4. If nums[mid] < nums[mid + 1], the peak is to the right: set lo = mid + 1.
  5. Otherwise, the peak is at mid or to the left: set hi = mid.
  6. When lo == hi, that index is a peak.
Why This Works: The boundary conditions nums[-1] = nums[n] = -∞ guarantee that a peak exists. At each step, we move toward the higher neighbor. Since we always go “uphill” and the array drops to -∞ at the edges, we must eventually reach a peak. This reduces the search space by half each iteration, giving O(log n).
Watch Out For:
  • Using hi = mid - 1 instead of hi = mid: This can skip the peak element itself. Since mid could be the peak, we must keep it in the search range.
  • Integer overflow when comparing adjacent elements: In languages like C++ with int, comparing adjacent elements is safe, but avoid subtracting them if values can be extreme.
  • Accessing nums[mid + 1] out of bounds: With hi = len - 1 and the loop condition lo < hi, mid is always strictly less than hi, so mid + 1 ≤ hi is in bounds.

#42 · Trapping Rain Water

Hard Two Pointers Greedy Stack Array

The Problem: Given n non-negative integers representing an elevation map where the width of each bar is 1, compute how much water it can trap after raining.

Example 1: height = [0,1,0,2,1,0,1,3,2,1,2,1] → 6

Example 2: height = [4,2,0,3,2,5] → 9

Brute Force Approach

For each position, find the maximum height to its left and right. Water trapped at that position = min(maxLeft, maxRight) - height[i]. O(n²) time, O(1) space.

    n, water = len(height), 0
    for i in range(n):
        left_max = max(height[:i+1])
        right_max = max(height[i:])
        water += min(left_max, right_max) - height[i]
    return water

Key Insight (Two Pointers): Use two pointers from each end. Maintain left_max and right_max. If left_max ≤ right_max, water at the left pointer is determined by left_max (the right side is guaranteed to be at least right_max), so add left_max - height[left] and advance left. Vice versa for the right pointer. This eliminates the need to scan for max in both directions.

How to Think About It

  1. Initialize left = 0, right = n - 1, left_max = 0, right_max = 0, water = 0.
  2. While left < right: compare height[left] and height[right].
  3. If height[left] ≤ height[right]: update left_max = max(left_max, height[left]). Add left_max - height[left] to water. Increment left.
  4. Else: update right_max = max(right_max, height[right]). Add right_max - height[right] to water. Decrement right.
  5. The key invariant: when processing the left pointer, we know the right side has a bar at least as tall as right_max, so water level is bounded by left_max.
Why This Works: Water at any position is limited by the shorter of the two tallest walls on either side. By advancing the pointer with the shorter wall, we guarantee that the water level at that position is determined entirely by the maintained max on that side — because the other side is provably at least as tall. This is a greedy choice: the shorter side is the bottleneck.
Watch Out For:
  • Empty or tiny arrays: If height has fewer than 3 elements, no water can be trapped. Handle gracefully.
  • Confusion about which pointer to advance: Always advance the pointer on the side with the smaller height. The taller side stays put because it might be the bottleneck for future positions.
  • Off-by-one with left_max/right_max update timing: Update the max BEFORE computing trapped water at that position, not after.

#763 · Partition Labels

Medium Greedy Hash Map String

The Problem: Given a string s, partition it into as many parts as possible so that each letter appears in at most one part. Return a list of the sizes of these parts.

Example 1: s = "ababcbacadefegdehijhklij" → [9, 7, 8]

Example 2: s = "eccbbbbdec" → [10]

Key Insight: For each character, find its LAST occurrence in the string. Then greedily extend the current partition to cover the last occurrence of every character seen so far. When the current index reaches the partition’s end boundary, cut — the partition is complete because no character in it appears beyond this point.

How to Think About It

  1. Build a map of each character’s last index: last[c] = i for each character c at position i.
  2. Initialize start = 0 and end = 0 to track the current partition boundaries.
  3. Iterate through the string. For each index i, update end = max(end, last[s[i]]).
  4. When i == end, the current partition is complete: append end - start + 1 to the result, and set start = end + 1.
  5. The greedy choice: we always extend the partition to the minimum required length, ensuring each character is fully contained.
Why This Works: By tracking the last occurrence of each character, we know the minimum extent a partition must cover. The greedy extension guarantees that once we reach the partition boundary, every character within it has no further occurrences. This produces the maximum number of partitions because we cut as early as possible.
Watch Out For:
  • Not updating end for every character: Every character in the current partition might extend its boundary. You must check last[s[i]] for ALL characters, not just new ones.
  • Forgetting the last partition: The loop naturally handles this because the last character’s index always equals end at that point.
  • Off-by-one in size calculation: Size is end - start + 1, not end - start.

Python Optimal Solution

        return result

Key Insight: Union-Find — initialize n components (each node is its own root). For each edge, union the two nodes. The answer is the number of remaining distinct components. With path compression + union by rank, each operation is nearly O(1).

How to Think About It (Union-Find)

  1. Initialize parent[i] = i and rank[i] = 0 for all nodes. Set count = n.
  2. find(x): follow parent pointers to the root, applying path compression — parent[x] = find(parent[x]).
  3. union(x, y): find roots of x and y. If different, attach the shorter tree under the taller one (union by rank) and decrement count.
  4. For each edge [u, v], call union(u, v).
  5. Return count — the number of distinct components remaining.
Why This Works: Each union operation merges two components into one. Starting with n components and processing all edges, the remaining count equals the number of connected components. Path compression flattens the tree, making subsequent find operations nearly constant time. Union by rank keeps trees balanced.
Watch Out For:
  • Forgetting path compression: Without it, find degrades to O(n) per call, making the overall solution O(n·E) instead of nearly O(E).
  • Not handling self-loops: If an edge connects a node to itself, find(u) == find(v), so no union happens. This is correct behavior.
  • Off-by-one in node numbering: Nodes are 0-indexed. Ensure your parent array has size n.

#684 · Redundant Connection

Medium Union-Find Graph Tree

The Problem: Given a graph that started as a tree with n nodes (labeled 1 to n), with one additional edge added, find the edge that can be removed to make the graph a tree again. If there are multiple answers, return the one that occurs last in the input.

Example 1: edges = [[1,2],[1,3],[2,3]] → [2,3] (removing [2,3] restores the tree)

Example 2: edges = [[1,2],[2,3],[3,4],[1,4],[1,5]] → [1,4]

Brute Force Approach

Try removing each edge (in reverse order), check if the remaining graph is a tree (connected with no cycles). Return the first edge whose removal produces a valid tree. O(E · (V+E)) time.

Python Brute Force

def findRedundantConnection(edges):
    n = len(edges)
    for skip in range(n - 1, -1, -1):
        adj = [[] for _ in range(n + 1)]
        for i, (u, v) in enumerate(edges):
            if i == skip:
                continue
            adj[u].append(v)
            adj[v].append(u)
        # Check if connected with BFS
        visited = set([1])
        queue = [1]
        while queue:
            node = queue.pop()
            for nei in adj[node]:
                if nei not in visited:
                    visited.add(nei)
                    queue.append(nei)
        if len(visited) == n:
            return edges[skip]
Key Insight: Process edges in order using Union-Find. The first edge that connects two already-connected nodes creates a cycle — that is the redundant edge. Since we process in order, the last such edge in the input is naturally found (the problem guarantees exactly one redundant edge).

How to Think About It

  1. Initialize Union-Find with n nodes (1-indexed).
  2. For each edge [u, v] in the input order:
  3. Call find(u) and find(v). If they have the same root, this edge creates a cycle — return [u, v].
  4. Otherwise, call union(u, v) to merge their components.
  5. The problem guarantees exactly one redundant edge, so we will always find it.
Why This Works: A tree with n nodes has exactly n-1 edges. Adding one more edge creates exactly one cycle. When we process edges sequentially with Union-Find, the edge that closes the cycle is the one where both endpoints are already in the same component. This is exactly the redundant edge.
Watch Out For:
  • 1-indexed nodes: The problem uses 1-indexed nodes. Size your parent array accordingly (n+1) or adjust indices.
  • Multiple valid answers: The problem asks for the edge appearing LAST in the input. Since Union-Find processes in order, the first cycle-creating edge we find IS the last one that could be removed (the one we encounter).
  • Confusing with directed graphs: This problem is undirected. The directed version (#685 Redundant Connection II) is significantly harder.

#212 · Word Search II

Hard Trie Backtracking DFS Matrix

The Problem: Given an m x n board of characters and a list of words, return all words that can be formed by sequentially adjacent cells (horizontally or vertically). The same cell may not be used more than once per word.

Example 1: board = [["o","a","a","n"],["e","t","a","e"],["i","h","k","r"],["i","f","l","v"]], words = ["oath","pea","eat","rain"] → ["eat","oath"]

Example 2: board = [["a","b"],["c","d"]], words = ["abcb"] → []

Brute Force Approach

For each word, run #79 Word Search (DFS backtracking from every cell). O(words · m · n · 4^L) where L = max word length. This repeats work for words sharing prefixes.

Python Brute Force

def findWords(board, words):
    rows, cols = len(board), len(board[0])
    result = []

    def dfs(r, c, word, idx, visited):
        if idx == len(word):
            return True
        if r < 0 or r >= rows or c < 0 or c >= cols:
            return False
        if (r, c) in visited or board[r][c] != word[idx]:
            return False
        visited.add((r, c))
        for dr, dc in [(0,1),(1,0),(-1,0),(0,-1)]:
            if dfs(r+dr, c+dc, word, idx+1, visited):
                visited.discard((r, c))
                return True
        visited.discard((r, c))
        return False

    for word in words:
        found = False
        for r in range(rows):
            for c in range(cols):
                if dfs(r, c, word, 0, set()):
                    found = True
                    break
            if found:
                break
        if found:
            result.append(word)
    return result
Key Insight: Build a Trie from the word list. Then DFS from each cell, walking the Trie simultaneously. When you reach a Trie node that marks a word end, add it to results. This shares prefix computation across ALL words — if “oath” and “oat” are both in the list, the “oat” traversal is done only once for both.

How to Think About It

  1. Build a Trie from the word list. Each leaf node stores the complete word.
  2. For each cell (r, c) on the board, if the character exists in the Trie root’s children, start a DFS.
  3. During DFS: mark the cell as visited (e.g., replace with ‘#’), move to the corresponding Trie child node.
  4. If the current Trie node has a word, add it to results and set it to None (prune to avoid duplicates).
  5. Recurse in all 4 directions. After recursion, restore the cell (backtrack).
  6. Optimization: After finding a word, prune empty Trie branches to speed up future searches.
Why This Works: The Trie acts as a shared prefix tree. Instead of searching for each word independently, a single DFS traversal from a cell simultaneously checks all words that could start with the characters along the path. The Trie’s structure ensures we abandon dead-end paths immediately (if no Trie child exists for the next character, we prune that DFS branch).
Watch Out For:
  • Duplicate results: The same word might be reachable from multiple paths. Set the word marker to None after finding it, or use a set for results.
  • Not restoring board cells: Forgetting to backtrack (restore the original character) breaks future searches from other starting positions.
  • TLE without Trie pruning: After finding all words under a branch, remove empty branches from the Trie. Without this optimization, LeetCode will TLE on large inputs.
  • Memory: dict-based Trie in Python is convenient but uses more memory than array-based. For competitive constraints, it is still sufficient.

    const rows = board.length, cols = board[0].length;
    const result: string[] = [];
    const dirs = [[0,1],[1,0],[-1,0],[0,-1]];

    function dfs(r: number, c: number, parent: Record<string, any>) {
        const ch = board[r][c];
        const node = parent[ch];
        if (!node) return;

        if (node['$']) {
            result.push(node['$']);
            delete node['$'];
        }

        board[r][c] = '#';
        for (const [dr, dc] of dirs) {
            const nr = r + dr, nc = c + dc;
            if (nr >= 0 && nr < rows && nc >= 0 && nc < cols && board[nr][nc] !== '#' && node[board[nr][nc]]) {
                dfs(nr, nc, node);
            }
        }
        board[r][c] = ch;

        if (Object.keys(node).length === 0) delete parent[ch];
    }

    for (let r = 0; r < rows; r++)
        for (let c = 0; c < cols; c++)
            if (trie[board[r][c]]) dfs(r, c, trie);

    return result;
}

impl Solution {
    pub fn find_words(mut board: Vec<Vec<char>>, words: Vec<String>) -> Vec<String> {
        let mut root = TrieNode::new();
        for w in &words {
            let mut node = &mut root;
            for &b in w.as_bytes() {
                node = node.children.entry(b).or_insert_with(TrieNode::new);
            }
            node.word = Some(w.clone());
        }

        let (rows, cols) = (board.len(), board[0].len());
        let mut result = Vec::new();

        for r in 0..rows {
            for c in 0..cols {
                Self::dfs(&mut board, r, c, &mut root, &mut result, rows, cols);
            }
        }
        result
    }

    fn dfs(board: &mut Vec<Vec<char>>, r: usize, c: usize,
           node: &mut TrieNode, result: &mut Vec<String>,
           rows: usize, cols: usize) {
        let ch = board[r][c];
        if ch == '#' { return; }
        if let Some(child) = node.children.get_mut(&(ch as u8)) {
            if let Some(word) = child.word.take() {
                result.push(word);
            }
            board[r][c] = '#';
            let dirs: [(i32,i32); 4] = [(0,1),(1,0),(-1,0),(0,-1)];
            for (dr, dc) in dirs {
                let (nr, nc) = (r as i32 + dr, c as i32 + dc);
                if nr >= 0 && nr < rows as i32 && nc >= 0 && nc < cols as i32 {
                    Self::dfs(board, nr as usize, nc as usize, child, result, rows, cols);
                }
            }
            board[r][c] = ch;
            if child.children.is_empty() && child.word.is_none() {
                node.children.remove(&(ch as u8));
            }
        }
    }
}

C++ Optimal Solution

class Solution {
    struct TrieNode {
        TrieNode* children[26] = {};
        string word;
    };

    void dfs(vector<vector<char>>& board, int r, int c,
             TrieNode* node, vector<string>& result) {
        char ch = board[r][c];
        if (ch == '#' || !node->children[ch - 'a']) return;
        TrieNode* child = node->children[ch - 'a'];

        if (!child->word.empty()) {
            result.push_back(child->word);
            child->word.clear();
        }

        board[r][c] = '#';
        int dirs[][2] = {{0,1},{1,0},{-1,0},{0,-1}};
        for (auto& [dr, dc] : dirs) {
            int nr = r + dr, nc = c + dc;
            if (nr >= 0 && nr < (int)board.size() &&
                nc >= 0 && nc < (int)board[0].size()) {
                dfs(board, nr, nc, child, result);
            }
        }
        board[r][c] = ch;

        // Prune
        bool empty = child->word.empty();
        for (auto* c : child->children) if (c) { empty = false; break; }
        if (empty) {
            delete child;
            node->children[ch - 'a'] = nullptr;
        }
    }

public:
    vector<string> findWords(vector<vector<char>>& board, vector<string>& words) {
        TrieNode* root = new TrieNode();
        for (auto& w : words) {
            TrieNode* node = root;
            for (char c : w) {
                if (!node->children[c - 'a'])
                    node->children[c - 'a'] = new TrieNode();
                node = node->children[c - 'a'];
            }
            node->word = w;
        }

        vector<string> result;
        for (int r = 0; r < board.size(); r++)
            for (int c = 0; c < board[0].size(); c++)
                dfs(board, r, c, root, result);
        return result;
    }
};

Language-Specific Features

  • Python: Dict-based Trie with setdefault is idiomatic. '$' as end marker. del for Trie pruning.
  • Go: Struct-based TrieNode with [26]*TrieNode children array. Set word = "" after finding to avoid duplicates.
  • TypeScript: Object-based Trie (Record<string, any>). delete operator for pruning. Object.keys().length to check if empty.
  • Rust: HashMap<u8, TrieNode> for children. .take() on Option to extract and clear the word in one step. Ownership requires careful &mut passing.
  • C++: Raw pointer-based Trie with new/delete. Fixed-size children array [26] initialized with {} (nullptr). delete and set to nullptr for pruning.
Brute: O(W · m · n · 4^L) time → Optimal: O(m · n · 4^L) time (shared prefixes), with Trie pruning much faster in practice

Connection: Combines #208 (Implement Trie) with #79 (Word Search) backtracking. The Trie replaces individual word searches with a single shared traversal. This pattern of “Trie + DFS” is powerful for any problem searching multiple patterns in a grid or string simultaneously.
