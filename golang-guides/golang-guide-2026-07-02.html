Pointers

Go Foundations • due today • rep 3

Quick Reminder

A pointer holds the address of a value. &x takes an address, *p dereferences it, and a nil pointer dereference panics. You pass a pointer when a function must mutate the caller's value or when copying a large struct is wasteful. new(T) allocates a zeroed T and returns a *T; &T{...} does the same with an initializer. Go has no pointer arithmetic (that lives in unsafe), and the compiler's escape analysis decides stack vs heap for you.

Key Pattern

package main

import "fmt"

type Counter struct{ n int }

// Pointer receiver: mutation is visible to the caller.
func (c *Counter) Inc() { c.n++ }

func zero(p *int) { *p = 0 } // writes through the pointer

func main() {
    c := Counter{}
    c.Inc() // Go auto-takes &c because Inc has a pointer receiver
    c.Inc()
    fmt.Println(c.n) // 2

    x := 42
    zero(&x)
    fmt.Println(x) // 0

    var p *int
    fmt.Println(p == nil) // true (dereferencing p here would panic)
}
Output
2 0 true
Challenge Exercise

Write swap(a, b *int) that exchanges two ints through their pointers, and a func appendID(s *[]int, id int) that appends to a slice via a pointer so the caller sees the grown slice header. Then explain the difference between a method with a value receiver called on an addressable variable vs an unaddressable map element (m["k"].Inc() won't compile for a pointer-receiver method) and why. Finally, describe when returning &T{} from a constructor is fine even though the local "escapes" the function.

#1 Pitfall: Taking the address of a loop variable (pre-Go 1.22) or of a map element. &m[k] does not compile because map entries are not addressable, and storing &v for a shared loop variable captured every iteration's final value. Copy into a fresh variable first, or index a slice (which is addressable).

bufio: Buffered I/O

I/O & Networking • overdue • rep 1

Quick Reminder

bufio wraps a Reader/Writer with an in-memory buffer so you make far fewer syscalls. bufio.Scanner is the idiomatic line/word reader — it splits input with a configurable SplitFunc (ScanLines by default) and hands you one token at a time. A bufio.Writer batches small writes; you must Flush() before the program exits or the tail of your output is lost.

Key Pattern

package main

import (
    "bufio"
    "fmt"
    "strings"
)

func main() {
    input := "alpha\nbravo\ncharlie"
    sc := bufio.NewScanner(strings.NewReader(input))
    sc.Split(bufio.ScanLines) // default; shown for clarity

    n := 0
    for sc.Scan() {
        n++
        fmt.Printf("%d: %s\n", n, sc.Text())
    }
    if err := sc.Err(); err != nil { // Scan() returns false on EOF or error
        panic(err)
    }
}
Output
1: alpha 2: bravo 3: charlie
Challenge Exercise

Read a large text stream line by line and count word frequencies, using bufio.Scanner with sc.Split(bufio.ScanWords). Then feed it a single line longer than bufio.MaxScanTokenSize (64 KiB) and observe the bufio.ErrTooLong error; fix it by calling sc.Buffer(make([]byte, 0, 1<<20), 1<<20). Finally, write the counts through a bufio.Writer and prove that omitting Flush() drops output.

#1 Pitfall: Forgetting bufio.Writer.Flush() (ideally defer w.Flush()), or ignoring scanner.Err() after the loop. A for sc.Scan() loop ends on both EOF and error — without checking Err() you silently treat a read failure as a clean end of input.

maps & slices Packages (Go 1.21+)

Modern Std Lib • overdue • rep 1

Quick Reminder

The generic slices and maps packages replace the copy-pasted helpers everyone used to hand-roll. slices.Sort, slices.Contains, slices.Index, slices.Equal, slices.Clone, and slices.BinarySearch cover the common slice chores; maps.Clone, maps.Equal, and maps.DeleteFunc do the same for maps. In Go 1.23, maps.Keys/maps.Values return iterators, which you materialize with slices.Sorted or slices.Collect.

Key Pattern

package main

import (
    "fmt"
    "maps"
    "slices"
)

func main() {
    xs := []int{3, 1, 2, 1}
    slices.Sort(xs)
    fmt.Println(xs)                       // [1 1 2 3]
    fmt.Println(slices.Contains(xs, 2))   // true
    fmt.Println(slices.Index(xs, 3))      // 3

    m := map[string]int{"b": 2, "a": 1, "c": 3}
    // maps.Keys returns an iterator (Go 1.23); sort it deterministically.
    keys := slices.Sorted(maps.Keys(m))
    fmt.Println(keys)                     // [a b c]

    clone := maps.Clone(m)
    fmt.Println(maps.Equal(m, clone))     // true
}
Output
[1 1 2 3] true 3 [a b c] true
Challenge Exercise

Given map[string][]int, produce a slice of {key, sum} pairs sorted by descending sum with ties broken alphabetically, using slices.SortFunc with a cmp.Or-style comparator. Then use slices.CompactFunc to dedupe adjacent equal entries and maps.DeleteFunc to drop empty slices from the map. Explain why slices.Clone is a shallow copy and what that means for a [][]int.

#1 Pitfall: Assuming maps.Keys(m) still returns a ready-made slice like the old golang.org/x/exp/maps did. In the standard library it is an iter.Seq[K] — passing it where a []K is expected fails to compile. Wrap it: slices.Collect(maps.Keys(m)) or slices.Sorted(maps.Keys(m)).

Advanced slog: Custom Handlers & Contextual Logging

Modern Std Lib • overdue • rep 1

Quick Reminder

log/slog separates the frontend (Logger, which builds records) from the backend (Handler, which formats and emits them). logger.With(...) returns a child logger that stamps every line with shared attributes; WithGroup namespaces them. For request-scoped context you attach attributes via a custom handler that reads from context.Context in its Handle method — so a trace ID rides along without threading it through every call.

Key Pattern

package main

import (
    "log/slog"
    "os"
)

func main() {
    // JSON backend; swap for slog.NewTextHandler without touching call sites.
    h := slog.NewJSONHandler(os.Stdout, &slog.HandlerOptions{
        Level: slog.LevelInfo,
    })
    base := slog.New(h)

    // Child logger carries shared attributes on every subsequent line.
    reqLog := base.With("request_id", "abc123", slog.String("svc", "api"))
    reqLog.Info("handling", "path", "/users", "method", "GET")
    reqLog.Warn("slow query", "ms", 812)
}
Output
{"time":"...","level":"INFO","msg":"handling","request_id":"abc123","svc":"api","path":"/users","method":"GET"} {"time":"...","level":"WARN","msg":"slow query","request_id":"abc123","svc":"api","ms":812}
Challenge Exercise

Write a ContextHandler that wraps a slog.Handler and, in Handle, pulls a traceID out of the context.Context and adds it as an attribute before delegating. Implement all four methods (Enabled, Handle, WithAttrs, WithGroup) by forwarding to the wrapped handler. Then set it as the default with slog.SetDefault and show that slog.InfoContext(ctx, "msg") now stamps every line with the trace ID. Explain why implementing WithAttrs/WithGroup correctly matters for performance.

#1 Pitfall: Using odd numbers of loose key/value args — logger.Info("msg", "key") with no value. slog emits a !BADKEY attribute instead of failing. Prefer the typed constructors (slog.String, slog.Int, slog.Any) in hot or critical paths to make the key/value pairing compiler-visible.

Integration Testing Patterns

Production Testing • overdue • rep 1

Quick Reminder

Integration tests exercise real collaborators — an actual HTTP server, a real database, the wired-together stack — rather than mocks. Keep them separate from fast unit tests with a build tag (//go:build integration) or a testing.Short() guard so go test -short skips them. httptest.NewServer spins up a real server on a random port; TestMain sets up and tears down shared resources (a container, a temp DB) once for the whole package.

Key Pattern

//go:build integration

package api

import (
    "io"
    "net/http"
    "net/http/httptest"
    "testing"
)

func TestHealth_Integration(t *testing.T) {
    if testing.Short() {
        t.Skip("skipping integration test in -short mode")
    }
    srv := httptest.NewServer(Handler()) // the real mux
    defer srv.Close()

    resp, err := http.Get(srv.URL + "/health")
    if err != nil {
        t.Fatal(err)
    }
    defer resp.Body.Close()

    body, _ := io.ReadAll(resp.Body)
    if got := string(body); got != "ok" {
        t.Fatalf("health = %q, want ok", got)
    }
}
Challenge Exercise

Stand up an integration test for a small JSON API: use TestMain to start a real dependency once (e.g. a temp SQLite file or a testcontainers Postgres), run httptest.NewServer against your actual router, POST a record, then GET it back and assert the round-trip. Gate the whole file with //go:build integration and run it via go test -tags=integration ./.... Explain why you seed and truncate the database in t.Cleanup rather than sharing state across subtests.

#1 Pitfall: Mixing slow, external-dependency integration tests into the default go test run so every commit pays the cost and flakes on a missing service. Isolate them behind a build tag or testing.Short(), and make setup fail loudly (skip with a clear message) when the dependency is unavailable rather than passing silently.

Dependency Injection with Wire

Production Architecture • overdue • rep 1

Quick Reminder

Google's wire is compile-time dependency injection: you declare providers (constructors) and Wire generates the boilerplate that calls them in the right order. Unlike reflection-based DI containers, there is no runtime magic — the generated wire_gen.go is plain Go you can read, and a missing dependency is a build error, not a 3am panic. You write an injector stub guarded by //go:build wireinject; running wire fills in the real body.

Key Pattern

//go:build wireinject

package main

import "github.com/google/wire"

// Providers: ordinary constructors.
func NewConfig() *Config              { return &Config{} }
func NewDB(c *Config) *DB             { return &DB{cfg: c} }
func NewServer(db *DB) *Server        { return &Server{db: db} }

// Injector stub: wire replaces the body in wire_gen.go.
func InitServer() *Server {
    wire.Build(NewConfig, NewDB, NewServer)
    return nil // overwritten by generated code
}
Challenge Exercise

Model a service with four providers where Server needs a *DB and a *Cache, and *DB needs a *Config. Group the infrastructure providers into a wire.NewSet and reference the set from the injector. Introduce a provider that returns (T, error) and show how the generated injector propagates that error. Then explain why Wire's compile-time approach catches a missing binding that a reflection-based container would only surface at runtime.

#1 Pitfall: Editing wire_gen.go by hand or forgetting to re-run wire after changing a provider signature. The generated file is derived state — your edits vanish on the next generation, and a stale wire_gen.go silently wires the old graph. Regenerate in CI and fail if the working tree changes.

Section 1 — Compression & Binary Encoding

compress/gzip & compress/flate: Streaming Compression

Std Lib

Why it matters: gzip is everywhere — HTTP response bodies, log rotation, .gz data files, backups. compress/gzip gives you a Writer that implements io.Writer and a Reader that implements io.Reader, so compression composes with any stream via io.Copy — a file, a network socket, an HTTP body — without buffering the whole payload in memory. Under the hood it wraps compress/flate (raw DEFLATE) and adds a gzip header, checksum, and length trailer.

Compress then decompress a stream

A gzip.Writer is an io.WriteCloser: write your bytes, then Close() to flush the trailer. Reading back is symmetric with gzip.NewReader.

package main

import (
    "bytes"
    "compress/gzip"
    "fmt"
    "io"
)

func main() {
    payload := []byte("hello, hello, hello, gophers")

    // --- compress into an in-memory buffer ---
    var buf bytes.Buffer
    zw := gzip.NewWriter(&buf)
    zw.Name = "greeting.txt" // optional header metadata
    if _, err := zw.Write(payload); err != nil {
        panic(err)
    }
    if err := zw.Close(); err != nil { // MUST close to flush the trailer
        panic(err)
    }

    // --- decompress it again ---
    zr, err := gzip.NewReader(&buf)
    if err != nil {
        panic(err)
    }
    out, _ := io.ReadAll(zr)
    zr.Close()

    fmt.Println(zr.Name)              // greeting.txt
    fmt.Println(len(out), "bytes")    // 28 bytes
    fmt.Println(string(out))          // hello, hello, hello, gophers
}
Output
greeting.txt 28 bytes hello, hello, hello, gophers

Choosing a compression level

gzip.NewWriterLevel trades speed for ratio. The named constants make intent explicit; BestSpeed for hot paths, BestCompression for archival.

package main

import (
    "bytes"
    "compress/gzip"
    "fmt"
    "strings"
)

func main() {
    data := []byte(strings.Repeat("go ", 1000)) // very compressible

    var buf bytes.Buffer
    zw, _ := gzip.NewWriterLevel(&buf, gzip.BestCompression)
    zw.Write(data)
    zw.Close()

    fmt.Println("original:  ", len(data)) // 3000
    fmt.Println("compressed:", buf.Len() < len(data)) // true
}
Output
original: 3000 compressed: true

Common Mistakes

Wrong
// Never closed -> the gzip trailer (CRC + size)
// is never written; readers see "unexpected EOF".
zw := gzip.NewWriter(&buf)
zw.Write(payload)
zw.Flush()          // flushes data, NOT the trailer
send(buf.Bytes())   // truncated / corrupt stream
Right
zw := gzip.NewWriter(&buf)
zw.Write(payload)
if err := zw.Close(); err != nil { // writes the trailer
    return err
}
send(buf.Bytes())

Key Rules

  • gzip.Writer is an io.WriteCloser — you must Close() (not just Flush()) to emit the CRC-32 and length trailer, or the archive is corrupt.
  • Compression composes with streams: io.Copy(gzip.NewWriter(dst), src) compresses a file or socket without loading it all into memory.
  • Pick a level with NewWriterLevel (BestSpeed … BestCompression, or DefaultCompression); reuse a writer across payloads with Reset(w) to avoid re-allocating.
  • Use compress/flate for raw DEFLATE (no gzip header), compress/zlib for the zlib wrapper; gzip is the right choice for files and HTTP.
Practice Exercise

Write gzipFile(src, dst string) error that streams a file through gzip.NewWriter with io.Copy (never reading the whole file into memory) and a matching gunzipFile. Then build an HTTP middleware that wraps the ResponseWriter in a gzip.Writer when the request's Accept-Encoding contains gzip, setting Content-Encoding: gzip and closing the writer in a deferred call. Explain why you must not also set Content-Length and why a sync.Pool of gzip writers helps under load.

Pitfall: Treating Flush() as a substitute for Close(). Flush pushes buffered data to the underlying writer so a reader can see it, but only Close writes the gzip footer. A stream that is flushed but never closed decompresses with ErrUnexpectedEOF or a CRC error. Always defer zw.Close() and check its error.
Connections: Builds directly on the io.Reader/io.Writer composition from the io package day; pairs with archive/zip (which uses DEFLATE per-entry) and with net/http transparent compression, and benefits from the sync.Pool allocation-reduction pattern.

archive/zip: Reading & Writing ZIP Archives

Std Lib

Why it matters: ZIP is the universal container — .xlsx/.docx office files, .jar/.apk bundles, plugin packages, and plain "download everything as a zip" endpoints are all ZIP archives. Unlike gzip (a single compressed stream), a ZIP holds many named entries with a central directory, so you can list and extract individual files without decompressing the whole thing. archive/zip reads and writes them with a small, stream-friendly API.

Writing a multi-file archive

A zip.Writer wraps any io.Writer. Each Create (or CreateHeader) returns an io.Writer for that entry's bytes; Close writes the central directory.

package main

import (
    "archive/zip"
    "bytes"
    "fmt"
)

func main() {
    var buf bytes.Buffer
    zw := zip.NewWriter(&buf)

    files := map[string]string{
        "readme.txt":    "hello, gopher",
        "data/nums.txt": "1,2,3",
    }
    for name, body := range files {
        w, _ := zw.Create(name) // name may include a path
        w.Write([]byte(body))
    }
    if err := zw.Close(); err != nil { // finalizes the central directory
        panic(err)
    }
    fmt.Println("archive bytes:", buf.Len() > 0) // true
}
Output
archive bytes: true

Reading entries back out

zip.NewReader needs the data and its length. Iterate r.File; each entry's Open() returns an io.ReadCloser for that file's contents.

package main

import (
    "archive/zip"
    "bytes"
    "fmt"
    "io"
)

func main() {
    // Build a tiny archive in memory (as above).
    var buf bytes.Buffer
    zw := zip.NewWriter(&buf)
    w1, _ := zw.Create("readme.txt")
    w1.Write([]byte("hello, gopher"))
    w2, _ := zw.Create("data/nums.txt")
    w2.Write([]byte("1,2,3"))
    zw.Close()

    // Now read it.
    r, _ := zip.NewReader(bytes.NewReader(buf.Bytes()), int64(buf.Len()))
    for _, f := range r.File {
        fmt.Printf("%-14s %d bytes\n", f.Name, f.UncompressedSize64)
    }

    rc, _ := r.File[0].Open()
    body, _ := io.ReadAll(rc)
    rc.Close()
    fmt.Println("--", r.File[0].Name, "--")
    fmt.Println(string(body))
}
Output
readme.txt 13 bytes data/nums.txt 5 bytes -- readme.txt -- hello, gopher

Common Mistakes

Wrong
// Extracting entry names straight onto disk lets a
// malicious "../../etc/passwd" escape the target dir.
for _, f := range r.File {
    dst := filepath.Join(outDir, f.Name)
    os.MkdirAll(filepath.Dir(dst), 0o755)
    // ... write to dst   // Zip Slip vulnerability
}
Right
for _, f := range r.File {
    dst := filepath.Join(outDir, f.Name)
    if !strings.HasPrefix(dst,
        filepath.Clean(outDir)+string(os.PathSeparator)) {
        return fmt.Errorf("unsafe path: %s", f.Name)
    }
    // ... safe to write to dst
}

Key Rules

  • Writing: Create(name) returns an io.Writer per entry; call zw.Close() exactly once at the end to write the central directory (missing it yields an unreadable file).
  • Reading: zip.NewReader(readerAt, size) or zip.OpenReader(path); iterate r.File, and f.Open() each entry — remember to close every ReadCloser.
  • Use CreateHeader with a zip.FileHeader to control the per-entry compression Method (zip.Store vs zip.Deflate), modification time, and mode.
  • Always validate extracted paths against your output root (the "Zip Slip" defense) before writing entries to disk.
Practice Exercise

Write zipDir(srcDir, zipPath string) error that walks a directory with filepath.WalkDir and adds every regular file under a relative path, then an unzip(zipPath, outDir string) error that extracts it with a Zip Slip guard. Serve a zip over HTTP by streaming a zip.NewWriter straight into the http.ResponseWriter (setting Content-Type: application/zip). Explain why zip.Store is preferable for already-compressed inputs (JPEGs, other zips) and how the central directory lets a reader seek to one entry without scanning the whole file.

Pitfall: "Zip Slip." A crafted archive with entry names like ../../../etc/cron.d/x will, if you blindly filepath.Join(outDir, f.Name) and write, land files outside your target directory. Always filepath.Clean and confirm the destination stays under outDir before creating any file — and reject absolute paths.
Connections: Shares DEFLATE with compress/gzip but adds a multi-entry directory; a zip.Reader also satisfies fs.FS, so it plugs into the io/fs abstraction from yesterday; path validation reuses the filepath and os.Root hardening ideas.

encoding/hex: Hexadecimal Encoding

Std Lib

Why it matters: hex is the lingua franca for showing raw bytes as text — hashes (sha256 digests), HMAC signatures, request IDs, wire dumps, and debugging binary protocols. encoding/hex converts []byte↔hex string in both directions, offers streaming Encoder/Decoder for large data, and gives you hex.Dump — the classic hexdump -C view — for inspecting buffers.

Encode, decode, and dump

Each byte becomes two lowercase hex digits, so the string length is always 2 × len(bytes). DecodeString reverses it and errors on odd length or non-hex characters.

package main

import (
    "encoding/hex"
    "fmt"
)

func main() {
    data := []byte("Go!")

    enc := hex.EncodeToString(data)
    fmt.Println(enc) // 476f21

    back, err := hex.DecodeString(enc)
    if err != nil {
        panic(err)
    }
    fmt.Println(string(back)) // Go!

    // Human-friendly canonical dump (offset | hex | ASCII).
    fmt.Print(hex.Dump([]byte("Gopher")))
}
Output
476f21 Go! 00000000 47 6f 70 68 65 72 |Gopher|

A real use: printing a hash digest

Hash functions return raw []byte; you almost always render them as hex before logging or comparing.

package main

import (
    "crypto/sha256"
    "encoding/hex"
    "fmt"
)

func main() {
    sum := sha256.Sum256([]byte("go")) // [32]byte
    fmt.Println(hex.EncodeToString(sum[:]))
}
Output
4cd0e21a9a0795a14ec9aa5f0e7d1abff0492565770e43eafdf1e3e8afed1f33

Common Mistakes

Wrong
// %x on a [N]byte array formats the array, and
// discarding the decode error hides bad input.
sum := sha256.Sum256(data)
s := fmt.Sprintf("%v", sum) // [251 53 ...] not hex!
raw, _ := hex.DecodeString(userInput) // silent on error
Right
sum := sha256.Sum256(data)
s := hex.EncodeToString(sum[:]) // slice the array first
raw, err := hex.DecodeString(userInput)
if err != nil {
    return fmt.Errorf("bad hex: %w", err)
}

Key Rules

  • hex.EncodeToString(b) / hex.DecodeString(s) are the everyday pair; output is lowercase and exactly twice the byte length.
  • Decoding fails on an odd-length string (hex.ErrLength) or any non-[0-9a-fA-F] byte — always check the error on untrusted input.
  • For streams use hex.NewEncoder(w)/hex.NewDecoder(r) so you never hold the whole hex string in memory; hex.Dump is for human debugging, not for wire format.
  • To render a fixed-size array (like a [32]byte digest), slice it first: hex.EncodeToString(sum[:]). fmt.Sprintf("%x", b) also works for a quick inline hex string.
Practice Exercise

Write fingerprint(r io.Reader) (string, error) that streams data through a sha256.New() hash (via io.Copy) and returns the hex digest — without buffering the input. Then write a tiny hexdump CLI that reads stdin and prints hex.Dump output. Finally, compare hex against encoding/base64: encode the same 32-byte digest both ways and explain when you'd choose hex (readability, fixed 2× size, case-insensitive) versus base64 (33% smaller, URL-safe variant).

Pitfall: Formatting a byte array with %v or Sprintf without %x and getting a decimal slice like [251 53 245 ...] instead of a hex digest — then storing or comparing that. Use hex.EncodeToString(sum[:]) (or %x) and remember a [N]byte array must be sliced to []byte before most byte APIs accept it.
Connections: The text-rendering counterpart to encoding/base64 and encoding/gob; pairs with the crypto hashing packages and with hash/fnv/crc32 from earlier days, and its streaming encoder mirrors the io-composition pattern used by gzip.

Section 2 — Web Server Building Blocks

mime/multipart: File Uploads & Form Data

Std Lib

Why it matters: HTML file uploads and many API clients send multipart/form-data — several named parts (form fields and files) packed into one body separated by a boundary string. mime/multipart both writes such bodies (for a client uploading a file) and reads them (for a server handling an upload). net/http builds on it: r.FormFile and r.ParseMultipartForm are thin wrappers over this package.

Writing a multipart body

A multipart.Writer generates the boundary and formats each part. Add plain fields with WriteField and file parts with CreateFormFile; Close() writes the terminating boundary. Grab FormDataContentType() for the request's Content-Type header.

package main

import (
    "bytes"
    "fmt"
    "io"
    "mime/multipart"
)

func main() {
    var body bytes.Buffer
    w := multipart.NewWriter(&body)

    w.WriteField("user", "ada")             // a normal form field
    fw, _ := w.CreateFormFile("doc", "hello.txt") // a file part
    fw.Write([]byte("file body"))
    w.Close()                               // MUST close to finish the body

    // The matching header a client would send:
    fmt.Println("Content-Type prefix:",
        w.FormDataContentType()[:30]) // multipart/form-data; boundary=

    // Read it back with the same boundary.
    r := multipart.NewReader(&body, w.Boundary())
    for {
        part, err := r.NextPart()
        if err == io.EOF {
            break
        }
        data, _ := io.ReadAll(part)
        fmt.Printf("%s (file=%q): %s\n",
            part.FormName(), part.FileName(), data)
    }
}
Output
Content-Type prefix: multipart/form-data; boundary= user (file=""): ada doc (file="hello.txt"): file body

Server side: handling an upload

In an http.Handler, ParseMultipartForm buffers small parts in memory and spills large ones to temp files; FormFile hands you the file plus its header.

func upload(w http.ResponseWriter, r *http.Request) {
    // Up to 10 MiB kept in memory; the rest spills to disk.
    if err := r.ParseMultipartForm(10 << 20); err != nil {
        http.Error(w, err.Error(), http.StatusBadRequest)
        return
    }
    file, header, err := r.FormFile("doc")
    if err != nil {
        http.Error(w, "missing file", http.StatusBadRequest)
        return
    }
    defer file.Close()

    fmt.Fprintf(w, "got %s (%d bytes), user=%s",
        header.Filename, header.Size, r.FormValue("user"))
}

Common Mistakes

Wrong
// Boundary set by hand / writer never closed ->
// the server can't find the final boundary.
w := multipart.NewWriter(&body)
w.CreateFormFile("doc", "a.txt")
// forgot w.Close()
req.Header.Set("Content-Type", "multipart/form-data")
Right
w := multipart.NewWriter(&body)
fw, _ := w.CreateFormFile("doc", "a.txt")
fw.Write(content)
w.Close() // writes the closing boundary
req.Header.Set("Content-Type", w.FormDataContentType())

Key Rules

  • Always Close() the multipart.Writer — it emits the closing boundary; an unclosed body is malformed and rejected by the receiver.
  • Set the request header from w.FormDataContentType(), never a hand-typed string — it carries the exact generated boundary.
  • On the server, bound memory with ParseMultipartForm(maxMemory) (larger parts stream to temp files) and always close the returned files; call r.MultipartForm.RemoveAll() to clean up temp files.
  • For huge uploads, read part-by-part with r.MultipartReader() instead of ParseMultipartForm so nothing large is buffered at once.
Practice Exercise

Build a client function postFile(url, field, filename string, data []byte) (*http.Response, error) that constructs a multipart body, sets the content type from FormDataContentType(), and POSTs it. Then write the matching handler that saves the upload to a temp dir and returns its size, and test the pair with httptest.NewServer. Finally, switch the handler to a streaming r.MultipartReader() loop and explain how it caps memory for a multi-gigabyte upload compared with ParseMultipartForm.

Pitfall: Reusing the same field/boundary by hand or forgetting Close(). Without the closing boundary the server's parser returns io.ErrUnexpectedEOF or "multipart: NextPart: EOF"; with a hand-written Content-Type the declared boundary won't match the body's actual boundary and no parts are found. Let the writer own the boundary end to end.
Connections: The machinery behind net/http's FormFile/ParseMultipartForm; parts are io.Readers so they compose with gzip and hashing, and testing the round-trip reuses the httptest patterns from the integration testing review.

net/http/httputil: Reverse Proxy & Debug Dumps

Std Lib

Why it matters: you rarely need nginx to put a Go service behind a proxy — httputil.ReverseProxy is a production-grade reverse proxy in the standard library, powering API gateways, load balancers, and local dev routers in a few lines. The same package's DumpRequest/DumpResponse helpers serialize a request or response to bytes for logging and debugging. It's the "glue" layer on top of net/http.

A one-host reverse proxy

NewSingleHostReverseProxy(target) returns an http.Handler that forwards every request to target, copying headers, streaming the body, and rewriting the Host. Mount it like any handler.

package main

import (
    "fmt"
    "io"
    "net/http"
    "net/http/httptest"
    "net/http/httputil"
    "net/url"
)

func main() {
    // A backend we'll proxy to.
    backend := httptest.NewServer(http.HandlerFunc(
        func(w http.ResponseWriter, r *http.Request) {
            fmt.Fprintf(w, "backend saw %s", r.URL.Path)
        }))
    defer backend.Close()

    target, _ := url.Parse(backend.URL)
    proxy := httputil.NewSingleHostReverseProxy(target)

    // The proxy is just an http.Handler.
    front := httptest.NewServer(proxy)
    defer front.Close()

    resp, _ := http.Get(front.URL + "/api/health")
    body, _ := io.ReadAll(resp.Body)
    resp.Body.Close()
    fmt.Println(string(body)) // backend saw /api/health
}
Output
backend saw /api/health

Customizing: rewrite the request, modify the response

Use Rewrite (Go 1.20+) to reshape the outbound request and ModifyResponse to touch what comes back — add headers, strip a path prefix, inject auth.

package main

import (
    "fmt"
    "net/http"
    "net/http/httptest"
    "net/http/httputil"
    "net/url"
)

func main() {
    backend := httptest.NewServer(http.HandlerFunc(
        func(w http.ResponseWriter, r *http.Request) {
            w.Write([]byte("ok"))
        }))
    defer backend.Close()
    target, _ := url.Parse(backend.URL)

    proxy := httputil.NewSingleHostReverseProxy(target)
    proxy.ModifyResponse = func(resp *http.Response) error {
        resp.Header.Set("X-Proxied-By", "go-guide")
        return nil
    }

    front := httptest.NewServer(proxy)
    defer front.Close()

    resp, _ := http.Get(front.URL)
    resp.Body.Close()
    fmt.Println(resp.Header.Get("X-Proxied-By")) // go-guide
}
Output
go-guide

Common Mistakes

Wrong
// Sharing the default transport's problems, and
// swallowing backend failures with no ErrorHandler:
// a dead backend returns a bare 502 with no context.
proxy := httputil.NewSingleHostReverseProxy(target)
http.ListenAndServe(":8080", proxy) // no timeouts, no logging
Right
proxy := httputil.NewSingleHostReverseProxy(target)
proxy.ErrorHandler = func(w http.ResponseWriter,
    r *http.Request, err error) {
    log.Printf("proxy %s: %v", r.URL.Path, err)
    http.Error(w, "upstream unavailable",
        http.StatusBadGateway)
}
srv := &http.Server{Addr: ":8080", Handler: proxy,
    ReadHeaderTimeout: 5 * time.Second}

Key Rules

  • ReverseProxy is an http.Handler — compose it with your mux and middleware; NewSingleHostReverseProxy is the quick path, a hand-built &ReverseProxy{Rewrite: ...} gives full control.
  • Set ErrorHandler so a failing upstream produces a logged, well-formed 502 instead of a bare default; set ModifyResponse to rewrite headers/bodies on the way out.
  • Give the proxy a custom Transport with sane timeouts and connection pooling; the zero-value default has no request timeout.
  • httputil.DumpRequest/DumpRequestOut/DumpResponse render messages to []byte for logging — pass body=false to skip large payloads.
Practice Exercise

Build a path-based router-proxy: requests to /a/* go to backend A and /b/* to backend B, each via its own ReverseProxy, using two httptest backends to verify routing. Add an ErrorHandler that returns 502 with a JSON body and a ModifyResponse that stamps a X-Upstream header. Then add request logging with httputil.DumpRequest(r, false). Explain why you should give the proxy an explicit http.Transport with timeouts rather than relying on the default, and how Rewrite differs from the older Director.

Pitfall: Running a ReverseProxy with the default (zero) Transport and no ErrorHandler in production. The default transport has no overall request timeout, so a slow or dead backend ties up goroutines and file descriptors, and without an ErrorHandler clients get an opaque 502. Always configure timeouts and a logging error handler for any internet-facing proxy.
Connections: Sits on top of the net/http Handler/Transport model and composes with the middleware-chain pattern; its debug dumps complement structured logging (see slog), and testing it reuses the httptest approach from integration testing.
