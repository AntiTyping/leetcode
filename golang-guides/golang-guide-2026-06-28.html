Slices Deep Dive

Go Foundations • due today • rep 3

Quick Reminder

A slice is a three-word header — pointer, length, capacity — over a backing array. Sub-slicing shares that array, so an append that stays within cap silently overwrites data still visible through another slice. The full-slice expression a[low:high:max] caps capacity to force the next append into a fresh array.

Key Pattern

package main

import "fmt"

func main() {
    a := []int{1, 2, 3, 4, 5}
    s := a[1:3]             // len 2, cap 4 (shares backing array)
    fmt.Println(s, len(s), cap(s))

    s = append(s, 99)      // fits in cap -> overwrites a[3]!
    fmt.Println("a now:", a)

    // Full-slice expression caps growth, forcing a copy on next append.
    safe := a[1:3:3]
    safe = append(safe, 42) // cap exceeded -> new array, a untouched
    fmt.Println("a still:", a)

    // Delete index 2 by shifting the tail down.
    b := []int{10, 20, 30, 40}
    b = append(b[:2], b[3:]...)
    fmt.Println("deleted:", b)
}
Output
[2 3] 2 4 a now: [1 2 3 99 5] a still: [1 2 3 99 5] deleted: [10 20 40]
Challenge Exercise

Write filterInPlace(xs []int, keep func(int) bool) []int that reuses xs's backing array (zero allocations) and returns the shortened slice. Then demonstrate the aliasing bug: create two slices over one array, append through the first, and show the second slice's values changed. Finally, use the three-index form to hand out a "read-only" sub-slice whose appends can never corrupt the parent, and explain how slices.Clip relates.

Pitfall: Returning s[:0] and appending to it while the caller still holds the original slice — both share one array, so your writes clobber the caller's data. When a function must not alter its input, copy first (append([]T(nil), s...)) or use the three-index form to bound capacity.

Interfaces

Go Foundations • due today • rep 3

Quick Reminder

An interface value is a pair: a dynamic type and a value. A type satisfies an interface implicitly by having its methods — no implements keyword. A type switch recovers the concrete type at runtime, and the empty interface any holds anything. Keep interfaces small and define them where they are consumed, not where the implementation lives.

Key Pattern

package main

import "fmt"

type Stringer interface {
    String() string
}

type Temp float64

func (t Temp) String() string { return fmt.Sprintf("%.1f C", float64(t)) }

func describe(v any) {
    switch x := v.(type) { // recover the dynamic type
    case Stringer:
        fmt.Println("stringer:", x.String())
    case int:
        fmt.Println("int:", x)
    default:
        fmt.Printf("other: %T\n", x)
    }
}

func main() {
    describe(Temp(21.5))
    describe(42)
    describe(true)

    var s Stringer = Temp(0)
    fmt.Println("nil check:", s == nil) // false: has a concrete type
}
Output
stringer: 21.5 C int: 42 other: bool nil check: false
Challenge Exercise

Define a small Shape interface (Area() float64) with Rect and Circle implementations, then write totalArea(shapes []Shape) float64. Next, reproduce the classic "typed nil in an interface" trap: return a (*MyError)(nil) as an error and show err != nil is true. Finally, explain when to accept an interface but return a concrete type ("accept interfaces, return structs").

Pitfall: Storing a nil pointer in an interface and comparing to nil. The interface holds a non-nil type (*T) with a nil value, so iface == nil is false. This is the #1 source of "my error check never fires" bugs — return a literal nil, never a typed nil pointer, from functions declared to return error.

fmt Package

Standard Library • overdue 5d • rep 2

Quick Reminder

Verbs drive formatting: %v (default), %+v (with field names), %#v (Go syntax), %T (type), %q (quoted), %d/%s/%f (typed). Width and precision (%8.3f, %-5d) pad and truncate. A type's String() string method is used by %v/%s, and Errorf with %w wraps an error for later errors.Is/As.

Key Pattern

package main

import "fmt"

type Point struct{ X, Y int }

func main() {
    p := Point{1, 2}
    fmt.Printf("%v\n", p)   // {1 2}
    fmt.Printf("%+v\n", p)  // {X:1 Y:2}
    fmt.Printf("%#v\n", p)  // main.Point{X:1, Y:2}
    fmt.Printf("%T\n", p)   // main.Point

    // Width, left-align, zero-pad + precision.
    fmt.Printf("%5d|%-5d|%08.3f\n", 42, 42, 3.14159)

    fmt.Println(fmt.Sprintf("%q", "hi\n")) // quoted, escapes shown
}
Output
{1 2} {X:1 Y:2} main.Point{X:1, Y:2} main.Point 42|42 |0003.142 "hi\n"
Challenge Exercise

Implement String() on a Money type so fmt.Println prints $12.50, then verify %v and %s both use it but %d does not. Build an error chain with fmt.Errorf("load config: %w", err) and recover the root with errors.Is. Finally, explain why implementing String() that itself calls fmt.Sprintf("%v", t) causes infinite recursion, and how to avoid it (convert to the underlying type first).

Pitfall: A String() or Error() method that formats the receiver with %v/%s re-invokes itself — infinite recursion and a stack overflow. Convert to the base type (fmt.Sprintf("%d", int(t))) or format individual fields instead of the whole value.

io Package

Standard Library • overdue 5d • rep 2

Quick Reminder

io.Reader and io.Writer are the two-method interfaces that make Go's I/O composable. io.Copy(dst, src) streams without buffering the whole payload; io.ReadAll slurps to a []byte; TeeReader mirrors reads into a second writer; MultiReader/MultiWriter fan in and out. Everything from files to HTTP bodies to strings.Reader plugs into the same plumbing.

Key Pattern

package main

import (
    "fmt"
    "io"
    "strings"
)

func main() {
    // Copy streams from any Reader to any Writer.
    var sb strings.Builder
    n, _ := io.Copy(&sb, strings.NewReader("hello world"))
    fmt.Println(n, sb.String())

    // TeeReader mirrors everything read into a second writer.
    var seen strings.Builder
    tee := io.TeeReader(strings.NewReader("abc"), &seen)
    io.ReadAll(tee)
    fmt.Println("teed:", seen.String())

    // MultiReader concatenates sources into one stream.
    mr := io.MultiReader(strings.NewReader("foo"), strings.NewReader("bar"))
    out, _ := io.ReadAll(mr)
    fmt.Println(string(out))
}
Output
11 hello world teed: abc foobar
Challenge Exercise

Write a function that copies a reader to a writer while computing its SHA-256 in a single pass, using io.MultiWriter(dst, hasher). Then use io.LimitReader to cap how many bytes you accept from an untrusted source. Finally, implement your own io.Reader that yields an infinite stream of a repeated byte and feed it to io.CopyN, explaining why Read must return (n, nil) with n > 0 rather than blocking.

Pitfall: Ignoring the byte count or io.EOF contract when writing a custom Reader. Read may return n > 0 and io.EOF in the same call; callers that only check the error before processing the bytes will silently drop the final chunk. Always process the returned n bytes first, then handle the error.

encoding/json

Standard Library • overdue 5d • rep 2

Quick Reminder

json.Marshal/Unmarshal map structs to JSON via exported fields and struct tags: json:"name" renames, ,omitempty drops zero values, and - skips a field entirely. Decoding into map[string]any handles unknown shapes — but all numbers arrive as float64. Use json.Decoder for streams and MarshalIndent for pretty output.

Key Pattern

package main

import (
    "encoding/json"
    "fmt"
)

type User struct {
    Name  string `json:"name"`
    Email string `json:"email,omitempty"` // dropped when empty
    Age   int    `json:"-"`               // never serialized
}

func main() {
    b, _ := json.Marshal(User{Name: "Ada", Age: 36})
    fmt.Println(string(b))

    var back User
    json.Unmarshal([]byte(`{"name":"Bo","email":"bo@x.io"}`), &back)
    fmt.Printf("%+v\n", back)

    // Unknown shape decodes into map[string]any; numbers become float64.
    var m map[string]any
    json.Unmarshal([]byte(`{"n":1,"ok":true}`), &m)
    fmt.Println(m["n"], m["ok"])
}
Output
{"name":"Ada"} {Name:Bo Email:bo@x.io Age:0} 1 true
Challenge Exercise

Add a time.Time field and a custom MarshalJSON/UnmarshalJSON pair on a wrapper type to emit a Unix timestamp instead of RFC 3339. Then decode a large JSON array with a streaming json.Decoder and Token() so you never hold the whole document in memory. Finally, explain why unexported fields are invisible to the encoder and why decoding a JSON number into an int field works but into an any yields float64.

Pitfall: Assuming omitempty omits "empty" structs or slices the way it does scalars. It only checks for the type's zero value — an empty-but-non-nil slice ([]int{}) and a zero-valued nested struct are not omitted. For structs, use a pointer field (*T) so nil triggers omitempty.

time Package

Standard Library • overdue 3d • rep 2

Quick Reminder

Layouts in Go use the reference time Mon Jan 2 15:04:05 MST 2006 (01/02 03:04:05PM '06) — you write the format by example, not with %Y. Duration is a typed int64 of nanoseconds with helpful constants (time.Hour). Add, Sub, and Since do arithmetic; always compare and store times in a known zone (usually UTC).

Key Pattern

package main

import (
    "fmt"
    "time"
)

func main() {
    // Reference layout: Mon Jan 2 15:04:05 MST 2006.
    t := time.Date(2026, time.July, 2, 9, 30, 0, 0, time.UTC)
    fmt.Println(t.Format("2006-01-02 15:04"))

    d := 90 * time.Minute
    fmt.Println(d, d.Hours())

    later := t.Add(48 * time.Hour)
    fmt.Println(later.Sub(t)) // duration between two times
    fmt.Println(later.Weekday())

    parsed, _ := time.Parse("2006-01-02", "2026-12-25")
    fmt.Println(parsed.YearDay())
}
Output
2026-07-02 09:30 1h30m0s 1.5 48h0m0s Saturday 359
Challenge Exercise

Parse an RFC 3339 timestamp with time.Parse(time.RFC3339, ...), convert it to a named zone with LoadLocation("America/New_York"), and format it back. Then write a time.Ticker loop that stops after N ticks (remember defer ticker.Stop()). Finally, explain the difference between a wall clock and the monotonic reading Go attaches to time.Now(), and why you should use time.Since for measuring elapsed time.

Pitfall: Writing the layout string with the wrong reference numbers (e.g. "YYYY-MM-DD" or "2020-01-01"). Only the exact reference values — year 2006, month 01, day 02, hour 15 (or 03), minute 04, second 05 — are recognized; any other digits are copied literally, so your dates format as garbage with no error.

Section 1 — CLI & Web Standard Library

flag: Command-Line Flag Parsing

Std Lib

Why it matters: flag is the standard-library way to give a program typed command-line options with automatic -h/-help usage text — no external dependency. It parses -name=value and -flag forms, coerces to the right type, applies defaults, and leaves the non-flag arguments in flag.Args(). For anything beyond a full subcommand CLI, it is all you need.

Define, parse, and read flags

Each flag.Type(name, default, usage) call returns a pointer you dereference after flag.Parse(). Parsing stops at the first non-flag argument, which then appears in flag.Args().

package main

import (
    "flag"
    "fmt"
)

func main() {
    verbose := flag.Bool("v", false, "verbose output")
    name := flag.String("name", "world", "name to greet")
    count := flag.Int("n", 1, "number of greetings")
    flag.Parse() // reads os.Args[1:]

    for i := 0; i < *count; i++ {
        fmt.Printf("hello %s\n", *name)
    }
    if *verbose {
        fmt.Println("leftover args:", flag.Args())
    }
}
Output (go run main.go -name=Ada -n=2 -v extra)
hello Ada hello Ada leftover args: [extra]

Bind into a struct with Var and a custom FlagSet

flag.IntVar/StringVar write straight into existing variables, and a private flag.NewFlagSet lets a subcommand parse its own slice of arguments without touching the global set.

package main

import (
    "flag"
    "fmt"
)

type Config struct {
    Port    int
    Host    string
}

func main() {
    var cfg Config
    fs := flag.NewFlagSet("serve", flag.ExitOnError)
    fs.IntVar(&cfg.Port, "port", 8080, "listen port")
    fs.StringVar(&cfg.Host, "host", "localhost", "bind host")

    // In a real CLI these come from os.Args; hard-coded here for the demo.
    fs.Parse([]string{"-port", "9090"})

    fmt.Printf("%s:%d\n", cfg.Host, cfg.Port)
}
Output
localhost:9090

Common Mistakes

Wrong
port := flag.Int("port", 80, "port")
// Read before Parse: always the default!
fmt.Println(*port) // 80, flags not parsed
flag.Parse()
Right
port := flag.Int("port", 80, "port")
flag.Parse()          // parse first
fmt.Println(*port)    // now reflects -port

Key Rules

  • flag.Bool/Int/String(...) return pointers; dereference them only after flag.Parse().
  • flag.XxxVar(&dst, ...) binds into an existing variable — ideal for populating a config struct.
  • Parsing halts at the first non-flag token; the rest are in flag.Args() / flag.NArg().
  • Use a dedicated flag.NewFlagSet per subcommand; flag.ExitOnError auto-prints usage and exits on a bad flag.
Practice Exercise

Build a tiny wc-style tool with -l (lines), -w (words), and -c (bytes) flags that reads the filenames left in flag.Args(). Then add a subcommand dispatcher: read os.Args[1] ("add" / "list"), and hand the remaining args to a per-command FlagSet. Finally, implement a custom flag type by satisfying flag.Value (String() + Set(string) error) that accepts a comma-separated list into a []string.

Pitfall: Go's flag package does not accept the GNU double-dash long form as a group (--verbose works, but -abc is a single flag named abc, not -a -b -c), and flags must come before positional args — tool file.txt -v leaves -v unparsed in Args(). If you need bundled short flags or interspersed args, reach for cobra/pflag.
Connections: The standard-library baseline that the cobra CLI framework builds on; pairs with environment-variable configuration for the classic "flags override env override defaults" precedence chain.

net/url: Parsing & Building URLs

Std Lib

Why it matters: hand-splitting URLs on / and ? breaks the moment a value contains an encoded space, ampersand, or unicode. net/url parses a URL into typed components, decodes percent-escapes for you, and — crucially — re-encodes query values safely when you build one. It is the correct tool anywhere you assemble an API request or route on a path.

Parse and inspect a URL

url.Parse returns a *url.URL with typed fields. u.Query() decodes the query string into url.Values (a map[string][]string) — values arrive already unescaped.

package main

import (
    "fmt"
    "net/url"
)

func main() {
    raw := "https://api.example.com:8443/v1/search?q=go+lang&page=2#results"
    u, err := url.Parse(raw)
    if err != nil {
        fmt.Println("parse:", err)
        return
    }
    fmt.Println("scheme:", u.Scheme)
    fmt.Println("host:", u.Host)         // host:port
    fmt.Println("hostname:", u.Hostname())
    fmt.Println("path:", u.Path)
    fmt.Println("fragment:", u.Fragment)

    q := u.Query()                       // url.Values
    fmt.Println("q:", q.Get("q"))        // "+" decoded to a space
}
Output
scheme: https host: api.example.com:8443 hostname: api.example.com path: /v1/search fragment: results q: go lang

Build a URL with safe encoding

Construct a url.URL and let url.Values.Encode() percent-escape special characters. Encode also sorts keys, so the output is deterministic.

package main

import (
    "fmt"
    "net/url"
)

func main() {
    base := &url.URL{
        Scheme: "https",
        Host:   "example.com",
        Path:   "/search",
    }
    v := url.Values{}
    v.Set("q", "a b&c")  // space -> +, ampersand -> %26
    v.Add("tag", "go")
    v.Add("tag", "cli")   // repeated key -> two params
    base.RawQuery = v.Encode()

    fmt.Println(base.String())

    ref, _ := url.Parse("../v2/status")
    fmt.Println(base.ResolveReference(ref)) // join relative to base
}
Output
https://example.com/search?q=a+b%26c&tag=go&tag=cli https://example.com/v2/status

Common Mistakes

Wrong
// String concat: breaks on special chars.
u := "https://x.io/s?q=" + userInput
// userInput "a&b" injects a bogus param
Right
v := url.Values{}
v.Set("q", userInput) // escaped for you
u := "https://x.io/s?" + v.Encode()

Key Rules

  • url.Parse gives typed fields; use Hostname()/Port() to split Host, and u.Query() to decode params.
  • Build query strings with url.Values + Encode() — never concatenate untrusted strings into a URL.
  • url.QueryEscape/PathEscape (and their Unescape partners) handle single components.
  • ResolveReference implements RFC 3986 relative resolution — the right way to follow a redirect or join a base and a link.
Practice Exercise

Write addQuery(rawURL string, params map[string]string) (string, error) that parses the URL, merges the new params into the existing query (preserving any already present), and returns the re-encoded string. Then parse a batch of user-supplied links and reject any whose Scheme is not http/https (an open-redirect guard). Finally, show how url.Parse vs url.ParseRequestURI differ on a relative path like /foo.

Pitfall: Reading u.Path when you need the raw, still-encoded path for routing. u.Path is decoded, so a literal %2F in a segment becomes / and can collapse two path segments into one — a real source of routing and security bugs. Use u.EscapedPath() (or u.RawPath) when the encoded form matters.
Connections: Feeds directly into net/http client requests (http.NewRequest takes a URL string) and the enhanced ServeMux routing; the encoding discipline mirrors the escaping concerns of encoding/base64.

encoding/base64: Binary-Safe Text

Std Lib

Why it matters: base64 turns arbitrary bytes into an ASCII-safe string so binary data can ride through text-only channels — JSON fields, URLs, HTTP headers, email, data URIs. Go ships four ready encodings: standard and URL-safe alphabets, each with a padded and a raw (no =) variant. Picking the right one is the whole game: +// are unsafe in URLs and filenames.

Encode and decode a round-trip

EncodeToString and DecodeString are the convenience helpers on a preconfigured *base64.Encoding. Decoding validates the input and returns an error on a corrupt payload.

package main

import (
    "encoding/base64"
    "fmt"
)

func main() {
    data := []byte("Hello, Go!")

    encoded := base64.StdEncoding.EncodeToString(data)
    fmt.Println("encoded:", encoded)

    decoded, err := base64.StdEncoding.DecodeString(encoded)
    if err != nil {
        fmt.Println("decode:", err)
        return
    }
    fmt.Println("decoded:", string(decoded))
}
Output
encoded: SGVsbG8sIEdvIQ== decoded: Hello, Go!

Standard vs URL-safe vs raw (no padding)

The URL alphabet swaps + / for - _; the Raw* encodings omit trailing = padding. Use RawURLEncoding for tokens that appear in URLs, cookies, or JWTs.

package main

import (
    "encoding/base64"
    "fmt"
)

func main() {
    data := []byte{0xff, 0xe0} // bytes that force + / in the output

    fmt.Println("std:", base64.StdEncoding.EncodeToString(data))    // + / , padded
    fmt.Println("url:", base64.URLEncoding.EncodeToString(data))    // - _ , padded
    fmt.Println("raw:", base64.RawURLEncoding.EncodeToString(data)) // - _ , no =
}
Output
std: /+A= url: _-A= raw: _-A

Common Mistakes

Wrong
// Encoded with URLEncoding, decoded
// with StdEncoding -> illegal char error
tok := base64.URLEncoding.EncodeToString(b)
base64.StdEncoding.DecodeString(tok)
Right
// Encode and decode with the SAME encoding.
tok := base64.URLEncoding.EncodeToString(b)
base64.URLEncoding.DecodeString(tok)

Key Rules

  • Four encodings: StdEncoding, URLEncoding, RawStdEncoding, RawURLEncoding — decode with the exact one you encoded with.
  • Output length is 4 * ceil(n/3) (padded); base64 is not compression — it inflates size by ~33%.
  • Base64 is encoding, not encryption — it hides nothing; anyone can decode it.
  • For streaming large data use base64.NewEncoder/NewDecoder over an io.Writer/Reader instead of buffering the whole payload.
Practice Exercise

Build a minimal, unsigned JWT-style token: JSON-encode a small header and payload, RawURLEncoding-encode each, and join with dots — then split and decode it back. Next, stream-encode a large file to base64 using base64.NewEncoder wrapping an os.File, confirming constant memory. Finally, decode a deliberately truncated string and show the specific CorruptInputError you get.

Pitfall: Putting StdEncoding output into a URL or filename. The +, /, and = characters are reserved/awkward there — / becomes a path separator, + can decode to a space, and = needs escaping. Always use URLEncoding (or RawURLEncoding) for anything that travels in a URL, cookie, or header.
Connections: Complements net/url escaping (both make bytes travel safely through text channels) and encoding/gob / JSON when you need to embed a binary blob inside a text format.

Section 2 — Serialization & Data Structures

encoding/gob: Go's Native Serialization

Std Lib

Why it matters: gob is Go's self-describing binary format for streaming Go values between Go programs — RPC payloads, on-disk caches, snapshots. It is more compact and faster than JSON and preserves Go types (including nested structs, maps, and slices) without hand-written tags, at the cost of being Go-only. When both ends of the wire are Go, gob is the path of least friction.

Encode and decode a value

A gob.Encoder writes to any io.Writer; a gob.Decoder reads from any io.Reader. The stream is self-describing — the decoder learns the field layout from the data itself.

package main

import (
    "bytes"
    "encoding/gob"
    "fmt"
)

type Point struct {
    X, Y int
    Tags []string
}

func main() {
    var buf bytes.Buffer

    enc := gob.NewEncoder(&buf)
    if err := enc.Encode(Point{X: 1, Y: 2, Tags: []string{"a", "b"}}); err != nil {
        fmt.Println("encode:", err)
        return
    }

    var p Point
    dec := gob.NewDecoder(&buf)
    if err := dec.Decode(&p); err != nil {
        fmt.Println("decode:", err)
        return
    }
    fmt.Printf("%+v\n", p)
    fmt.Println("wrote bytes:", buf.Len() > 0)
}
Output
{X:1 Y:2 Tags:[a b]} wrote bytes: true

Interface values need registration

To send a value through an interface field, register each concrete type with gob.Register so the decoder can reconstruct it. Without it, decoding fails with "name not registered for interface".

package main

import (
    "bytes"
    "encoding/gob"
    "fmt"
)

type Shape interface{ Area() float64 }

type Circle struct{ R float64 }

func (c Circle) Area() float64 { return 3.14 * c.R * c.R }

func main() {
    gob.Register(Circle{}) // concrete type behind the interface

    var buf bytes.Buffer
    var in Shape = Circle{R: 2}
    gob.NewEncoder(&buf).Encode(&in)

    var out Shape
    gob.NewDecoder(&buf).Decode(&out)
    fmt.Printf("%.2f\n", out.Area())
}
Output
12.56

Common Mistakes

Wrong
// Only exported fields are encoded;
// 'secret' is silently dropped.
type T struct{ ID int; secret string }
// decoded T always has secret == ""
Right
// Export every field you need on the wire.
type T struct{ ID int; Secret string }
// Secret now survives the round-trip

Key Rules

  • Only exported fields are transmitted; unexported fields are silently ignored on both ends.
  • The decoder is tolerant: missing fields keep their zero value and unknown fields are skipped — enabling loose schema evolution.
  • Call gob.Register(T{}) for every concrete type sent through an interface value.
  • Gob is Go-to-Go only and not a stable cross-language format — use JSON or protobuf when the other side isn't Go.
Practice Exercise

Persist an in-memory cache: gob-encode a map[string]User to a file, then reload it on startup and verify equality. Next, demonstrate schema evolution — encode with a struct that has 2 fields, decode into a struct that adds a 3rd, and confirm the new field is its zero value with no error. Finally, benchmark gob vs encoding/json on the same 10k-element slice and compare both encoded size and time.

Pitfall: Treating gob as a durable, versioned archive format. It has no schema version, isn't guaranteed byte-stable across Go releases, and can't be read by non-Go tools. For long-term storage or public APIs prefer a self-describing, cross-language format (JSON, protobuf); reserve gob for ephemeral Go-to-Go transfer where speed and convenience win.
Connections: The serialization sibling of encoding/json (same encoder/decoder shape, different trade-offs) and the classic payload format for net/rpc; combine with encoding/base64 to embed a gob blob inside a text field.

container/ring: Circular Lists

Data Structures

Why it matters: container/ring is a fixed-size circular doubly linked list — every element has a Next() and Prev(), and the last wraps to the first. That makes it a natural fit for ring buffers: the "last N samples", round-robin scheduling over a fixed set of workers, or a rolling window where writing a new value overwrites the oldest with zero allocation and no shifting.

Fill a ring and walk it

ring.New(n) allocates a ring of n linked elements. Assign to .Value and step with .Next(); Do visits every element once, starting from the current one.

package main

import (
    "container/ring"
    "fmt"
)

func main() {
    r := ring.New(5) // circular list of 5 elements

    for i := 0; i < r.Len(); i++ {
        r.Value = i
        r = r.Next() // after r.Len() steps we're back at the start
    }

    sum := 0
    r.Do(func(v any) { // walk the whole ring in order
        sum += v.(int)
    })
    fmt.Println("sum:", sum)

    fmt.Println("start:", r.Value)          // back at element 0
    fmt.Println("two ahead:", r.Move(2).Value)
}
Output
sum: 10 start: 0 two ahead: 2

A fixed-memory "last N samples" buffer

Because the ring never grows, feeding a stream through it keeps only the most recent window values — the oldest is overwritten automatically.

package main

import (
    "container/ring"
    "fmt"
)

func main() {
    const window = 3
    r := ring.New(window)

    for _, sample := range []int{10, 20, 30, 40, 50} {
        r.Value = sample
        r = r.Next() // overwrite oldest as new samples arrive
    }

    // Ring now holds only the last 3 samples, oldest-first from here.
    r.Do(func(v any) {
        if v != nil {
            fmt.Print(v, " ")
        }
    })
    fmt.Println()
}
Output
30 40 50

Common Mistakes

Wrong
// Ignoring the return of Next/Move:
// these do NOT mutate r in place.
r.Next()          // result discarded
fmt.Println(r.Value) // still the old node
Right
// Reassign: navigation returns a new node.
r = r.Next()
fmt.Println(r.Value) // the next node's value

Key Rules

  • ring.New(n) builds an n-element circular list; there is no separate "list" handle — every node is the ring.
  • Next(), Prev(), and Move(n) return a node — reassign, they don't mutate the receiver's position.
  • Do(f) iterates all elements forward from the current one; Len() is O(n) — cache it if you loop on it.
  • Link and Unlink splice rings together and remove runs of elements when you need a resizable circular structure.
Practice Exercise

Implement a MovingAverage over the last k readings backed by a ring.Ring, with an Add(x float64) method and an Avg() float64 that walks the ring. Then build a round-robin load balancer: a ring of backend addresses where each request advances the cursor with Next(). Finally, contrast the ring approach with a plain slice + modulo index, and explain when the linked structure actually pays off (hint: cheap Link/Unlink).

Pitfall: Assuming ring.New(n) zeroes Value to something meaningful — every element's Value starts as nil (any). If you read the ring before fully populating it, Do hands you nil values and a type assertion like v.(int) panics. Guard with a nil check, or track how many slots you've actually written.
Connections: Completes the container trio alongside container/list (recency/insertion order) and container/heap (priority order) — ring covers fixed-size cyclic access that a plain slice handles only with manual modulo bookkeeping.
