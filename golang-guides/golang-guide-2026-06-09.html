encoding/json

Standard Library • overdue 3 days • rep 0

Quick Reminder

json.Marshal converts Go structs to JSON bytes; json.Unmarshal does the reverse. Struct tags like `json:"name,omitempty"` control field names and behaviour. json.NewEncoder/json.NewDecoder work with streams. json.RawMessage defers parsing for heterogeneous payloads. Custom (un)marshalers let you control serialization via MarshalJSON() and UnmarshalJSON().

Key Pattern

type User struct {
    Name  string `json:"name"`
    Email string `json:"email,omitempty"`
    Age   int    `json:"age"`
}

data, _ := json.Marshal(User{Name: "Alice", Age: 30})
// {"name":"Alice","age":30}

var u User
json.Unmarshal(data, &u)
Challenge Exercise

Implement a custom UnmarshalJSON for a FlexString type that can be either a JSON string ("hello") or a JSON object ({"value":"hello","lang":"en"}). In both cases, extract the string value.

#1 Pitfall: Only exported (uppercase) fields are visible to encoding/json. A field like name string (lowercase) is silently ignored during Marshal/Unmarshal.

fmt Package

Standard Library • overdue 3 days • rep 0

Quick Reminder

%v default format, %+v adds field names for structs, %#v Go syntax representation, %T type. The Stringer interface (String() string) customizes how types print. fmt.Fprintf writes to any io.Writer. fmt.Sprintf returns a formatted string. fmt.Errorf creates formatted errors with %w for wrapping.

Key Pattern

type Point struct{ X, Y int }

func (p Point) String() string {
    return fmt.Sprintf("Point(%d, %d)", p.X, p.Y)
}

fmt.Println(Point{3, 4}) // Point(3, 4)
Challenge Exercise

Implement the Stringer interface on a custom type so that fmt.Println(Point{3, 4}) outputs Point(3, 4). Then implement GoStringer so %#v outputs main.Point{X:3, Y:4}.

#1 Pitfall: Infinite recursion if your String() method calls fmt.Sprintf("%s", p) on the receiver — this calls String() again. Use fmt.Sprintf("Point(%d, %d)", p.X, p.Y) with primitive fields instead.

io Package

Standard Library • overdue 3 days • rep 0

Quick Reminder

io.Reader (Read(p []byte) (n int, err error)) and io.Writer are the fundamental stream abstractions. io.Copy transfers between them efficiently. io.TeeReader duplicates a read stream. io.Pipe creates an in-memory synchronous pipe. io.ReadAll reads everything to a byte slice. io.LimitReader caps reads at N bytes.

Key Pattern

// TeeReader: read from src, copy to both dst and stdout
src := strings.NewReader("hello world")
var buf bytes.Buffer
tee := io.TeeReader(src, &buf)

io.Copy(os.Stdout, tee)  // prints: hello world
fmt.Println(buf.String()) // also: hello world
Challenge Exercise

Create a TeeReader that copies data to both a file and stdout while reading from a source io.Reader. Read from the tee and verify both destinations received identical data.

#1 Pitfall: Not closing writers, especially io.Pipe writers. If the PipeWriter is never closed, the reader goroutine blocks forever, causing a goroutine leak.

os & filepath

Standard Library • overdue 3 days • rep 0

Quick Reminder

os.Create/os.Open/os.OpenFile for file operations. os.ReadFile/os.WriteFile for simple reads/writes. filepath.Join builds paths portably. filepath.Walk/filepath.WalkDir traverse directories. os.Getenv/os.Setenv for environment variables. filepath.Ext, filepath.Base, filepath.Dir extract path components.

Key Pattern

filepath.Walk(".", func(path string, info os.FileInfo, err error) error {
    if err != nil {
        return err
    }
    if filepath.Ext(path) == ".go" {
        fmt.Printf("%s (%d bytes)\n", path, info.Size())
    }
    return nil
})
Challenge Exercise

Walk a directory and list all .go files sorted by size (largest first). Print each file's path and size. Use filepath.WalkDir and sort.Slice.

#1 Pitfall: Using string concatenation (dir + "/" + file) instead of filepath.Join(dir, file) — breaks on Windows and mishandles trailing slashes.

Init Functions

Functions & Control Flow • overdue 3 days • rep 1

Quick Reminder

Each Go file can have one or more init() functions. They run automatically before main(), after all package-level variables are initialized. Import order determines package init order: if A imports B which imports C, then C's inits run first, then B's, then A's. Within a package, files are processed in alphabetical order by filename.

Key Pattern

var config map[string]string

func init() {
    // Runs before main(), good for one-time setup
    config = map[string]string{
        "env":  "production",
        "port": "8080",
    }
}
Challenge Exercise

Explain the initialization order when package A imports B which imports C, and each has an init() function. Then explain what happens if two files in the same package (a.go and z.go) each define an init().

#1 Pitfall: Relying on init() order across files in the same package — order is alphabetical by filename, which is fragile. Prefer explicit initialization in main() when order matters.

Variadic Functions

Functions & Control Flow • overdue 3 days • rep 1

Quick Reminder

Variadic parameters (...T) accept zero or more arguments of type T. Inside the function, the parameter is a []T slice. Pass a slice with slice... syntax. append is the most common variadic function. Only the last parameter can be variadic.

Key Pattern

func Merge[T any](slices ...[]T) []T {
    var result []T
    for _, s := range slices {
        result = append(result, s...)
    }
    return result
}

merged := Merge([]int{1, 2}, []int{3, 4}, []int{5})
// [1 2 3 4 5]
Challenge Exercise

Write a generic Merge function that takes variadic slices and returns a single merged slice. Test it with []int and []string inputs.

#1 Pitfall: Modifying the underlying array when passing a slice to a variadic function with .... The variadic parameter shares the same backing array as the original slice, so append inside the function can corrupt the caller's data.

Builder Pattern — Fluent APIs in Go

Advanced Patterns • New

Why It Matters

When functional options feel too abstract, the builder pattern provides a familiar fluent API for constructing complex objects step by step. Builders are especially useful for query construction, request building, and any scenario where the construction process is multi-step with validation at the end.

Core Concepts

  • Method chaining — each setter returns *Builder so calls chain: b.SetX(1).SetY(2).Build()
  • Pointer receivers — required so methods mutate the same builder instance
  • Build() method — validates accumulated state and returns the final object (or an error)
  • Error accumulation — collect errors during building, report them all at Build() time
  • Builder vs Functional Options — builder for multi-step construction with ordering; functional options for unordered configuration

Simple Query Builder

package main

import (
    "fmt"
    "strings"
)

type QueryBuilder struct {
    table      string
    columns    []string
    conditions []string
    orderBy    string
    limit      int
    err        error
}

func NewQueryBuilder() *QueryBuilder {
    return &QueryBuilder{limit: -1}
}

func (qb *QueryBuilder) Select(cols ...string) *QueryBuilder {
    qb.columns = cols
    return qb
}

func (qb *QueryBuilder) From(table string) *QueryBuilder {
    qb.table = table
    return qb
}

func (qb *QueryBuilder) Where(cond string) *QueryBuilder {
    qb.conditions = append(qb.conditions, cond)
    return qb
}

func (qb *QueryBuilder) OrderBy(col string) *QueryBuilder {
    qb.orderBy = col
    return qb
}

func (qb *QueryBuilder) Limit(n int) *QueryBuilder {
    if n < 0 {
        qb.err = fmt.Errorf("limit must be non-negative, got %d", n)
    }
    qb.limit = n
    return qb
}

func (qb *QueryBuilder) Build() (string, error) {
    if qb.err != nil {
        return "", qb.err
    }
    if qb.table == "" {
        return "", fmt.Errorf("table is required")
    }

    cols := "*"
    if len(qb.columns) > 0 {
        cols = strings.Join(qb.columns, ", ")
    }

    query := fmt.Sprintf("SELECT %s FROM %s", cols, qb.table)

    if len(qb.conditions) > 0 {
        query += " WHERE " + strings.Join(qb.conditions, " AND ")
    }
    if qb.orderBy != "" {
        query += " ORDER BY " + qb.orderBy
    }
    if qb.limit >= 0 {
        query += fmt.Sprintf(" LIMIT %d", qb.limit)
    }

    return query, nil
}

func main() {
    query, err := NewQueryBuilder().
        Select("name", "email", "age").
        From("users").
        Where("age > 18").
        Where("active = true").
        OrderBy("name ASC").
        Limit(10).
        Build()

    if err != nil {
        fmt.Println("Error:", err)
        return
    }
    fmt.Println(query)
}
Output
SELECT name, email, age FROM users WHERE age > 18 AND active = true ORDER BY name ASC LIMIT 10

HTTP Request Builder

package main

import (
    "bytes"
    "encoding/json"
    "fmt"
    "net/http"
    "time"
)

type RequestBuilder struct {
    method  string
    url     string
    headers map[string]string
    body    interface{}
    timeout time.Duration
    errs    []string
}

func NewRequest() *RequestBuilder {
    return &RequestBuilder{
        method:  "GET",
        headers: make(map[string]string),
        timeout: 30 * time.Second,
    }
}

func (rb *RequestBuilder) Method(m string) *RequestBuilder {
    rb.method = m
    return rb
}

func (rb *RequestBuilder) URL(url string) *RequestBuilder {
    rb.url = url
    return rb
}

func (rb *RequestBuilder) Header(key, val string) *RequestBuilder {
    rb.headers[key] = val
    return rb
}

func (rb *RequestBuilder) JSON(v interface{}) *RequestBuilder {
    rb.body = v
    rb.headers["Content-Type"] = "application/json"
    return rb
}

func (rb *RequestBuilder) Timeout(d time.Duration) *RequestBuilder {
    rb.timeout = d
    return rb
}

func (rb *RequestBuilder) Build() (*http.Request, error) {
    if rb.url == "" {
        rb.errs = append(rb.errs, "URL is required")
    }
    if len(rb.errs) > 0 {
        return nil, fmt.Errorf("build errors: %v", rb.errs)
    }

    var buf bytes.Buffer
    if rb.body != nil {
        if err := json.NewEncoder(&buf).Encode(rb.body); err != nil {
            return nil, fmt.Errorf("encoding body: %w", err)
        }
    }

    req, err := http.NewRequest(rb.method, rb.url, &buf)
    if err != nil {
        return nil, err
    }

    for k, v := range rb.headers {
        req.Header.Set(k, v)
    }
    return req, nil
}

func main() {
    req, err := NewRequest().
        Method("POST").
        URL("https://api.example.com/users").
        Header("Authorization", "Bearer token123").
        JSON(map[string]string{"name": "Alice"}).
        Timeout(5 * time.Second).
        Build()

    if err != nil {
        fmt.Println("Error:", err)
        return
    }
    fmt.Printf("%s %s\n", req.Method, req.URL)
    fmt.Printf("Content-Type: %s\n", req.Header.Get("Content-Type"))
    fmt.Printf("Authorization: %s\n", req.Header.Get("Authorization"))
}
Output
POST https://api.example.com/users Content-Type: application/json Authorization: Bearer token123

Builder vs Functional Options

package main

import "fmt"

// --- Functional Options approach ---
type ServerConfig struct {
    Host string
    Port int
}

type ServerOption func(*ServerConfig)

func WithHost(h string) ServerOption {
    return func(c *ServerConfig) { c.Host = h }
}

func WithPort(p int) ServerOption {
    return func(c *ServerConfig) { c.Port = p }
}

func NewServerFO(opts ...ServerOption) *ServerConfig {
    c := &ServerConfig{Host: "localhost", Port: 8080}
    for _, o := range opts {
        o(c)
    }
    return c
}

// --- Builder approach ---
type ServerBuilder struct {
    config ServerConfig
}

func NewServerBuilder() *ServerBuilder {
    return &ServerBuilder{config: ServerConfig{Host: "localhost", Port: 8080}}
}

func (b *ServerBuilder) Host(h string) *ServerBuilder {
    b.config.Host = h
    return b
}

func (b *ServerBuilder) Port(p int) *ServerBuilder {
    b.config.Port = p
    return b
}

func (b *ServerBuilder) Build() *ServerConfig {
    return &b.config
}

func main() {
    // Functional options: unordered, concise
    s1 := NewServerFO(WithHost("0.0.0.0"), WithPort(9090))
    fmt.Printf("FO:      %s:%d\n", s1.Host, s1.Port)

    // Builder: step-by-step, discoverable via IDE autocomplete
    s2 := NewServerBuilder().Host("0.0.0.0").Port(9090).Build()
    fmt.Printf("Builder: %s:%d\n", s2.Host, s2.Port)
}
Output
FO: 0.0.0.0:9090 Builder: 0.0.0.0:9090

Common Mistakes

Do: Use pointer receivers for chaining
func (b *Builder) Name(n string) *Builder {
    b.name = n
    return b // same instance
}
Don't: Use value receivers
func (b Builder) Name(n string) Builder {
    b.name = n
    return b // copies every call!
}
Do: Validate in Build()
func (b *Builder) Build() (*Obj, error) {
    if b.name == "" {
        return nil, errors.New("name required")
    }
    return &Obj{name: b.name}, nil
}
Don't: Skip validation
func (b *Builder) Build() *Obj {
    return &Obj{name: b.name}
    // name could be empty!
}

Key Rules

  • Always use pointer receivers so method chaining mutates the same builder instance
  • Validate accumulated state in Build(), not in individual setters (unless you want early failures)
  • Prefer functional options for simple configuration; use builders for multi-step construction with ordering
  • Collect errors during building and report them all at Build() time for better developer experience
  • Builders are great for query construction, request building, and complex object assembly
Connection: Builders use pointer receivers (Structs & Methods) for chaining. Compare with Functional Options from Day 6 — both solve configuration problems, but builders give IDE discoverability while functional options are more idiomatic for library APIs. Builders pair with error handling for validation in Build().

Repository Pattern — Interface-Based Data Access

Advanced Patterns • New

Why It Matters

Abstracting data access behind an interface makes business logic testable without a real database and allows swapping storage backends (Postgres, in-memory, file-based) without changing service code. This is one of the most important patterns for production Go services.

Core Concepts

  • Repository interface — a small interface defining CRUD operations (defined at the consumer, not the provider)
  • Concrete implementations — PostgresRepo, InMemoryRepo, etc. that satisfy the interface
  • Dependency injection — the service accepts the interface, not a concrete type
  • In-memory implementation — a real (not mock) implementation for tests that uses maps or slices
  • Interface segregation — keep the interface small; split read and write interfaces if needed

Repository Interface Definition

package main

import (
    "context"
    "errors"
    "fmt"
    "time"
)

// Domain model
type Task struct {
    ID        string
    Title     string
    Done      bool
    CreatedAt time.Time
}

// Repository interface — defined where it's CONSUMED
type TaskRepository interface {
    Create(ctx context.Context, task *Task) error
    GetByID(ctx context.Context, id string) (*Task, error)
    List(ctx context.Context) ([]*Task, error)
    Update(ctx context.Context, task *Task) error
    Delete(ctx context.Context, id string) error
}

// Sentinel errors for the domain
var (
    ErrTaskNotFound = errors.New("task not found")
    ErrTaskExists   = errors.New("task already exists")
)

func main() {
    fmt.Println("TaskRepository interface defined with 5 CRUD methods")
}
Output
TaskRepository interface defined with 5 CRUD methods

In-Memory Implementation (for Tests)

package main

import (
    "context"
    "fmt"
    "sync"
    "time"
)

type InMemoryTaskRepo struct {
    mu    sync.RWMutex
    tasks map[string]*Task
}

func NewInMemoryTaskRepo() *InMemoryTaskRepo {
    return &InMemoryTaskRepo{tasks: make(map[string]*Task)}
}

func (r *InMemoryTaskRepo) Create(ctx context.Context, task *Task) error {
    r.mu.Lock()
    defer r.mu.Unlock()
    if _, exists := r.tasks[task.ID]; exists {
        return ErrTaskExists
    }
    r.tasks[task.ID] = task
    return nil
}

func (r *InMemoryTaskRepo) GetByID(ctx context.Context, id string) (*Task, error) {
    r.mu.RLock()
    defer r.mu.RUnlock()
    task, ok := r.tasks[id]
    if !ok {
        return nil, ErrTaskNotFound
    }
    return task, nil
}

func (r *InMemoryTaskRepo) List(ctx context.Context) ([]*Task, error) {
    r.mu.RLock()
    defer r.mu.RUnlock()
    result := make([]*Task, 0, len(r.tasks))
    for _, t := range r.tasks {
        result = append(result, t)
    }
    return result, nil
}

func (r *InMemoryTaskRepo) Update(ctx context.Context, task *Task) error {
    r.mu.Lock()
    defer r.mu.Unlock()
    if _, exists := r.tasks[task.ID]; !exists {
        return ErrTaskNotFound
    }
    r.tasks[task.ID] = task
    return nil
}

func (r *InMemoryTaskRepo) Delete(ctx context.Context, id string) error {
    r.mu.Lock()
    defer r.mu.Unlock()
    if _, exists := r.tasks[id]; !exists {
        return ErrTaskNotFound
    }
    delete(r.tasks, id)
    return nil
}

func main() {
    repo := NewInMemoryTaskRepo()
    ctx := context.Background()

    // Create
    task := &Task{ID: "1", Title: "Learn Go", CreatedAt: time.Now()}
    repo.Create(ctx, task)

    // Read
    found, _ := repo.GetByID(ctx, "1")
    fmt.Printf("Found: %s (done: %v)\n", found.Title, found.Done)

    // Update
    found.Done = true
    repo.Update(ctx, found)

    // List
    all, _ := repo.List(ctx)
    fmt.Printf("Total tasks: %d\n", len(all))

    // Delete
    repo.Delete(ctx, "1")
    _, err := repo.GetByID(ctx, "1")
    fmt.Printf("After delete: %v\n", err)
}
Output
Found: Learn Go (done: false) Total tasks: 1 After delete: task not found

Service Using the Repository

package main

import (
    "context"
    "fmt"
    "time"
)

type TaskService struct {
    repo TaskRepository
}

func NewTaskService(repo TaskRepository) *TaskService {
    return &TaskService{repo: repo}
}

func (s *TaskService) CreateTask(ctx context.Context, id, title string) (*Task, error) {
    task := &Task{
        ID:        id,
        Title:     title,
        Done:      false,
        CreatedAt: time.Now(),
    }
    if err := s.repo.Create(ctx, task); err != nil {
        return nil, fmt.Errorf("creating task: %w", err)
    }
    return task, nil
}

func (s *TaskService) CompleteTask(ctx context.Context, id string) error {
    task, err := s.repo.GetByID(ctx, id)
    if err != nil {
        return fmt.Errorf("finding task: %w", err)
    }
    task.Done = true
    if err := s.repo.Update(ctx, task); err != nil {
        return fmt.Errorf("updating task: %w", err)
    }
    return nil
}

func main() {
    // In production: NewTaskService(postgresRepo)
    // In tests:      NewTaskService(NewInMemoryTaskRepo())
    repo := NewInMemoryTaskRepo()
    svc := NewTaskService(repo)
    ctx := context.Background()

    task, _ := svc.CreateTask(ctx, "1", "Write tests")
    fmt.Printf("Created: %s (done: %v)\n", task.Title, task.Done)

    svc.CompleteTask(ctx, "1")
    updated, _ := repo.GetByID(ctx, "1")
    fmt.Printf("After complete: %s (done: %v)\n", updated.Title, updated.Done)
}
Output
Created: Write tests (done: false) After complete: Write tests (done: true)

Common Mistakes

Do: Small, focused interface
type TaskRepository interface {
    Create(ctx context.Context, t *Task) error
    GetByID(ctx context.Context, id string) (*Task, error)
}
Don't: Leak storage into interface
type TaskRepository interface {
    ExecSQL(query string) error
    BeginTx() (*sql.Tx, error)
    // SQL-specific = can't swap backends
}
Do: Use domain errors
var ErrTaskNotFound = errors.New("task not found")

// Both Postgres and InMemory return this
return nil, ErrTaskNotFound
Don't: Leak storage errors
// Returning sql.ErrNoRows to the service
// couples service to SQL
return nil, sql.ErrNoRows

Key Rules

  • Define the repository interface where it is consumed (in the service package), not where it is implemented
  • Keep the interface small — split into TaskReader and TaskWriter if the full CRUD interface is too large
  • Use domain-level errors (ErrTaskNotFound), not storage-level errors (sql.ErrNoRows)
  • Always accept context.Context as the first parameter for cancellation and timeout support
  • The in-memory implementation is a real implementation, not a mock — it can serve as both test helper and cache
Connection: The repository pattern builds directly on Interfaces (implicit satisfaction) and Mocking from Day 6. It uses Context for cancellation-aware data access. The in-memory implementation uses sync.RWMutex for thread safety. Domain errors use Error Handling patterns with sentinel errors and %w wrapping.

reflect Package — Runtime Type Inspection

Advanced Patterns • New

Why It Matters

The reflect package enables runtime type inspection — needed for serialization, ORMs, validation frameworks, and generic data processing. But it is slow (10-100x slower than direct access) and should be used sparingly. When generics can solve the problem, prefer generics.

Core Concepts

  • reflect.TypeOf(v) — returns the reflect.Type, describing the type of v
  • reflect.ValueOf(v) — returns the reflect.Value, holding the actual value of v
  • Kind — the underlying category: reflect.Struct, reflect.Ptr, reflect.Slice, etc.
  • Struct field iteration — Type.NumField(), Type.Field(i), reading struct tags via Tag.Get("json")
  • Setting values — must pass a pointer; use .Elem() to dereference, then .Set()
  • reflect.DeepEqual — recursive comparison including unexported fields

TypeOf and ValueOf Basics

package main

import (
    "fmt"
    "reflect"
)

func main() {
    x := 42
    s := "hello"
    xs := []int{1, 2, 3}

    fmt.Printf("x:  Type=%-12s Kind=%-8s Value=%v\n",
        reflect.TypeOf(x), reflect.TypeOf(x).Kind(), reflect.ValueOf(x))
    fmt.Printf("s:  Type=%-12s Kind=%-8s Value=%v\n",
        reflect.TypeOf(s), reflect.TypeOf(s).Kind(), reflect.ValueOf(s))
    fmt.Printf("xs: Type=%-12s Kind=%-8s Value=%v\n",
        reflect.TypeOf(xs), reflect.TypeOf(xs).Kind(), reflect.ValueOf(xs))
}
Output
x: Type=int Kind=int Value=42 s: Type=string Kind=string Value=hello xs: Type=[]int Kind=slice Value=[1 2 3]

Iterating Struct Fields and Tags

package main

import (
    "fmt"
    "reflect"
)

type User struct {
    Name  string `json:"name" validate:"required"`
    Email string `json:"email,omitempty" validate:"email"`
    Age   int    `json:"age" validate:"min=0,max=150"`
}

func InspectStruct(v interface{}) {
    t := reflect.TypeOf(v)
    if t.Kind() == reflect.Ptr {
        t = t.Elem()
    }

    fmt.Printf("Struct: %s (%d fields)\n", t.Name(), t.NumField())
    for i := 0; i < t.NumField(); i++ {
        f := t.Field(i)
        jsonTag := f.Tag.Get("json")
        validateTag := f.Tag.Get("validate")
        fmt.Printf("  %-8s %-8s json=%q validate=%q\n",
            f.Name, f.Type, jsonTag, validateTag)
    }
}

func main() {
    InspectStruct(User{})
}
Output
Struct: User (3 fields) Name string json="name" validate="required" Email string json="email,omitempty" validate="email" Age int json="age" validate="min=0,max=150"

Struct-to-Map Converter

package main

import (
    "fmt"
    "reflect"
    "strings"
)

func StructToMap(v interface{}) map[string]interface{} {
    result := make(map[string]interface{})
    val := reflect.ValueOf(v)
    typ := reflect.TypeOf(v)

    if typ.Kind() == reflect.Ptr {
        val = val.Elem()
        typ = typ.Elem()
    }

    for i := 0; i < typ.NumField(); i++ {
        field := typ.Field(i)
        if !field.IsExported() {
            continue // skip unexported fields
        }

        // Use json tag as key if present
        key := field.Name
        if jsonTag := field.Tag.Get("json"); jsonTag != "" {
            parts := strings.Split(jsonTag, ",")
            if parts[0] != "" && parts[0] != "-" {
                key = parts[0]
            }
        }

        result[key] = val.Field(i).Interface()
    }
    return result
}

type Product struct {
    Name  string  `json:"name"`
    Price float64 `json:"price"`
    SKU   string  `json:"sku"`
}

func main() {
    p := Product{Name: "Widget", Price: 9.99, SKU: "W-001"}
    m := StructToMap(p)
    for k, v := range m {
        fmt.Printf("  %s: %v\n", k, v)
    }
}
Output
name: Widget price: 9.99 sku: W-001

Setting Values via Reflect (Requires Pointer)

package main

import (
    "fmt"
    "reflect"
)

func FillDefaults(ptr interface{}) {
    val := reflect.ValueOf(ptr).Elem() // must be pointer
    typ := val.Type()

    for i := 0; i < val.NumField(); i++ {
        field := val.Field(i)
        if !field.CanSet() {
            continue
        }
        if field.Kind() == reflect.String && field.String() == "" {
            tag := typ.Field(i).Tag.Get("default")
            if tag != "" {
                field.SetString(tag)
            } else {
                field.SetString("N/A")
            }
        }
    }
}

type Config struct {
    Host    string `default:"localhost"`
    Port    string `default:"8080"`
    Region  string // no default tag
    Version string
}

func main() {
    c := &Config{Version: "1.0"} // Version is already set
    FillDefaults(c)
    fmt.Printf("Host: %s\n", c.Host)
    fmt.Printf("Port: %s\n", c.Port)
    fmt.Printf("Region: %s\n", c.Region)
    fmt.Printf("Version: %s\n", c.Version)
}
Output
Host: localhost Port: 8080 Region: N/A Version: 1.0

Benchmarking Reflect vs Direct Access

package main

import (
    "fmt"
    "reflect"
    "time"
)

type Point struct{ X, Y int }

func main() {
    p := Point{X: 10, Y: 20}
    iterations := 10_000_000

    // Direct access
    start := time.Now()
    sum := 0
    for i := 0; i < iterations; i++ {
        sum += p.X + p.Y
    }
    directDur := time.Since(start)

    // Reflect access
    start = time.Now()
    val := reflect.ValueOf(p)
    sum = 0
    for i := 0; i < iterations; i++ {
        sum += int(val.Field(0).Int()) + int(val.Field(1).Int())
    }
    reflectDur := time.Since(start)

    fmt.Printf("Direct:  %v\n", directDur)
    fmt.Printf("Reflect: %v\n", reflectDur)
    fmt.Printf("Reflect is ~%.0fx slower\n", float64(reflectDur)/float64(directDur))
}
Output (approximate)
Direct: 7ms Reflect: 320ms Reflect is ~46x slower

Common Mistakes

Do: Pass pointer to Set values
val := reflect.ValueOf(&s).Elem()
val.Field(0).SetString("new")
// Works: pointer allows mutation
Don't: Forget Elem() on pointer
val := reflect.ValueOf(&s)
val.Field(0).SetString("new")
// PANIC: val is *Struct, not Struct
// Need val.Elem().Field(0)
Do: Use generics when possible
func Max[T constraints.Ordered](a, b T) T {
    if a > b { return a }
    return b
}
Don't: Use reflect for type-safe ops
func Max(a, b interface{}) interface{} {
    va := reflect.ValueOf(a)
    vb := reflect.ValueOf(b)
    // 10 lines of Kind-checking...
}

Key Rules

  • Use reflect for serialization, ORMs, validation, and code generation — not for everyday logic
  • Always check Kind() before accessing fields to avoid panics
  • Setting values requires a pointer: reflect.ValueOf(&v).Elem()
  • Only exported fields can be read/set via reflect — unexported fields cause panics on Set()
  • Prefer generics over reflect when the type set is known at compile time
  • Reflect is 10-100x slower than direct access — cache reflected types when possible
Connection: reflect is the engine behind encoding/json's Marshal/Unmarshal (which reads struct tags via reflect). The Struct-to-Map converter above is essentially what json.Marshal does internally. The unsafe package (next concept) provides even lower-level access when reflect is too slow.

unsafe Package — Escape Hatches

Advanced Patterns • New

Why It Matters

The unsafe package provides escape hatches for performance-critical code and FFI (foreign function interface). It bypasses Go's type system entirely. You should almost never use it — but understanding it helps you read standard library internals and performance-critical code.

Core Concepts

  • unsafe.Pointer — a special pointer type that can be converted to/from any pointer type
  • unsafe.Sizeof(v) — returns the size in bytes of the value's type (not including referenced data)
  • unsafe.Alignof(v) — returns the alignment of the value's type
  • unsafe.Offsetof(s.f) — returns the byte offset of field f within struct s
  • 6 valid conversion patterns for unsafe.Pointer (defined in the unsafe package docs)
  • Zero-copy string/[]byte conversion — fast but dangerous if the source is mutated

Sizeof for Different Types

package main

import (
    "fmt"
    "unsafe"
)

func main() {
    var b bool
    var i8 int8
    var i32 int32
    var i64 int64
    var f64 float64
    var s string
    var sl []int
    var p *int

    fmt.Printf("bool:    %d bytes\n", unsafe.Sizeof(b))
    fmt.Printf("int8:    %d bytes\n", unsafe.Sizeof(i8))
    fmt.Printf("int32:   %d bytes\n", unsafe.Sizeof(i32))
    fmt.Printf("int64:   %d bytes\n", unsafe.Sizeof(i64))
    fmt.Printf("float64: %d bytes\n", unsafe.Sizeof(f64))
    fmt.Printf("string:  %d bytes (header: pointer + length)\n", unsafe.Sizeof(s))
    fmt.Printf("[]int:   %d bytes (header: pointer + length + cap)\n", unsafe.Sizeof(sl))
    fmt.Printf("*int:    %d bytes\n", unsafe.Sizeof(p))
}
Output (64-bit system)
bool: 1 bytes int8: 1 bytes int32: 4 bytes int64: 8 bytes float64: 8 bytes string: 16 bytes (header: pointer + length) []int: 24 bytes (header: pointer + length + cap) *int: 8 bytes

Struct Memory Layout and Padding

package main

import (
    "fmt"
    "unsafe"
)

// Bad layout: padding wastes memory
type BadLayout struct {
    A bool   // 1 byte + 7 padding
    B int64  // 8 bytes
    C bool   // 1 byte + 7 padding
    D int64  // 8 bytes
} // Total: 32 bytes

// Good layout: fields ordered by decreasing alignment
type GoodLayout struct {
    B int64  // 8 bytes
    D int64  // 8 bytes
    A bool   // 1 byte
    C bool   // 1 byte + 6 padding
} // Total: 24 bytes

func main() {
    fmt.Printf("BadLayout:  %d bytes\n", unsafe.Sizeof(BadLayout{}))
    fmt.Printf("GoodLayout: %d bytes\n", unsafe.Sizeof(GoodLayout{}))
    fmt.Printf("Saved: %d bytes per instance\n",
        unsafe.Sizeof(BadLayout{})-unsafe.Sizeof(GoodLayout{}))

    // Show field offsets
    var bl BadLayout
    fmt.Printf("\nBadLayout offsets:\n")
    fmt.Printf("  A (bool):  offset=%d\n", unsafe.Offsetof(bl.A))
    fmt.Printf("  B (int64): offset=%d\n", unsafe.Offsetof(bl.B))
    fmt.Printf("  C (bool):  offset=%d\n", unsafe.Offsetof(bl.C))
    fmt.Printf("  D (int64): offset=%d\n", unsafe.Offsetof(bl.D))
}
Output
BadLayout: 32 bytes GoodLayout: 24 bytes Saved: 8 bytes per instance BadLayout offsets: A (bool): offset=0 B (int64): offset=8 C (bool): offset=16 D (int64): offset=24

Converting Between Pointer Types

package main

import (
    "fmt"
    "math"
    "unsafe"
)

func main() {
    // View float64 bits as uint64
    f := math.Pi
    bits := *(*uint64)(unsafe.Pointer(&f))
    fmt.Printf("Pi as float64: %f\n", f)
    fmt.Printf("Pi as uint64:  %d\n", bits)
    fmt.Printf("Pi hex bits:   0x%x\n", bits)

    // Convert back
    f2 := *(*float64)(unsafe.Pointer(&bits))
    fmt.Printf("Recovered:     %f\n", f2)

    // Verify: same as math.Float64bits
    fmt.Printf("math.Float64bits: %d\n", math.Float64bits(f))
}
Output
Pi as float64: 3.141593 Pi as uint64: 4614256656552045848 Pi hex bits: 0x400921fb54442d18 Recovered: 3.141593 math.Float64bits: 4614256656552045848

Zero-Copy String to []byte (Dangerous!)

package main

import (
    "fmt"
    "unsafe"
)

// StringToBytes converts a string to []byte without copying.
// WARNING: The returned byte slice MUST NOT be modified!
// Strings in Go are immutable; modifying the bytes is undefined behavior.
func StringToBytes(s string) []byte {
    return unsafe.Slice(unsafe.StringData(s), len(s))
}

// BytesToString converts []byte to string without copying.
// WARNING: The original byte slice MUST NOT be modified after this call!
func BytesToString(b []byte) string {
    return unsafe.String(unsafe.SliceData(b), len(b))
}

func main() {
    s := "hello world"
    b := StringToBytes(s)
    fmt.Printf("String: %q\n", s)
    fmt.Printf("Bytes:  %v\n", b)
    fmt.Printf("Same data? No copy performed.\n")

    // Safe because we don't modify b
    s2 := BytesToString(b)
    fmt.Printf("Back to string: %q\n", s2)
}
Output
String: "hello world" Bytes: [104 101 108 108 111 32 119 111 114 108 100] Same data? No copy performed. Back to string: "hello world"

Common Mistakes

Do: Use unsafe only when necessary
// Standard library internal:
// optimizing hot path in encoding/json
// or interfacing with C via cgo
Don't: Use unsafe for convenience
// "I don't want to write a type conversion"
x := *(*int)(unsafe.Pointer(&myFloat))
// Use math.Float64bits instead!
Do: Order struct fields by size
type Good struct {
    A int64 // 8
    B int32 // 4
    C bool  // 1
} // 16 bytes (with padding)
Don't: Assume no padding
type Bad struct {
    C bool  // 1 + 7 padding
    A int64 // 8
    B int32 // 4 + 4 padding
} // 24 bytes!

Key Rules

  • Almost never use unsafe in application code — it is for standard library internals, performance-critical paths, and CGo
  • Zero-copy string/[]byte conversions are dangerous: mutating the bytes after conversion is undefined behavior
  • Order struct fields by decreasing size to minimize padding and save memory
  • unsafe.Sizeof returns the size of the value's type header, not the total allocated memory (e.g., a string header is 16 bytes regardless of string length)
  • The Go garbage collector may move objects — storing unsafe.Pointer in a uintptr and converting back later is unsafe
Connection: unsafe.Sizeof helps understand the memory layout that the reflect package (previous concept) inspects at runtime. The zero-copy conversion techniques appear internally in encoding/json and the standard library. Struct padding knowledge complements Benchmarks — optimizing hot structs for cache-line alignment can measurably improve performance.

Code Generation — go generate

Advanced Patterns • New

Why It Matters

Code generation eliminates boilerplate, keeps generated code in sync with a source of truth, and enables patterns impossible with generics alone. The go generate command scans for //go:generate directives and runs the specified commands to produce or update Go source files.

Core Concepts

  • //go:generate command args... — a directive comment that triggers command args... when you run go generate
  • stringer — generates String() methods for integer-based enum types (iota constants)
  • enumer — generates String(), parsing, and iteration utilities for enums
  • mockgen — generates mock implementations of interfaces for testing
  • text/template — Go's built-in template engine for custom code generation
  • go generate ./... — runs all generate directives recursively

Using stringer for Enum Types

package main

// To use in a real project:
// 1. Install: go install golang.org/x/tools/cmd/stringer@latest
// 2. Add the directive below (uncommented)
// 3. Run: go generate ./...

// go:generate stringer -type=Color

type Color int

const (
    Red Color = iota
    Green
    Blue
    Yellow
)

// Without stringer, you'd write this manually:
func (c Color) String() string {
    names := [...]string{"Red", "Green", "Blue", "Yellow"}
    if c < Red || c > Yellow {
        return "Unknown"
    }
    return names[c]
}

// stringer generates exactly this kind of code automatically.

Complete Example: Enum with String Methods

package main

import "fmt"

type Status int

const (
    StatusPending Status = iota
    StatusRunning
    StatusComplete
    StatusFailed
)

// In production, stringer generates this.
// Here we write it manually to show the pattern.
var statusNames = map[Status]string{
    StatusPending:  "Pending",
    StatusRunning:  "Running",
    StatusComplete: "Complete",
    StatusFailed:   "Failed",
}

func (s Status) String() string {
    if name, ok := statusNames[s]; ok {
        return name
    }
    return fmt.Sprintf("Status(%d)", s)
}

// ParseStatus converts a string back to a Status.
func ParseStatus(s string) (Status, error) {
    for k, v := range statusNames {
        if v == s {
            return k, nil
        }
    }
    return 0, fmt.Errorf("unknown status: %q", s)
}

func main() {
    s := StatusRunning
    fmt.Printf("Status: %s\n", s)
    fmt.Printf("Value:  %d\n", s)

    parsed, err := ParseStatus("Complete")
    if err != nil {
        fmt.Println("Error:", err)
        return
    }
    fmt.Printf("Parsed: %s (value: %d)\n", parsed, parsed)

    // Invalid status
    _, err = ParseStatus("Invalid")
    fmt.Printf("Parse error: %v\n", err)
}
Output
Status: Running Value: 1 Parsed: Complete (value: 2) Parse error: unknown status: "Invalid"

Using mockgen for Interface Mocks

package main

import "fmt"

// In a real project, you'd add this directive:
// //go:generate mockgen -source=cache.go -destination=mock_cache.go -package=main

// Cache is the interface we want to generate a mock for.
type Cache interface {
    Get(key string) (string, bool)
    Set(key string, value string) error
    Delete(key string) error
}

// mockgen would generate a MockCache struct with:
// - EXPECT() for setting expectations
// - Implementation of all interface methods
// - Call tracking and verification

// Example of what mockgen generates (simplified):
type MockCache struct {
    GetFunc    func(key string) (string, bool)
    SetFunc    func(key, value string) error
    DeleteFunc func(key string) error
}

func (m *MockCache) Get(key string) (string, bool) { return m.GetFunc(key) }
func (m *MockCache) Set(key, value string) error    { return m.SetFunc(key, value) }
func (m *MockCache) Delete(key string) error        { return m.DeleteFunc(key) }

func main() {
    // In tests, use the generated mock:
    mock := &MockCache{
        GetFunc: func(key string) (string, bool) {
            if key == "hello" {
                return "world", true
            }
            return "", false
        },
        SetFunc: func(key, value string) error {
            fmt.Printf("Mock Set(%q, %q)\n", key, value)
            return nil
        },
        DeleteFunc: func(key string) error { return nil },
    }

    val, ok := mock.Get("hello")
    fmt.Printf("Get(hello): %q, found=%v\n", val, ok)
    mock.Set("foo", "bar")
}
Output
Get(hello): "world", found=true Mock Set("foo", "bar")

Custom Generator with text/template

package main

import (
    "fmt"
    "os"
    "text/template"
)

// In a real project, this would be a separate generator program
// invoked by //go:generate go run gen.go

const enumTemplate = `// Code generated by enum-gen. DO NOT EDIT.
package {{.Package}}

import "fmt"

var _{{.TypeName}}_names = map[{{.TypeName}}]string{
{{- range .Values}}
    {{$.TypeName}}{{.}}: "{{.}}",
{{- end}}
}

func (v {{.TypeName}}) String() string {
    if name, ok := _{{.TypeName}}_names[v]; ok {
        return name
    }
    return fmt.Sprintf("{{.TypeName}}(%d)", v)
}
`

type EnumDef struct {
    Package  string
    TypeName string
    Values   []string
}

func main() {
    def := EnumDef{
        Package:  "main",
        TypeName: "Priority",
        Values:   []string{"Low", "Medium", "High", "Critical"},
    }

    tmpl := template.Must(template.New("enum").Parse(enumTemplate))
    fmt.Println("Generated code:")
    fmt.Println("---")
    tmpl.Execute(os.Stdout, def)
    fmt.Println("---")
}

go generate Workflow

// Step 1: Add directive to your source file
//go:generate stringer -type=Color
//go:generate mockgen -source=repo.go -destination=mock_repo.go

// Step 2: Run go generate
// $ go generate ./...

// Step 3: Generated files appear alongside source
// color_string.go  (from stringer)
// mock_repo.go     (from mockgen)

// Step 4: Commit generated files to version control
// $ git add color_string.go mock_repo.go

// Step 5: In CI, verify generated code is fresh
// $ go generate ./...
// $ git diff --exit-code
// (fails if generated code is stale)

Common Mistakes

Do: Commit generated code
// Generated files in version control:
// - Builds work without generator tools
// - CI can verify freshness with git diff
// - Code reviewers see what changed
Don't: Gitignore generated code
// If generated code is gitignored:
// - Builds need generator tools installed
// - Can't verify CI without running gen
// - Harder to review changes
Do: Add "DO NOT EDIT" header
// Code generated by mygen. DO NOT EDIT.
package main

// Go tools recognize this pattern
Don't: Hand-edit generated files
// Someone edited mock_repo.go manually
// Next go generate overwrites their changes
// Always edit the source or template

Key Rules

  • Use stringer for any integer-based enum type — saves manual boilerplate and stays in sync
  • Use mockgen for generating mocks of large interfaces — write manual mocks for small ones
  • Always commit generated code to version control and verify freshness in CI
  • Include the // Code generated ... DO NOT EDIT. header so tools and humans know not to modify it
  • Prefer generics over code generation when the type system can express the abstraction
  • Custom generators with text/template are powerful for domain-specific boilerplate
Connection: stringer generates String() methods that implement the Stringer interface from the fmt Package. mockgen generates implementations of Interfaces, supporting the Mocking patterns from Day 6. The reflect package (previous concept) powers many generators that inspect struct tags. Code generation is the compile-time complement to reflect's runtime inspection.
