Error Handling

Go Foundations • 2 days ago • rep 1

Quick Reminder

Go handles errors as values returned from functions, not exceptions. The error interface has a single Error() string method. Use fmt.Errorf with %w to wrap errors, errors.Is to compare sentinel errors, and errors.As to extract typed errors. Always check returned errors immediately.

Key Pattern

// Wrap errors with context
result, err := doSomething()
if err != nil {
    return fmt.Errorf("doSomething failed: %w", err)
}

// Sentinel error checking
if errors.Is(err, os.ErrNotExist) {
    // handle file not found
}

// Type assertion on errors
var pathErr *os.PathError
if errors.As(err, &pathErr) {
    fmt.Println("failed path:", pathErr.Path)
}
Challenge Exercise

Create a ValidationError type that holds a field name and a message. Implement the error interface. Write a ValidateUser(name, email string) error function that returns a wrapped ValidationError. In the caller, use errors.As to extract the field name and print a user-friendly message.

#1 Pitfall: Using == to compare errors instead of errors.Is — this breaks when errors are wrapped with %w. Always use errors.Is for sentinel errors and errors.As for typed errors.

First-class Functions

Functions & Control Flow • 2 days ago • rep 0

Quick Reminder

Functions in Go are first-class values — they can be assigned to variables, passed as arguments, returned from other functions, and stored in data structures. Closures capture variables from their enclosing scope by reference. Function types are declared with type HandlerFunc func(args) returns.

Key Pattern

// Function as parameter (strategy pattern)
func Apply(nums []int, fn func(int) int) []int {
    result := make([]int, len(nums))
    for i, n := range nums {
        result[i] = fn(n)
    }
    return result
}

// Closure capturing external state
func Counter() func() int {
    count := 0
    return func() int {
        count++
        return count
    }
}

// Middleware pattern
func WithLogging(next http.HandlerFunc) http.HandlerFunc {
    return func(w http.ResponseWriter, r *http.Request) {
        log.Println(r.Method, r.URL.Path)
        next(w, r)
    }
}
Challenge Exercise

Write a Compose[T any](fns ...func(T) T) func(T) T function that returns a single function applying all fns in left-to-right order. Test by composing double, addOne, and square on integers. Verify that Compose(double, addOne, square)(3) returns 49 (3*2=6, 6+1=7, 7*7=49).

#1 Pitfall: Closures capture variables by reference, not by value. In a loop, all closures may share the same variable unless you shadow it or pass it as a parameter (critical in pre-Go 1.22 code).

Defer, Panic, Recover

Functions & Control Flow • 2 days ago • rep 0

Quick Reminder

defer schedules a function call to run when the surrounding function returns (LIFO order). panic triggers a runtime error that unwinds the stack. recover catches a panic — it only works inside a defer. Use defer for cleanup (files, locks, connections). Panic only for truly unrecoverable situations.

Key Pattern

// Defer for cleanup (LIFO order)
func processFile(path string) error {
    f, err := os.Open(path)
    if err != nil {
        return err
    }
    defer f.Close() // guaranteed cleanup

    // ... process file
    return nil
}

// Recover from panic in deferred function
func safeDiv(a, b int) (result int, err error) {
    defer func() {
        if r := recover(); r != nil {
            err = fmt.Errorf("recovered: %v", r)
        }
    }()
    return a / b, nil
}
Challenge Exercise

Write a SafeExecute(fn func()) (panicked bool, value interface{}) function that runs fn and returns whether it panicked plus the panic value. Use it to safely execute a series of functions, collecting which ones panicked. Print a summary like "3/5 functions completed successfully."

#1 Pitfall: Deferred function arguments are evaluated immediately, not at execution time. defer fmt.Println(x) captures the current value of x. To capture the final value, use a closure: defer func() { fmt.Println(x) }().

Init Functions

Functions & Control Flow • 2 days ago • rep 0

Quick Reminder

init() functions run automatically before main(), in dependency order (imported packages first). A single file can have multiple init() functions. They take no arguments and return nothing. Use them for setup that cannot be done with variable declarations. Prefer explicit initialization when possible for testability.

Key Pattern

package config

import "os"

var DatabaseURL string
var Debug bool

func init() {
    DatabaseURL = os.Getenv("DATABASE_URL")
    if DatabaseURL == "" {
        DatabaseURL = "localhost:5432"
    }
    Debug = os.Getenv("DEBUG") == "true"
}

// Multiple inits run in source order
func init() {
    // validate configuration
    if DatabaseURL == "" {
        panic("DATABASE_URL must not be empty after init")
    }
}

#1 Pitfall: Relying on init() for heavy or fallible setup makes testing hard because you cannot control when or whether it runs. Prefer explicit Setup() functions that return errors.

Variadic Functions

Functions & Control Flow • 2 days ago • rep 0

Quick Reminder

Variadic functions accept a variable number of arguments using ... syntax. The variadic parameter is received as a slice. Use s... to expand a slice into variadic arguments. Only the last parameter can be variadic. Common in fmt.Println, append, and option patterns.

Key Pattern

// Basic variadic
func Sum(nums ...int) int {
    total := 0
    for _, n := range nums {
        total += n
    }
    return total
}

// Expanding a slice with ...
values := []int{1, 2, 3}
Sum(values...) // expands slice into individual args

// Functional options pattern
type Server struct { port int; timeout time.Duration }
type Option func(*Server)
func WithPort(p int) Option { return func(s *Server) { s.port = p } }
func NewServer(opts ...Option) *Server {
    s := &Server{port: 8080}
    for _, opt := range opts {
        opt(s)
    }
    return s
}
Challenge Exercise

Write a Pipeline(input string, transforms ...func(string) string) string function that applies transforms left-to-right. Create transforms: ToUpper, TrimSpace, AddBrackets (wraps in []). Verify Pipeline(" hello ", TrimSpace, ToUpper, AddBrackets) returns "[HELLO]".

#1 Pitfall: Passing a slice to a variadic function without ... — Sum(mySlice) fails to compile. You must write Sum(mySlice...) to expand the slice.

Type Parameters (Generics)

Functions & Control Flow • 2 days ago • rep 0

Quick Reminder

Go 1.18+ supports generics via type parameters in square brackets. Constraints restrict which types are allowed: any for all types, comparable for types supporting ==/!=, and custom interface constraints for operator requirements. Type inference often allows omitting explicit type arguments.

Key Pattern

// Generic function with constraint
func Filter[T any](s []T, predicate func(T) bool) []T {
    var result []T
    for _, v := range s {
        if predicate(v) {
            result = append(result, v)
        }
    }
    return result
}

// Generic type
type Stack[T any] struct {
    items []T
}
func (s *Stack[T]) Push(val T) { s.items = append(s.items, val) }
func (s *Stack[T]) Pop() (T, bool) {
    if len(s.items) == 0 {
        var zero T
        return zero, false
    }
    val := s.items[len(s.items)-1]
    s.items = s.items[:len(s.items)-1]
    return val, true
}
Challenge Exercise

Write a generic Set[T comparable] type with methods Add(T), Remove(T), Contains(T) bool, Union(Set[T]) Set[T], and Intersection(Set[T]) Set[T]. Use a map[T]struct{} internally. Test with both int and string sets.

#1 Pitfall: Trying to use operators like < or + with any constraint — you need cmp.Ordered or a custom constraint that includes the required types. The any constraint only supports assignment and interface method calls.

Concurrency & Standard Library — 6 Concepts

Concurrency Patterns

Concurrency • New

Go's concurrency primitives (goroutines, channels, select) combine into powerful, reusable patterns. These patterns solve real-world problems like distributing work, collecting results, limiting throughput, and managing resource access. Mastering them is what separates Go beginners from proficient Go programmers.

Why It Matters

Production systems need more than raw goroutines and channels. You need structured ways to distribute work across cores, collect results safely, limit concurrency to avoid overwhelming downstream services, and control throughput. These patterns are the building blocks of every serious Go server, CLI tool, and data pipeline.

Fan-out/Fan-in: Distribute Work, Collect Results

package main

import (
    "fmt"
    "sync"
)

// producer sends jobs to the jobs channel
func producer(jobs chan<- int, count int) {
    for i := 1; i <= count; i++ {
        jobs <- i
    }
    close(jobs)
}

// worker reads from jobs, writes results
func worker(id int, jobs <-chan int, results chan<- string, wg *sync.WaitGroup) {
    defer wg.Done()
    for job := range jobs {
        result := fmt.Sprintf("worker %d processed job %d (squared=%d)", id, job, job*job)
        results <- result
    }
}

func main() {
    const numWorkers = 3
    const numJobs = 9

    jobs := make(chan int, numJobs)
    results := make(chan string, numJobs)

    // Fan-out: launch multiple workers reading from the same channel
    var wg sync.WaitGroup
    for w := 1; w <= numWorkers; w++ {
        wg.Add(1)
        go worker(w, jobs, results, &wg)
    }

    // Producer sends jobs
    go producer(jobs, numJobs)

    // Close results when all workers are done
    go func() {
        wg.Wait()
        close(results)
    }()

    // Fan-in: collect all results from the single results channel
    for r := range results {
        fmt.Println(r)
    }
    fmt.Println("All done!")
}
Output (order varies)
worker 3 processed job 2 (squared=4) worker 1 processed job 1 (squared=1) worker 2 processed job 3 (squared=9) worker 1 processed job 5 (squared=25) worker 3 processed job 4 (squared=16) worker 2 processed job 6 (squared=36) worker 1 processed job 7 (squared=49) worker 3 processed job 8 (squared=64) worker 2 processed job 9 (squared=81) All done!

Pipeline: Chain Stages with Channels

package main

import "fmt"

// Stage 1: source generates numbers
func source(nums ...int) <-chan int {
    out := make(chan int)
    go func() {
        for _, n := range nums {
            out <- n
        }
        close(out)
    }()
    return out
}

// Stage 2: transform doubles each number
func transform(in <-chan int) <-chan int {
    out := make(chan int)
    go func() {
        for n := range in {
            out <- n * 2
        }
        close(out)
    }()
    return out
}

// Stage 3: sink formats as strings
func sink(in <-chan int) <-chan string {
    out := make(chan string)
    go func() {
        for n := range in {
            out <- fmt.Sprintf("result: %d", n)
        }
        close(out)
    }()
    return out
}

func main() {
    // Chain: source -> transform -> sink
    numbers := source(1, 2, 3, 4, 5)
    doubled := transform(numbers)
    formatted := sink(doubled)

    for s := range formatted {
        fmt.Println(s)
    }
}
Output
result: 2 result: 4 result: 6 result: 8 result: 10

Worker Pool: Fixed Number of Goroutines

package main

import (
    "fmt"
    "math/rand"
    "sync"
    "time"
)

type Job struct {
    ID   int
    Data string
}

type Result struct {
    JobID    int
    Output   string
    Duration time.Duration
}

func workerPool(numWorkers int, jobs []Job) []Result {
    jobCh := make(chan Job, len(jobs))
    resultCh := make(chan Result, len(jobs))

    // Start fixed number of workers
    var wg sync.WaitGroup
    for w := 0; w < numWorkers; w++ {
        wg.Add(1)
        go func(id int) {
            defer wg.Done()
            for job := range jobCh {
                start := time.Now()
                // Simulate variable-length work
                time.Sleep(time.Duration(rand.Intn(50)) * time.Millisecond)
                resultCh <- Result{
                    JobID:    job.ID,
                    Output:   fmt.Sprintf("worker-%d processed %q", id, job.Data),
                    Duration: time.Since(start),
                }
            }
        }(w)
    }

    // Send all jobs
    for _, j := range jobs {
        jobCh <- j
    }
    close(jobCh)

    // Close results when workers finish
    go func() {
        wg.Wait()
        close(resultCh)
    }()

    // Collect results
    var results []Result
    for r := range resultCh {
        results = append(results, r)
    }
    return results
}

func main() {
    jobs := []Job{
        {1, "alpha"}, {2, "beta"}, {3, "gamma"},
        {4, "delta"}, {5, "epsilon"}, {6, "zeta"},
    }

    results := workerPool(3, jobs)
    for _, r := range results {
        fmt.Printf("Job %d: %s (%v)\n", r.JobID, r.Output, r.Duration.Round(time.Millisecond))
    }
}
Output (order and durations vary)
Job 1: worker-0 processed "alpha" (12ms) Job 2: worker-1 processed "beta" (35ms) Job 3: worker-2 processed "gamma" (8ms) Job 4: worker-2 processed "delta" (27ms) Job 5: worker-0 processed "epsilon" (41ms) Job 6: worker-1 processed "zeta" (19ms)

Rate Limiting with time.Ticker

package main

import (
    "fmt"
    "time"
)

func main() {
    // Steady rate limiter: 1 event per 200ms
    requests := []string{"req-1", "req-2", "req-3", "req-4", "req-5"}

    // Steady rate: use a ticker
    limiter := time.NewTicker(200 * time.Millisecond)
    defer limiter.Stop()

    start := time.Now()
    for _, req := range requests {
        <-limiter.C // wait for the next tick
        fmt.Printf("[%6dms] handled %s\n",
            time.Since(start).Milliseconds(), req)
    }

    fmt.Println()

    // Bursty rate limiter: allow 3 immediate, then 1/200ms
    bursty := make(chan time.Time, 3)
    // Pre-fill burst capacity
    for i := 0; i < 3; i++ {
        bursty <- time.Now()
    }
    // Refill at steady rate
    go func() {
        for t := range time.NewTicker(200 * time.Millisecond).C {
            bursty <- t
        }
    }()

    moreRequests := []string{"burst-1", "burst-2", "burst-3", "burst-4", "burst-5"}
    start = time.Now()
    for _, req := range moreRequests {
        <-bursty
        fmt.Printf("[%6dms] handled %s\n",
            time.Since(start).Milliseconds(), req)
    }
}
Output (times approximate)
[ 200ms] handled req-1 [ 400ms] handled req-2 [ 600ms] handled req-3 [ 800ms] handled req-4 [ 1000ms] handled req-5 [ 0ms] handled burst-1 [ 0ms] handled burst-2 [ 0ms] handled burst-3 [ 200ms] handled burst-4 [ 400ms] handled burst-5

Semaphore: Buffered Channel as Counting Semaphore

package main

import (
    "fmt"
    "sync"
    "time"
)

type Semaphore chan struct{}

func NewSemaphore(max int) Semaphore {
    return make(Semaphore, max)
}

func (s Semaphore) Acquire() {
    s <- struct{}{} // blocks when buffer is full
}

func (s Semaphore) Release() {
    <-s // frees a slot
}

func main() {
    sem := NewSemaphore(3) // max 3 concurrent operations
    var wg sync.WaitGroup

    for i := 1; i <= 8; i++ {
        wg.Add(1)
        go func(id int) {
            defer wg.Done()

            sem.Acquire()
            defer sem.Release()

            fmt.Printf("[%s] task %d started\n",
                time.Now().Format("15:04:05.000"), id)
            time.Sleep(300 * time.Millisecond)
            fmt.Printf("[%s] task %d done\n",
                time.Now().Format("15:04:05.000"), id)
        }(i)
    }

    wg.Wait()
    fmt.Println("All tasks complete")
}

Common Mistakes

Do: Close results channel after all workers finish
go func() {
    wg.Wait()
    close(results)
}()
for r := range results { ... }
Don't: Close results before workers are done
close(results) // workers may still be sending!
// panic: send on closed channel
for r := range results { ... }
Do: Use buffered channels for known work sizes
jobs := make(chan Job, len(allJobs))
// prevents blocking when sending jobs
Don't: Use unbuffered when producer is synchronous
jobs := make(chan Job) // blocks on each send
// deadlock if no worker is reading yet!

Key Rules

  • Fan-out: multiple goroutines reading from the same channel distributes work automatically
  • Fan-in: multiple goroutines writing to the same channel collects results into one stream
  • Pipelines: each stage is a function that takes <-chan T and returns <-chan U
  • Worker pools: launch N goroutines reading from a jobs channel for bounded concurrency
  • A buffered channel of struct{} makes an efficient counting semaphore
  • Rate limiting: time.Ticker for steady rate; pre-filled buffered channel for burst + steady
Connection: These patterns build on Goroutines, Channels, and Select Statement from Day 3. The worker pool pattern extends into errgroup (next) for error-aware concurrency. The semaphore pattern is what errgroup.SetLimit uses internally. Pipelines compose well with the io Package (later today) for streaming data processing.

errgroup

Concurrency • New

The golang.org/x/sync/errgroup package provides a higher-level abstraction for managing groups of goroutines that can fail. It combines sync.WaitGroup functionality with error propagation and context-based cancellation, solving the most common concurrent error-handling pattern in one clean API.

Why It Matters

sync.WaitGroup has a critical gap: it cannot propagate errors. When launching multiple goroutines, you need to know if any failed and typically want to cancel remaining work when one fails. errgroup solves this elegantly, making it the go-to choice for production concurrent code.

Basic errgroup Usage

package main

import (
    "fmt"
    "math/rand"
    "time"

    "golang.org/x/sync/errgroup"
)

func fetchURL(url string) (string, error) {
    // Simulate network latency
    time.Sleep(time.Duration(rand.Intn(100)) * time.Millisecond)
    if url == "https://bad.example.com" {
        return "", fmt.Errorf("failed to fetch %s: connection refused", url)
    }
    return fmt.Sprintf("content from %s", url), nil
}

func main() {
    urls := []string{
        "https://api.example.com/users",
        "https://api.example.com/posts",
        "https://api.example.com/comments",
    }

    var g errgroup.Group
    results := make([]string, len(urls))

    for i, url := range urls {
        i, url := i, url // capture for goroutine
        g.Go(func() error {
            content, err := fetchURL(url)
            if err != nil {
                return err
            }
            results[i] = content // safe: each goroutine writes to its own index
            return nil
        })
    }

    // Wait for all goroutines; returns the first error (or nil)
    if err := g.Wait(); err != nil {
        fmt.Println("Error:", err)
    } else {
        for _, r := range results {
            fmt.Println(r)
        }
    }
}
Output
content from https://api.example.com/users content from https://api.example.com/posts content from https://api.example.com/comments

errgroup with Context Cancellation

package main

import (
    "context"
    "fmt"
    "time"

    "golang.org/x/sync/errgroup"
)

func fetchData(ctx context.Context, source string, delay time.Duration) (string, error) {
    select {
    case <-time.After(delay):
        return fmt.Sprintf("data from %s", source), nil
    case <-ctx.Done():
        return "", fmt.Errorf("%s cancelled: %w", source, ctx.Err())
    }
}

func main() {
    // WithContext: if any goroutine returns an error, ctx is cancelled
    g, ctx := errgroup.WithContext(context.Background())

    sources := map[string]time.Duration{
        "database":    50 * time.Millisecond,
        "cache":       20 * time.Millisecond,
        "bad-service": 0, // will fail immediately
    }

    results := make(map[string]string)
    var mu sync.Mutex // protect the map — or use a slice

    for name, delay := range sources {
        name, delay := name, delay
        g.Go(func() error {
            if name == "bad-service" {
                return fmt.Errorf("bad-service: connection timeout")
            }
            data, err := fetchData(ctx, name, delay)
            if err != nil {
                return err
            }
            mu.Lock()
            results[name] = data
            mu.Unlock()
            return nil
        })
    }

    if err := g.Wait(); err != nil {
        fmt.Println("Pipeline failed:", err)
        fmt.Println("Partial results collected:", len(results))
    }
}

Note: Add import "sync" to the imports. The context cancellation means goroutines still running when one fails will see ctx.Done() fire and can exit early.

Output
Pipeline failed: bad-service: connection timeout Partial results collected: 1

errgroup.SetLimit for Bounded Concurrency

package main

import (
    "fmt"
    "time"

    "golang.org/x/sync/errgroup"
)

func processItem(id int) error {
    fmt.Printf("[%s] processing item %d\n",
        time.Now().Format("15:04:05.000"), id)
    time.Sleep(200 * time.Millisecond) // simulate work
    return nil
}

func main() {
    var g errgroup.Group
    g.SetLimit(3) // at most 3 goroutines run concurrently

    for i := 1; i <= 9; i++ {
        i := i
        g.Go(func() error { // blocks if 3 already running
            return processItem(i)
        })
    }

    if err := g.Wait(); err != nil {
        fmt.Println("Error:", err)
    } else {
        fmt.Println("All items processed!")
    }
}

Comparison: sync.WaitGroup vs errgroup

package main

import (
    "fmt"
    "sync"

    "golang.org/x/sync/errgroup"
)

// With sync.WaitGroup — manual error handling, no cancellation
func withWaitGroup() error {
    var wg sync.WaitGroup
    errCh := make(chan error, 3) // must size correctly

    for i := 0; i < 3; i++ {
        wg.Add(1)
        go func(id int) {
            defer wg.Done()
            if id == 1 {
                errCh <- fmt.Errorf("task %d failed", id)
                return
            }
            fmt.Printf("WG: task %d ok\n", id)
        }(i)
    }

    wg.Wait()
    close(errCh)

    for err := range errCh {
        return err // return first error
    }
    return nil
}

// With errgroup — clean and automatic
func withErrgroup() error {
    var g errgroup.Group

    for i := 0; i < 3; i++ {
        i := i
        g.Go(func() error {
            if i == 1 {
                return fmt.Errorf("task %d failed", i)
            }
            fmt.Printf("EG: task %d ok\n", i)
            return nil
        })
    }

    return g.Wait() // returns first error automatically
}

func main() {
    fmt.Println("--- WaitGroup ---")
    if err := withWaitGroup(); err != nil {
        fmt.Println("WG error:", err)
    }

    fmt.Println("\n--- errgroup ---")
    if err := withErrgroup(); err != nil {
        fmt.Println("EG error:", err)
    }
}
Output (order varies)
--- WaitGroup --- WG: task 0 ok WG: task 2 ok WG error: task 1 failed --- errgroup --- EG: task 0 ok EG: task 2 ok EG error: task 1 failed

Common Mistakes

Do: Use WithContext for cancellation
g, ctx := errgroup.WithContext(ctx)
g.Go(func() error {
    // check ctx.Done() for early exit
    select {
    case <-ctx.Done():
        return ctx.Err()
    default:
        return doWork()
    }
})
Don't: Ignore the context from WithContext
g, _ := errgroup.WithContext(ctx)
g.Go(func() error {
    return doExpensiveWork()
    // ignores cancellation, wastes resources
})
Do: Capture loop variables
for i, url := range urls {
    i, url := i, url // shadow!
    g.Go(func() error {
        results[i] = fetch(url)
        return nil
    })
}
Don't: Share loop variable (pre-Go 1.22)
for i, url := range urls {
    g.Go(func() error {
        results[i] = fetch(url) // race!
        return nil
    })
}

Key Rules

  • g.Go() launches a goroutine; g.Wait() blocks until all finish and returns the first non-nil error
  • errgroup.WithContext cancels the derived context on the first error — remaining goroutines should check ctx.Done()
  • g.SetLimit(n) bounds concurrent goroutines — g.Go blocks when the limit is reached
  • Each goroutine writes to its own slice index (safe) or uses a mutex for shared maps
  • Only the first error is returned; if you need all errors, collect them with a mutex
Connection: errgroup is the production-ready evolution of the Concurrency Patterns (worker pool, fan-out) combined with the Context package's cancellation. It replaces the manual sync.WaitGroup + error channel pattern from sync Package. The SetLimit method is equivalent to the semaphore pattern.

Standard Library Mastery • New

The fmt package implements formatted I/O with functions analogous to C's printf and scanf. It handles printing, string formatting, error creation, and scanning. Every Go program uses it, and understanding its verbs and interfaces unlocks powerful custom formatting.

package main

func main() {

    fmt.Printf("[%10d]\n", 42)      // right-aligned

Sprintf, Fprintf, and Errorf

package main

import (
    "errors"
    "fmt"
    "os"
    "strings"
)

var ErrNotFound = errors.New("not found")

func findUser(id int) (string, error) {
    if id <= 0 {
        // %w wraps the error — allows errors.Is/As to unwrap
        return "", fmt.Errorf("findUser(id=%d): %w", id, ErrNotFound)
    }
    return fmt.Sprintf("User_%d", id), nil
}

func main() {
    // Sprintf: format to string
    greeting := fmt.Sprintf("Hello, %s! You have %d messages.", "Alice", 3)
    fmt.Println(greeting)

    // Fprintf: format to any io.Writer
    fmt.Fprintf(os.Stderr, "Warning: %s\n", "disk almost full")

    // Write to a strings.Builder
    var b strings.Builder
    fmt.Fprintf(&b, "(%d, %d)", 10, 20)
    fmt.Fprintf(&b, " + (%d, %d)", 30, 40)
    fmt.Println(b.String())

    // Errorf with %w for error wrapping
    _, err := findUser(-1)
    if err != nil {
        fmt.Println("Error:", err)
        fmt.Println("Is ErrNotFound?", errors.Is(err, ErrNotFound))
    }
}

Hello, Alice! You have 3 messages. Warning: disk almost full (10, 20) + (30, 40) Error: findUser(id=-1): not found Is ErrNotFound? true

Common Mistakes

Do: Use %w to wrap errors
return fmt.Errorf("open config: %w", err)
// errors.Is and errors.As work through the chain
Don't: Use %v for error wrapping
return fmt.Errorf("open config: %v", err)
// loses the error chain! errors.Is returns false
Do: Use %+v for struct debugging
fmt.Printf("config: %+v\n", cfg)
// shows field names: {Port:8080 Host:localhost}
Don't: Rely on %v for debugging structs
fmt.Printf("config: %v\n", cfg)
// just shows values: {8080 localhost}
// which field is which?

Key Rules

  • %v is the default format; %+v adds field names; %#v gives Go syntax
  • %T prints the type — invaluable for debugging interfaces
  • Implement String() string (the Stringer interface) for human-readable output of your types
  • Use %w in Errorf to wrap errors; %v loses the error chain
  • Width and precision: %10d (width 10), %-10s (left-align), %.2f (2 decimal places), %05d (zero-pad)
Connection: The Stringer interface connects to Interfaces (implicit satisfaction). Fprintf writes to any io.Writer, linking directly to the io Package (next). Errorf with %w is the primary tool for the Error Handling pattern. Width/precision formatting is essential for CLI tools using os & filepath.

io Package

Standard Library Mastery • New

The io package defines the fundamental I/O interfaces: Reader and Writer. These two interfaces are the backbone of Go's I/O model — files, network connections, HTTP bodies, buffers, compressors, and encryptors all implement them. Understanding io.Reader and io.Writer unlocks composable, efficient streaming.

Why It Matters

Nearly every Go program reads or writes data. The io.Reader/io.Writer abstraction means you can write code that works with files, network streams, in-memory buffers, or compressed data — all without changing a line. Composition of readers and writers is how Go achieves powerful data pipelines without heavy frameworks.

io.LimitReader and io.Pipe

package main

import (
    "fmt"
    "io"
    "strings"
)

func main() {
    // LimitReader: restrict how many bytes can be read
    src := strings.NewReader("This is a very long string that we want to limit")
    limited := io.LimitReader(src, 10)

    data, _ := io.ReadAll(limited)
    fmt.Printf("Limited read: %q (%d bytes)\n", data, len(data))

    // Pipe: in-memory pipe connecting a writer to a reader
    pr, pw := io.Pipe()

    // Writer goroutine
    go func() {
        defer pw.Close()
        for i := 1; i <= 3; i++ {
            fmt.Fprintf(pw, "message %d\n", i)
        }
    }()

    // Reader (main goroutine)
    received, _ := io.ReadAll(pr)
    fmt.Print("Pipe received:\n", string(received))
}
Output
Limited read: "This is a " (10 bytes) Pipe received: message 1 message 2 message 3

Composing Readers: A Counting Reader

package main

import (
    "fmt"
    "io"
    "strings"
)

// CountingReader wraps an io.Reader and counts bytes read
type CountingReader struct {
    reader    io.Reader
    BytesRead int64
}

func NewCountingReader(r io.Reader) *CountingReader {
    return &CountingReader{reader: r}
}

func (cr *CountingReader) Read(p []byte) (int, error) {
    n, err := cr.reader.Read(p)
    cr.BytesRead += int64(n)
    return n, err
}

func main() {
    src := strings.NewReader("Hello, composed readers! This is efficient.")
    counter := NewCountingReader(src)

    // Use the counting reader as a regular reader
    data, _ := io.ReadAll(counter)
    fmt.Printf("Data: %q\n", string(data))
    fmt.Printf("Total bytes read: %d\n", counter.BytesRead)
}
Output
Data: "Hello, composed readers! This is efficient." Total bytes read: 44

Common Mistakes

Do: Check for io.EOF properly
n, err := r.Read(buf)
if n > 0 {
    process(buf[:n]) // process data first!
}
if err == io.EOF {
    break // then check for EOF
}
Don't: Ignore data on EOF read
n, err := r.Read(buf)
if err != nil {
    break // may discard last chunk!
}
process(buf[:n])
Do: Use io.ReadAll for small data
data, err := io.ReadAll(resp.Body)
// appropriate for known-small responses
Don't: ReadAll unbounded data
data, _ := io.ReadAll(untrustedReader)
// could consume all memory!
// Use io.LimitReader first

Key Rules

  • io.Reader has one method: Read(p []byte) (n int, err error). Always process n bytes before checking err
  • io.Writer has one method: Write(p []byte) (n int, err error). An error means not all bytes were written
  • io.Copy streams data efficiently without buffering the entire content in memory
  • Compose readers by wrapping: LimitReader, TeeReader, or your own decorator
  • io.Pipe creates a synchronous in-memory pipe — a write blocks until a corresponding read occurs
  • Always close writers/pipes when done to signal EOF to the reader side
Connection: io.Reader/io.Writer are the interfaces that os & filepath (next) builds on — files implement both. fmt.Fprintf writes to any io.Writer, connecting to the fmt Package. encoding/json (later) uses NewEncoder(w io.Writer) and NewDecoder(r io.Reader) for streaming JSON. Custom readers compose the same way Interfaces compose.

os & filepath

Standard Library Mastery • New

The os package provides platform-independent operating system functionality: file I/O, environment variables, process management. The path/filepath package handles file path manipulation in a way that works across operating systems. Together they are essential for any program that interacts with the filesystem.

Why It Matters

File operations, directory management, and path handling are fundamental to CLIs, build tools, configuration loaders, log rotators, and data processors. Go's standard library makes these operations safe and portable. Understanding the difference between one-shot I/O and streaming I/O is key to writing efficient programs.

File Operations: Open, Create, ReadFile, WriteFile

package main

import (
    "fmt"
    "os"
)

func main() {
    // One-shot write (Go 1.16+)
    content := []byte("Hello from Go!\nLine 2\nLine 3\n")
    err := os.WriteFile("/tmp/go-demo.txt", content, 0644)
    if err != nil {
        fmt.Println("write error:", err)
        return
    }
    fmt.Println("File written successfully")

    // One-shot read (Go 1.16+)
    data, err := os.ReadFile("/tmp/go-demo.txt")
    if err != nil {
        fmt.Println("read error:", err)
        return
    }
    fmt.Printf("Read %d bytes:\n%s", len(data), data)

    // Streaming read with os.Open
    f, err := os.Open("/tmp/go-demo.txt")
    if err != nil {
        fmt.Println("open error:", err)
        return
    }
    defer f.Close()

    buf := make([]byte, 10)
    n, _ := f.Read(buf)
    fmt.Printf("\nFirst %d bytes: %q\n", n, buf[:n])

    // File info
    info, _ := f.Stat()
    fmt.Printf("Name: %s, Size: %d bytes, Mode: %s\n",
        info.Name(), info.Size(), info.Mode())

    // Cleanup
    os.Remove("/tmp/go-demo.txt")
}

package main

import (
    "fmt"
    "os"
    "path/filepath"
)

func main() {
    // Path manipulation
    p := filepath.Join("home", "user", "docs", "file.tar.gz")

    fmt.Println("Dir:    ", filepath.Dir(p))
    fmt.Println("Base:   ", filepath.Base(p))
    fmt.Println("Ext:    ", filepath.Ext(p))

    // Clean normalizes paths
    messy := "/home/user/../user/./docs//file.txt"

    abs, _ := filepath.Abs(".")

    os.MkdirAll("/tmp/walk-demo/src", 0755)
    os.WriteFile("/tmp/walk-demo/main.go", []byte("package main"), 0644)
    os.WriteFile("/tmp/walk-demo/src/lib.go", []byte("package src"), 0644)
    os.WriteFile("/tmp/walk-demo/README.md", []byte("# Demo"), 0644)

    fmt.Println("\nWalking /tmp/walk-demo:")
    filepath.WalkDir("/tmp/walk-demo", func(path string, d os.DirEntry, err error) error {
        if err != nil {
            return err
        }
        rel, _ := filepath.Rel("/tmp/walk-demo", path)
        if d.IsDir() {
            fmt.Printf("  [DIR]  %s\n", rel)
        } else {
            fmt.Printf("  [FILE] %s (ext: %s)\n", rel, filepath.Ext(rel))
        }
        return nil
    })

    // Glob: pattern matching
    matches, _ := filepath.Glob("/tmp/walk-demo/*.go")
    fmt.Println("\n*.go files:", matches)

    os.RemoveAll("/tmp/walk-demo")
}

// breaks on Windows!
Do: Check errors from file operations
data, err := os.ReadFile(path)
if err != nil {
    return fmt.Errorf("read %s: %w", path, err)
}
Don't: Ignore file operation errors
data, _ := os.ReadFile(path)
// data may be nil — crash ahead!

Key Rules

  • os.ReadFile/os.WriteFile for small, one-shot file I/O (Go 1.16+)
  • os.Open/os.Create/os.OpenFile for streaming or fine-grained control — always defer f.Close()
  • Use filepath.Join for portable path construction — never concatenate with "/"
  • filepath.WalkDir (Go 1.16+) is faster than filepath.Walk and should be preferred
  • Check existence with os.Stat + os.IsNotExist(err), not by trying to open the file
  • Use os.LookupEnv to distinguish "not set" from "set to empty string"
Connection: os.File implements both io.Reader and io.Writer, connecting directly to the io Package. File paths parsed by filepath are used by os.Open and os.Create. Environment variables loaded by os.Getenv often feed into encoding/json (next) for config parsing. The Error Handling patterns (sentinel errors, wrapping) are essential for robust file operations.

encoding/json

Standard Library Mastery • New

The encoding/json package provides efficient JSON encoding and decoding. It maps between Go structs and JSON using struct tags, supports custom marshal/unmarshal behavior, and offers both in-memory (Marshal/Unmarshal) and streaming (Encoder/Decoder) APIs.

Why It Matters

JSON is the lingua franca of web APIs, configuration files, and data interchange. Every Go web service, CLI tool that talks to REST APIs, or program that reads JSON configs needs this package. Understanding struct tags, custom marshalers, and streaming decode is essential for production Go code.

Marshal and Unmarshal: Basic Usage

package main

import (
    "encoding/json"
    "fmt"
)

type User struct {
    Name    string `json:"name"`
    Email   string `json:"email"`
    Age     int    `json:"age"`
    Active  bool   `json:"active"`
}

func main() {
    // Marshal: struct -> JSON
    user := User{
        Name:   "Alice",
        Email:  "alice@example.com",
        Age:    30,
        Active: true,
    }

    data, err := json.Marshal(user)
    if err != nil {
        fmt.Println("marshal error:", err)
        return
    }
    fmt.Println("JSON:", string(data))

    // Pretty print
    pretty, _ := json.MarshalIndent(user, "", "  ")
    fmt.Println("\nPretty:\n" + string(pretty))

    // Unmarshal: JSON -> struct
    jsonStr := `{"name":"Bob","email":"bob@example.com","age":25,"active":false}`
    var bob User
    err = json.Unmarshal([]byte(jsonStr), &bob)
    if err != nil {
        fmt.Println("unmarshal error:", err)
        return
    }
    fmt.Printf("\nParsed: %+v\n", bob)
}
Output
JSON: {"name":"Alice","email":"alice@example.com","age":30,"active":true} Pretty: { "name": "Alice", "email": "alice@example.com", "age": 30, "active": true } Parsed: {Name:Bob Email:bob@example.com Age:25 Active:false}

Struct Tags: omitempty, "-", and Naming

package main

import (
    "encoding/json"
    "fmt"
)

type Product struct {
    ID       int      `json:"id"`
    Name     string   `json:"name"`
    Price    float64  `json:"price"`
    Discount float64  `json:"discount,omitempty"` // omit if zero value
    Internal string   `json:"-"`                   // never include in JSON
    Tags     []string `json:"tags,omitempty"`      // omit if nil/empty
}

func main() {
    // With all fields set
    p1 := Product{
        ID: 1, Name: "Widget", Price: 9.99,
        Discount: 2.50, Internal: "secret", Tags: []string{"sale", "new"},
    }
    j1, _ := json.MarshalIndent(p1, "", "  ")
    fmt.Println("Full product:")
    fmt.Println(string(j1))

    // With zero-value fields (omitempty in action)
    p2 := Product{ID: 2, Name: "Gadget", Price: 19.99}
    j2, _ := json.MarshalIndent(p2, "", "  ")
    fmt.Println("\nMinimal product:")
    fmt.Println(string(j2))
}
Output
Full product: { "id": 1, "name": "Widget", "price": 9.99, "discount": 2.5, "tags": [ "sale", "new" ] } Minimal product: { "id": 2, "name": "Gadget", "price": 19.99 }

Custom MarshalJSON and UnmarshalJSON

package main

import (
    "encoding/json"
    "fmt"
    "time"
)

type Event struct {
    Name string    `json:"name"`
    Date EventDate `json:"date"`
}

// Custom type with special JSON format
type EventDate struct {
    time.Time
}

func (d EventDate) MarshalJSON() ([]byte, error) {
    formatted := d.Format("2006-01-02")
    return json.Marshal(formatted) // wrap in quotes
}

func (d *EventDate) UnmarshalJSON(data []byte) error {
    var s string
    if err := json.Unmarshal(data, &s); err != nil {
        return err
    }
    t, err := time.Parse("2006-01-02", s)
    if err != nil {
        return err
    }
    d.Time = t
    return nil
}

func main() {
    // Marshal with custom date format
    event := Event{
        Name: "Go Conference",
        Date: EventDate{time.Date(2026, 6, 15, 0, 0, 0, 0, time.UTC)},
    }

    data, _ := json.MarshalIndent(event, "", "  ")
    fmt.Println("Marshaled:")
    fmt.Println(string(data))

    // Unmarshal with custom date format
    jsonStr := `{"name":"Workshop","date":"2026-07-20"}`
    var parsed Event
    json.Unmarshal([]byte(jsonStr), &parsed)
    fmt.Printf("\nUnmarshaled: %s on %s\n",
        parsed.Name, parsed.Date.Format("Monday, January 2, 2006"))
}
Output
Marshaled: { "name": "Go Conference", "date": "2026-06-15" } Unmarshaled: Workshop on Monday, July 20, 2026

json.RawMessage: Delay Parsing

package main

import (
    "encoding/json"
    "fmt"
)

type Envelope struct {
    Type string          `json:"type"`
    Data json.RawMessage `json:"data"` // parse later based on Type
}

type UserData struct {
    Name  string `json:"name"`
    Email string `json:"email"`
}

type OrderData struct {
    OrderID int     `json:"order_id"`
    Total   float64 `json:"total"`
}

func main() {
    messages := []string{
        `{"type":"user","data":{"name":"Alice","email":"a@b.com"}}`,
        `{"type":"order","data":{"order_id":42,"total":99.95}}`,
    }

    for _, msg := range messages {
        var env Envelope
        json.Unmarshal([]byte(msg), &env)

        switch env.Type {
        case "user":
            var u UserData
            json.Unmarshal(env.Data, &u)
            fmt.Printf("User: %s (%s)\n", u.Name, u.Email)
        case "order":
            var o OrderData
            json.Unmarshal(env.Data, &o)
            fmt.Printf("Order: #%d total=$%.2f\n", o.OrderID, o.Total)
        }
    }
}
Output
User: Alice (a@b.com) Order: #42 total=$99.95

Streaming with json.Encoder and json.Decoder

package main

import (
    "bytes"
    "encoding/json"
    "fmt"
    "strings"
)

type LogEntry struct {
    Level   string `json:"level"`
    Message string `json:"message"`
}

func main() {
    // Encoder: stream JSON objects to a writer
    var buf bytes.Buffer
    enc := json.NewEncoder(&buf)

    entries := []LogEntry{
        {"info", "server started"},
        {"warn", "high memory usage"},
        {"error", "connection lost"},
    }

    for _, entry := range entries {
        enc.Encode(entry) // writes JSON + newline
    }

    fmt.Println("Encoded (newline-delimited JSON):")
    fmt.Print(buf.String())

    // Decoder: stream JSON objects from a reader
    fmt.Println("\nDecoded:")
    dec := json.NewDecoder(strings.NewReader(buf.String()))
    for dec.More() {
        var entry LogEntry
        if err := dec.Decode(&entry); err != nil {
            fmt.Println("decode error:", err)
            break
        }
        fmt.Printf("  [%s] %s\n", entry.Level, entry.Message)
    }
}
Output
Encoded (newline-delimited JSON): {"level":"info","message":"server started"} {"level":"warn","message":"high memory usage"} {"level":"error","message":"connection lost"} Decoded: [info] server started [warn] high memory usage [error] connection lost

Dynamic JSON with map[string]interface{}

package main

import (
    "encoding/json"
    "fmt"
)

func main() {
    // Parse unknown JSON into a map
    raw := `{
        "name": "Alice",
        "age": 30,
        "scores": [95, 87, 92],
        "address": {"city": "Portland", "state": "OR"}
    }`

    var data map[string]interface{}
    json.Unmarshal([]byte(raw), &data)

    // Access fields (type assertions required)
    name := data["name"].(string)
    age := data["age"].(float64) // JSON numbers are float64!
    fmt.Printf("Name: %s, Age: %.0f\n", name, age)

    // Nested access
    addr := data["address"].(map[string]interface{})
    fmt.Printf("City: %s\n", addr["city"])

    // Array access
    scores := data["scores"].([]interface{})
    for i, s := range scores {
        fmt.Printf("Score %d: %.0f\n", i+1, s.(float64))
    }

    // json.Number for precise numbers
    dec := json.NewDecoder(strings.NewReader(`{"price": 19.99}`))
    dec.UseNumber()
    var m map[string]interface{}
    dec.Decode(&m)
    price, _ := m["price"].(json.Number).Float64()
    fmt.Printf("Price: %.2f\n", price)
}

Note: Add "strings" to the imports for the json.Number example.

Output
Name: Alice, Age: 30 City: Portland Score 1: 95 Score 2: 87 Score 3: 92 Price: 19.99

Common Mistakes

Do: Export fields for JSON
type User struct {
    Name  string `json:"name"`  // exported
    Email string `json:"email"` // exported
}
Don't: Use unexported fields
type User struct {
    name  string `json:"name"`  // unexported!
    email string `json:"email"` // silently ignored
}
Do: Use Decoder for streams/HTTP
dec := json.NewDecoder(resp.Body)
var result APIResponse
err := dec.Decode(&result)
// memory efficient: no intermediate []byte
Don't: ReadAll + Unmarshal for HTTP
data, _ := io.ReadAll(resp.Body) // buffered twice
var result APIResponse
json.Unmarshal(data, &result) // wastes memory

Key Rules

  • Struct fields must be exported (capitalized) to be visible to the JSON encoder/decoder
  • Use json:"name" tags to control JSON field names; omitempty to skip zero values; "-" to exclude
  • Implement MarshalJSON/UnmarshalJSON for custom serialization (dates, enums, etc.)
  • Use json.RawMessage to delay parsing when the structure depends on a discriminator field
  • Prefer json.NewDecoder/json.NewEncoder for streaming — avoid ReadAll + Unmarshal for HTTP bodies
  • JSON numbers decode as float64 in interface{} — use json.Number for precise handling
Connection: json.NewEncoder/json.NewDecoder work with the io Package's io.Writer/io.Reader interfaces. Struct tags relate to Structs & Methods. Custom marshalers use the Interfaces pattern (implicit satisfaction). json.RawMessage is powerful with the Error Handling pattern for progressive parsing with meaningful errors.
