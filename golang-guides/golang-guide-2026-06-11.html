Functional Options Pattern

Advanced Patterns • 3 days overdue • rep 0

Quick Reminder

Option functions that configure a struct, applied in the constructor. Define type Option func(*Config) and create WithXxx functions that return an Option. The constructor iterates over options and applies each one, allowing callers to customize only what they need while defaults handle the rest.

Key Pattern

type Option func(*Server)

func WithTimeout(d time.Duration) Option {
    return func(s *Server) { s.timeout = d }
}

func WithMaxConns(n int) Option {
    return func(s *Server) { s.maxConns = n }
}

func NewServer(addr string, opts ...Option) *Server {
    s := &Server{
        addr:     addr,
        timeout:  30 * time.Second, // default
        maxConns: 100,              // default
    }
    for _, opt := range opts {
        opt(s)
    }
    return s
}

// Usage: clean, self-documenting API
srv := NewServer(":8080",
    WithTimeout(10*time.Second),
    WithMaxConns(500),
)
Challenge Exercise

Create a NewHTTPClient constructor using the functional options pattern. Support options for WithBaseURL(string), WithRetries(int), and WithHeader(key, value string) (which should be additive — multiple calls add multiple headers). Defaults: no base URL, 0 retries, empty headers map.

#1 Pitfall: Forgetting to initialize the config struct with sensible defaults before applying options. If you skip defaults, zero-values (0 for ints, "" for strings) become the "configured" values when no option is provided, leading to subtle bugs like 0-second timeouts.

httptest

Testing & Tooling • 3 days overdue • rep 0

Quick Reminder

httptest.NewRecorder() captures handler responses without starting a server. httptest.NewServer(handler) starts a real local HTTP server for integration tests. The recorder approach is fast and deterministic; the server approach tests the full HTTP stack including middleware.

Key Pattern

func TestHealthHandler(t *testing.T) {
    req := httptest.NewRequest("GET", "/health", nil)
    w := httptest.NewRecorder()

    healthHandler(w, req)

    resp := w.Result()
    if resp.StatusCode != http.StatusOK {
        t.Errorf("got %d, want 200", resp.StatusCode)
    }

    body, _ := io.ReadAll(resp.Body)
    if string(body) != `{"status":"ok"}` {
        t.Errorf("unexpected body: %s", body)
    }
}

// Full server test
func TestAPI(t *testing.T) {
    srv := httptest.NewServer(myRouter())
    defer srv.Close()

    resp, err := http.Get(srv.URL + "/api/users")
    // ...assertions...
}
Challenge Exercise

Write a handler that returns JSON {"count": N} where N increments on each call (use a closure). Test it with httptest.NewRecorder — call the handler 3 times and verify the count increments correctly.

#1 Pitfall: Forgetting defer srv.Close() when using httptest.NewServer. Leaked test servers hold open ports and goroutines, causing flaky tests and resource exhaustion in CI. Use t.Cleanup(srv.Close) as an alternative.

Middleware Chains

Advanced Patterns • 3 days overdue • rep 0

Quick Reminder

Middleware wraps an http.Handler to add cross-cutting behavior (logging, auth, CORS, recovery). The signature func(http.Handler) http.Handler makes middleware composable. Each middleware calls the next handler in the chain, with pre-processing before and post-processing after.

Key Pattern

func logging(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        start := time.Now()
        next.ServeHTTP(w, r)
        slog.Info("request",
            "method", r.Method,
            "path", r.URL.Path,
            "duration", time.Since(start),
        )
    })
}

func recovery(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        defer func() {
            if err := recover(); err != nil {
                http.Error(w, "Internal Server Error", 500)
            }
        }()
        next.ServeHTTP(w, r)
    })
}

// Chain: recovery -> logging -> handler
mux := http.NewServeMux()
mux.HandleFunc("/", homeHandler)
wrapped := recovery(logging(mux))
Challenge Exercise

Write a requireAuth middleware that checks for an Authorization header. If missing, return 401 with {"error":"unauthorized"}. If present, call the next handler. Chain it with a logging middleware and test both paths with httptest.

#1 Pitfall: Writing to http.ResponseWriter after calling next.ServeHTTP(w, r) when the inner handler has already written headers. Once WriteHeader is called, subsequent calls are silently ignored, and writing headers after the body has started causes a superfluous WriteHeader warning.

Concurrency Patterns

Concurrency • 3 days overdue • rep 1

Quick Reminder

Fan-out/fan-in splits work across goroutines and merges results. Pipelines chain stages via channels. Worker pools use a fixed number of goroutines reading from a shared job channel. Rate limiting uses time.Ticker or golang.org/x/time/rate to throttle throughput.

Key Pattern

// Worker pool with buffered job channel
func workerPool(jobs <-chan int, results chan<- int, numWorkers int) {
    var wg sync.WaitGroup
    for w := 0; w < numWorkers; w++ {
        wg.Add(1)
        go func() {
            defer wg.Done()
            for j := range jobs {
                results <- j * j // process job
            }
        }()
    }
    go func() {
        wg.Wait()
        close(results)
    }()
}

jobs := make(chan int, 100)
results := make(chan int, 100)
workerPool(jobs, results, 5)

for i := 1; i <= 20; i++ {
    jobs <- i
}
close(jobs)

for r := range results {
    fmt.Println(r)
}
Challenge Exercise

Build a pipeline with three stages: (1) generate URLs, (2) fan-out to 3 workers that simulate HTTP fetches with time.Sleep, (3) fan-in results and print them. Use channels and sync.WaitGroup to coordinate.

#1 Pitfall: Closing the results channel in the wrong goroutine or before all workers finish. Always close in a separate goroutine that waits on the WaitGroup, never from the main goroutine that is still reading results (that causes deadlock or premature termination).

Context

Concurrency • 3 days overdue • rep 1

Quick Reminder

context.WithCancel for manual cancellation, context.WithTimeout for deadline propagation, context.WithValue for request-scoped data (use sparingly). Always pass ctx as the first parameter. Cancellation propagates to all child contexts. Check ctx.Err() or <-ctx.Done() in long-running operations.

Key Pattern

func fetchData(ctx context.Context, url string) ([]byte, error) {
    req, err := http.NewRequestWithContext(ctx, "GET", url, nil)
    if err != nil {
        return nil, err
    }
    resp, err := http.DefaultClient.Do(req)
    if err != nil {
        return nil, err // includes context cancellation
    }
    defer resp.Body.Close()
    return io.ReadAll(resp.Body)
}

// Caller controls the timeout
ctx, cancel := context.WithTimeout(context.Background(), 5*time.Second)
defer cancel()

data, err := fetchData(ctx, "https://api.example.com/data")
if errors.Is(err, context.DeadlineExceeded) {
    log.Println("request timed out")
}
Challenge Exercise

Write a function that launches 3 goroutines, each simulating work for a random duration (1-5 seconds). Use context.WithTimeout set to 3 seconds. Print which goroutines completed vs. which were cancelled. Use select with ctx.Done().

#1 Pitfall: Forgetting to call cancel() from WithCancel or WithTimeout. Even if the timeout fires, the cancel function must still be called to release resources. Always defer cancel() immediately after creating the context.

reflect Package

Advanced Patterns • 2 days overdue • rep 0

Quick Reminder

reflect.TypeOf(v) returns the type, reflect.ValueOf(v) returns the value. Use .Kind() to check the underlying kind (struct, slice, map, etc.). Iterate struct fields with NumField() and Field(i). Parse struct tags with field.Tag.Get("json"). Reflection is powerful but slow — prefer code generation for hot paths.

Key Pattern

func printFields(v interface{}) {
    val := reflect.ValueOf(v)
    typ := val.Type()

    for i := 0; i < val.NumField(); i++ {
        field := typ.Field(i)
        value := val.Field(i)
        tag := field.Tag.Get("json")
        fmt.Printf("%s (%s) = %v [json:%q]\n",
            field.Name, field.Type, value, tag)
    }
}

type User struct {
    Name  string `json:"name"`
    Age   int    `json:"age"`
    Email string `json:"email,omitempty"`
}

printFields(User{"Alice", 30, "alice@example.com"})
// Name (string) = Alice [json:"name"]
// Age (int) = 30 [json:"age"]
// Email (string) = alice@example.com [json:"email,omitempty"]
Challenge Exercise

Write a function toMap(v interface{}) map[string]interface{} that uses reflection to convert any struct into a map. Use the json struct tag as the map key (fall back to the field name if no tag). Skip fields tagged with json:"-".

#1 Pitfall: Calling reflect.ValueOf(&v).Elem() vs. reflect.ValueOf(v). If you pass a non-pointer, you get a non-settable value — calling .Set() panics. To modify a struct through reflection, you must pass a pointer and call .Elem() to get the underlying settable value.

gRPC & Protocol Buffers

Advanced Networking • New

Why It Matters

gRPC is the dominant RPC framework for microservice communication. It uses Protocol Buffers (protobuf) for efficient binary serialization, HTTP/2 for multiplexed streams, and code generation for type-safe client/server stubs. Compared to REST+JSON, gRPC offers smaller payloads, bidirectional streaming, and strongly-typed contracts that catch integration errors at compile time rather than runtime.

Core Concepts

  • .proto files — define messages (data) and services (RPCs) in a language-neutral schema
  • protoc compiler — generates Go code with protoc-gen-go and protoc-gen-go-grpc
  • Unary RPC — single request, single response (like a function call)
  • Server streaming — single request, stream of responses
  • Client streaming — stream of requests, single response
  • Bidirectional streaming — both sides send streams simultaneously
  • Interceptors — middleware for gRPC (auth, logging, metrics)

Defining a Proto File

// user.proto
syntax = "proto3";

package user;
option go_package = "myapp/proto/user";

message User {
    string id = 1;
    string name = 2;
    string email = 3;
    int32 age = 4;
}

message GetUserRequest {
    string id = 1;
}

message ListUsersRequest {
    int32 page_size = 1;
    string page_token = 2;
}

message ListUsersResponse {
    repeated User users = 1;
    string next_page_token = 2;
}

service UserService {
    // Unary RPC
    rpc GetUser(GetUserRequest) returns (User);

    // Server streaming RPC
    rpc ListUsers(ListUsersRequest) returns (stream User);
}

Generating Go Code

// go install google.golang.org/grpc/cmd/protoc-gen-go-grpc@latest

// Generate code:

Implementing the Server

package main

import (
    "context"
    "fmt"
    "log"
    "net"

    pb "myapp/proto/user"

    "google.golang.org/grpc"
    "google.golang.org/grpc/codes"
    "google.golang.org/grpc/status"
)

type userServer struct {
    pb.UnimplementedUserServiceServer
    users map[string]*pb.User
}

func (s *userServer) GetUser(ctx context.Context, req *pb.GetUserRequest) (*pb.User, error) {
    user, ok := s.users[req.Id]
    if !ok {
        return nil, status.Errorf(codes.NotFound, "user %s not found", req.Id)
    }
    return user, nil
}

func (s *userServer) ListUsers(req *pb.ListUsersRequest, stream pb.UserService_ListUsersServer) error {
    for _, user := range s.users {
        if err := stream.Send(user); err != nil {
            return err
        }
    }
    return nil
}

func main() {
    lis, err := net.Listen("tcp", ":50051")
    if err != nil {
        log.Fatalf("failed to listen: %v", err)
    }

    srv := grpc.NewServer()
    pb.RegisterUserServiceServer(srv, &userServer{
        users: map[string]*pb.User{
            "1": {Id: "1", Name: "Alice", Email: "alice@example.com", Age: 30},
            "2": {Id: "2", Name: "Bob", Email: "bob@example.com", Age: 25},
        },
    })

    fmt.Println("gRPC server listening on :50051")
    if err := srv.Serve(lis); err != nil {
        log.Fatalf("failed to serve: %v", err)
    }
}

Implementing the Client

package main

import (
    "context"
    "fmt"
    "io"
    "log"
    "time"

    pb "myapp/proto/user"

    "google.golang.org/grpc"
    "google.golang.org/grpc/credentials/insecure"
)

func main() {
    conn, err := grpc.NewClient("localhost:50051",
        grpc.WithTransportCredentials(insecure.NewCredentials()),
    )
    if err != nil {
        log.Fatalf("failed to connect: %v", err)
    }
    defer conn.Close()

    client := pb.NewUserServiceClient(conn)

    // Unary call
    ctx, cancel := context.WithTimeout(context.Background(), 5*time.Second)
    defer cancel()

    user, err := client.GetUser(ctx, &pb.GetUserRequest{Id: "1"})
    if err != nil {
        log.Fatalf("GetUser failed: %v", err)
    }
    fmt.Printf("User: %s (%s)\n", user.Name, user.Email)

    // Server streaming
    stream, err := client.ListUsers(ctx, &pb.ListUsersRequest{PageSize: 10})
    if err != nil {
        log.Fatalf("ListUsers failed: %v", err)
    }

    for {
        user, err := stream.Recv()
        if err == io.EOF {
            break
        }
        if err != nil {
            log.Fatalf("stream error: %v", err)
        }
        fmt.Printf("  - %s (%s)\n", user.Name, user.Email)
    }
}

Interceptors (Middleware for gRPC)

// Unary server interceptor for logging
func loggingInterceptor(
    ctx context.Context,
    req interface{},
    info *grpc.UnaryServerInfo,
    handler grpc.UnaryHandler,
) (interface{}, error) {
    start := time.Now()
    resp, err := handler(ctx, req)
    slog.Info("gRPC call",
        "method", info.FullMethod,
        "duration", time.Since(start),
        "error", err,
    )
    return resp, err
}

// Apply interceptor when creating the server
srv := grpc.NewServer(
    grpc.UnaryInterceptor(loggingInterceptor),
)

// Chain multiple interceptors
srv := grpc.NewServer(
    grpc.ChainUnaryInterceptor(
        loggingInterceptor,
        authInterceptor,
        recoveryInterceptor,
    ),
)

Common Mistakes

Do: Use status codes correctly
return nil, status.Errorf(
    codes.NotFound,
    "user %s not found", id,
)
// Client checks:
// status.Code(err) == codes.NotFound
Don't: Return plain errors
return nil, fmt.Errorf("user not found")
// Client gets codes.Unknown,
// loses semantic meaning
Do: Embed UnimplementedServer
type myServer struct {
    pb.UnimplementedUserServiceServer
}
// Forward-compatible: new RPCs
// get a default "unimplemented" response
Don't: Implement interface directly
type myServer struct{}
// Adding new RPCs to the .proto
// breaks compilation immediately

Key Rules

  • Always embed Unimplemented*Server for forward compatibility when new RPCs are added to the proto
  • Use gRPC status codes (codes.NotFound, codes.InvalidArgument) instead of plain errors for semantic error handling
  • Pass context.Context through all RPC calls — it carries deadlines, cancellation, and metadata
  • Use interceptors for cross-cutting concerns (auth, logging, tracing) — they are the gRPC equivalent of HTTP middleware
  • Prefer gRPC over REST for internal service-to-service communication; use REST/gRPC-Gateway for public-facing APIs
Connection: gRPC interceptors are the direct equivalent of Middleware Chains from HTTP. The Context package carries deadlines and metadata across gRPC boundaries. Protobuf code generation connects to the Code Generation concept. OpenTelemetry (later in this guide) integrates with gRPC interceptors for distributed tracing.

WebSocket in Go

Advanced Networking • New

Why It Matters

WebSockets enable full-duplex, persistent connections between client and server — essential for real-time features like chat, live dashboards, notifications, and collaborative editing. Unlike HTTP request/response, a WebSocket connection stays open for bidirectional streaming, eliminating the overhead of repeated connection setup and enabling server-initiated pushes.

Core Concepts

  • HTTP Upgrade — WebSocket starts as an HTTP request that "upgrades" to the WebSocket protocol
  • gorilla/websocket — the most widely used Go WebSocket library
  • Upgrader — configures the HTTP-to-WebSocket upgrade (buffer sizes, origin check)
  • Message types — text (TextMessage) and binary (BinaryMessage)
  • Concurrent access — one reader goroutine + one writer goroutine per connection
  • Ping/pong — keep-alive mechanism to detect dead connections
  • Hub pattern — central coordinator for broadcasting to multiple clients

Basic WebSocket Server

package main

import (
    "fmt"
    "log"
    "net/http"

    "github.com/gorilla/websocket"
)

var upgrader = websocket.Upgrader{
    ReadBufferSize:  1024,
    WriteBufferSize: 1024,
    CheckOrigin: func(r *http.Request) bool {
        return true // In production, validate the origin!
    },
}

func wsHandler(w http.ResponseWriter, r *http.Request) {
    conn, err := upgrader.Upgrade(w, r, nil)
    if err != nil {
        log.Println("upgrade error:", err)
        return
    }
    defer conn.Close()

    for {
        msgType, msg, err := conn.ReadMessage()
        if err != nil {
            log.Println("read error:", err)
            break
        }
        fmt.Printf("received: %s\n", msg)

        // Echo the message back
        if err := conn.WriteMessage(msgType, msg); err != nil {
            log.Println("write error:", err)
            break
        }
    }
}

func main() {
    http.HandleFunc("/ws", wsHandler)
    fmt.Println("WebSocket server on :8080")
    log.Fatal(http.ListenAndServe(":8080", nil))
}

Concurrent Read/Write with Goroutines

package main

import (
    "fmt"
    "log"
    "sync"
    "time"

    "github.com/gorilla/websocket"
)

type Client struct {
    conn    *websocket.Conn
    send    chan []byte
    mu      sync.Mutex // protects conn.WriteMessage
}

func (c *Client) readPump() {
    defer c.conn.Close()

    c.conn.SetReadDeadline(time.Now().Add(60 * time.Second))
    c.conn.SetPongHandler(func(string) error {
        c.conn.SetReadDeadline(time.Now().Add(60 * time.Second))
        return nil
    })

    for {
        _, msg, err := c.conn.ReadMessage()
        if err != nil {
            if websocket.IsUnexpectedCloseError(err,
                websocket.CloseGoingAway,
                websocket.CloseNormalClosure,
            ) {
                log.Printf("unexpected close: %v", err)
            }
            break
        }
        fmt.Printf("received: %s\n", msg)
    }
}

func (c *Client) writePump() {
    ticker := time.NewTicker(30 * time.Second)
    defer func() {
        ticker.Stop()
        c.conn.Close()
    }()

    for {
        select {
        case msg, ok := <-c.send:
            if !ok {
                // Channel closed
                c.conn.WriteMessage(websocket.CloseMessage, []byte{})
                return
            }
            c.mu.Lock()
            err := c.conn.WriteMessage(websocket.TextMessage, msg)
            c.mu.Unlock()
            if err != nil {
                return
            }
        case <-ticker.C:
            // Send ping to keep connection alive
            c.mu.Lock()
            err := c.conn.WriteMessage(websocket.PingMessage, nil)
            c.mu.Unlock()
            if err != nil {
                return
            }
        }
    }
}

Hub Pattern for Broadcasting

package main

import "sync"

type Hub struct {
    clients    map[*Client]bool
    broadcast  chan []byte
    register   chan *Client
    unregister chan *Client
    mu         sync.RWMutex
}

func NewHub() *Hub {
    return &Hub{
        clients:    make(map[*Client]bool),
        broadcast:  make(chan []byte, 256),
        register:   make(chan *Client),
        unregister: make(chan *Client),
    }
}

func (h *Hub) Run() {
    for {
        select {
        case client := <-h.register:
            h.mu.Lock()
            h.clients[client] = true
            h.mu.Unlock()

        case client := <-h.unregister:
            h.mu.Lock()
            if _, ok := h.clients[client]; ok {
                delete(h.clients, client)
                close(client.send)
            }
            h.mu.Unlock()

        case msg := <-h.broadcast:
            h.mu.RLock()
            for client := range h.clients {
                select {
                case client.send <- msg:
                default:
                    // Client buffer full, disconnect
                    close(client.send)
                    delete(h.clients, client)
                }
            }
            h.mu.RUnlock()
        }
    }
}

// In handler:
// hub.register <- client
// go client.readPump()
// go client.writePump()

Testing WebSocket Handlers

package main

import (
    "net/http"
    "net/http/httptest"
    "strings"
    "testing"

    "github.com/gorilla/websocket"
)

func TestWebSocketEcho(t *testing.T) {
    // Create test server
    srv := httptest.NewServer(http.HandlerFunc(wsHandler))
    defer srv.Close()

    // Convert http:// to ws://
    wsURL := "ws" + strings.TrimPrefix(srv.URL, "http") + "/ws"

    // Connect
    ws, _, err := websocket.DefaultDialer.Dial(wsURL, nil)
    if err != nil {
        t.Fatalf("dial error: %v", err)
    }
    defer ws.Close()

    // Send message
    msg := "hello, websocket!"
    if err := ws.WriteMessage(websocket.TextMessage, []byte(msg)); err != nil {
        t.Fatalf("write error: %v", err)
    }

    // Read echo
    _, got, err := ws.ReadMessage()
    if err != nil {
        t.Fatalf("read error: %v", err)
    }

    if string(got) != msg {
        t.Errorf("got %q, want %q", got, msg)
    }
}

Common Mistakes

Do: One reader + one writer goroutine
go client.readPump()   // reads
go client.writePump()  // writes
// WebSocket connections are NOT
// safe for concurrent writes
Don't: Write from multiple goroutines
// Multiple goroutines calling WriteMessage
// concurrently causes data corruption
go func() { conn.WriteMessage(...) }()
go func() { conn.WriteMessage(...) }()
Do: Validate origin in production
upgrader := websocket.Upgrader{
    CheckOrigin: func(r *http.Request) bool {
        origin := r.Header.Get("Origin")
        return origin == "https://myapp.com"
    },
}
Don't: Accept all origins
CheckOrigin: func(r *http.Request) bool {
    return true // CSRF vulnerability!
}

Key Rules

  • Use exactly one goroutine for reading and one for writing per WebSocket connection — never write from multiple goroutines
  • Implement ping/pong keepalives to detect dead connections and avoid resource leaks
  • Always validate the CheckOrigin function in production to prevent cross-site WebSocket hijacking
  • Use a buffered send channel per client and disconnect slow consumers whose buffer fills up
  • The Hub pattern centralizes broadcast logic — use it whenever multiple clients need the same updates
Connection: WebSocket handlers are http.Handler functions that upgrade the connection — they integrate with Middleware Chains for auth/logging before the upgrade. The Hub pattern uses Goroutines, Channels, and Select from the concurrency toolkit. Context can carry request-scoped data through the upgrade. Graceful Shutdown must close all active WebSocket connections.

Profiling with pprof

Observability & Profiling • New

Why It Matters

When your Go service is slow or using too much memory, guessing is expensive. pprof gives you data-driven answers: which functions consume CPU time, where heap allocations happen, and whether goroutines are leaking. Go's built-in profiling tools are production-safe with minimal overhead, making them a first-line tool for diagnosing performance issues in live systems.

Core Concepts

  • net/http/pprof — import for side-effect to register profiling HTTP endpoints on DefaultServeMux
  • runtime/pprof — programmatic API for writing profiles to files (benchmarks, CLI tools)
  • CPU profile — samples the call stack at ~100Hz to find hot paths
  • Memory (heap) profile — tracks allocations; inuse shows current memory, alloc shows cumulative
  • Goroutine profile — snapshot of all goroutines' stack traces (leak detection)
  • Block profile — where goroutines block on synchronization primitives
  • go tool pprof — interactive CLI for analyzing profiles (top, list, web)

Enabling pprof in an HTTP Server

package main

import (

Goroutine Leak Detection

package main

import (
    "fmt"
    "net/http"
    _ "net/http/pprof"
    "runtime"
    "time"
)

func leakyServer() {
    for i := 0; i < 100; i++ {
        go func(id int) {
            ch := make(chan struct{}) // never closed!
            <-ch                     // blocks forever = goroutine leak
        }(i)
    }
}

func main() {
    go func() {
        http.ListenAndServe(":6060", nil) // pprof endpoint
    }()

    leakyServer()

    time.Sleep(time.Second)
    fmt.Printf("Goroutines: %d\n", runtime.NumGoroutine())
    // Output: Goroutines: 103 (100 leaked + main + pprof + ...)

    // Debug with:
    // go tool pprof http://localhost:6060/debug/pprof/goroutine
    // (pprof) top
    // (pprof) traces

    select {} // keep running for pprof
}

Using go tool pprof

// Capture a 30-second CPU profile from a running server:
// go tool pprof http://localhost:6060/debug/pprof/profile?seconds=30

// Interactive commands:
// (pprof) top 10          - top 10 CPU consumers
// (pprof) list funcName   - source-level annotation
// (pprof) web             - open flame graph in browser
// (pprof) png > out.png   - export call graph as image

// Compare two profiles (before/after optimization):
// go tool pprof -diff_base old.prof new.prof

// Analyze from benchmark tests:
// go test -cpuprofile cpu.prof -memprofile mem.prof -bench .
// go tool pprof cpu.prof

Common Mistakes

Do: Serve pprof on a separate port
// pprof on internal port only
go http.ListenAndServe(":6060", nil)

// App on public port
http.ListenAndServe(":8080", appMux)
Don't: Expose pprof publicly
import _ "net/http/pprof"

// Registers on DefaultServeMux
http.ListenAndServe(":8080", nil)
// /debug/pprof is now public!
// Leaks internal details, DoS risk
Do: GC before heap profile
runtime.GC() // Clear unreachable objects
pprof.WriteHeapProfile(f)
// Shows only live allocations
Don't: Profile without GC
pprof.WriteHeapProfile(f)
// May include garbage not yet collected,
// giving misleading allocation data

Key Rules

  • Never expose /debug/pprof on a public-facing port — serve it on an internal-only listener or behind authentication
  • Call runtime.GC() before taking a heap profile to ensure you see only live objects, not pending garbage
  • Use go test -bench . -cpuprofile cpu.prof to profile specific benchmarks — this is the easiest way to start
  • For goroutine leak detection, compare runtime.NumGoroutine() before and after operations — it should return to baseline
  • The web command in go tool pprof generates a flame graph — install graphviz to enable graph visualization
Connection: Profiling connects to Benchmarks (profile specific benchmark functions), Memory Management (escape analysis + sync.Pool optimizations verified via pprof), and Concurrency Patterns (goroutine profiles catch leaked workers). Use with OpenTelemetry for a complete observability stack: tracing for latency, pprof for resource usage.

OpenTelemetry & Tracing

Observability & Profiling • New

Why It Matters

In a microservices architecture, a single user request can traverse dozens of services. When something is slow, you need to trace the request's path across all services and identify which service or database call is the bottleneck. OpenTelemetry (OTel) is the industry-standard, vendor-neutral framework for distributed tracing, metrics, and logs. It replaces older tools like Jaeger client libraries and OpenCensus with a unified API.

Core Concepts

  • Trace — the complete journey of a request across services, identified by a trace ID
  • Span — a single unit of work within a trace (e.g., "handle HTTP request", "query database")
  • Span attributes — key-value metadata attached to spans (user ID, HTTP status, SQL query)
  • Context propagation — trace/span IDs transmitted via HTTP headers or gRPC metadata
  • Exporter — sends telemetry data to a backend (Jaeger, OTLP, Zipkin, console)
  • TracerProvider — creates tracers; configured once at application startup
  • Metrics — counters, histograms, and gauges for quantitative measurements

Setting Up the OTel SDK

package main

import (
    "context"
    "fmt"
    "log"

    "go.opentelemetry.io/otel"
    "go.opentelemetry.io/otel/attribute"
    "go.opentelemetry.io/otel/exporters/stdout/stdouttrace"
    "go.opentelemetry.io/otel/sdk/resource"
    sdktrace "go.opentelemetry.io/otel/sdk/trace"
    semconv "go.opentelemetry.io/otel/semconv/v1.24.0"
    "go.opentelemetry.io/otel/trace"
)

func initTracer() (*sdktrace.TracerProvider, error) {
    // Console exporter for development
    exporter, err := stdouttrace.New(stdouttrace.WithPrettyPrint())
    if err != nil {
        return nil, fmt.Errorf("creating exporter: %w", err)
    }

    tp := sdktrace.NewTracerProvider(
        sdktrace.WithBatcher(exporter),
        sdktrace.WithResource(resource.NewWithAttributes(
            semconv.SchemaURL,
            semconv.ServiceNameKey.String("my-service"),
            semconv.ServiceVersionKey.String("1.0.0"),
        )),
    )

    otel.SetTracerProvider(tp)
    return tp, nil
}

func main() {
    tp, err := initTracer()
    if err != nil {
        log.Fatal(err)
    }
    defer tp.Shutdown(context.Background())

    // Now use otel.Tracer() anywhere in your code
    tracer := otel.Tracer("my-service")
    ctx, span := tracer.Start(context.Background(), "main")
    defer span.End()

    doWork(ctx)
}

func doWork(ctx context.Context) {
    tracer := otel.Tracer("my-service")
    _, span := tracer.Start(ctx, "doWork")
    defer span.End()

    span.SetAttributes(
        attribute.String("work.type", "computation"),
        attribute.Int("work.items", 42),
    )

    fmt.Println("doing work...")
}

HTTP Middleware for Automatic Tracing

package main

import (
    "context"
    "fmt"
    "net/http"

    "go.opentelemetry.io/otel"
    "go.opentelemetry.io/otel/attribute"
    "go.opentelemetry.io/otel/propagation"
    semconv "go.opentelemetry.io/otel/semconv/v1.24.0"
)

// Tracing middleware: creates a span for each HTTP request
func tracingMiddleware(next http.Handler) http.Handler {
    tracer := otel.Tracer("http-server")

    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        // Extract trace context from incoming headers
        ctx := otel.GetTextMapPropagator().Extract(
            r.Context(),
            propagation.HeaderCarrier(r.Header),
        )

        ctx, span := tracer.Start(ctx, fmt.Sprintf("%s %s", r.Method, r.URL.Path))
        defer span.End()

        // Add HTTP attributes
        span.SetAttributes(
            semconv.HTTPMethodKey.String(r.Method),
            semconv.HTTPURLKey.String(r.URL.String()),
            attribute.String("http.client_ip", r.RemoteAddr),
        )

        // Pass traced context to the next handler
        next.ServeHTTP(w, r.WithContext(ctx))
    })
}

func userHandler(w http.ResponseWriter, r *http.Request) {
    ctx := r.Context()

    // Child span for database query
    tracer := otel.Tracer("http-server")
    _, span := tracer.Start(ctx, "db.query")
    span.SetAttributes(
        attribute.String("db.statement", "SELECT * FROM users WHERE id = ?"),
        attribute.String("db.system", "postgresql"),
    )
    // ... execute query ...
    span.End()

    fmt.Fprintln(w, `{"user": "Alice"}`)
}

Context Propagation Across Services

package main

import (
    "context"
    "net/http"

    "go.opentelemetry.io/otel"
    "go.opentelemetry.io/otel/propagation"
)

func init() {
    // Set up W3C Trace Context propagation (standard)
    otel.SetTextMapPropagator(
        propagation.NewCompositeTextMapPropagator(
            propagation.TraceContext{},
            propagation.Baggage{},
        ),
    )
}

// When calling another service, inject trace context into headers
func callService(ctx context.Context, url string) (*http.Response, error) {
    req, err := http.NewRequestWithContext(ctx, "GET", url, nil)
    if err != nil {
        return nil, err
    }

    // Inject trace context into outgoing request headers
    otel.GetTextMapPropagator().Inject(ctx, propagation.HeaderCarrier(req.Header))

    // The downstream service extracts these headers to continue the trace
    return http.DefaultClient.Do(req)
}

Metrics: Counters, Histograms, Gauges

package main

import (
    "context"
    "math/rand"
    "time"

    "go.opentelemetry.io/otel"
    "go.opentelemetry.io/otel/metric"
)

func setupMetrics() {
    meter := otel.Meter("my-service")

    // Counter: total requests
    requestCount, _ := meter.Int64Counter("http.requests.total",
        metric.WithDescription("Total HTTP requests"),
    )

    // Histogram: request duration
    requestDuration, _ := meter.Float64Histogram("http.request.duration_ms",
        metric.WithDescription("HTTP request duration in milliseconds"),
    )

    // Gauge (via callback): active connections
    meter.Int64ObservableGauge("http.active_connections",
        metric.WithDescription("Current active connections"),
        metric.WithInt64Callback(func(_ context.Context, o metric.Int64Observer) error {
            o.Observe(int64(getActiveConnections()))
            return nil
        }),
    )

    // Usage in handler:
    // requestCount.Add(ctx, 1, metric.WithAttributes(...))
    // requestDuration.Record(ctx, elapsed.Milliseconds(), ...)
    _ = requestCount
    _ = requestDuration
}

func getActiveConnections() int {
    return rand.Intn(100)
}

func main() {
    setupMetrics()
    time.Sleep(time.Second)
}

Common Mistakes

Do: End spans with defer
ctx, span := tracer.Start(ctx, "operation")
defer span.End()
// Span ends even if function panics
Don't: Forget to end spans
ctx, span := tracer.Start(ctx, "operation")
// ... code ...
// Forgot span.End()!
// Span is never exported, trace is broken
Do: Propagate context through calls
func getUser(ctx context.Context, id string) {
    _, span := tracer.Start(ctx, "getUser")
    defer span.End()
    db.QueryContext(ctx, "SELECT ...")
}
Don't: Drop the context
func getUser(id string) { // no ctx!
    db.Query("SELECT ...")
    // No trace propagation, orphaned span
}

Key Rules

  • Always defer span.End() immediately after tracer.Start() — unleaked spans break trace visibility
  • Pass context.Context through every function in the call chain — it carries the active span for parent-child linking
  • Set up W3C Trace Context propagation (propagation.TraceContext{}) so traces cross service boundaries via HTTP headers
  • Use attribute.String/Int/Bool to enrich spans with searchable metadata — but avoid high-cardinality attributes (e.g., user IDs in metrics)
  • Call TracerProvider.Shutdown(ctx) on application exit to flush all pending spans to the exporter
Connection: OTel tracing middleware integrates with Middleware Chains and gRPC Interceptors. Context is the carrier for trace propagation. Combine with pprof for a complete observability stack: OTel for distributed request tracing, pprof for in-process resource profiling. Structured Logging with slog can include trace IDs for correlated log-trace analysis.

Embedding & go:embed

Observability & Profiling • New

Why It Matters

Go's //go:embed directive (Go 1.16+) embeds files and directories into the compiled binary at build time. This enables true single-binary deployments: your web server includes its HTML templates, static assets, migration scripts, and config files with zero external dependencies. No more "file not found" errors in production because someone forgot to copy the templates directory.

Core Concepts

  • //go:embed — compiler directive that embeds file contents into a variable
  • embed.FS — read-only filesystem interface for embedded directory trees
  • embed.String / embed.Byte — embed single files as string or []byte
  • Glob patterns — //go:embed static/* embeds all files matching the pattern
  • Works with net/http, html/template, io/fs — standard library integration
  • Build-time resolution — files must exist when go build runs; missing files cause compile errors

Embedding Single Files

package main

import (
    _ "embed"
    "fmt"
)

// Embed a single file as a string
//go:embed version.txt
var version string

// Embed a single file as bytes
//go:embed config/defaults.json
var defaultConfig []byte

func main() {
    fmt.Println("Version:", version)
    fmt.Println("Config:", string(defaultConfig))
}

// version.txt contains: v1.2.3
// config/defaults.json contains: {"port": 8080, "debug": false}
Output
Version: v1.2.3 Config: {"port": 8080, "debug": false}

Embedding Directory Trees with embed.FS

package main

import (
    "embed"
    "fmt"
    "io/fs"
)

//go:embed templates/*.html
var templateFS embed.FS

//go:embed static/*
var staticFS embed.FS

func main() {
    // List all embedded files
    fs.WalkDir(templateFS, ".", func(path string, d fs.DirEntry, err error) error {
        if err != nil {
            return err
        }
        if !d.IsDir() {
            fmt.Println("embedded:", path)
        }
        return nil
    })

    // Read a specific file
    data, err := templateFS.ReadFile("templates/index.html")
    if err != nil {
        panic(err)
    }
    fmt.Println("index.html:", len(data), "bytes")
}
Output
embedded: templates/index.html embedded: templates/layout.html embedded: templates/404.html index.html: 1234 bytes

Serving Static Files with net/http

package main

import (
    "embed"
    "fmt"
    "io/fs"
    "log"
    "net/http"
)

//go:embed static/*
var staticFiles embed.FS

func main() {
    // Strip the "static" prefix so /style.css maps to static/style.css
    staticSub, err := fs.Sub(staticFiles, "static")
    if err != nil {
        log.Fatal(err)
    }

    mux := http.NewServeMux()

    // Serve embedded static files
    mux.Handle("/static/", http.StripPrefix("/static/",
        http.FileServer(http.FS(staticSub)),
    ))

    mux.HandleFunc("/", func(w http.ResponseWriter, r *http.Request) {
        fmt.Fprintln(w, "Home page - static assets are embedded!")
    })

    fmt.Println("Server on :8080 with embedded static files")
    log.Fatal(http.ListenAndServe(":8080", mux))
}

// Requests to /static/style.css serve the embedded static/style.css
// Requests to /static/js/app.js serve the embedded static/js/app.js

Embedding Multiple Patterns and Conditional Embedding

package main

import (
    "embed"
    "fmt"
)

// Multiple patterns on separate directives
//go:embed migrations/*.sql
//go:embed seeds/*.sql
var sqlFiles embed.FS

// Embed everything in a directory (including subdirectories)
//go:embed assets
var allAssets embed.FS

var documentation embed.FS

    if err != nil {
        fmt.Println("Error:", err)
        return
    }
    fmt.Println("Migration:")
    fmt.Println(string(data))

    entries, _ := sqlFiles.ReadDir("migrations")
    for _, e := range entries {
        fmt.Println(" -", e.Name())
    }
}

Common Mistakes

Do: Use fs.Sub to strip prefix
//go:embed static/*
var staticFS embed.FS

sub, _ := fs.Sub(staticFS, "static")
http.Handle("/", http.FileServer(http.FS(sub)))
// /style.css -> static/style.css
Don't: Serve with directory prefix
//go:embed static/*
var staticFS embed.FS

http.Handle("/", http.FileServer(http.FS(staticFS)))
// URL must be /static/style.css -- awkward!
Do: Import embed for string/[]byte
import _ "embed" // blank import required!

//go:embed version.txt
var version string
Don't: Forget the embed import
// Missing: import _ "embed"

//go:embed version.txt
var version string
// Compile error: //go:embed requires
// import of "embed"

Key Rules

  • The //go:embed directive must immediately precede the variable declaration — no blank lines between them
  • For string and []byte embedding, you must have import _ "embed" even though the package is not referenced directly
  • Embedded files are read-only — embed.FS implements fs.ReadFileFS and fs.ReadDirFS but has no write methods
  • Use fs.Sub() to strip directory prefixes when serving files via http.FileServer
  • Paths in //go:embed are relative to the source file's directory and cannot use .. to escape it
Connection: go:embed creates single-binary deployments that simplify Cross-Compilation (no file dependencies to copy). Pairs with html/template for server-side rendering and net/http for serving static assets. The embed.FS type implements the io/fs interfaces, connecting to the io Package. Database migrations embedded this way work with the database/sql pattern.
