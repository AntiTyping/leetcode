Type Parameters (Generics)

Functions & Control Flow • overdue 4 days • rep 1

Quick Reminder

Type parameters let you write functions and types that work with multiple types: func Map[T, U any](s []T, f func(T) U) []U. Constraints limit which types are allowed — any allows all types, comparable allows ==/!=, and custom constraint interfaces use ~ for underlying type matching (e.g., ~int | ~float64).

Key Pattern

func Filter[T any](s []T, predicate func(T) bool) []T {
    var result []T
    for _, v := range s {
        if predicate(v) {
            result = append(result, v)
        }
    }
    return result
}

nums := []int{1, 2, 3, 4, 5, 6}
evens := Filter(nums, func(n int) bool { return n%2 == 0 })
// [2 4 6]
Challenge Exercise

Write a generic Filter[T] function that takes a slice and a predicate, returning a new slice containing only elements where the predicate returns true. Test it with both []int and []string.

#1 Pitfall: You cannot use == on a type parameter constrained by any. If you need equality checks, constrain with comparable: func Contains[T comparable](s []T, target T) bool.

Benchmarks

Testing & Tooling • overdue 2 days • rep 0

Quick Reminder

Benchmark functions start with Benchmark and take *testing.B. The loop runs b.N times (adjusted automatically). Use b.ResetTimer() to exclude setup time, b.ReportAllocs() to track allocations, and b.RunParallel for concurrent benchmarks. Run with go test -bench=. -benchmem.

Key Pattern

var result string // package-level to prevent compiler optimization

func BenchmarkConcat(b *testing.B) {
    b.ReportAllocs()
    for i := 0; i < b.N; i++ {
        s := ""
        for j := 0; j < 100; j++ {
            s += "x"
        }
        result = s
    }
}

func BenchmarkBuilder(b *testing.B) {
    b.ReportAllocs()
    for i := 0; i < b.N; i++ {
        var sb strings.Builder
        for j := 0; j < 100; j++ {
            sb.WriteString("x")
        }
        result = sb.String()
    }
}
Challenge Exercise

Write benchmarks comparing two approaches to string concatenation: += in a loop vs strings.Builder. Assign the result to a package-level variable to prevent the compiler from optimizing the code away.

#1 Pitfall: The compiler can optimize away benchmark code if the result is unused. Always assign the final result to a package-level var result to prevent dead-code elimination.

net/http

Standard Library • overdue 2 days • rep 0

Quick Reminder

Handlers implement http.Handler (interface with ServeHTTP(w, r)) or use http.HandlerFunc adapter. http.ServeMux routes requests. Middleware wraps handlers: func logging(next http.Handler) http.Handler. The http.Client has no default timeout — always set one. Go 1.22+ ServeMux supports method and path parameters: GET /users/{id}.

Key Pattern

func timingMiddleware(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        start := time.Now()
        next.ServeHTTP(w, r)
        w.Header().Set("X-Response-Time", time.Since(start).String())
    })
}

mux := http.NewServeMux()
mux.HandleFunc("GET /health", healthHandler)
http.ListenAndServe(":8080", timingMiddleware(mux))
Challenge Exercise

Write an HTTP middleware that adds an X-Response-Time header to every response, measuring how long the handler took to execute. Chain it with a simple health-check handler.

#1 Pitfall: The default http.Client{} has no timeout, meaning requests can hang indefinitely. Always create a client with an explicit timeout: &http.Client{Timeout: 10 * time.Second}.

sort Package

Standard Library • overdue 2 days • rep 0

Quick Reminder

sort.Slice sorts in-place with a custom less function. sort.SliceStable preserves order of equal elements. sort.Search performs binary search on a sorted slice. slices.SortFunc (Go 1.21+) is the generic alternative. For multi-field sorting, chain comparisons in the less function.

Key Pattern

type Employee struct {
    Name string
    Age  int
    Dept string
}

employees := []Employee{
    {"Alice", 30, "Eng"}, {"Bob", 25, "Eng"}, {"Carol", 30, "Sales"},
}

// Sort by age ascending, then name ascending
sort.SliceStable(employees, func(i, j int) bool {
    if employees[i].Age != employees[j].Age {
        return employees[i].Age < employees[j].Age
    }
    return employees[i].Name < employees[j].Name
})
Challenge Exercise

Sort a slice of structs by multiple fields: first by department (ascending), then by age (descending). Use sort.SliceStable to preserve relative order within ties.

#1 Pitfall: sort.Search requires the slice to already be sorted, and its predicate sense is inverted from what you might expect — it finds the smallest index i where f(i) is true, so you write func(i int) bool { return s[i] >= target }, not s[i] == target.

testing Package: Table-Driven Tests, Subtests

Testing • overdue 2 days • rep 0

Quick Reminder

Table-driven tests use a slice of test cases iterated with for _, tc := range tests. Subtests with t.Run(name, func(t *testing.T) {...}) let you run/filter individual cases. t.Parallel() runs subtests concurrently. Use t.Helper() in assertion helpers to get correct line numbers in failure output.

Key Pattern

func TestParseURL(t *testing.T) {
    tests := []struct {
        name    string
        input   string
        wantErr bool
    }{
        {"valid http", "http://example.com", false},
        {"valid https", "https://example.com/path", false},
        {"empty string", "", true},
        {"no scheme", "example.com", true},
    }

    for _, tc := range tests {
        tc := tc // capture for parallel (pre-Go 1.22)
        t.Run(tc.name, func(t *testing.T) {
            t.Parallel()
            _, err := url.Parse(tc.input)
            if (err != nil) != tc.wantErr {
                t.Errorf("Parse(%q) error = %v, wantErr %v",
                    tc.input, err, tc.wantErr)
            }
        })
    }
}
Challenge Exercise

Write table-driven tests for a URL parser function. Include at least 4 test cases covering valid URLs, empty strings, and missing schemes. Use t.Run for subtests and t.Parallel().

#1 Pitfall: Loop variable capture in parallel subtests (pre-Go 1.22). Without tc := tc inside the loop, all parallel subtests share the last value of tc. Go 1.22+ fixes this with per-iteration scoping, but the tc := tc pattern remains common in existing code.

time Package

Standard Library • overdue 2 days • rep 0

Quick Reminder

time.Now() returns current time. time.Duration represents elapsed time. time.Timer fires once after a delay; time.Ticker fires repeatedly. Formatting uses Go's reference time Mon Jan 2 15:04:05 MST 2006 (1/2 3:04:05 PM '06 -0700). time.Parse and time.Format both use this reference time, NOT format strings like %Y-%m-%d.

Key Pattern

now := time.Now()

// ISO 8601 format
iso := now.Format("2006-01-02T15:04:05Z07:00")
fmt.Println(iso) // 2026-06-10T14:30:00-04:00

// Parse a date string
t, _ := time.Parse("2006-01-02", "2026-06-10")
fmt.Println(t) // 2026-06-10 00:00:00 +0000 UTC

// Duration arithmetic
deadline := now.Add(24 * time.Hour)
remaining := time.Until(deadline)
fmt.Printf("%.0f hours left\n", remaining.Hours())
Challenge Exercise

Format the current time as ISO 8601 (2006-01-02T15:04:05Z07:00). Then parse the string "June 10, 2026 3:04pm" using the correct reference-time layout. Print both results.

#1 Pitfall: Go's reference time is Mon Jan 2 15:04:05 MST 2006 — these are NOT arbitrary values. They encode 1=month, 2=day, 3=hour(12h), 4=minute, 5=second, 6=year, 7=timezone offset. Using any other numbers (e.g., "2026-06-10") produces garbage.

bufio: Buffered I/O

I/O & Networking • New

Why It Matters

Reading or writing one byte at a time through system calls is extremely slow. The bufio package wraps io.Reader and io.Writer with in-memory buffers, batching I/O operations for dramatically better performance. bufio.Scanner is the standard way to process text line by line.

Core Concepts

  • bufio.NewReader(r) — wraps an io.Reader with a default 4096-byte buffer
  • bufio.Scanner — reads tokens (lines by default) from an io.Reader; call Scan() in a loop, Text() for the current token
  • Scanner.Split(splitFunc) — customize tokenization: bufio.ScanLines (default), bufio.ScanWords, bufio.ScanBytes, or a custom SplitFunc
  • bufio.NewWriter(w) — buffers writes; must call Flush() to ensure all data is written
  • bufio.ReadWriter — combines a buffered reader and writer into one struct

Line-by-Line Processing with Scanner

package main

import (
    "bufio"
    "fmt"
    "strings"
)

func main() {
    input := "line one\nline two\nline three\n"
    scanner := bufio.NewScanner(strings.NewReader(input))

    lineNum := 0
    for scanner.Scan() {
        lineNum++
        fmt.Printf("%d: %s\n", lineNum, scanner.Text())
    }

    if err := scanner.Err(); err != nil {
        fmt.Println("Error:", err)
    }
}
Output
1: line one 2: line two 3: line three

Word Scanning and Custom Split Functions

package main

import (
    "bufio"
    "fmt"
    "strings"
)

func main() {
    input := "Go is a statically typed language"
    scanner := bufio.NewScanner(strings.NewReader(input))
    scanner.Split(bufio.ScanWords)

    words := []string{}
    for scanner.Scan() {
        words = append(words, scanner.Text())
    }
    fmt.Printf("Word count: %d\n", len(words))
    fmt.Printf("Words: %v\n", words)
}
Output
Word count: 6 Words: [Go is a statically typed language]

Custom SplitFunc: Splitting on Commas

package main

import (
    "bufio"
    "bytes"
    "fmt"
    "strings"
)

// splitOnComma is a custom SplitFunc that splits input on commas
func splitOnComma(data []byte, atEOF bool) (advance int, token []byte, err error) {
    if atEOF && len(data) == 0 {
        return 0, nil, nil
    }
    if i := bytes.IndexByte(data, ','); i >= 0 {
        return i + 1, bytes.TrimSpace(data[:i]), nil
    }
    if atEOF {
        return len(data), bytes.TrimSpace(data), nil
    }
    return 0, nil, nil // request more data
}

func main() {
    input := "Alice, Bob, Carol, Dave"
    scanner := bufio.NewScanner(strings.NewReader(input))
    scanner.Split(splitOnComma)

    for scanner.Scan() {
        fmt.Printf("Name: %q\n", scanner.Text())
    }
}
Output
Name: "Alice" Name: "Bob" Name: "Carol" Name: "Dave"

Buffered Writer

package main

import (
    "bufio"
    "fmt"
    "os"
)

func main() {
    w := bufio.NewWriter(os.Stdout)

    fmt.Fprintln(w, "buffered line 1")
    fmt.Fprintln(w, "buffered line 2")
    fmt.Fprintln(w, "buffered line 3")

    // Nothing printed yet! Data is in the buffer.
    // Flush sends it to the underlying writer (stdout).
    w.Flush()

    fmt.Printf("Buffer size: %d bytes\n", w.Available()+w.Buffered())
}
Output
buffered line 1 buffered line 2 buffered line 3 Buffer size: 4096 bytes

Common Mistakes

Do: Always Flush buffered writers
w := bufio.NewWriter(file)
defer w.Flush() // ensures all data written
fmt.Fprintln(w, "data")
Don't: Forget to Flush
w := bufio.NewWriter(file)
fmt.Fprintln(w, "data")
// data may remain in buffer!
// file closed without Flush = data lost
Do: Check Scanner.Err()
for scanner.Scan() {
    process(scanner.Text())
}
if err := scanner.Err(); err != nil {
    log.Fatal(err)
}
Don't: Ignore Scanner errors
for scanner.Scan() {
    process(scanner.Text())
}
// scanner stopped — was it EOF or error?
// no way to know without checking Err()

Key Rules

  • bufio.Scanner is the idiomatic way to read text line by line — prefer it over manual ReadString('\n')
  • Always call Flush() on a bufio.Writer before closing the underlying writer — use defer w.Flush()
  • The default scanner buffer is 64KB; for longer lines use scanner.Buffer(buf, maxSize)
  • Always check scanner.Err() after the scan loop — Scan() returns false for both EOF and errors
  • Custom SplitFunc must return (0, nil, nil) to request more data when the current buffer is incomplete
Connection: bufio wraps the io.Reader and io.Writer interfaces from Day 7's io Package. The bufio.Scanner is essential for the TCP Servers concept later in this guide, where you read lines from network connections. Buffered writers pair naturally with os & filepath file operations.

regexp: Regular Expressions

I/O & Networking • New

Why It Matters

Go's regexp package uses RE2 syntax, which guarantees linear-time matching (no catastrophic backtracking). It is the standard tool for pattern matching, validation, and text extraction. Compiled patterns (regexp.MustCompile) should be reused as package-level variables for performance.

Core Concepts

  • regexp.Compile(pattern) — returns (*Regexp, error); use when the pattern might be invalid
  • regexp.MustCompile(pattern) — panics on invalid pattern; use for compile-time-known patterns as package-level vars
  • FindString / FindAllString — find first/all matches
  • FindStringSubmatch / FindAllStringSubmatch — include capture group results
  • ReplaceAllString / ReplaceAllStringFunc — replace matches with a string or computed result
  • Named groups: (?P<name>pattern) — accessed via SubexpNames() and FindStringSubmatch

Basic Matching and Finding

package main

import (
    "fmt"
    "regexp"
)

// Compile once at package level for reuse
var emailRe = regexp.MustCompile(`\b[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Z|a-z]{2,}\b`)

func main() {
    text := "Contact alice@example.com or bob@test.org for info."

    // MatchString: does the pattern match?
    fmt.Println("Has email:", emailRe.MatchString(text))

    // FindString: first match
    fmt.Println("First:", emailRe.FindString(text))

    // FindAllString: all matches (-1 = no limit)
    all := emailRe.FindAllString(text, -1)
    fmt.Println("All:", all)
}
Output
Has email: true First: alice@example.com All: [alice@example.com bob@test.org]

Capture Groups and Named Groups

package main

import (
    "fmt"
    "regexp"
)

var logRe = regexp.MustCompile(
    `(?P<timestamp>\d{4}-\d{2}-\d{2})\s+(?P<level>\w+)\s+(?P<message>.+)`,
)

func parseLogLine(line string) map[string]string {
    match := logRe.FindStringSubmatch(line)
    if match == nil {
        return nil
    }

    result := make(map[string]string)
    for i, name := range logRe.SubexpNames() {
        if i != 0 && name != "" {
            result[name] = match[i]
        }
    }
    return result
}

func main() {
    lines := []string{
        "2026-06-10 INFO Server started on port 8080",
        "2026-06-10 ERROR Connection refused",
    }

    for _, line := range lines {
        parsed := parseLogLine(line)
        fmt.Printf("[%s] %s: %s\n", parsed["timestamp"], parsed["level"], parsed["message"])
    }
}
Output
[2026-06-10] INFO: Server started on port 8080 [2026-06-10] ERROR: Connection refused

Replacement with ReplaceAllStringFunc

package main

import (
    "fmt"
    "regexp"
    "strings"
)

var wordRe = regexp.MustCompile(`\b\w+\b`)

func main() {
    text := "hello world from go"

    // Simple replacement
    censored := regexp.MustCompile(`\b\w{5,}\b`).ReplaceAllString(text, "***")
    fmt.Println("Censored:", censored)

    // Functional replacement: capitalize each word
    titled := wordRe.ReplaceAllStringFunc(text, func(s string) string {
        return strings.ToUpper(s[:1]) + s[1:]
    })
    fmt.Println("Titled:", titled)
}
Output
Censored: *** *** from go Titled: Hello World From Go

Common Mistakes

Do: Compile once, reuse
var re = regexp.MustCompile(`\d+`)

func process(s string) string {
    return re.ReplaceAllString(s, "#")
    // compiled once, used many times
}
Don't: Compile in a loop
func process(s string) string {
    re := regexp.MustCompile(`\d+`)
    return re.ReplaceAllString(s, "#")
    // recompiles every call!
}
Do: Use MustCompile for constants
// Known-good pattern: panic is fine
var re = regexp.MustCompile(`^\d{4}-\d{2}-\d{2}$`)
Don't: MustCompile user input
// User-supplied pattern could be invalid
re := regexp.MustCompile(userInput)
// PANIC if userInput is malformed!
// Use Compile() and handle the error

Key Rules

  • Use regexp.MustCompile for compile-time-known patterns as package-level variables
  • Use regexp.Compile for user-supplied patterns and handle the error
  • Go's RE2 engine guarantees linear-time matching — no catastrophic backtracking, but no backreferences either
  • FindAllString(text, -1) returns all matches; pass a positive n to limit results
  • Named groups with (?P<name>...) make complex patterns self-documenting
  • ReplaceAllStringFunc lets you compute replacements dynamically based on each match
Connection: Regular expressions are commonly used with bufio.Scanner to process structured text line by line. The named-group parsing pattern above complements encoding/json for parsing unstructured text that does not fit a JSON schema. In production, compiled regexps are often package-level vars, similar to init functions for one-time setup.

database/sql — Standard Database Interface

I/O & Networking • New

Why It Matters

The database/sql package provides a generic interface around SQL databases. It manages connection pools automatically, supports prepared statements for safety and performance, and integrates with context.Context for timeouts and cancellation. Actual database drivers (PostgreSQL, MySQL, SQLite) plug in via the database/sql/driver interface.

Core Concepts

  • sql.Open(driver, dsn) — returns a *sql.DB (a connection pool, not a single connection); does not connect immediately
  • Connection pool tuning — SetMaxOpenConns, SetMaxIdleConns, SetConnMaxLifetime
  • QueryRow — returns at most one row; use .Scan() to read columns
  • Query — returns *sql.Rows; iterate with rows.Next() + rows.Scan(); always defer rows.Close()
  • Exec — for INSERT/UPDATE/DELETE; returns sql.Result with RowsAffected() and LastInsertId()
  • sql.Null* types — sql.NullString, sql.NullInt64 for nullable columns
  • Transactions — db.BeginTx(ctx, opts) returns *sql.Tx; call tx.Commit() or tx.Rollback()

Opening and Configuring a Connection Pool

package main

import (
    "context"
    "database/sql"
    "fmt"
    "time"

    _ "github.com/mattn/go-sqlite3" // driver registers itself via init()
)

func main() {
    db, err := sql.Open("sqlite3", ":memory:")
    if err != nil {
        panic(err)
    }
    defer db.Close()

    // Configure connection pool
    db.SetMaxOpenConns(25)
    db.SetMaxIdleConns(5)
    db.SetConnMaxLifetime(5 * time.Minute)

    // Verify connectivity
    ctx, cancel := context.WithTimeout(context.Background(), 3*time.Second)
    defer cancel()

    if err := db.PingContext(ctx); err != nil {
        panic(err)
    }
    fmt.Println("Connected to database")
}
Output
Connected to database

Query, QueryRow, and Exec

package main

import (
    "context"
    "database/sql"
    "fmt"

    _ "github.com/mattn/go-sqlite3"
)

type User struct {
    ID    int64
    Name  string
    Email sql.NullString // nullable column
}

func main() {
    db, _ := sql.Open("sqlite3", ":memory:")
    defer db.Close()
    ctx := context.Background()

    // Create table
    db.ExecContext(ctx, `CREATE TABLE users (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        name TEXT NOT NULL,
        email TEXT
    )`)

    // Exec: INSERT
    result, _ := db.ExecContext(ctx,
        "INSERT INTO users (name, email) VALUES (?, ?)",
        "Alice", "alice@example.com")
    id, _ := result.LastInsertId()
    fmt.Printf("Inserted ID: %d\n", id)

    db.ExecContext(ctx,
        "INSERT INTO users (name, email) VALUES (?, ?)",
        "Bob", nil) // NULL email

    // QueryRow: single row
    var user User
    err := db.QueryRowContext(ctx,
        "SELECT id, name, email FROM users WHERE id = ?", 1,
    ).Scan(&user.ID, &user.Name, &user.Email)
    if err == sql.ErrNoRows {
        fmt.Println("Not found")
    } else if err != nil {
        panic(err)
    } else {
        fmt.Printf("User: %s (email: %s)\n", user.Name, user.Email.String)
    }

    // Query: multiple rows
    rows, _ := db.QueryContext(ctx, "SELECT id, name, email FROM users")
    defer rows.Close()

    for rows.Next() {
        var u User
        rows.Scan(&u.ID, &u.Name, &u.Email)
        email := "NULL"
        if u.Email.Valid {
            email = u.Email.String
        }
        fmt.Printf("  [%d] %s (%s)\n", u.ID, u.Name, email)
    }
    if err := rows.Err(); err != nil {
        panic(err)
    }
}
Output
Inserted ID: 1 User: Alice (email: alice@example.com) [1] Alice (alice@example.com) [2] Bob (NULL)

Transactions

package main

import (
    "context"
    "database/sql"
    "fmt"

    _ "github.com/mattn/go-sqlite3"
)

func transferFunds(ctx context.Context, db *sql.DB, from, to int64, amount float64) error {
    tx, err := db.BeginTx(ctx, nil)
    if err != nil {
        return fmt.Errorf("begin tx: %w", err)
    }
    // Always rollback if commit hasn't happened
    defer tx.Rollback()

    // Debit
    res, err := tx.ExecContext(ctx,
        "UPDATE accounts SET balance = balance - ? WHERE id = ? AND balance >= ?",
        amount, from, amount)
    if err != nil {
        return fmt.Errorf("debit: %w", err)
    }
    rows, _ := res.RowsAffected()
    if rows == 0 {
        return fmt.Errorf("insufficient funds or account not found")
    }

    // Credit
    _, err = tx.ExecContext(ctx,
        "UPDATE accounts SET balance = balance + ? WHERE id = ?",
        amount, to)
    if err != nil {
        return fmt.Errorf("credit: %w", err)
    }

    return tx.Commit()
}

func main() {
    db, _ := sql.Open("sqlite3", ":memory:")
    defer db.Close()
    ctx := context.Background()

    db.ExecContext(ctx, `CREATE TABLE accounts (id INTEGER PRIMARY KEY, balance REAL)`)
    db.ExecContext(ctx, "INSERT INTO accounts VALUES (1, 100.0), (2, 50.0)")

    err := transferFunds(ctx, db, 1, 2, 30.0)
    if err != nil {
        fmt.Println("Transfer failed:", err)
        return
    }

    var b1, b2 float64
    db.QueryRowContext(ctx, "SELECT balance FROM accounts WHERE id = 1").Scan(&b1)
    db.QueryRowContext(ctx, "SELECT balance FROM accounts WHERE id = 2").Scan(&b2)
    fmt.Printf("Account 1: $%.2f\n", b1)
    fmt.Printf("Account 2: $%.2f\n", b2)
}
Output
Account 1: $70.00 Account 2: $80.00

Common Mistakes

Do: Use parameterized queries
db.QueryContext(ctx,
    "SELECT * FROM users WHERE name = ?",
    userName) // safe from SQL injection
Don't: Interpolate strings
db.QueryContext(ctx,
    "SELECT * FROM users WHERE name = '"+userName+"'")
// SQL INJECTION vulnerability!
Do: Close rows when done
rows, _ := db.QueryContext(ctx, query)
defer rows.Close()
for rows.Next() { /* ... */ }
if err := rows.Err(); err != nil { /* handle */ }
Don't: Forget rows.Close()
rows, _ := db.QueryContext(ctx, query)
for rows.Next() { /* ... */ }
// connection leaked! pool exhausted over time

Key Rules

  • sql.Open returns a pool, not a connection — it is safe to share across goroutines and should be long-lived
  • Always use parameterized queries (? or $1) to prevent SQL injection — never interpolate user input
  • Always defer rows.Close() after Query and check rows.Err() after the loop
  • Use sql.NullString / sql.NullInt64 for nullable columns — plain Go types cannot represent SQL NULL
  • Transactions: defer tx.Rollback() is safe even after a successful tx.Commit() (it becomes a no-op)
  • Use Context variants (QueryContext, ExecContext) for timeout and cancellation support
Connection: database/sql drivers register themselves via init functions (the blank import _ "driver/pkg" triggers the driver's init()). The Repository Pattern from Day 7 provides the abstraction layer over database/sql for testable data access. Context integration enables request-scoped timeouts for queries.

TCP Servers — Low-Level Networking

I/O & Networking • New

Why It Matters

While net/http handles HTTP, many protocols (database drivers, message queues, custom protocols) use raw TCP. Go's net package makes writing TCP servers straightforward with net.Listen and goroutine-per-connection handling. Understanding TCP servers deepens your understanding of how net/http works under the hood.

Core Concepts

  • net.Listen("tcp", ":port") — creates a listener that accepts incoming TCP connections
  • listener.Accept() — blocks until a new connection arrives; returns net.Conn
  • net.Conn — implements both io.Reader and io.Writer; read/write raw bytes
  • Goroutine-per-connection — go handleConn(conn) after each Accept for concurrent handling
  • bufio.Scanner on a connection — the standard way to read line-based protocols
  • Graceful shutdown — close the listener to stop accepting, drain existing connections

Echo Server

package main

import (
    "bufio"
    "fmt"
    "log"
    "net"
    "strings"
)

func handleConn(conn net.Conn) {
    defer conn.Close()
    addr := conn.RemoteAddr().String()
    log.Printf("New connection from %s", addr)

    scanner := bufio.NewScanner(conn)
    for scanner.Scan() {
        line := scanner.Text()
        if strings.ToLower(line) == "quit" {
            fmt.Fprintln(conn, "Goodbye!")
            return
        }
        // Echo back the line
        fmt.Fprintf(conn, "Echo: %s\n", line)
    }

    if err := scanner.Err(); err != nil {
        log.Printf("Connection error from %s: %v", addr, err)
    }
    log.Printf("Connection closed: %s", addr)
}

func main() {
    listener, err := net.Listen("tcp", ":9000")
    if err != nil {
        log.Fatal(err)
    }
    defer listener.Close()
    log.Println("Echo server listening on :9000")

    for {
        conn, err := listener.Accept()
        if err != nil {
            log.Printf("Accept error: %v", err)
            continue
        }
        go handleConn(conn) // handle each connection concurrently
    }
}
Output (server log)
Echo server listening on :9000 New connection from 127.0.0.1:54321 Connection closed: 127.0.0.1:54321

TCP Chat Server

package main

import (
    "bufio"
    "fmt"
    "log"
    "net"
    "sync"
)

type ChatServer struct {
    mu      sync.Mutex
    clients map[net.Conn]string
}

func NewChatServer() *ChatServer {
    return &ChatServer{clients: make(map[net.Conn]string)}
}

func (s *ChatServer) broadcast(sender net.Conn, msg string) {
    s.mu.Lock()
    defer s.mu.Unlock()
    for conn := range s.clients {
        if conn != sender {
            fmt.Fprintln(conn, msg)
        }
    }
}

func (s *ChatServer) addClient(conn net.Conn, name string) {
    s.mu.Lock()
    s.clients[conn] = name
    s.mu.Unlock()
}

func (s *ChatServer) removeClient(conn net.Conn) {
    s.mu.Lock()
    delete(s.clients, conn)
    s.mu.Unlock()
}

func (s *ChatServer) handle(conn net.Conn) {
    defer conn.Close()

    fmt.Fprint(conn, "Enter your name: ")
    scanner := bufio.NewScanner(conn)
    if !scanner.Scan() {
        return
    }
    name := scanner.Text()

    s.addClient(conn, name)
    defer s.removeClient(conn)

    s.broadcast(conn, fmt.Sprintf("*** %s has joined ***", name))
    log.Printf("%s connected", name)

    for scanner.Scan() {
        msg := scanner.Text()
        s.broadcast(conn, fmt.Sprintf("%s: %s", name, msg))
    }

    s.broadcast(conn, fmt.Sprintf("*** %s has left ***", name))
    log.Printf("%s disconnected", name)
}

func main() {
    server := NewChatServer()
    listener, err := net.Listen("tcp", ":9001")
    if err != nil {
        log.Fatal(err)
    }
    defer listener.Close()
    log.Println("Chat server listening on :9001")

    for {
        conn, err := listener.Accept()
        if err != nil {
            log.Printf("Accept error: %v", err)
            continue
        }
        go server.handle(conn)
    }
}

TCP Client

package main

import (
    "bufio"
    "fmt"
    "net"
    "time"
)

func main() {
    // Connect with timeout
    conn, err := net.DialTimeout("tcp", "localhost:9000", 5*time.Second)
    if err != nil {
        fmt.Println("Connection failed:", err)
        return
    }
    defer conn.Close()

    // Set read/write deadlines
    conn.SetDeadline(time.Now().Add(30 * time.Second))

    // Send a message
    fmt.Fprintln(conn, "Hello, server!")

    // Read response
    scanner := bufio.NewScanner(conn)
    if scanner.Scan() {
        fmt.Println("Server says:", scanner.Text())
    }

    // Disconnect
    fmt.Fprintln(conn, "quit")
}

Common Mistakes

Do: Close connections in handlers
func handle(conn net.Conn) {
    defer conn.Close() // always close
    // process connection...
}
Don't: Forget to close
func handle(conn net.Conn) {
    // process connection...
    // no Close() = file descriptor leak
    // eventually "too many open files"
}
Do: Set deadlines on connections
conn.SetDeadline(time.Now().Add(30 * time.Second))
// or set per-operation:
conn.SetReadDeadline(time.Now().Add(10 * time.Second))
Don't: Use no deadline
scanner := bufio.NewScanner(conn)
scanner.Scan() // blocks forever if client
// disconnects without closing cleanly

Key Rules

  • Always defer conn.Close() in the handler goroutine to prevent file descriptor leaks
  • Launch a new goroutine per connection: go handleConn(conn) after Accept()
  • Use bufio.Scanner for line-based protocols; use io.Read for binary protocols
  • Set deadlines (SetDeadline, SetReadDeadline, SetWriteDeadline) to prevent hung connections
  • Protect shared state (client lists, broadcast channels) with sync.Mutex or channels
  • Use net.DialTimeout on the client side to avoid blocking indefinitely on connect
Connection: TCP servers use goroutines for concurrent connection handling and sync.Mutex for shared state protection (both from the Concurrency week). net.Conn implements io.Reader/io.Writer, so all io Package and bufio patterns apply directly to network connections. The Graceful Shutdown concept (next) builds on this foundation.

Graceful Shutdown — Clean Termination

I/O & Networking • New

Why It Matters

Production services must shut down cleanly when receiving termination signals (SIGINT, SIGTERM). Graceful shutdown means: stop accepting new requests, wait for in-flight requests to complete (with a timeout), release resources, and exit. Without it, active requests get dropped, database transactions are left half-complete, and data can be corrupted.

Core Concepts

  • os.Signal / signal.Notify — register to receive OS signals (SIGINT from Ctrl+C, SIGTERM from kill)
  • context.WithCancel — create a cancellable context; cancel it when the signal arrives
  • http.Server.Shutdown(ctx) — gracefully shuts down the HTTP server: stops accepting, waits for in-flight requests
  • sync.WaitGroup — track in-flight goroutines; Wait() blocks until all are done
  • Shutdown timeout — use context.WithTimeout to force-close after a deadline if draining takes too long

HTTP Server with Graceful Shutdown

package main

import (
    "context"
    "fmt"
    "log"
    "net/http"
    "os"
    "os/signal"
    "syscall"
    "time"
)

func main() {
    mux := http.NewServeMux()
    mux.HandleFunc("/", func(w http.ResponseWriter, r *http.Request) {
        // Simulate work
        time.Sleep(2 * time.Second)
        fmt.Fprintln(w, "Hello, World!")
    })
    mux.HandleFunc("/health", func(w http.ResponseWriter, r *http.Request) {
        w.WriteHeader(http.StatusOK)
        fmt.Fprintln(w, "OK")
    })

    server := &http.Server{
        Addr:         ":8080",
        Handler:      mux,
        ReadTimeout:  10 * time.Second,
        WriteTimeout: 15 * time.Second,
        IdleTimeout:  60 * time.Second,
    }

    // Start server in a goroutine
    go func() {
        log.Println("Server starting on :8080")
        if err := server.ListenAndServe(); err != http.ErrServerClosed {
            log.Fatalf("Server error: %v", err)
        }
    }()

    // Wait for interrupt signal
    quit := make(chan os.Signal, 1)
    signal.Notify(quit, syscall.SIGINT, syscall.SIGTERM)
    sig := <-quit
    log.Printf("Received signal: %v. Shutting down...", sig)

    // Give in-flight requests 30 seconds to complete
    ctx, cancel := context.WithTimeout(context.Background(), 30*time.Second)
    defer cancel()

    if err := server.Shutdown(ctx); err != nil {
        log.Printf("Forced shutdown: %v", err)
    }

    log.Println("Server stopped gracefully")
}
Output
Server starting on :8080 Received signal: interrupt. Shutting down... Server stopped gracefully

TCP Server with Graceful Shutdown

package main

import (
    "bufio"
    "context"
    "fmt"
    "log"
    "net"
    "os"
    "os/signal"
    "sync"
    "syscall"
    "time"
)

type TCPServer struct {
    listener net.Listener
    wg       sync.WaitGroup
    ctx      context.Context
    cancel   context.CancelFunc
}

func NewTCPServer(addr string) (*TCPServer, error) {
    listener, err := net.Listen("tcp", addr)
    if err != nil {
        return nil, err
    }
    ctx, cancel := context.WithCancel(context.Background())
    return &TCPServer{
        listener: listener,
        ctx:      ctx,
        cancel:   cancel,
    }, nil
}

func (s *TCPServer) handleConn(conn net.Conn) {
    defer s.wg.Done()
    defer conn.Close()

    scanner := bufio.NewScanner(conn)
    for scanner.Scan() {
        select {
        case <-s.ctx.Done():
            fmt.Fprintln(conn, "Server shutting down. Goodbye!")
            return
        default:
            fmt.Fprintf(conn, "Echo: %s\n", scanner.Text())
        }
    }
}

func (s *TCPServer) Serve() {
    log.Printf("TCP server listening on %s", s.listener.Addr())

    for {
        conn, err := s.listener.Accept()
        if err != nil {
            select {
            case <-s.ctx.Done():
                return // shutdown requested
            default:
                log.Printf("Accept error: %v", err)
                continue
            }
        }
        s.wg.Add(1)
        go s.handleConn(conn)
    }
}

func (s *TCPServer) Shutdown(timeout time.Duration) {
    log.Println("Initiating graceful shutdown...")

    // 1. Stop accepting new connections
    s.cancel()
    s.listener.Close()

    // 2. Wait for in-flight connections with timeout
    done := make(chan struct{})
    go func() {
        s.wg.Wait()
        close(done)
    }()

    select {
    case <-done:
        log.Println("All connections drained")
    case <-time.After(timeout):
        log.Println("Shutdown timed out, forcing close")
    }
}

func main() {
    server, err := NewTCPServer(":9000")
    if err != nil {
        log.Fatal(err)
    }

    go server.Serve()

    // Wait for signal
    quit := make(chan os.Signal, 1)
    signal.Notify(quit, syscall.SIGINT, syscall.SIGTERM)
    <-quit

    server.Shutdown(10 * time.Second)
    log.Println("Server stopped")
}
Output
TCP server listening on [::]:9000 Initiating graceful shutdown... All connections drained Server stopped

Worker Pool with Graceful Shutdown

package main

import (
    "context"
    "fmt"
    "log"
    "os"
    "os/signal"
    "sync"
    "syscall"
    "time"
)

func worker(ctx context.Context, id int, jobs <-chan int, wg *sync.WaitGroup) {
    defer wg.Done()
    for {
        select {
        case <-ctx.Done():
            log.Printf("Worker %d: shutting down", id)
            return
        case job, ok := <-jobs:
            if !ok {
                log.Printf("Worker %d: no more jobs", id)
                return
            }
            log.Printf("Worker %d: processing job %d", id, job)
            time.Sleep(500 * time.Millisecond) // simulate work
        }
    }
}

func main() {
    ctx, cancel := context.WithCancel(context.Background())
    var wg sync.WaitGroup
    jobs := make(chan int, 100)

    // Start workers
    for i := 1; i <= 3; i++ {
        wg.Add(1)
        go worker(ctx, i, jobs, &wg)
    }

    // Produce jobs in background
    go func() {
        for i := 1; ; i++ {
            select {
            case <-ctx.Done():
                close(jobs)
                return
            case jobs <- i:
            }
            time.Sleep(200 * time.Millisecond)
        }
    }()

    // Wait for signal
    quit := make(chan os.Signal, 1)
    signal.Notify(quit, syscall.SIGINT, syscall.SIGTERM)
    <-quit

    fmt.Println("\nShutting down workers...")
    cancel() // signal all workers to stop

    wg.Wait()
    fmt.Println("All workers stopped")
}
Output
Worker 1: processing job 1 Worker 2: processing job 2 Worker 3: processing job 3 Worker 1: processing job 4 Shutting down workers... Worker 2: shutting down Worker 3: shutting down Worker 1: shutting down All workers stopped

Common Mistakes

Do: Use buffered signal channel
quit := make(chan os.Signal, 1)
signal.Notify(quit, syscall.SIGINT)
// buffer size 1: signal is not lost if
// we're not yet reading from the channel
Don't: Use unbuffered signal channel
quit := make(chan os.Signal)
signal.Notify(quit, syscall.SIGINT)
// unbuffered: if signal arrives before
// we read, it's DROPPED!
Do: Set a shutdown timeout
ctx, cancel := context.WithTimeout(
    context.Background(), 30*time.Second)
defer cancel()
server.Shutdown(ctx)
// won't wait forever
Don't: Wait forever for drain
server.Shutdown(context.Background())
// if a client hangs, server never exits
// deploy pipeline blocks, no rollback

Key Rules

  • Always use a buffered channel of size 1 for signal.Notify to avoid missing signals
  • Handle both SIGINT (Ctrl+C) and SIGTERM (deployment systems like Kubernetes send SIGTERM)
  • Set a shutdown timeout with context.WithTimeout — never wait indefinitely for in-flight requests
  • http.Server.Shutdown stops accepting new connections and waits for in-flight ones — check the returned error
  • Use sync.WaitGroup to track goroutines that must finish before exit
  • The shutdown sequence is: stop accepting, drain in-flight, release resources, exit
Connection: Graceful shutdown combines nearly every concurrency primitive: goroutines for concurrent handling, channels for signal delivery, select for multiplexing, context for cancellation propagation, sync.WaitGroup for completion tracking, and sync.Mutex for shared state. The TCP Server and net/http concepts provide the server patterns being gracefully shut down.
