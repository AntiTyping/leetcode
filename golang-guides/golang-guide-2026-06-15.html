errgroup

Concurrency • 7 days ago • rep 1

Quick Reminder

errgroup.Group manages a group of goroutines. If any returns an error, g.Wait() returns the first error. Combine with errgroup.WithContext for automatic cancellation on first failure.

Key Pattern

g, ctx := errgroup.WithContext(context.Background())
for _, url := range urls {
    url := url
    g.Go(func() error {
        req, _ := http.NewRequestWithContext(ctx, "GET", url, nil)
        resp, err := http.DefaultClient.Do(req)
        if err != nil { return err }
        defer resp.Body.Close()
        return nil
    })
}
if err := g.Wait(); err != nil {
    log.Fatal(err)
}
Challenge Exercise

Write a function that fetches N URLs concurrently using errgroup with a limit of 3 concurrent goroutines (g.SetLimit(3)). Return the first error encountered.

#1 Pitfall: Not using the context from errgroup.WithContext — other goroutines keep running even after one fails.

Mocking: Interfaces for Testability

Testing & Tooling • 7 days ago • rep 0

Quick Reminder

Define small interfaces at the point of use. In production, pass the real implementation; in tests, pass a mock struct that satisfies the interface. No mocking framework needed.

Key Pattern

type UserStore interface {
    GetUser(id string) (*User, error)
}

type userService struct { store UserStore }

// In test:
type mockStore struct { users map[string]*User }
func (m *mockStore) GetUser(id string) (*User, error) {
    u, ok := m.users[id]
    if !ok { return nil, ErrNotFound }
    return u, nil
}
Challenge Exercise

Define a Cache interface with Get(key string) (string, bool) and Set(key, value string). Write a service that uses it, then create a mock implementation for testing that tracks call counts.

#1 Pitfall: Defining interfaces in the package that implements them (Java-style). In Go, define interfaces where they're consumed.

Maps

Go Foundations • 11 days ago • rep 2

Quick Reminder

map[K]V provides O(1) lookup. Iteration order is random. Use comma-ok to check existence. Maps are not safe for concurrent access — use sync.Map or a mutex.

Key Pattern

m := map[string]int{"a": 1, "b": 2}
if v, ok := m["c"]; ok {
    fmt.Println(v)
} else {
    fmt.Println("not found")
}
delete(m, "a")
Challenge Exercise

Write a WordFrequency(text string) map[string]int function that counts word frequencies (case-insensitive), then return the top 3 most frequent words.

#1 Pitfall: Concurrent reads/writes to a map cause a fatal panic — always protect shared maps with sync.RWMutex.

Strings & Runes

Go Foundations • 11 days ago • rep 2

Quick Reminder

Go strings are UTF-8 encoded byte slices. len(s) gives byte count, not character count. Use []rune(s) or utf8.RuneCountInString(s) for character count. strings.Builder for efficient concatenation.

Key Pattern

s := "Hello, 世界"
fmt.Println(len(s))                    // 13 (bytes)
fmt.Println(utf8.RuneCountInString(s)) // 9 (characters)
for i, r := range s {
    fmt.Printf("%d: %c\n", i, r)
}
var b strings.Builder
b.WriteString("Hello")
b.WriteString(" World")
fmt.Println(b.String())
Challenge Exercise

Write a function ReverseString(s string) string that correctly reverses a UTF-8 string (not just bytes). "Hello, 世界" → "界世 ,olleH".

#1 Pitfall: Using s[i] gives a byte, not a character — use []rune(s) for character access.

Structs & Methods

Go Foundations • 11 days ago • rep 2

Quick Reminder

Use value receivers for read-only methods on small structs. Use pointer receivers when you need to mutate or the struct is large. Embedding promotes methods.

Key Pattern

type Circle struct { Radius float64 }
func (c Circle) Area() float64 { return math.Pi * c.Radius * c.Radius }
func (c *Circle) Scale(factor float64) { c.Radius *= factor }

type Shape struct {
    Circle // embedding
    Color string
}
s := Shape{Circle{5}, "red"}
fmt.Println(s.Area()) // promoted method
Challenge Exercise

Create a Stack[T any] generic struct with Push, Pop, Peek, Len, and IsEmpty methods. Use pointer receivers for mutations.

#1 Pitfall: Defining a method with a value receiver that tries to mutate the struct — the change is lost because it modifies a copy.

text/template & html/template

Standard Library

Go's template packages power config generation, email templates, and HTML rendering. html/template auto-escapes to prevent XSS — crucial for web apps.

Example 1 — Basic text template

package main

import (
    "os"
    "text/template"
)

func main() {
    tmpl := template.Must(template.New("greeting").Parse(
        "Hello, {{.Name}}! You have {{.Count}} messages.\n"))

    data := struct {
        Name  string
        Count int
    }{"Alice", 5}

    tmpl.Execute(os.Stdout, data)
}
Output
Hello, Alice! You have 5 messages.

Example 2 — Conditionals, loops, and pipelines

package main

import (
    "os"
    "strings"
    "text/template"
)

func main() {
    funcMap := template.FuncMap{
        "upper": strings.ToUpper,
    }

    tmpl := template.Must(template.New("report").Funcs(funcMap).Parse(`
{{- range .Items}}
- {{.Name | upper}}: ${{printf "%.2f" .Price}}
{{- end}}
{{if gt (len .Items) 3}}(and more...){{end}}
Total: {{len .Items}} items
`))

    data := struct {
        Items []struct{ Name string; Price float64 }
    }{
        Items: []struct{ Name string; Price float64 }{
            {"Widget", 9.99},
            {"Gadget", 24.50},
        },
    }
    tmpl.Execute(os.Stdout, data)
}
Output
- WIDGET: $9.99 - GADGET: $24.50 Total: 2 items

Example 3 — html/template for safe HTML

package main

import (
    "html/template"
    "os"
)

func main() {
    tmpl := template.Must(template.New("page").Parse(`
<!DOCTYPE html>
<html>
<body>
    <h1>{{.Title}}</h1>
    <p>{{.Content}}</p>
    <a href="/user/{{.UserID}}">Profile</a>
</body>
</html>`))

    data := struct {
        Title   string
        Content string
        UserID  string
    }{
        Title:   "Welcome",
        Content: "<script>alert('xss')</script>",  // auto-escaped!
        UserID:  "42",
    }
    tmpl.Execute(os.Stdout, data)
}
Output
<!DOCTYPE html> <html> <body> <h1>Welcome</h1> <p>&lt;script&gt;alert(&#39;xss&#39;)&lt;/script&gt;</p> <a href="/user/42">Profile</a> </body> </html>

Example 4 — Nested templates and blocks

package main

import (
    "os"
    "text/template"
)

func main() {
    base := `{{define "base"}}Header
{{template "content" .}}
Footer{{end}}`

    page := `{{define "content"}}Page: {{.Title}}{{end}}`

    tmpl := template.Must(template.New("").Parse(base))
    template.Must(tmpl.Parse(page))

    tmpl.ExecuteTemplate(os.Stdout, "base", struct{ Title string }{"Home"})
}
Output
Header Page: Home Footer

Common Mistakes

Don't: text/template for HTML
// No XSS protection!
import "text/template"
tmpl.Execute(w, userInput)
Do: html/template for HTML
// Auto-escapes by context
import "html/template"
tmpl.Execute(w, userInput)
Don't: Ignore parse errors
tmpl, _ := template.New("x").Parse(s)
// silent failure on bad template
Do: Use template.Must
tmpl := template.Must(
    template.New("x").Parse(s))
// panics early on bad templates
Don't: Ignore Execute error
tmpl.Execute(w, data)
// partial output already written
Do: Execute to buffer first
var buf bytes.Buffer
if err := tmpl.Execute(&buf, data); err != nil {
    http.Error(w, "error", 500)
    return
}
buf.WriteTo(w)

Key Rules

  • Use html/template for all HTML output — it auto-escapes by context (HTML, JS, URL, CSS).
  • Use template.FuncMap to add custom functions — register before Parse().
  • Use {{- }} and {{ -}} to trim whitespace around template actions.

crypto Package & TLS

Security

Every production Go service needs cryptographic operations — hashing passwords, generating tokens, signing data, and configuring TLS. Go's crypto packages are battle-tested and used by the standard net/http server.

Example 1 — Hashing with crypto/sha256

package main

import (
    "crypto/sha256"
    "encoding/hex"
    "fmt"
)

func main() {
    data := []byte("hello world")
    hash := sha256.Sum256(data)
    fmt.Println(hex.EncodeToString(hash[:]))
    // a591a6d40bf420404a011733cfb7b190d62c65bf0bcda32b57b277d9ad9f146e

    // Incremental hashing for large data
    h := sha256.New()
    h.Write([]byte("hello "))
    h.Write([]byte("world"))
    fmt.Println(hex.EncodeToString(h.Sum(nil)))
    // same hash
}

Example 2 — Secure random with crypto/rand

package main

import (
    "crypto/rand"
    "encoding/base64"
    "encoding/hex"
    "fmt"
    "math/big"
)

func main() {
    // Random bytes
    token := make([]byte, 32)
    rand.Read(token)
    fmt.Println(hex.EncodeToString(token))

    // URL-safe token
    fmt.Println(base64.URLEncoding.EncodeToString(token))

    // Random integer in [0, max)
    n, _ := rand.Int(rand.Reader, big.NewInt(100))
    fmt.Println("Random 0-99:", n)
}

Example 3 — HMAC for message authentication

package main

import (
    "crypto/hmac"
    "crypto/sha256"
    "encoding/hex"
    "fmt"
)

func sign(key, message []byte) string {
    mac := hmac.New(sha256.New, key)
    mac.Write(message)
    return hex.EncodeToString(mac.Sum(nil))
}

func verify(key, message []byte, signature string) bool {
    expected := sign(key, message)
    return hmac.Equal([]byte(expected), []byte(signature))
}

func main() {
    key := []byte("my-secret-key")
    msg := []byte("transfer $100 to Alice")

    sig := sign(key, msg)
    fmt.Println("Signature:", sig)
    fmt.Println("Valid:", verify(key, msg, sig))       // true
    fmt.Println("Tampered:", verify(key, []byte("transfer $1000"), sig)) // false
}

Example 4 — TLS configuration for HTTP server

package main

import (
    "crypto/tls"
    "fmt"
    "net/http"
)

func main() {
    tlsConfig := &tls.Config{
        MinVersion: tls.VersionTLS13,
        CurvePreferences: []tls.CurveID{
            tls.X25519,
            tls.CurveP256,
        },
    }

    server := &http.Server{
        Addr:      ":8443",
        TLSConfig: tlsConfig,
        Handler: http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
            fmt.Fprintln(w, "Hello, TLS!")
        }),
    }

    // In production: server.ListenAndServeTLS("cert.pem", "key.pem")
    fmt.Println("Server configured with TLS 1.3+")
}

Common Mistakes

Don't: math/rand for security
import "math/rand"
token := rand.Intn(1000000)
// predictable!
Do: crypto/rand for security
import "crypto/rand"
token := make([]byte, 32)
rand.Read(token)
// cryptographically secure
Don't: Compare HMACs with ==
if computedMAC == expectedMAC {
    // timing attack!
}
Do: Use hmac.Equal()
if hmac.Equal(computedMAC, expectedMAC) {
    // constant-time comparison
}
Don't: Allow old TLS
tlsConfig := &tls.Config{
    // MinVersion defaults to TLS 1.0
}
Do: Require TLS 1.3
tlsConfig := &tls.Config{
    MinVersion: tls.VersionTLS13,
}

Key Rules

  • crypto/rand for all security-critical randomness. math/rand only for non-security use.
  • Use hmac.Equal() instead of == for constant-time comparison.
  • Set MinVersion: tls.VersionTLS13 in production TLS configs.

Advanced Runtime

sync/atomic — Atomic Operations

Concurrency

Atomic operations provide lock-free thread safety for simple counters, flags, and pointers. They're much faster than mutexes for single-value operations and are the building block for lock-free data structures.

Example 1 — Atomic counter

package main

import (
    "fmt"
    "sync"
    "sync/atomic"
)

func main() {
    var counter atomic.Int64
    var wg sync.WaitGroup

    for i := 0; i < 1000; i++ {
        wg.Add(1)
        go func() {
            defer wg.Done()
            counter.Add(1)
        }()
    }

    wg.Wait()
    fmt.Println("Counter:", counter.Load()) // always 1000
}
Output
Counter: 1000

Example 2 — Atomic bool for shutdown flag

package main

import (
    "fmt"
    "sync/atomic"
    "time"
)

func main() {
    var shutdown atomic.Bool

    // Worker goroutine
    go func() {
        for !shutdown.Load() {
            fmt.Println("Working...")
            time.Sleep(100 * time.Millisecond)
        }
        fmt.Println("Worker stopped")
    }()

    time.Sleep(350 * time.Millisecond)
    shutdown.Store(true)
    time.Sleep(150 * time.Millisecond)
}
Output
Working... Working... Working... Worker stopped

Example 3 — atomic.Value for config hot-reload

package main

import (
    "fmt"
    "sync/atomic"
)

type Config struct {
    MaxConns int
    Debug    bool
}

func main() {
    var config atomic.Value
    config.Store(&Config{MaxConns: 100, Debug: false})

    // Read (concurrent-safe, lock-free)
    cfg := config.Load().(*Config)
    fmt.Printf("MaxConns: %d, Debug: %v\n", cfg.MaxConns, cfg.Debug)

    // Hot reload — store a whole new config
    config.Store(&Config{MaxConns: 200, Debug: true})

    cfg2 := config.Load().(*Config)
    fmt.Printf("MaxConns: %d, Debug: %v\n", cfg2.MaxConns, cfg2.Debug)
}
Output
MaxConns: 100, Debug: false MaxConns: 200, Debug: true

Example 4 — CompareAndSwap for lock-free patterns

package main

import (
    "fmt"
    "sync"
    "sync/atomic"
)

func main() {
    var max atomic.Int64
    max.Store(0)

    var wg sync.WaitGroup
    values := []int64{5, 3, 9, 1, 7, 8, 2, 6, 4, 10}

    for _, v := range values {
        v := v
        wg.Add(1)
        go func() {
            defer wg.Done()
            for {
                old := max.Load()
                if v <= old { return }
                if max.CompareAndSwap(old, v) { return }
            }
        }()
    }

    wg.Wait()
    fmt.Println("Max:", max.Load()) // 10
}

Common Mistakes

Don't: Atomic for multi-field updates
var x, y atomic.Int64
// Can't atomically update both
x.Store(1)
y.Store(2) // not atomic together
Do: Mutex for related values
var mu sync.Mutex
mu.Lock()
x = 1
y = 2 // both updated atomically
mu.Unlock()
Don't: Store nil in atomic.Value
var v atomic.Value
v.Store(nil) // panics!
Do: Store typed nil pointer
var v atomic.Value
v.Store((*Config)(nil)) // OK
Don't: Mix atomic and non-atomic
var counter int64
atomic.AddInt64(&counter, 1)
fmt.Println(counter) // data race!
Do: All access must be atomic
var counter atomic.Int64
counter.Add(1)
fmt.Println(counter.Load()) // safe

Key Rules

  • Use atomic types (atomic.Int64, atomic.Bool, atomic.Pointer[T]) for single-value concurrent access — faster than Mutex.
  • Use atomic.Value for swapping entire structs (config reload, cache replacement).
  • Use Mutex when you need to atomically update multiple related fields.

cgo — Calling C from Go

Interop

cgo lets Go programs call C libraries — essential for wrapping existing C/C++ libraries (SQLite, OpenSSL, system APIs). Understanding cgo trade-offs helps decide when to use it vs. pure Go alternatives.

Example 1 — Basic cgo

package main

/*
#include <stdio.h>
#include <stdlib.h>

void greet(const char* name) {
    printf("Hello from C, %s!\n", name);
}
*/
import "C"
import (
    "fmt"
    "unsafe"
)

func main() {
    name := C.CString("Alice")
    defer C.free(unsafe.Pointer(name))

    C.greet(name)

    // C arithmetic
    result := C.int(40) + C.int(2)
    fmt.Println("C says:", int(result))
}
Output
Hello from C, Alice! C says: 42

Example 2 — Passing data between Go and C

package main

/*
#include <stdlib.h>
#include <string.h>

int sum_array(int* arr, int len) {
    int total = 0;
    for (int i = 0; i < len; i++) {
        total += arr[i];
    }
    return total;
}

char* reverse_string(const char* s) {
    int len = strlen(s);
    char* result = (char*)malloc(len + 1);
    for (int i = 0; i < len; i++) {
        result[i] = s[len - 1 - i];
    }
    result[len] = '\0';
    return result;
}
*/
import "C"
import (
    "fmt"
    "unsafe"
)

func main() {
    // Pass a Go slice to C
    goSlice := []C.int{10, 20, 30, 40, 50}
    total := C.sum_array(&goSlice[0], C.int(len(goSlice)))
    fmt.Println("Sum:", int(total)) // 150

    // Get a C string back, free it when done
    input := C.CString("Hello, Go!")
    defer C.free(unsafe.Pointer(input))

    reversed := C.reverse_string(input)
    defer C.free(unsafe.Pointer(reversed))
    fmt.Println("Reversed:", C.GoString(reversed)) // !oG ,olleH
}

Example 3 — Wrapping a C library (math)

package main

/*
#cgo LDFLAGS: -lm
#include <math.h>
*/
import "C"
import "fmt"

func main() {
    // Call C math functions
    result := C.sqrt(C.double(144))
    fmt.Printf("sqrt(144) = %.0f\n", float64(result)) // 12

    result2 := C.pow(C.double(2), C.double(10))
    fmt.Printf("2^10 = %.0f\n", float64(result2)) // 1024
}

Common Mistakes

Don't: Forget to free C.CString
name := C.CString("Alice")
C.greet(name)
// memory leak!
Do: Defer C.free immediately
name := C.CString("Alice")
defer C.free(unsafe.Pointer(name))
C.greet(name)
Don't: Blank line before import "C"
/*
#include <stdio.h>
*/

import "C"  // cgo won't see the C code!
Do: Comment and import contiguous
/*
#include <stdio.h>
*/
import "C"  // no blank line between
Don't: cgo when pure Go exists
// mattn/go-sqlite3 uses cgo
// disables cross-compilation
import _ "github.com/mattn/go-sqlite3"
Do: Prefer pure Go
// modernc.org/sqlite is pure Go
// cross-compiles, no cgo needed
import _ "modernc.org/sqlite"

Key Rules

  • cgo has significant overhead per call (~100ns vs ~5ns for Go calls) — avoid it in hot paths.
  • cgo disables cross-compilation (CGO_ENABLED=0 for static builds). Consider the deployment impact.
  • Always free C-allocated memory. Use defer C.free(unsafe.Pointer(...)) immediately after allocation.

go/ast & go/parser — Analyzing Go Code

Tooling

Go's standard library includes packages for parsing Go source code into ASTs. This powers custom linters, code generators, refactoring tools, and IDE features — all without third-party dependencies.

Example 1 — Parsing a Go file

package main

import (
    "fmt"
    "go/ast"
    "go/parser"
    "go/token"
)

func main() {
    src := `
package example

import "fmt"

func Hello(name string) string {
    return fmt.Sprintf("Hello, %s!", name)
}

func Add(a, b int) int {
    return a + b
}
`
    fset := token.NewFileSet()
    file, err := parser.ParseFile(fset, "example.go", src, parser.ParseComments)
    if err != nil { panic(err) }

    fmt.Println("Package:", file.Name.Name)
    for _, decl := range file.Decls {
        if fn, ok := decl.(*ast.FuncDecl); ok {
            fmt.Printf("Function: %s (line %d)\n",
                fn.Name.Name, fset.Position(fn.Pos()).Line)
        }
    }
}
Output
Package: example Function: Hello (line 6) Function: Add (line 10)

Example 2 — Walking the AST with ast.Inspect

package main

import (
    "fmt"
    "go/ast"
    "go/parser"
    "go/token"
)

func main() {
    src := `package main
func main() {
    x := 10
    if x > 5 {
        for i := 0; i < x; i++ {
            fmt.Println(i)
        }
    }
}`
    fset := token.NewFileSet()
    file, _ := parser.ParseFile(fset, "", src, 0)

    counts := map[string]int{}
    ast.Inspect(file, func(n ast.Node) bool {
        switch n.(type) {
        case *ast.IfStmt:
            counts["if"]++
        case *ast.ForStmt:
            counts["for"]++
        case *ast.RangeStmt:
            counts["range"]++
        case *ast.CallExpr:
            counts["call"]++
        }
        return true
    })

    for kind, count := range counts {
        fmt.Printf("%s: %d\n", kind, count)
    }
}
Output
if: 1 for: 1 call: 1

Example 3 — Extracting struct fields and tags

package main

import (
    "fmt"
    "go/ast"
    "go/parser"
    "go/token"
    "strings"
)

func main() {
    src := `package models
type User struct {
    ID    int    ` + "`" + `json:"id" db:"user_id"` + "`" + `
    Name  string ` + "`" + `json:"name" db:"full_name"` + "`" + `
    Email string ` + "`" + `json:"email"` + "`" + `
}
`
    fset := token.NewFileSet()
    file, _ := parser.ParseFile(fset, "", src, parser.ParseComments)

    ast.Inspect(file, func(n ast.Node) bool {
        ts, ok := n.(*ast.TypeSpec)
        if !ok { return true }

        st, ok := ts.Type.(*ast.StructType)
        if !ok { return true }

        fmt.Printf("Struct: %s\n", ts.Name.Name)
        for _, field := range st.Fields.List {
            name := field.Names[0].Name
            tag := ""
            if field.Tag != nil {
                tag = strings.Trim(field.Tag.Value, "`")
            }
            fmt.Printf("  %s: %s\n", name, tag)
        }
        return true
    })
}
Output
Struct: User ID: json:"id" db:"user_id" Name: json:"name" db:"full_name" Email: json:"email"

Example 4 — Finding function complexity (counting branches)

package main

import (
    "fmt"
    "go/ast"
    "go/parser"
    "go/token"
)

func cyclomaticComplexity(fn *ast.FuncDecl) int {
    complexity := 1
    ast.Inspect(fn.Body, func(n ast.Node) bool {
        switch n.(type) {
        case *ast.IfStmt, *ast.ForStmt, *ast.RangeStmt,
             *ast.CaseClause, *ast.CommClause:
            complexity++
        case *ast.BinaryExpr:
            if be, ok := n.(*ast.BinaryExpr); ok {
                if be.Op.String() == "&&" || be.Op.String() == "||" {
                    complexity++
                }
            }
        }
        return true
    })
    return complexity
}

func main() {
    src := `package main
func process(x int) string {
    if x < 0 { return "negative" }
    if x == 0 { return "zero" }
    for i := 0; i < x; i++ {
        if i%2 == 0 && i > 5 { return "special" }
    }
    return "positive"
}`
    fset := token.NewFileSet()
    file, _ := parser.ParseFile(fset, "", src, 0)

    for _, decl := range file.Decls {
        if fn, ok := decl.(*ast.FuncDecl); ok {
            fmt.Printf("%s: complexity = %d\n",
                fn.Name.Name, cyclomaticComplexity(fn))
        }
    }
}
Output
process: complexity = 5

Common Mistakes

Don't: Skip token.NewFileSet()
file, _ := parser.ParseFile(nil, "", src, 0)
// positions will be wrong!
Do: Always create a FileSet
fset := token.NewFileSet()
file, _ := parser.ParseFile(fset, "", src, 0)
// positions are correct
Don't: Modify AST during Inspect
ast.Inspect(file, func(n ast.Node) bool {
    if id, ok := n.(*ast.Ident); ok {
        id.Name = "renamed" // subtle bugs!
    }
    return true
})
Do: Collect changes, apply after
var renames []*ast.Ident
ast.Inspect(file, func(n ast.Node) bool {
    if id, ok := n.(*ast.Ident); ok {
        renames = append(renames, id)
    }
    return true
})
for _, id := range renames {
    id.Name = "renamed"
}
Don't: Forget ParseComments flag
file, _ := parser.ParseFile(fset, "", src, 0)
// file.Comments is nil
Do: Include flag when needed
file, _ := parser.ParseFile(fset, "", src,
    parser.ParseComments)
// doc comments available

Key Rules

  • go/parser + go/ast work on syntax only — no type information. For types, use go/types and golang.org/x/tools/go/packages.
  • Use ast.Inspect for simple read-only analysis. Use ast.Walk with a custom visitor for more control.
  • The AST matches Go's grammar closely — each construct has a corresponding ast.* type.
