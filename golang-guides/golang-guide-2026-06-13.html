Slices Deep Dive

Foundations • 3 days overdue • rep 2

Quick Reminder

A slice is a descriptor containing a pointer to an underlying array, a length, and a capacity. make([]T, len, cap) allocates the array. append may allocate a new array when capacity is exceeded, breaking shared references. Slice expressions a[low:high:max] control both length and capacity, preventing unintended overwrites of shared backing arrays.

Key Pattern

// make vs literal
s1 := make([]int, 0, 10) // len=0, cap=10
s2 := []int{1, 2, 3}     // len=3, cap=3

// append grows capacity when needed
s1 = append(s1, 1, 2, 3)
fmt.Println(len(s1), cap(s1)) // 3, 10

// copy is independent of source
src := []int{1, 2, 3}
dst := make([]int, len(src))
copy(dst, src)
dst[0] = 99 // src is unchanged

// Full slice expression limits capacity
original := []int{1, 2, 3, 4, 5}
sub := original[1:3:3] // len=2, cap=2
sub = append(sub, 99)  // allocates new array
// original is unchanged: [1 2 3 4 5]
Challenge Exercise

Write a function removeDuplicates(s []int) []int that removes duplicates in-place (modifying the original slice) and returns the deduplicated slice. Do not allocate a new slice. Then demonstrate with a test that the original backing array is modified. Hint: use two pointers.

#1 Pitfall: Appending to a sub-slice without a full slice expression (a[lo:hi:hi]). When the sub-slice shares capacity with the original, append overwrites elements in the original that you thought were safe. Always use the three-index slice expression to cap the capacity of sub-slices.

Interfaces

Foundations • due today • rep 2

Quick Reminder

Go interfaces are satisfied implicitly — no implements keyword. A type satisfies an interface if it has all the required methods. The empty interface interface{} (or any) accepts any value. Use type assertions (v.(Type)) to extract the concrete type, and type switches (switch v := x.(type)) to branch on multiple types. Small, focused interfaces (1-3 methods) are idiomatic.

Key Pattern

type Writer interface {
    Write(p []byte) (n int, err error)
}

type Logger struct{ prefix string }

func (l *Logger) Write(p []byte) (int, error) {
    fmt.Printf("[%s] %s", l.prefix, p)
    return len(p), nil
}

// Logger implicitly satisfies io.Writer

// Type assertion
var w Writer = &Logger{prefix: "APP"}
if l, ok := w.(*Logger); ok {
    fmt.Println("prefix:", l.prefix)
}

// Type switch
func describe(i interface{}) string {
    switch v := i.(type) {
    case int:
        return fmt.Sprintf("int: %d", v)
    case string:
        return fmt.Sprintf("string: %q", v)
    case error:
        return fmt.Sprintf("error: %v", v)
    default:
        return fmt.Sprintf("unknown: %T", v)
    }
}
Challenge Exercise

Define a Shape interface with Area() float64 and Perimeter() float64. Implement it for Circle and Rectangle. Write a function printShapeInfo(s Shape) that prints the area and perimeter. Then add a type switch that prints an extra detail specific to each concrete type (e.g., radius for Circle, aspect ratio for Rectangle).

#1 Pitfall: A nil interface value and an interface holding a nil pointer are different. var w io.Writer is a nil interface (w == nil is true). But var p *Logger; var w io.Writer = p is a non-nil interface holding a nil pointer (w == nil is false). This trips up error checks: a function returning a concrete nil pointer as an error interface will never equal nil.

sync Package

Concurrency • 4 days overdue • rep 1

Quick Reminder

sync.Mutex provides exclusive locking; sync.RWMutex allows multiple concurrent readers but only one writer. sync.WaitGroup waits for a collection of goroutines to finish. sync.Once ensures a function runs exactly once (thread-safe singletons). sync.Map is a concurrent map optimized for read-heavy workloads — prefer a regular map with RWMutex for most use cases.

Key Pattern

// Mutex-protected counter
type SafeCounter struct {
    mu sync.Mutex
    v  map[string]int
}

func (c *SafeCounter) Inc(key string) {
    c.mu.Lock()
    defer c.mu.Unlock()
    c.v[key]++
}

func (c *SafeCounter) Value(key string) int {
    c.mu.Lock()
    defer c.mu.Unlock()
    return c.v[key]
}

// sync.Once for lazy initialization
var (
    instance *Database
    once     sync.Once
)

func GetDB() *Database {
    once.Do(func() {
        instance = connectToDatabase()
    })
    return instance
}

// WaitGroup for fan-out
var wg sync.WaitGroup
for _, url := range urls {
    wg.Add(1)
    go func(u string) {
        defer wg.Done()
        fetch(u)
    }(url)
}
wg.Wait()
Challenge Exercise

Build a thread-safe Cache struct using sync.RWMutex with methods Get(key string) (string, bool), Set(key, value string), and Delete(key string). Get should use RLock for concurrent reads. Write a benchmark comparing your RWMutex cache with a Mutex-only version under read-heavy load (90% reads, 10% writes).

#1 Pitfall: Copying a sync.Mutex after first use. Mutexes must not be copied because the copy retains the lock state of the original. Always pass mutex-containing structs by pointer, and use go vet to detect accidental copies.

httptest

Testing • due today • rep 1

Quick Reminder

httptest.NewRecorder() captures handler responses without starting a server. httptest.NewServer(handler) starts a real local HTTP server for integration tests. The recorder approach is fast and deterministic; the server approach tests the full HTTP stack including middleware.

Key Pattern

func TestHealthHandler(t *testing.T) {
    req := httptest.NewRequest("GET", "/health", nil)
    w := httptest.NewRecorder()

    healthHandler(w, req)

    resp := w.Result()
    if resp.StatusCode != http.StatusOK {
        t.Errorf("got %d, want 200", resp.StatusCode)
    }

    body, _ := io.ReadAll(resp.Body)
    if string(body) != `{"status":"ok"}` {
        t.Errorf("unexpected body: %s", body)
    }
}

// Full server test with middleware
func TestAPIWithMiddleware(t *testing.T) {
    handler := logging(auth(myRouter()))
    srv := httptest.NewServer(handler)
    t.Cleanup(srv.Close)

    resp, err := http.Get(srv.URL + "/api/users")
    if err != nil {
        t.Fatal(err)
    }
    defer resp.Body.Close()
    // ...assertions...
}
Challenge Exercise

Write a JSON API handler POST /users that accepts {"name": "Alice"} and returns {"id": 1, "name": "Alice"} with status 201. Test it with httptest.NewRecorder, verifying the status code, content-type header, and response body. Also test the error case when the request body is invalid JSON (should return 400).

#1 Pitfall: Forgetting defer srv.Close() (or t.Cleanup(srv.Close)) when using httptest.NewServer. Leaked test servers hold open ports and goroutines, causing flaky tests and resource exhaustion in CI.

Functional Options Pattern

Patterns • due today • rep 1

Quick Reminder

Option functions that configure a struct, applied in the constructor. Define type Option func(*Config) and create WithXxx functions that return an Option. The constructor iterates over options and applies each one, allowing callers to customize only what they need while defaults handle the rest.

Key Pattern

type Option func(*Server)

func WithTimeout(d time.Duration) Option {
    return func(s *Server) { s.timeout = d }
}

func WithMaxConns(n int) Option {
    return func(s *Server) { s.maxConns = n }
}

func NewServer(addr string, opts ...Option) *Server {
    s := &Server{
        addr:     addr,
        timeout:  30 * time.Second, // default
        maxConns: 100,              // default
    }
    for _, opt := range opts {
        opt(s)
    }
    return s
}

// Usage: clean, self-documenting API
srv := NewServer(":8080",
    WithTimeout(10*time.Second),
    WithMaxConns(500),
)
Challenge Exercise

Create a NewLogger constructor using functional options. Support WithLevel(string), WithOutput(io.Writer), and WithJSON(bool). Defaults: level "info", output os.Stdout, JSON false. Add validation: if an invalid level is passed, the option should set an internal error field. The constructor should return (*Logger, error).

#1 Pitfall: Forgetting to initialize the config struct with sensible defaults before applying options. If you skip defaults, zero-values (0 for ints, "" for strings) become the "configured" values when no option is provided, leading to subtle bugs like 0-second timeouts.

gRPC & Protocol Buffers

Networking • due today • rep 0

Quick Reminder

gRPC uses Protocol Buffers for binary serialization and HTTP/2 for transport. Define services and messages in .proto files, generate Go code with protoc. Four RPC types: unary, server streaming, client streaming, bidirectional. Use status.Errorf(codes.NotFound, ...) for semantic errors. Interceptors provide middleware (auth, logging, tracing). Always embed Unimplemented*Server for forward compatibility.

Key Pattern

// Server implementation
type userServer struct {
    pb.UnimplementedUserServiceServer
    users map[string]*pb.User
}

func (s *userServer) GetUser(ctx context.Context, req *pb.GetUserRequest) (*pb.User, error) {
    user, ok := s.users[req.Id]
    if !ok {
        return nil, status.Errorf(codes.NotFound, "user %s not found", req.Id)
    }
    return user, nil
}

// Client call with timeout
ctx, cancel := context.WithTimeout(context.Background(), 5*time.Second)
defer cancel()

user, err := client.GetUser(ctx, &pb.GetUserRequest{Id: "1"})
if s, ok := status.FromError(err); ok && s.Code() == codes.NotFound {
    log.Println("user not found")
}

// Interceptor for logging
srv := grpc.NewServer(
    grpc.ChainUnaryInterceptor(loggingInterceptor, authInterceptor),
)
Challenge Exercise

Write a unary server interceptor that extracts a request-id from gRPC metadata. If missing, generate a UUID and inject it into the context. Log the request-id alongside the method name and duration. Chain it with a recovery interceptor that catches panics and returns codes.Internal.

#1 Pitfall: Using plain fmt.Errorf for gRPC errors instead of status.Errorf. Plain errors are mapped to codes.Unknown on the client side, losing all semantic meaning. Always use the google.golang.org/grpc/status package for gRPC-aware error handling.

Generics Advanced: Type Sets & Constraints

Modern Go • New

Why It Matters

Go 1.18 introduced generics, but the real power lies in advanced constraint features: type sets let you restrict generic parameters to specific types or underlying types, enabling safe arithmetic and comparison operations. Understanding type sets, the ~ operator, and constraint composition is essential for writing reusable, type-safe libraries that go beyond simple any or comparable constraints.

Core Concepts

  • Type sets — constraints define a set of types, not just methods
  • Union constraints — int | float64 | string restricts to specific types
  • ~ operator — ~int matches any type with underlying type int (including type MyInt int)
  • comparable — built-in constraint for types that support == and !=
  • any — alias for interface{}, accepts all types
  • Type inference — compiler can often infer type parameters from arguments

Type Sets and Union Constraints

// Constraint using union types
type Number interface {
    ~int | ~int8 | ~int16 | ~int32 | ~int64 |
    ~float32 | ~float64
}

func Sum[T Number](nums []T) T {
    var total T
    for _, n := range nums {
        total += n
    }
    return total
}

// Works with concrete types AND named types
type Celsius float64
temps := []Celsius{20.5, 22.3, 19.8}
avg := Sum(temps) / Celsius(len(temps))
fmt.Printf("Average: %.1f C\n", avg)

// Works with standard types too
ints := []int{1, 2, 3, 4, 5}
fmt.Println(Sum(ints)) // 15
Output
Average: 20.9 C 15

Building Generic Data Structures

// Generic Set using comparable constraint
type Set[T comparable] struct {
    items map[T]struct{}
}

func NewSet[T comparable](vals ...T) *Set[T] {
    s := &Set[T]{items: make(map[T]struct{})}
    for _, v := range vals {
        s.Add(v)
    }
    return s
}

func (s *Set[T]) Add(v T)           { s.items[v] = struct{}{} }
func (s *Set[T]) Contains(v T) bool { _, ok := s.items[v]; return ok }
func (s *Set[T]) Remove(v T)        { delete(s.items, v) }
func (s *Set[T]) Len() int          { return len(s.items) }

func (s *Set[T]) Union(other *Set[T]) *Set[T] {
    result := NewSet[T]()
    for v := range s.items {
        result.Add(v)
    }
    for v := range other.items {
        result.Add(v)
    }
    return result
}

// Usage - type parameter inferred from arguments
names := NewSet("Alice", "Bob", "Charlie")
names.Add("Diana")
fmt.Println(names.Contains("Bob"))   // true
fmt.Println(names.Len())             // 4

Result Type for Error Handling

// Generic Result type
type Result[T any] struct {
    value T
    err   error
}

func Ok[T any](v T) Result[T]    { return Result[T]{value: v} }
func Err[T any](e error) Result[T] { return Result[T]{err: e} }

func (r Result[T]) Unwrap() (T, error) { return r.value, r.err }
func (r Result[T]) IsOk() bool         { return r.err == nil }

func (r Result[T]) Map(fn func(T) T) Result[T] {
    if r.err != nil {
        return r
    }
    return Ok(fn(r.value))
}

// Usage
func divide(a, b float64) Result[float64] {
    if b == 0 {
        return Err[float64](errors.New("division by zero"))
    }
    return Ok(a / b)
}

result := divide(10, 3).Map(func(v float64) float64 {
    return math.Round(v*100) / 100
})
if v, err := result.Unwrap(); err != nil {
    fmt.Println("error:", err)
} else {
    fmt.Printf("result: %.2f\n", v) // result: 3.33
}

Common Mistakes

Do: Use ~T for named types
type Ordered interface {
    ~int | ~float64 | ~string
}

// Accepts: int, MyInt, float64, etc.
func Max[T Ordered](a, b T) T {
    if a > b { return a }
    return b
}
Don't: Forget ~ for named types
type Ordered interface {
    int | float64 | string
}

// FAILS: type MyInt int does NOT
// satisfy "int" (only exact int)

Key Rules

  • Use ~T in constraints to match named types with the same underlying type — without ~, only exact types match
  • Go does not support generic methods (methods cannot introduce new type parameters); use generic functions or generic types instead
  • Type inference works for function arguments but not for return types — you may need to specify type parameters explicitly when the return type is ambiguous
  • Prefer comparable over any when you need == or map keys — it documents intent and catches bugs at compile time
  • Keep constraints focused: a constraint with 20 types in a union is a code smell — consider if generics are the right tool
Connection: Advanced generics build on the Type Parameters basics from Day 8. The comparable constraint connects to Interfaces (implicit satisfaction). The Set[T] data structure uses Maps internally. The Result[T, E] pattern connects to Error Handling and the Functional Options pattern (both aim to make APIs safer).

Fuzz Testing (Go 1.18+)

Modern Go • New

Why It Matters

Fuzz testing automatically generates random inputs to find bugs that hand-written tests miss. Go 1.18 integrated fuzzing directly into the testing package — no third-party tools needed. Fuzzers excel at finding edge cases in parsers, encoders, validators, and any function that processes untrusted input. A single fuzz run can uncover panics, infinite loops, and logic errors that would take thousands of manually written test cases to find.

Core Concepts

  • func FuzzXxx(f *testing.F) — fuzz test function signature
  • f.Add() — seed the corpus with initial examples (guides the fuzzer)
  • f.Fuzz() — the fuzz target that receives generated inputs
  • Supported seed types: string, []byte, int, float64, bool, rune
  • Crash files saved to testdata/fuzz/FuzzXxx/ — automatically replayed as regression tests
  • Run: go test -fuzz=FuzzXxx -fuzztime=30s

Basic Fuzz Test

// reverse.go
func Reverse(s string) string {
    runes := []rune(s)
    for i, j := 0, len(runes)-1; i < j; i, j = i+1, j-1 {
        runes[i], runes[j] = runes[j], runes[i]
    }
    return string(runes)
}

// reverse_test.go
func FuzzReverse(f *testing.F) {
    // Seed corpus with interesting inputs
    f.Add("hello")
    f.Add("world")
    f.Add("")
    f.Add("a")
    f.Add("Go is fun!")

    f.Fuzz(func(t *testing.T, s string) {
        // Property 1: reversing twice gives the original
        rev := Reverse(s)
        doubleRev := Reverse(rev)
        if s != doubleRev {
            t.Errorf("Reverse(Reverse(%q)) = %q, want %q", s, doubleRev, s)
        }

        // Property 2: length is preserved
        if len(rev) != len(s) {
            t.Errorf("len(Reverse(%q)) = %d, want %d", s, len(rev), len(s))
        }

        // Property 3: must be valid UTF-8
        if !utf8.ValidString(s) {
            return // skip invalid UTF-8 inputs
        }
        if !utf8.ValidString(rev) {
            t.Errorf("Reverse(%q) produced invalid UTF-8: %q", s, rev)
        }
    })
}

Running the Fuzzer

// go test -fuzz=FuzzReverse -fuzztime=30s

// Run with verbose output

Fuzzing a Parser

// Parse structured data: "key=value;key=value"
func ParsePairs(input string) (map[string]string, error) {
    result := make(map[string]string)
    if input == "" {
        return result, nil
    }

    pairs := strings.Split(input, ";")
    for _, pair := range pairs {
        parts := strings.SplitN(pair, "=", 2)
        if len(parts) != 2 {
            return nil, fmt.Errorf("invalid pair: %q", pair)
        }
        key := strings.TrimSpace(parts[0])
        if key == "" {
            return nil, fmt.Errorf("empty key in pair: %q", pair)
        }
        result[key] = strings.TrimSpace(parts[1])
    }
    return result, nil
}

func FuzzParsePairs(f *testing.F) {
    f.Add("name=Alice;age=30")
    f.Add("key=value")
    f.Add("")
    f.Add("a=b;c=d;e=f")

    f.Fuzz(func(t *testing.T, input string) {
        result, err := ParsePairs(input)
        if err != nil {
            return // parser correctly rejected invalid input
        }

        // If it parsed, verify round-trip consistency
        for k, v := range result {
            if k == "" {
                t.Error("parsed an empty key")
            }
            _ = v // value can be empty
        }
    })
}

Common Mistakes

Do: Test properties, not specific values
f.Fuzz(func(t *testing.T, n int) {
    result := Abs(n)
    if result < 0 {
        t.Error("Abs returned negative")
    }
    // Property: result >= 0
})
Don't: Test exact outputs in fuzz
f.Fuzz(func(t *testing.T, n int) {
    if Abs(-5) != 5 {
        t.Error("wrong") // Not fuzzing!
    }
    // Hardcoded input ignores the
    // generated value entirely
})

Key Rules

  • Fuzz tests verify properties (round-trip, invariants, no panics) — not specific input/output pairs
  • Seed the corpus with edge cases: empty strings, single characters, unicode, max values, boundary conditions
  • Skip invalid inputs gracefully with early return rather than t.Fatal — not every random input should be valid
  • Crash files in testdata/fuzz/ are regression tests — commit them to version control
  • Best targets for fuzzing: parsers, serializers, validators, encoding/decoding functions, anything processing untrusted input
Connection: Fuzz testing extends the testing Package with automated input generation. Use it alongside Table-Driven Tests (seed corpus acts like a mini test table). Combines with Error Handling to verify error paths. The encoding/json package is a classic fuzzing target — fuzz marshal/unmarshal round-trips.

Range-over-func Iterators (Go 1.23+)

Modern Go • New

Why It Matters

Go 1.23 introduced range-over-function iterators, enabling custom iteration with the familiar for range syntax. Before this, custom iteration required channels (goroutine overhead), callback-based APIs, or manual Next() methods. Function iterators are zero-allocation, composable, and integrate directly with for range, making Go's iteration story as clean as its concurrency story.

Core Concepts

  • iter.Seq[V] — func(yield func(V) bool) — single-value iterator
  • iter.Seq2[K, V] — func(yield func(K, V) bool) — key-value iterator
  • The yield function is called for each element; return false to stop iteration
  • break in a for range loop causes yield to return false
  • slices.Values(), maps.Keys(), maps.Values() — stdlib iterators
  • iter.Pull() — convert a push iterator to a pull-based (stateful) iterator

Basic Iterator

package main

import (
    "fmt"
    "iter"
)

// Countdown returns an iterator that yields n, n-1, ..., 1
func Countdown(n int) iter.Seq[int] {
    return func(yield func(int) bool) {
        for i := n; i > 0; i-- {
            if !yield(i) {
                return // caller broke out of loop
            }
        }
    }
}

func main() {
    // Use with for range
    for v := range Countdown(5) {
        fmt.Println(v)
    }
    // 5, 4, 3, 2, 1

    // Break works naturally
    for v := range Countdown(100) {
        if v <= 3 {
            break // yield returns false, iterator stops
        }
        fmt.Println(v)
    }
}
Output
5 4 3 2 1 100 99 ... 4

Key-Value Iterators with Seq2

// Enumerate adds an index to any iterator
func Enumerate[V any](seq iter.Seq[V]) iter.Seq2[int, V] {
    return func(yield func(int, V) bool) {
        i := 0
        for v := range seq {
            if !yield(i, v) {
                return
            }
            i++
        }
    }
}

// Iterate over a linked list
type Node[T any] struct {
    Value T
    Next  *Node[T]
}

func (n *Node[T]) All() iter.Seq[T] {
    return func(yield func(T) bool) {
        for curr := n; curr != nil; curr = curr.Next {
            if !yield(curr.Value) {
                return
            }
        }
    }
}

// Usage
head := &Node[string]{Value: "a", Next: &Node[string]{Value: "b", Next: &Node[string]{Value: "c"}}}
for i, v := range Enumerate(head.All()) {
    fmt.Printf("%d: %s\n", i, v)
}
// 0: a
// 1: b
// 2: c

Composable Iterator Combinators

// Filter returns elements where pred is true
func Filter[V any](seq iter.Seq[V], pred func(V) bool) iter.Seq[V] {
    return func(yield func(V) bool) {
        for v := range seq {
            if pred(v) {
                if !yield(v) {
                    return
                }
            }
        }
    }
}

// Map transforms each element
func Map[V, U any](seq iter.Seq[V], fn func(V) U) iter.Seq[U] {
    return func(yield func(U) bool) {
        for v := range seq {
            if !yield(fn(v)) {
                return
            }
        }
    }
}

// Take returns the first n elements
func Take[V any](seq iter.Seq[V], n int) iter.Seq[V] {
    return func(yield func(V) bool) {
        i := 0
        for v := range seq {
            if i >= n {
                return
            }
            if !yield(v) {
                return
            }
            i++
        }
    }
}

// Compose: take first 3 even squares
squares := Map(slices.Values([]int{1, 2, 3, 4, 5, 6, 7, 8}), func(n int) int {
    return n * n
})
evens := Filter(squares, func(n int) bool { return n%2 == 0 })
first3 := Take(evens, 3)

for v := range first3 {
    fmt.Println(v) // 4, 16, 36
}

Pull Iterators

// iter.Pull converts push-based to pull-based iteration
seq := slices.Values([]string{"alpha", "beta", "gamma"})
next, stop := iter.Pull(seq)
defer stop() // must call stop to release resources

v1, ok1 := next() // "alpha", true
v2, ok2 := next() // "beta", true
fmt.Println(v1, ok1) // alpha true
fmt.Println(v2, ok2) // beta true

// Useful for interleaving two iterators (Zip)
func Zip[A, B any](sa iter.Seq[A], sb iter.Seq[B]) iter.Seq2[A, B] {
    return func(yield func(A, B) bool) {
        nextB, stopB := iter.Pull(sb)
        defer stopB()

        for a := range sa {
            b, ok := nextB()
            if !ok {
                return
            }
            if !yield(a, b) {
                return
            }
        }
    }
}

Common Mistakes

Do: Check yield return value
func Generate(n int) iter.Seq[int] {
    return func(yield func(int) bool) {
        for i := 0; i < n; i++ {
            if !yield(i) {
                return // respect break
            }
        }
    }
}
Don't: Ignore yield's return
func Generate(n int) iter.Seq[int] {
    return func(yield func(int) bool) {
        for i := 0; i < n; i++ {
            yield(i) // ignores break!
            // continues generating
            // after caller stopped
        }
    }
}

Key Rules

  • Always check yield's return value — returning false means the caller used break and iteration must stop
  • Iterators are lazy — no work happens until you range over them, and they stop early if you break
  • Use iter.Pull when you need stateful, step-by-step access to an iterator (e.g., Zip, interleaving)
  • Always defer stop() when using iter.Pull — failing to stop leaks the iterator goroutine
  • Prefer iter.Seq/iter.Seq2 over channels for iteration — zero allocation, no goroutine overhead
Connection: Range-over-func replaces channel-based iteration from Concurrency Patterns with zero-allocation alternatives. Combines with Generics for type-safe combinator libraries. The slices and maps packages use these iterators extensively. Pull iterators connect to the sync Package (they internally use goroutine synchronization).

Structured Concurrency Patterns

Modern Go • New

Why It Matters

Unstructured goroutine usage leads to leaks, orphaned work, and error swallowing. Structured concurrency ensures that every goroutine has a clear owner, a cancellation path, and proper error propagation. Go's errgroup, context, and singleflight packages provide the building blocks for concurrency that is safe by construction.

Core Concepts

  • Context-based lifecycle — every goroutine receives a context and respects its cancellation
  • errgroup with derived contexts — cancels all goroutines when any one fails
  • Worker pool with cancellation — context.Done() drains work on shutdown
  • Pipeline with cleanup — each stage cleans up when the pipeline is cancelled
  • Goroutine leak prevention — always provide an exit path (context or done channel)
  • singleflight — deduplicate concurrent requests for the same resource

errgroup with Derived Context

import "golang.org/x/sync/errgroup"

func fetchAllPages(ctx context.Context, urls []string) ([]string, error) {
    // g.Go goroutines are cancelled if any one fails
    g, ctx := errgroup.WithContext(ctx)
    results := make([]string, len(urls))

    for i, url := range urls {
        i, url := i, url // capture loop variables
        g.Go(func() error {
            req, err := http.NewRequestWithContext(ctx, "GET", url, nil)
            if err != nil {
                return fmt.Errorf("creating request for %s: %w", url, err)
            }
            resp, err := http.DefaultClient.Do(req)
            if err != nil {
                return fmt.Errorf("fetching %s: %w", url, err)
            }
            defer resp.Body.Close()

            body, err := io.ReadAll(resp.Body)
            if err != nil {
                return fmt.Errorf("reading %s: %w", url, err)
            }
            results[i] = string(body) // each goroutine writes its own index
            return nil
        })
    }

    if err := g.Wait(); err != nil {
        return nil, err // first error cancels all others
    }
    return results, nil
}

Worker Pool with Context Cancellation

func processJobs(ctx context.Context, jobs []Job, workers int) error {
    g, ctx := errgroup.WithContext(ctx)
    jobCh := make(chan Job)

    // Producer: send jobs until cancelled
    g.Go(func() error {
        defer close(jobCh)
        for _, j := range jobs {
            select {
            case jobCh <- j:
            case <-ctx.Done():
                return ctx.Err()
            }
        }
        return nil
    })

    // Workers: process jobs until channel closes or context cancels
    for w := 0; w < workers; w++ {
        g.Go(func() error {
            for j := range jobCh {
                select {
                case <-ctx.Done():
                    return ctx.Err()
                default:
                }
                if err := j.Process(ctx); err != nil {
                    return err // cancels all other workers
                }
            }
            return nil
        })
    }

    return g.Wait()
}

singleflight: Deduplicating Concurrent Requests

import "golang.org/x/sync/singleflight"

type UserCache struct {
    mu    sync.RWMutex
    cache map[string]*User
    group singleflight.Group
}

func (uc *UserCache) GetUser(ctx context.Context, id string) (*User, error) {
    // Fast path: check cache
    uc.mu.RLock()
    if user, ok := uc.cache[id]; ok {
        uc.mu.RUnlock()
        return user, nil
    }
    uc.mu.RUnlock()

    // singleflight: if 100 goroutines request user "123",
    // only ONE database query runs; all 100 get the result
    v, err, shared := uc.group.Do(id, func() (interface{}, error) {
        user, err := fetchUserFromDB(ctx, id)
        if err != nil {
            return nil, err
        }

        // Cache the result
        uc.mu.Lock()
        uc.cache[id] = user
        uc.mu.Unlock()

        return user, nil
    })

    if err != nil {
        return nil, err
    }

    if shared {
        log.Printf("user %s: result shared across concurrent requests", id)
    }

    return v.(*User), nil
}

Preventing Goroutine Leaks

// BAD: goroutine leaks if nobody reads from ch
func leaky() <-chan int {
    ch := make(chan int)
    go func() {
        for i := 0; ; i++ {
            ch <- i // blocks forever if nobody reads
        }
    }()
    return ch
}

// GOOD: context cancellation stops the goroutine
func safe(ctx context.Context) <-chan int {
    ch := make(chan int)
    go func() {
        defer close(ch)
        for i := 0; ; i++ {
            select {
            case ch <- i:
            case <-ctx.Done():
                return // goroutine exits cleanly
            }
        }
    }()
    return ch
}

// Usage
ctx, cancel := context.WithCancel(context.Background())
ch := safe(ctx)
fmt.Println(<-ch) // 0
fmt.Println(<-ch) // 1
cancel()           // goroutine exits, no leak

Common Mistakes

Do: Use errgroup for fork-join
g, ctx := errgroup.WithContext(ctx)
for _, item := range items {
    item := item
    g.Go(func() error {
        return process(ctx, item)
    })
}
return g.Wait() // all errors collected
Don't: Fire-and-forget goroutines
for _, item := range items {
    go func(it Item) {
        process(context.TODO(), it)
        // error is lost
        // no cancellation
        // may outlive caller
    }(item)
}

Key Rules

  • Every goroutine must have an owner that waits for it and handles its errors — errgroup is the standard tool for this
  • Use errgroup.WithContext to get automatic cancellation when any goroutine fails
  • Always check ctx.Done() in long-running goroutine loops — it is the universal "time to stop" signal
  • Use singleflight to deduplicate concurrent requests for the same resource (cache stampede prevention)
  • Never launch a goroutine without a way to stop it — pass a context, a done channel, or close its input channel
Connection: Structured concurrency formalizes the Concurrency Patterns (fan-out/fan-in, worker pools) with proper lifecycle management. Builds on Context for cancellation propagation and sync Package for synchronization. The singleflight pattern pairs with httptest for testing cache behavior under concurrent load.

Advanced Error Handling: errors.Join & Sentinel Errors

Modern Go • New

Why It Matters

Go 1.13 introduced error wrapping with %w, and Go 1.20 added errors.Join for combining multiple errors. Together with sentinel errors, errors.Is, and errors.As, these tools create a robust error handling system. Mastering these patterns lets you build error chains that preserve context for debugging while enabling programmatic error checking at any level of the call stack.

Core Concepts

  • Sentinel errors — predefined error values: var ErrNotFound = errors.New("not found")
  • errors.Is(err, target) — checks if any error in the chain matches target
  • errors.As(err, &target) — extracts a specific error type from the chain
  • fmt.Errorf("...: %w", err) — wraps an error, preserving the chain
  • errors.Join(err1, err2, ...) — combines multiple errors into one (Go 1.20+)
  • Unwrap() []error — custom types can unwrap to multiple errors (multi-error)

Sentinel Errors

package storage

import "errors"

// Sentinel errors are package-level variables
var (
    ErrNotFound     = errors.New("storage: not found")
    ErrUnauthorized = errors.New("storage: unauthorized")
    ErrConflict     = errors.New("storage: conflict")
)

func GetUser(id string) (*User, error) {
    user, ok := db[id]
    if !ok {
        return nil, fmt.Errorf("GetUser(%s): %w", id, ErrNotFound)
    }
    return user, nil
}

// Caller checks using errors.Is (traverses the entire wrap chain)
user, err := storage.GetUser("123")
if errors.Is(err, storage.ErrNotFound) {
    // handle not found specifically
    http.Error(w, "User not found", 404)
} else if err != nil {
    // handle other errors
    http.Error(w, "Internal error", 500)
}

Custom Error Types with errors.As

// Custom error type carries structured data
type ValidationError struct {
    Field   string
    Message string
}

func (e *ValidationError) Error() string {
    return fmt.Sprintf("validation: %s - %s", e.Field, e.Message)
}

func ValidateUser(u User) error {
    if u.Name == "" {
        return &ValidationError{Field: "name", Message: "is required"}
    }
    if u.Age < 0 || u.Age > 150 {
        return &ValidationError{Field: "age", Message: "must be 0-150"}
    }
    return nil
}

// Extract with errors.As
err := ValidateUser(User{Name: "", Age: -1})
var ve *ValidationError
if errors.As(err, &ve) {
    fmt.Printf("Field %q failed: %s\n", ve.Field, ve.Message)
    // Field "name" failed: is required
}

// Works through wrapped errors too
wrapped := fmt.Errorf("creating user: %w", err)
if errors.As(wrapped, &ve) {
    fmt.Println(ve.Field) // "name" -- still found through the chain
}

errors.Join: Combining Multiple Errors

// Validate multiple fields, collect all errors
func ValidateConfig(cfg Config) error {
    var errs []error

    if cfg.Host == "" {
        errs = append(errs, fmt.Errorf("host: %w", ErrRequired))
    }
    if cfg.Port < 1 || cfg.Port > 65535 {
        errs = append(errs, fmt.Errorf("port: must be 1-65535"))
    }
    if cfg.Timeout <= 0 {
        errs = append(errs, fmt.Errorf("timeout: must be positive"))
    }

    return errors.Join(errs...) // nil if errs is empty
}

// errors.Join returns nil when given no non-nil errors
err := ValidateConfig(Config{Host: "", Port: 0, Timeout: -1})
if err != nil {
    fmt.Println(err)
    // host: required
    // port: must be 1-65535
    // timeout: must be positive
}

// errors.Is works through joined errors!
if errors.Is(err, ErrRequired) {
    fmt.Println("at least one required field is missing")
}

Multi-Error Unwrap

// Custom type that wraps multiple errors
type MultiError struct {
    Errors []error
}

func (m *MultiError) Error() string {
    msgs := make([]string, len(m.Errors))
    for i, e := range m.Errors {
        msgs[i] = e.Error()
    }
    return strings.Join(msgs, "; ")
}

// Implement Unwrap() []error for errors.Is/As compatibility
func (m *MultiError) Unwrap() []error {
    return m.Errors
}

// Now errors.Is and errors.As traverse all wrapped errors
me := &MultiError{Errors: []error{
    fmt.Errorf("db: %w", ErrNotFound),
    fmt.Errorf("cache: %w", ErrUnauthorized),
}}

fmt.Println(errors.Is(me, ErrNotFound))     // true
fmt.Println(errors.Is(me, ErrUnauthorized)) // true

Common Mistakes

Do: Use %w for wrapping
// Wraps: errors.Is/As can find ErrNotFound
return fmt.Errorf("GetUser(%s): %w", id, ErrNotFound)

// Caller can check:
if errors.Is(err, ErrNotFound) { ... }
Don't: Use %v (breaks the chain)
// Does NOT wrap: chain is broken
return fmt.Errorf("GetUser(%s): %v", id, ErrNotFound)

// Caller's check FAILS:
if errors.Is(err, ErrNotFound) { ... } // false!
Do: Sentinel errors as package vars
var ErrNotFound = errors.New("not found")

// Stable identity for comparison
errors.Is(err, ErrNotFound) // works
Don't: Create errors inline
if errors.Is(err, errors.New("not found")) {
    // ALWAYS false!
    // errors.New creates a new pointer
    // each time, different identity
}

Key Rules

  • Use %w to wrap errors when callers should be able to inspect the chain with errors.Is/errors.As
  • Use %v to format errors when you want to hide the implementation detail (opaque error)
  • Define sentinel errors as package-level var (not const) — errors.New returns a pointer, and identity comparison requires a stable address
  • errors.Join returns nil when all arguments are nil — safe to use with errors.Join(errs...) on an empty slice
  • Implement Unwrap() []error on custom multi-error types so errors.Is and errors.As traverse all contained errors
Connection: Builds on Error Handling basics (error interface, custom errors). errors.Join pairs with errgroup for collecting errors from concurrent operations. gRPC status errors require different patterns (status.Errorf) but errors.Is still works for wrapped status errors. The Functional Options pattern can use validation errors to report bad configuration.
